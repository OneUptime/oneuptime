To verify the agent is running:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "kubernetes-agent.name" . }}

To check collector logs:
  kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "kubernetes-agent.name" . }} -c otel-collector
{{- if .Values.ebpf.enabled }}

To check the eBPF instrumentation logs:
  kubectl logs -n {{ .Release.Namespace }} -l component=ebpf-instrument -c ebpf-instrument --tail=100

To turn eBPF auto-instrumentation off:
  helm upgrade {{ .Release.Name }} oneuptime/kubernetes-agent -n {{ .Release.Namespace }} --reuse-values --set ebpf.enabled=false
{{- end }}

Your cluster should appear in OneUptime within a few minutes at:
  {{ .Values.oneuptime.url }}/dashboard/<project-id>/kubernetes

If it stays "Disconnected" or no data shows up (most often a wrong/revoked
ingestion key — which is silently dropped, so the collector logs look clean),
run the diagnostic script for a one-shot root-cause verdict:
  curl -fsSL https://raw.githubusercontent.com/OneUptime/oneuptime/master/HelmChart/Public/kubernetes-agent/troubleshoot.sh | bash -s -- -n {{ .Release.Namespace }}

Or validate your ingestion key directly (200 = valid, 401 = unknown/revoked):
  curl -i -H "x-oneuptime-token: <YOUR_API_KEY>" {{ .Values.oneuptime.url }}/otlp/v1/validate
