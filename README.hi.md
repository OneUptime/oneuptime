## अपने पूरे ऑब्ज़र्वेबिलिटी स्टैक को बदलें

OneUptime मॉनिटरिंग, अलर्टिंग, इंसिडेंट रिस्पॉन्स और ऑब्ज़र्वेबिलिटी को एक ही ओपन-सोर्स ऐप में लाता है — ताकि आप दर्जनों अलग-अलग टूल्स के लिए पैसे चुकाना (और उन्हें आपस में जोड़ना) बंद कर दें।

| इसके बजाय… | OneUptime का उपयोग करें… |
|---|---|
| Pingdom / UptimeRobot | **अपटाइम मॉनिटरिंग** — दुनिया भर से वेबसाइट, API, पिंग, पोर्ट, SSL, DNS और सिंथेटिक चेक |
| StatusPage.io | **स्टेटस पेज** — सब्सक्राइबर्स के साथ ब्रांडेड सार्वजनिक और निजी स्टेटस पेज |
| PagerDuty / Opsgenie | **ऑन-कॉल और अलर्ट** — शेड्यूल, एस्केलेशन नीतियाँ, SMS / कॉल / पुश / Slack |
| Incident.io | **इंसिडेंट मैनेजमेंट** — घोषित करें, ट्राइएज करें, संवाद करें और पोस्ट-मॉर्टम करें |
| Datadog / New Relic | **APM और मेट्रिक्स** — ट्रेस, डैशबोर्ड और सर्विस परफ़ॉर्मेंस |
| Loggly | **लॉग मैनेजमेंट** — लॉग एकत्र करें, खोजें और उन पर अलर्ट पाएँ |
| Sentry | **एरर ट्रैकिंग** — पूरे स्टैक ट्रेस और संदर्भ के साथ एक्सेप्शन |

यह सब **100% ओपन सोर्स (Apache 2.0)** है और सेल्फ-होस्ट करने के लिए मुफ़्त है।

## ⚡ क्विक स्टार्ट

### ☁️ OneUptime Cloud — आसान तरीका

कोई सेटअप नहीं, हमेशा अपडेटेड, और यह ओपन-सोर्स प्रोजेक्ट को फंड करता है।

**→ [oneuptime.com पर मुफ़्त साइन अप करें](https://oneuptime.com)**

### 🐳 Docker Compose के साथ सेल्फ-होस्ट करें

आपकी ज़रूरत की हर चीज़ एक ही सर्वर पर (Debian / Ubuntu / RHEL, Docker + Docker Compose)। होमलैब और छोटी टीमों के लिए बढ़िया — एक Raspberry Pi भी काम करता है।

```bash
# 1. Clone the release branch
git clone --depth 1 --single-branch --branch release https://github.com/OneUptime/oneuptime.git
cd oneuptime

# 2. Create your config (then edit it — set strong, random secrets!)
cp config.example.env config.env

# 3. Start everything
npm start
```

OneUptime अब **http://localhost** पर चल रहा है — इसे खोलें और अपना पहला अकाउंट बनाएँ।

📖 पूरी गाइड: [Docker Compose इंस्टॉल](/App/FeatureSet/Docs/Content/en/installation/docker-compose.md) · [साइज़िंग और आवश्यकताएँ](/App/FeatureSet/Docs/Content/en/installation/sizing.md)

### ☸️ प्रोडक्शन के लिए Helm के साथ Kubernetes

```bash
helm repo add oneuptime https://helm-chart.oneuptime.com
helm install oneuptime oneuptime/oneuptime
```

📖 पूरी इंस्टॉल निर्देश और वैल्यूज़ [Artifact Hub पर →](https://artifacthub.io/packages/helm/oneuptime/oneuptime)

> **किसी मौजूदा इंस्टॉल को अपग्रेड कर रहे हैं?** [अपग्रेड गाइड](/App/FeatureSet/Docs/Content/en/installation/upgrading.md) देखें।

### 🖥️ इंफ़्रास्ट्रक्चर मॉनिटरिंग

कॉपी-पेस्ट, **OpenTelemetry-आधारित** एजेंट जोड़ें ताकि आपकी सर्विसेज़ जिस पर चलती हैं उस सब पर नज़र रखी जा सके — तैयार अलर्ट टेम्पलेट सहित:

- **सर्वर और VMs** — Linux, macOS और Windows से CPU, मेमोरी, डिस्क, नेटवर्क, प्रोसेस और लॉग। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/host-otel-collector.md)
- **Kubernetes** — एक `helm install` नोड/पॉड/कंटेनर/क्लस्टर मेट्रिक्स, इवेंट, लॉग और eBPF ट्रेस व सर्विस मैप भेजता है। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/kubernetes-agent.md)
- **Docker** — एक ही एजेंट हर कंटेनर को स्वतः खोज लेता है और मेट्रिक्स व लॉग भेजता है। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/docker-host.md)
- **Podman** — Podman के Docker-संगत सॉकेट के माध्यम से वही एक-एजेंट ऑटो-डिस्कवरी। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/podman-host.md)
- **Proxmox** — नोड, VMs, कंटेनर, स्टोरेज, HA स्थिति, बैकअप कवरेज और रेप्लिकेशन हेल्थ। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/proxmox.md)
- **Ceph** — क्लस्टर हेल्थ, कैपेसिटी पूर्वानुमान और OSD/पूल/PG/मॉनिटर विज़िबिलिटी। [डॉक्स →](/App/FeatureSet/Docs/Content/en/telemetry/ceph.md)

## 💡 OneUptime क्यों?

हमारा मिशन सरल है: **डाउनटाइम कम करना और अधिक प्रोडक्ट्स को सफल होने में मदद करना।** सात वेंडरों को आपस में जोड़-तोड़ कर टिकाने के बजाय, आपको एक ऐसा प्लेटफ़ॉर्म मिलता है जो आपको यह समझने में मदद करता है कि चीज़ें *क्यों* टूटती हैं, इंसिडेंट पर तेज़ी से प्रतिक्रिया देता है और परिचालन के बोझ को कम करता है — पूरी तरह ओपन सोर्स, ताकि आपके डेटा और आपके स्टैक पर आपका ही स्वामित्व हो।
