# 变量与筛选

变量把单张仪表板变成模板。给仪表板加一个 `service` 变量,同样的图表就会为 `checkout`、`payments` 或 `search` 重新渲染——访客从顶部下拉中选择,而不是你去建三张几乎相同的仪表板。

## 变量类型

在 **仪表板 → 设置 → 变量** 下添加变量。每个变量有一个名称(在组件中以 `{{name}}` 引用)、一个可选的标签和一个类型。

### 自定义列表

一个静态下拉框。选项由你自己输入。

适用于:选项少且固定时。`environment` 取值 `prod, staging, dev`。`region` 取值 `us-east-1, eu-west-1, ap-south-1`。

### 查询

选项来自对你数据的查询。

适用于:选项随时间变化、你希望下拉框跟上时。"过去 24 小时内出现过的每个客户 ID"。查询会对你项目的数据运行,结果作为下拉项。

### 文本输入

一个自由文本字段。访客输入什么就用什么。

适用于:你想让仪表板像搜索工具一样使用时。按 IP 地址、请求 ID 或任何自由格式的值过滤。

### 遥测属性

选项是你遥测数据在仪表板时间范围内某个属性的不同取值。

配置 **属性键**(例如 `service.name`、`host.name`、`k8s.cluster.name`)。下拉框会填入你日志、指标和追踪中所见的每个不同取值。

适用于:选项与你的遥测中已经发送的标签匹配时。这是最常用的类型,因为它会自动更新——当你发布一个打了 `service.name = inventory` 标签的新服务,这个名字会出现在下拉里,你不必修改仪表板。

## 多选

每个变量都可以允许多选。开启后,访客可以选一个或多个值;仪表板会筛选为它们中的任意一个。

什么时候用多选:你想要"把 checkout 和 payments 放在一起比较"而不离开仪表板时。当跨所选值的算式不成立时(例如对平均值求平均)避免使用。

## 默认值

每个变量都可以有一个默认值。访客修改之前,仪表板用默认值渲染。对公开仪表板来说,默认值就是访客最初看到的内容。

## 如何在组件中使用变量

任何接受筛选条件的地方——指标的 `WHERE`、列表的筛选条件、日志流的属性匹配——你都可以用 `{{variable_name}}`。

例如,按服务过滤的图表:

```
service.name = '{{service}}'
```

下拉设为 `checkout` 时,图表筛选到 checkout 服务。访客切到 `payments` 时,图表为 payments 重新渲染。

对于 **遥测属性** 变量,OneUptime 知道这个变量对应的是哪个属性,并会把筛选应用到所有使用相同属性的组件——你不必逐个手动修改组件。

## 时间范围

仪表板头部有一个全局时间范围。每个指标类组件都对这个窗口进行查询。选项:

- **预设**——过去 1 小时、24 小时、7 天、30 天、90 天(取决于你的数据保留期)。
- **自定义**——选择开始和结束时间。

时间范围是仪表板 URL 的一部分——分享 URL 就是在分享窗口。在事件中很有用:把时间范围钉到"今天 UTC 10:00–10:30",然后把链接粘到事件频道。

## 刷新间隔

时间范围旁边,选择组件重新查询的频率:

- **关闭**——页面加载时只查询一次。
- **5s / 10s / 30s / 1m / 5m / 15m**——自动刷新。

自动刷新适合挂墙屏幕或实时事件视图。在排查问题时关闭它,让画面在你查看的过程中保持不动。

## 组合起来

一张以服务为模板的仪表板通常有:

1. 一个类型为 **遥测属性** 的 `service` 变量,对应 `service.name`。默认:你最关注的服务。多选关闭(这样图表每次只显示一个)。
2. 一个类型为 **自定义列表** 的 `environment` 变量。默认:`prod`。
3. 一个类型为 **遥测属性** 的 `cluster` 变量,对应 `k8s.cluster.name`。多选打开(这样可以跨集群比较)。
4. 在筛选条件里引用这些变量的组件。

结果:一张仪表板覆盖所有服务,顶部三个下拉。
