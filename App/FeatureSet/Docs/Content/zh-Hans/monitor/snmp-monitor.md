# SNMP 监控器

SNMP（简单网络管理协议）监控允许您通过查询 SNMP OID（对象标识符）来监控交换机、路由器、防火墙和其他网络基础设施等网络设备。

## 概述

SNMP 监控器使用 OID 查询网络设备的特定管理信息。这使您能够：

- 监控设备可用性和健康状况
- 跟踪接口统计信息（流量、错误、状态）
- 监控系统指标（CPU、内存、运行时间）
- 检查自定义厂商特定的 OID
- 基于 OID 值设置告警

## 创建 SNMP 监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **SNMP** 作为监控器类型
4. 按照下面的说明配置 SNMP 设置

## 配置选项

### 基本设置

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| SNMP 版本 | 协议版本：v1、v2c 或 v3 | 是 |
| 主机名/IP | SNMP 设备的主机名或 IP 地址 | 是 |
| 端口 | SNMP 端口（默认：161） | 是 |

### 认证

#### SNMP v1/v2c

对于 SNMP v1 和 v2c，您只需提供 community string：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| Community String | SNMP community string（例如"public"） | 是 |

#### SNMP v3

SNMPv3 提供增强的安全性，包括认证和加密：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 安全级别 | noAuthNoPriv、authNoPriv 或 authPriv | 是 |
| 用户名 | SNMPv3 用户名 | 是 |
| 认证协议 | MD5、SHA、SHA256 或 SHA512 | 如果是 authNoPriv 或 authPriv |
| 认证密钥 | 认证密码 | 如果是 authNoPriv 或 authPriv |
| 加密协议 | DES、AES 或 AES256 | 如果是 authPriv |
| 加密密钥 | 隐私/加密密码 | 如果是 authPriv |

### 要监控的 OID

添加您要从设备查询的 OID。对于每个 OID，您可以指定：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| OID | 数字 OID（例如 1.3.6.1.2.1.1.1.0） | 是 |
| 名称 | OID 的友好名称（例如 sysDescr） | 否 |
| 描述 | 此 OID 代表内容的描述 | 否 |

### 常用 OID 模板

OneUptime 为常用的 OID 提供模板：

#### System MIB

| OID | 名称 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.1.1.0 | sysDescr | 系统描述 |
| 1.3.6.1.2.1.1.3.0 | sysUpTime | 系统运行时间（时钟滴答数） |
| 1.3.6.1.2.1.1.5.0 | sysName | 系统名称 |
| 1.3.6.1.2.1.1.6.0 | sysLocation | 系统位置 |
| 1.3.6.1.2.1.1.4.0 | sysContact | 系统联系人 |

#### Interface MIB

| OID | 名称 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.2.1.0 | ifNumber | 网络接口数量 |
| 1.3.6.1.2.1.2.2.1.8.X | ifOperStatus | 接口操作状态（X = 接口索引） |
| 1.3.6.1.2.1.2.2.1.10.X | ifInOctets | 输入字节数（X = 接口索引） |
| 1.3.6.1.2.1.2.2.1.16.X | ifOutOctets | 输出字节数（X = 接口索引） |

#### Host Resources MIB

| OID | 名称 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.25.1.1.0 | hrSystemUptime | 主机系统运行时间 |
| 1.3.6.1.2.1.25.1.5.0 | hrSystemNumUsers | 用户数量 |
| 1.3.6.1.2.1.25.1.6.0 | hrSystemProcesses | 运行进程数量 |
| 1.3.6.1.2.1.25.3.3.1.2.X | hrProcessorLoad | CPU 负载（X = 处理器索引） |

### 高级设置

| 字段 | 描述 | 默认值 |
|------|------|--------|
| 超时 | 等待响应的时间（毫秒） | 5000 |
| 重试次数 | 失败时的重试次数 | 3 |

## 监控标准

您可以设置标准来检查 SNMP 响应并触发告警或事件。

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| SNMP 设备是否在线 | 检查设备是否响应 SNMP 查询 |
| SNMP 响应时间 | 检查查询响应时间（毫秒） |
| SNMP OID 值 | 检查特定 OID 返回的值 |
| SNMP OID 是否存在 | 检查 OID 是否返回值（非空） |

### 示例标准

#### 检查设备是否在线
- **检查项**：SNMP 设备是否在线
- **过滤类型**：True

#### 当响应时间超过阈值时发出告警
- **检查项**：SNMP 响应时间（毫秒）
- **过滤类型**：大于
- **值**：1000

#### 检查接口状态
- **检查项**：SNMP OID 值
- **OID**：1.3.6.1.2.1.2.2.1.8.1
- **过滤类型**：等于
- **值**：1（1 = 正常，2 = 故障）

#### 检查 CPU 负载阈值
- **检查项**：SNMP OID 值
- **OID**：1.3.6.1.2.1.25.3.3.1.2.1
- **过滤类型**：大于
- **值**：80

## 使用监控器密钥

出于安全原因，您可以将 community string 和 SNMPv3 凭据等敏感信息存储为密钥。

### 添加密钥

1. 前往 **项目设置** -> **监控器密钥** -> **创建监控器密钥**
2. 添加您的密钥（例如 community string 或 SNMPv3 密码）
3. 选择应有权访问此密钥的 SNMP 监控器

### 在 SNMP 配置中使用密钥

在任何敏感字段中使用 `{{monitorSecrets.SECRET_NAME}}` 语法：

- **Community String**：`{{monitorSecrets.SnmpCommunity}}`
- **SNMPv3 认证密钥**：`{{monitorSecrets.SnmpAuthKey}}`
- **SNMPv3 加密密钥**：`{{monitorSecrets.SnmpPrivKey}}`

## 告警模板变量

创建事件或告警模板时，您可以使用以下变量：

| 变量 | 描述 |
|------|------|
| `{{isOnline}}` | 设备是否在线（true/false） |
| `{{responseTimeInMs}}` | 查询响应时间（毫秒） |
| `{{failureCause}}` | 查询失败时的错误消息 |
| `{{oidResponses}}` | OID 响应对象数组 |
| `{{OID_NAME}}` | 按名称获取特定 OID 的值（例如 `{{sysUpTime}}`） |

## 故障排查

### 常见问题

#### 设备不响应
- 验证设备 IP/主机名是否正确
- 检查设备上是否启用了 SNMP
- 验证防火墙规则是否允许 UDP 端口 161
- 确认 community string 是否正确

#### 认证失败（v3）
- 验证用户名、认证协议和认证密钥
- 确保安全级别与设备配置匹配
- 检查 authPriv 级别的加密协议和密钥是否正确

#### OID 未找到
- 验证 OID 是否受您的设备支持
- 检查 OID 是否需要加载特定的 MIB
- 尝试使用 snmpget/snmpwalk 工具直接查询 OID

### 测试 SNMP 连接性

设置监控之前，您可以使用命令行工具测试 SNMP 连接性：

```bash
# SNMP v2c
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# SNMP v3 (authPriv)
snmpget -v3 -u username -l authPriv -a SHA -A authpassword -x AES -X privpassword 192.168.1.1 1.3.6.1.2.1.1.1.0
```

## 最佳实践

1. **尽可能使用 SNMPv3** — 它提供认证和加密以增强安全性
2. **将凭据存储为密钥** — 切勿硬编码 community string 或密码
3. **仅监控必要的 OID** — 只查询所需内容以减少网络开销
4. **设置适当的超时** — 网络设备的响应时间可能各不相同
5. **使用描述性 OID 名称** — 使告警消息更易于理解
6. **部署前先测试** — 在创建监控器之前验证 SNMP 连接性
