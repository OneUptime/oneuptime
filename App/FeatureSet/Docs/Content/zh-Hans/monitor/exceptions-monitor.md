# 异常监控器

异常监控允许您监控应用程序异常和错误，当异常数量超过配置的阈值时触发告警。OneUptime 在一个时间窗口内评估来自您遥测服务的异常数据。

## 概述

异常监控器统计并过滤符合特定条件的异常。这使您能够：

- 对应用程序中的异常峰值发出告警
- 监控特定的异常类型
- 按错误消息搜索异常
- 分别跟踪已解决和活跃的异常
- 从错误模式中检测应用程序稳定性问题

## 创建异常监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **异常** 作为监控器类型
4. 选择要监控的遥测服务
5. 根据需要配置异常过滤器和标准

## 配置选项

### 遥测服务

选择一个或多个服务来监控其异常。服务必须通过 OpenTelemetry 向 OneUptime 发送异常数据。

### 异常过滤器

| 过滤器 | 描述 | 是否必填 |
|--------|------|---------|
| 异常类型 | 按异常类型名称过滤（例如 `NullPointerException`、`TypeError`） | 否 |
| 消息 | 在异常消息中进行文本搜索 | 否 |
| 包含已解决 | 包含已标记为已解决的异常（默认：false） | 否 |
| 包含已归档 | 包含已归档的异常（默认：false） | 否 |
| 时间窗口 | 搜索异常的时间范围（秒，默认：60） | 否 |

## 监控标准

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| 异常数量 | 时间窗口内符合过滤条件的异常数量 |

### 过滤类型

- **大于** — 异常数量超过阈值
- **小于** — 异常数量低于阈值
- **大于或等于** — 异常数量等于或超过阈值
- **小于或等于** — 异常数量等于或低于阈值
- **等于** — 异常数量完全匹配
- **不等于** — 异常数量不匹配

### 示例标准

#### 当 60 秒内超过 10 个异常时发出告警

- **时间窗口**：60 秒
- **检查项**：异常数量
- **过滤类型**：大于
- **值**：10

#### 对任何 NullPointerException 发出告警

- **异常类型**：`NullPointerException`
- **时间窗口**：60 秒
- **检查项**：异常数量
- **过滤类型**：大于
- **值**：0

#### 监控包含特定消息的异常

- **消息**：`out of memory`
- **时间窗口**：300 秒
- **检查项**：异常数量
- **过滤类型**：大于
- **值**：0

## 设置要求

异常监控需要您的应用程序通过 OpenTelemetry 向 OneUptime 发送异常数据。设置说明请参见 [OpenTelemetry](/docs/telemetry/open-telemetry) 文档。
