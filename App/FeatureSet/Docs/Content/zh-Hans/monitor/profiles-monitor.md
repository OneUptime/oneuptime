# 性能分析监控器

性能分析监控允许您监控来自应用程序的持续分析数据，并根据分析数量和模式触发告警。OneUptime 在时间窗口内评估来自遥测服务的分析数据。

## 概述

性能分析监控器统计并过滤符合特定条件的分析数据。这使您能够：

- 监控来自应用程序的持续分析数据
- 按类型过滤分析（CPU、内存、goroutine 等）
- 跟踪分析量和模式
- 对分析异常发出告警
- 按自定义分析属性过滤

## 创建性能分析监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **性能分析** 作为监控器类型
4. 选择要监控的遥测服务
5. 根据需要配置分析过滤器和标准

## 配置选项

### 遥测服务

选择一个或多个服务来监控其分析数据。服务必须通过 OpenTelemetry 向 OneUptime 发送持续分析数据。

### 分析过滤器

| 过滤器 | 描述 | 是否必填 |
|--------|------|---------|
| 分析类型 | 按分析类型名称过滤（例如 CPU、内存、goroutine） | 否 |
| 属性 | 用于过滤自定义分析属性的键值对 | 否 |
| 时间窗口 | 搜索分析数据的时间范围（秒，默认：60） | 否 |

## 监控标准

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| 分析数量 | 时间窗口内符合过滤条件的分析数量 |

### 过滤类型

- **大于** — 分析数量超过阈值
- **小于** — 分析数量低于阈值
- **大于或等于** — 分析数量等于或超过阈值
- **小于或等于** — 分析数量等于或低于阈值
- **等于** — 分析数量完全匹配
- **不等于** — 分析数量不匹配

### 示例标准

#### 当 5 分钟内未收到分析数据时发出告警

- **时间窗口**：300 秒
- **检查项**：分析数量
- **过滤类型**：等于
- **值**：0

## 设置要求

性能分析监控需要您的应用程序通过 OpenTelemetry 向 OneUptime 发送持续分析数据。设置说明请参见 [OpenTelemetry](/docs/telemetry/open-telemetry) 文档。
