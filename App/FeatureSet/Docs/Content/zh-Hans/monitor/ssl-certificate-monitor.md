# SSL 证书监控器

SSL 证书监控允许您监控网站和服务上 SSL/TLS 证书的有效性和过期情况。OneUptime 定期检查您的证书，并在证书过期之前或检测到任何问题时提醒您。

## 概述

SSL 证书监控器连接到您的 HTTPS 端点并检查 SSL/TLS 证书。这使您能够：

- 监控证书过期日期
- 检测已过期或即将过期的证书
- 识别自签名证书
- 验证证书有效性
- 防止因证书过期导致的服务中断

## 创建 SSL 证书监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **SSL 证书** 作为监控器类型
4. 输入要检查的 HTTPS 端点的 URL
5. 根据需要配置监控标准

## 配置选项

### URL

输入您要监控其 SSL 证书的端点的完整 HTTPS URL（例如 `https://example.com` 或 `https://example.com:8443`）。

## 监控标准

您可以配置标准来判断证书状态何时处于在线、降级或离线，基于以下条件：

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| 是否在线 | 服务器是否可达 |
| 是否有效证书 | 证书是否有效（未过期、非自签名） |
| 是否为自签名证书 | 证书是否为自签名 |
| 证书是否已过期 | 证书是否已过期 |
| 是否为无效证书 | 证书是否无效 |
| 在 X 小时内过期 | 证书过期前的小时数 |
| 在 X 天内过期 | 证书过期前的天数 |
| 请求是否超时 | 连接是否超时 |

### 过滤类型

对于 **是否在线**、**是否有效证书**、**是否为自签名证书**、**证书是否已过期**、**是否为无效证书** 和 **请求是否超时**：

- **True** — 条件为真
- **False** — 条件为假

对于 **在 X 小时内过期** 和 **在 X 天内过期**：

- **大于** — 过期时间距离超过指定值
- **小于** — 过期时间距离少于指定值
- **大于或等于** — 过期时间距离等于或超过指定值
- **小于或等于** — 过期时间距离等于或少于指定值
- **等于** — 过期时间完全匹配
- **不等于** — 过期时间不匹配

### 示例标准

#### 当证书在 30 天内过期时标记为降级

- **检查项**：在 X 天内过期
- **过滤类型**：小于
- **值**：30

#### 当证书已过期时标记为离线

- **检查项**：证书是否已过期
- **过滤类型**：True

#### 当证书为自签名时发出告警

- **检查项**：是否为自签名证书
- **过滤类型**：True

#### 当证书无效时标记为离线

- **检查项**：是否为无效证书
- **过滤类型**：True

## 最佳实践

1. **设置多个阈值** — 在过期前 30 天使用降级状态，在过期前 7 天使用离线状态，给自己足够的续签时间
2. **监控所有端点** — 如果您有多个域名或子域名，为每个创建一个监控器
3. **包含非标准端口** — 不要忘记在非标准端口上运行 HTTPS 的服务
4. **续签后监控** — 证书续签后，验证监控器确认其有效
