# 워크플로 작성하기

워크플로를 만들려면 **Workflows → Create Workflow**를 열고 이름을 입력한 다음 **Builder** 탭을 클릭합니다. 빈 캔버스가 나타나며, 여기서 자동화를 빌드할 수 있습니다.

## 캔버스

Builder는 드래그 앤 드롭 캔버스입니다. 사이드 팔레트에서 블록을 추가하고, 선으로 서로 연결한 다음, 각 블록을 클릭하여 동작을 구성합니다. 변경 사항은 자동으로 저장됩니다. 저장이 완료되면 상단에 표시기가 나타납니다.

모든 워크플로는 시작 부분에 하나의 **트리거**로 시작합니다. 나머지는 모두 동작을 수행하는 **컴포넌트**입니다.

## 블록에 포함된 항목

| 필드 | 설명 |
| --- | --- |
| **Title** | 캔버스에 표시되는 이름입니다. 복잡한 워크플로를 더 쉽게 읽을 수 있도록 이름을 변경합니다. |
| **Settings** | 블록이 작동하는 데 필요한 정보입니다. URL, Slack 채널, 메시지 본문 등이 있습니다. 필수 필드는 별표로 표시됩니다. |
| **Input** | 왼쪽의 점입니다. 이전 블록의 선이 들어옵니다. |
| **Outputs** | 오른쪽의 점입니다. 다음 블록으로 선이 나갑니다. 많은 블록에는 **success**와 **error** 출력이 각각 있어서 두 경우를 모두 처리할 수 있습니다. |

## 블록 연결하기

한 블록의 출력 점에서 다음 블록의 입력 점으로 끌어다 놓습니다. 그려진 선이 다음에 실행될 항목을 결정합니다.

- **success**에서 연결하면 이전 블록이 정상 동작한 경우에만 다음 블록이 실행됩니다.
- **error**에서 연결하면 이전 블록이 실패한 경우에만 다음 블록이 실행됩니다.
- 출력을 연결하지 않으면 해당 경로는 거기서 끝납니다.

하나의 출력을 여러 블록에 연결할 수 있습니다. 해당 시점부터 모두 동시에 실행됩니다.

## 블록 구성하기

블록을 클릭하면 사이드에 설정이 열립니다. 각 설정에는 적절한 입력 방식(텍스트 필드, 드롭다운, 코드 편집기, 토글 등)이 제공됩니다.

대부분의 텍스트 필드는 변수를 받습니다. 이를 통해 한 블록에서 다음 블록으로 데이터가 흐릅니다. 구문에 대해서는 [변수](/docs/workflows/variables)를 참조하세요.

## 첫 워크플로

캔버스를 가장 빠르게 익히는 방법은 다음과 같습니다.

1. **Manual** 트리거를 캔버스에 끌어다 놓습니다.
2. 그 옆에 **Log** 컴포넌트(**Utils** 아래)를 끌어다 놓습니다. 트리거를 Log 컴포넌트에 연결합니다.
3. Log 블록의 메시지 필드에 `Hello from {{Manual.JSON.name}}`을 입력합니다.
4. 저장하고 워크플로를 켭니다.
5. **Run Manually**를 클릭하고 입력으로 `{ "name": "Ada" }`를 붙여넣은 후 제출합니다.
6. **Logs** 탭을 엽니다. 가장 최근 실행에 `Hello from Ada`가 표시됩니다.

이 과정 — 끌고, 연결하고, 구성하고, 실행하고, 로그를 확인 — 이 모든 워크플로를 빌드하는 방식입니다.

## 저장 및 켜기

캔버스는 작업하는 동안 자동으로 저장됩니다. 별도의 "게시" 단계는 없습니다.

하지만 워크플로는 Settings에서 **Enabled**가 켜져 있어야만 실제로 실행됩니다. 새 워크플로는 비활성화 상태로 시작됩니다. 이 스위치를 안전망으로 활용하세요. 빌드한 후 **Run Manually**로 테스트하고, 로그를 확인한 다음 켜시면 됩니다.

워크플로를 삭제하지 않고 일시 중지하려면 **Enabled**를 끄세요. 이미 진행 중인 실행은 완료되며, 새 실행은 시작되지 않습니다.

## 정리

- 블록을 끌어서 이동합니다. 레이아웃은 저장되므로 다음 사람이 동일한 배치를 볼 수 있습니다.
- 선을 오른쪽 클릭하여 삭제할 수 있습니다. 블록을 오른쪽 클릭하면 삭제하거나 복제할 수 있습니다.
- 넓은 워크플로는 실행 방향으로 읽힐 수 있도록 왼쪽에서 오른쪽으로 배치하세요.
