# 워크플로우 작성

**워크플로우 → Create Workflow**에서 워크플로우를 만들고, 이름과 선택적 설명을 부여한 뒤, **Builder** 탭을 열어 캔버스 위에 노드를 끌어다 놓기 시작합니다.

## 캔버스

Builder는 확대·이동이 가능한 그래프입니다. 구성 요소 팔레트에서 노드를 추가하고, 엣지로 연결하고, 사이드 패널에서 각 노드의 인자를 설정합니다. 헤더의 저장 표시기는 가장 최근 편집이 영구 저장되었는지 알려줍니다.

워크플로우는 항상 정확히 하나의 **트리거** 노드로 시작합니다. 트리거에는 입력 포트가 없습니다 — 실행이 여기서 시작됩니다. 그 아래로 이어지는 모든 것은 **구성 요소**입니다.

## 노드의 구조

모든 노드는 다음을 가집니다:

| 항목 | 용도 |
| --- | --- |
| **제목** | 캔버스에 표시되는 라벨. 기본값은 구성 요소 이름이며, 복잡한 워크플로우를 읽기 쉽게 만들기 위해 덮어쓸 수 있다. |
| **인자** | 구성 요소가 작업을 하기 위해 필요한 설정 — URL, Slack 채널, JavaScript 스니펫 등. 필수 인자는 별표로 표시된다. |
| **입력 포트** | 노드 왼쪽의 소켓으로 들어오는 엣지가 도착한다. 구성 요소는 `in`이라는 입력 포트 하나를 가진다; 트리거는 없다. |
| **출력 포트** | 오른쪽의 소켓으로 나가는 엣지가 시작된다. 구성 요소는 `success`, `error`, `yes`, `no` 같은 포트를 정의한다. |
| **반환값** | 노드가 생산하는 데이터 — 출력 포트의 페이로드. 하위 노드는 이를 `{{NodeId.fieldName}}`으로 참조한다. |

## 노드 연결

출력 포트에서 하위 노드의 입력 포트로 끌어 엣지를 만듭니다. `success`에서 시작된 엣지는 상위 노드가 성공했을 때만 그 분기를 실행합니다; `error`에서 시작된 엣지는 실패했을 때만 실행됩니다. 포트를 연결하지 않으면 그 분기는 단순히 끝납니다.

팬 아웃이 가능합니다: 하나의 출력 포트가 여러 하위 노드로 공급될 수 있으며, 그 시점부터 모두 병렬로 실행됩니다.

## 인자 설정

노드를 클릭하면 사이드 패널이 열립니다. 각 인자는 타입별 에디터를 가집니다:

- **Text / URL / Email / Number / Password** — 단일 라인 입력.
- **JSON** — 문법 강조와 검증 표시기가 있는 JSON 에디터.
- **JavaScript** — **Custom Code** 구성 요소가 사용하는 스니펫용 코드 에디터.
- **Markdown / HTML** — 이메일과 메시지 구성 요소를 위한 리치 텍스트 본문.
- **CronTab** — 스케줄 표현식(Schedule 트리거가 사용).
- **Boolean** — 토글.
- **Select / Query** — 고정된 값 집합이나 모델 스타일 쿼리를 받는 필드용 드롭다운.

어떤 텍스트 필드든 변수 보간을 허용합니다 — 규칙은 [변수](/docs/workflows/variables)를 참고하세요.

## 최소한의 첫 워크플로우

캔버스에 가장 빠르게 익숙해지는 방법:

1. **Manual** 트리거를 놓습니다.
2. **Log** 구성 요소(**Utils** 아래)를 놓습니다. 트리거의 출력 포트를 Log 구성 요소의 입력 포트에 연결합니다.
3. Log 구성 요소의 인자에 `Hello from {{Manual.JSON.name}}`을 입력합니다.
4. 저장하고 워크플로우를 활성화합니다.
5. **Run Manually**를 클릭하고 입력으로 `{ "name": "Ada" }`를 붙여 넣고 제출합니다.
6. **Logs** 탭을 엽니다. 가장 최근 실행은 Log 노드의 캡처된 출력 `Hello from Ada`를 보여줍니다.

이 왕복 — 드래그, 배선, 설정, 실행, 점검 — 이 모든 워크플로우 작성의 리듬입니다.

## 저장, 활성화, 프로덕션에서 테스트

워크플로우는 `Workflow.graph` 컬럼에 JSON 그래프로 저장됩니다. Builder는 편집 중 저장하며, 헤더의 저장 표시기는 가장 최근 변경이 서버에 도달했을 때를 보여줍니다. 별도의 "publish" 단계는 없습니다.

다만: 워크플로우는 **isEnabled**가 켜져 있을 때만 트리거를 발동합니다. 새 워크플로우는 비활성 상태로 출시됩니다. 이 플래그를 "프로덕션 준비 완료" 스위치로 취급하세요 — 만들고, **Run Manually**로 샘플 페이로드로 드라이 런을 실행하고, **Logs**를 보고, 그제서야 Enable을 켭니다.

워크플로우를 삭제하지 않고 일시 정지해야 한다면(예: 관련 없는 인시던트 중에), **Settings**에서 **isEnabled**를 끕니다. 진행 중인 실행은 계속됩니다; 새 실행은 시작되지 않습니다.

## 재정렬과 재구성

- 노드를 끌어 위치를 바꿉니다. 위치는 그래프에 저장되어 다음에 캔버스를 여는 사람도 같은 레이아웃을 봅니다.
- 엣지를 우클릭해 삭제합니다; 노드를 우클릭하면 삭제와 복제 옵션이 나옵니다.
- 넓은 워크플로우는 왼쪽에서 오른쪽으로 배치하여 실행 방향이 읽기 방향과 일치하도록 합니다.
