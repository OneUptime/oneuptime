# 워크플로 개요

워크플로를 사용하면 코드를 작성하지 않고도 OneUptime의 작업을 자동화할 수 있습니다. 캔버스에 몇 개의 블록을 끌어다 놓고 서로 연결하기만 하면, 인시던트가 발생하거나 스케줄이 실행되거나 다른 도구가 OneUptime에 데이터를 보낼 때마다 자동으로 동작하는 자동화가 완성됩니다.

워크플로는 프로젝트의 백그라운드 도우미라고 생각하시면 됩니다. 이벤트에 반응하고, 다른 도구와 소통하며, 사용자가 본업에 집중하는 동안 조용히 모든 것을 동기화 상태로 유지합니다.

## 워크플로로 할 수 있는 일

- **OneUptime을 다른 도구와 연결** — 인시던트를 Slack으로 보내고, Jira 티켓을 만들고, 사용 중인 스택의 webhook으로 게시합니다.
- **OneUptime에서 발생하는 일에 반응** — 심각한 인시던트가 생성되면 온콜 팀에 알리고 자동으로 티켓을 엽니다.
- **스케줄에 따라 작업 실행** — 5분마다, 매일 밤, 매주 월요일 아침 등 원하는 주기로 실행할 수 있습니다.
- **외부에서 데이터 받기** — 고유 URL을 통해 다른 시스템이 OneUptime으로 데이터를 푸시하도록 허용합니다.
- **공통 자동화 재사용** — 한 번 만들어 두면 다른 어떤 워크플로에서도 호출할 수 있습니다.

## 워크플로의 동작 방식

모든 워크플로는 세 가지 요소로 구성됩니다.

1. **트리거** — 워크플로를 시작하는 요소입니다. 수동 버튼, 스케줄, 들어오는 webhook 또는 OneUptime 내부 이벤트(예: 새 인시던트 생성)가 될 수 있습니다.
2. **하나 이상의 컴포넌트** — 워크플로가 수행하는 작업입니다. 메시지 보내기, HTTP 호출, 빠른 확인, 조건 분기 등이 있습니다.
3. **이들 간의 연결선** — 한 블록에서 다음 블록으로 선을 그어 실행 순서를 결정합니다.

이 모든 작업은 캔버스에서 시각적으로 구성합니다. 대부분의 워크플로는 코딩이 필요 없으며, 필요한 경우 JavaScript 코드 조각을 넣을 수도 있습니다.

## 핵심 용어

| 용어 | 의미 |
| --- | --- |
| **워크플로** | 자동화 전체 — 이름, 캔버스, 그리고 켜고 끌 수 있는 스위치로 구성됩니다. |
| **트리거** | 첫 번째 블록입니다. 워크플로가 언제 실행될지 결정합니다. 모든 워크플로에는 정확히 하나의 트리거가 있습니다. |
| **컴포넌트** | 액션 블록입니다. 메시지를 보내거나, 요청을 만들거나, 조건을 확인합니다. |
| **실행(Run)** | 워크플로의 한 번의 실행입니다. 타임스탬프와 각 블록의 출력이 함께 저장됩니다. |
| **전역 변수** | API 키와 같은 값을 한 번 저장하고 모든 워크플로에서 재사용합니다. |

## OneUptime에서 워크플로 찾기

왼쪽 내비게이션에서 **Workflows**를 엽니다. 거기에서 다음 항목을 확인할 수 있습니다.

- **Workflows** — 워크플로 목록입니다. 새 워크플로를 만들거나 기존 워크플로를 엽니다.
- **Builder 탭** — 워크플로를 설계하는 캔버스입니다.
- **Logs 탭** — 이 워크플로의 모든 실행 내역과 세부 정보를 확인합니다.
- **Settings 탭** — 이름, 설명, 소유자, 라벨, 활성화/비활성화를 설정합니다.
- **Global Variables** — 모든 워크플로에서 공유되는 값입니다.
- **Runs & Logs** — 프로젝트 내 모든 워크플로의 실행 이력입니다.

## 첫 워크플로 만들기

1. **만들기** — 워크플로의 이름과 짧은 설명을 입력합니다.
2. **트리거 선택** — 수동, 스케줄, webhook 또는 OneUptime 이벤트 중 선택합니다.
3. **컴포넌트 추가** — 액션을 캔버스에 끌어다 놓고 연결합니다.
4. **테스트** — **Run Manually**를 클릭하고 로그에서 결과를 확인합니다.
5. **켜기** — 준비가 되면 Settings에서 **Enabled** 스위치를 켭니다.

## 간단한 예시

심각한 인시던트가 생성될 때마다 Slack에 게시하고 싶다고 가정해 보겠습니다.

1. "Critical incidents to Slack"이라는 워크플로를 만듭니다.
2. **Incident → On Create** 트리거를 선택합니다.
3. **Conditions** 블록을 추가합니다. 인시던트 제목에 "Sev 1"이 포함되어 있는지 확인하도록 설정합니다.
4. **Yes** 분기에서 **Slack** 블록을 추가합니다. 채널을 선택하고 메시지를 작성합니다.
5. 워크플로를 켭니다.

다음에 누군가가 제목에 "Sev 1"이 포함된 인시던트를 열면 Slack에 알림이 표시됩니다.

## 워크플로와 OneUptime의 다른 기능 간의 관계

- **모니터**는 문제를 찾아내고, **인시던트**는 이를 기록하며, **워크플로**는 이에 반응합니다.
- **런북**은 사람이 따라 하는 단계별 가이드입니다. 워크플로는 무인 자동화입니다. 사람이 결정을 내려야 할 때는 런북을, 단계가 자동으로 진행될 때는 워크플로를 사용하세요.
- **워크스페이스 연결**(Slack, Teams)은 워크플로가 메시지를 보내는 위치입니다.
