# 포트 모니터

포트 모니터링을 통해 호스트의 특정 TCP 또는 UDP 포트의 가용성을 모니터링할 수 있습니다. OneUptime은 주기적으로 지정된 포트에 연결을 시도하고 열려 있고 응답하는지 확인합니다.

## 개요

포트 모니터는 특정 네트워크 포트가 연결을 받아들이는지 테스트합니다. 이를 통해 다음이 가능합니다:

- 특정 포트에서 서비스 가용성 모니터링
- 포트 응답 시간 추적
- 데이터베이스, 메일 서버 및 애플리케이션 서버와 같은 서비스가 실행 중인지 확인
- 사용자에게 영향을 미치기 전에 서비스 장애 감지

## 포트 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **포트**를 선택합니다
4. 호스트 이름 또는 IP 주소와 포트 번호를 입력합니다
5. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### 호스트 이름 또는 IP 주소

대상 호스트의 호스트 이름 또는 IP 주소를 입력합니다 (예: `example.com` 또는 `192.168.1.1`).

### 포트

모니터링할 포트 번호를 입력합니다 (1–65535). 일반적인 예:

| 포트 | 서비스 |
|------|---------|
| 22 | SSH |
| 25 | SMTP |
| 80 | HTTP |
| 443 | HTTPS |
| 3306 | MySQL |
| 5432 | PostgreSQL |
| 6379 | Redis |
| 27017 | MongoDB |

## 모니터링 기준

다음을 기반으로 포트가 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다:

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| 온라인 여부 | 포트가 열려 있고 연결을 받아들이는지 여부 |
| 응답 시간 (ms) | 밀리초 단위의 연결 설정 시간 |
| 요청 타임아웃 여부 | 연결 시도가 타임아웃되었는지 여부 |

### 필터 유형

**온라인 여부** 및 **요청 타임아웃 여부**:

- **True** — 조건이 참임
- **False** — 조건이 거짓임

**응답 시간**:

- **초과** — 응답 시간이 임계값을 초과함
- **미만** — 응답 시간이 임계값보다 낮음
- **이상** — 응답 시간이 임계값 이상임
- **이하** — 응답 시간이 임계값 이하임
- **동일** — 응답 시간이 정확히 일치함
- **다름** — 응답 시간이 일치하지 않음
- **시간에 따른 평가** — 시간 창에 걸쳐 집계 (평균, 합계, 최대, 최소, 모든 값, 임의 값)를 사용하여 평가

### 기준 예시

#### 포트가 닫혀 있는 경우 오프라인으로 표시

- **확인 대상**: 온라인 여부
- **필터 유형**: False

#### 연결 시간이 500ms를 초과하는 경우 알림

- **확인 대상**: 응답 시간 (ms)
- **필터 유형**: 초과
- **값**: 500

#### 연결이 느린 경우 저하로 표시

- **확인 대상**: 응답 시간 (ms)
- **필터 유형**: 초과
- **값**: 200
