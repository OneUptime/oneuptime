# 로그 모니터

로그 모니터링을 통해 애플리케이션 로그를 모니터링하고 로그 패턴, 카운트 및 심각도 수준을 기반으로 알림을 트리거할 수 있습니다. OneUptime은 텔레메트리 서비스의 로그를 평가하고 구성된 기준에 따라 확인합니다.

## 개요

로그 모니터는 시간 창에 걸쳐 특정 필터와 일치하는 로그를 검색하고 카운트합니다. 이를 통해 다음이 가능합니다:

- 오류 로그 급증에 대한 알림
- 특정 로그 패턴 또는 메시지 모니터링
- 심각도 수준별 로그 볼륨 추적
- 서비스, 속성 및 내용별로 로그 필터링
- 로그 패턴에서 애플리케이션 문제 감지

## 로그 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **로그**를 선택합니다
4. 모니터링할 텔레메트리 서비스를 선택합니다
5. 필요에 따라 로그 필터 및 기준을 구성합니다

## 구성 옵션

### 텔레메트리 서비스

로그를 모니터링할 하나 이상의 서비스를 선택합니다. 서비스는 OpenTelemetry를 통해 OneUptime으로 로그를 전송해야 합니다.

### 로그 필터

| 필터 | 설명 | 필수 여부 |
|--------|-------------|----------|
| 심각도 수준 | 로그 심각도로 필터링 (ERROR, WARN, INFO, DEBUG 등) | 아니요 |
| 본문 | 로그 메시지 본문 내 텍스트 검색 | 아니요 |
| 속성 | 커스텀 로그 속성에 대해 필터링할 키-값 쌍 | 아니요 |
| 시간 창 | 로그를 검색할 범위 (초 단위, 기본값: 60) | 아니요 |

### 심각도 수준

하나 이상의 심각도 수준으로 로그를 필터링합니다:

- **FATAL** / **EMERGENCY** / **CRITICAL**
- **ERROR**
- **WARN** / **WARNING**
- **INFO** / **INFORMATIONAL**
- **DEBUG**
- **TRACE**
- **UNSPECIFIED**

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| 로그 카운트 | 시간 창에서 필터와 일치하는 로그 수 |

### 필터 유형

- **초과** — 로그 카운트가 임계값을 초과함
- **미만** — 로그 카운트가 임계값보다 낮음
- **이상** — 로그 카운트가 임계값 이상임
- **이하** — 로그 카운트가 임계값 이하임
- **동일** — 로그 카운트가 정확히 일치함
- **다름** — 로그 카운트가 일치하지 않음

### 기준 예시

#### 60초 내에 100개 이상의 오류 로그가 발생하는 경우 알림

- **심각도 수준**: ERROR
- **시간 창**: 60초
- **확인 대상**: 로그 카운트
- **필터 유형**: 초과
- **값**: 100

#### 치명적 로그가 나타나는 경우 알림

- **심각도 수준**: FATAL
- **시간 창**: 60초
- **확인 대상**: 로그 카운트
- **필터 유형**: 초과
- **값**: 0

#### 특정 오류 메시지가 포함된 로그 모니터링

- **본문**: `database connection timeout`
- **시간 창**: 300초
- **확인 대상**: 로그 카운트
- **필터 유형**: 초과
- **값**: 5

## 설정 요구 사항

로그 모니터링을 위해서는 애플리케이션이 OpenTelemetry를 통해 OneUptime으로 로그를 전송해야 합니다. 설정 지침은 [OpenTelemetry](/docs/telemetry/open-telemetry) 문서를 참조하십시오.
