# SNMP 모니터

SNMP (Simple Network Management Protocol) 모니터링을 통해 SNMP OID (Object Identifier)를 쿼리하여 스위치, 라우터, 방화벽 및 기타 네트워크 인프라와 같은 네트워크 장치를 모니터링할 수 있습니다.

## 개요

SNMP 모니터는 OID를 사용하여 관리 정보를 위한 네트워크 장치를 쿼리합니다. 이를 통해 다음이 가능합니다:

- 장치 가용성 및 상태 모니터링
- 인터페이스 통계 추적 (트래픽, 오류, 상태)
- 시스템 메트릭 모니터링 (CPU, 메모리, 가동 시간)
- 커스텀 공급업체별 OID 확인
- OID 값을 기반으로 알림 설정

## SNMP 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **SNMP**를 선택합니다
4. 아래에 설명된 SNMP 설정을 구성합니다

## 구성 옵션

### 기본 설정

| 필드           | 설명                                 | 필수 여부 |
| -------------- | ------------------------------------ | --------- |
| SNMP 버전      | 프로토콜 버전: v1, v2c 또는 v3       | 예        |
| 호스트 이름/IP | SNMP 장치의 호스트 이름 또는 IP 주소 | 예        |
| 포트           | SNMP 포트 (기본값: 161)              | 예        |

### 인증

#### SNMP v1/v2c

SNMP v1 및 v2c의 경우 커뮤니티 문자열만 제공하면 됩니다:

| 필드            | 설명                                | 필수 여부 |
| --------------- | ----------------------------------- | --------- |
| 커뮤니티 문자열 | SNMP 커뮤니티 문자열 (예: "public") | 예        |

#### SNMP v3

SNMPv3는 인증 및 암호화로 향상된 보안을 제공합니다:

| 필드                    | 설명                                   | 필수 여부                       |
| ----------------------- | -------------------------------------- | ------------------------------- |
| 보안 수준               | noAuthNoPriv, authNoPriv 또는 authPriv | 예                              |
| 사용자 이름             | SNMPv3 사용자 이름                     | 예                              |
| 인증 프로토콜           | MD5, SHA, SHA256 또는 SHA512           | authNoPriv 또는 authPriv인 경우 |
| 인증 키                 | 인증 비밀번호                          | authNoPriv 또는 authPriv인 경우 |
| 개인 정보 보호 프로토콜 | DES, AES 또는 AES256                   | authPriv인 경우                 |
| 개인 정보 보호 키       | 개인 정보 보호/암호화 비밀번호         | authPriv인 경우                 |

### 모니터링할 OID

장치에서 쿼리할 OID를 추가합니다. 각 OID에 대해 다음을 지정할 수 있습니다:

| 필드 | 설명                             | 필수 여부 |
| ---- | -------------------------------- | --------- |
| OID  | 숫자 OID (예: 1.3.6.1.2.1.1.1.0) | 예        |
| 이름 | OID의 친숙한 이름 (예: sysDescr) | 아니요    |
| 설명 | 이 OID가 나타내는 것의 설명      | 아니요    |

### 일반적인 OID 템플릿

OneUptime은 일반적으로 모니터링되는 OID에 대한 템플릿을 제공합니다:

#### 시스템 MIB

| OID               | 이름        | 설명                       |
| ----------------- | ----------- | -------------------------- |
| 1.3.6.1.2.1.1.1.0 | sysDescr    | 시스템 설명                |
| 1.3.6.1.2.1.1.3.0 | sysUpTime   | 시스템 가동 시간 (틱 단위) |
| 1.3.6.1.2.1.1.5.0 | sysName     | 시스템 이름                |
| 1.3.6.1.2.1.1.6.0 | sysLocation | 시스템 위치                |
| 1.3.6.1.2.1.1.4.0 | sysContact  | 시스템 연락처              |

#### 인터페이스 MIB

| OID                    | 이름         | 설명                                         |
| ---------------------- | ------------ | -------------------------------------------- |
| 1.3.6.1.2.1.2.1.0      | ifNumber     | 네트워크 인터페이스 수                       |
| 1.3.6.1.2.1.2.2.1.8.X  | ifOperStatus | 인터페이스 운영 상태 (X = 인터페이스 인덱스) |
| 1.3.6.1.2.1.2.2.1.10.X | ifInOctets   | 입력 바이트 (X = 인터페이스 인덱스)          |
| 1.3.6.1.2.1.2.2.1.16.X | ifOutOctets  | 출력 바이트 (X = 인터페이스 인덱스)          |

#### 호스트 리소스 MIB

| OID                      | 이름              | 설명                           |
| ------------------------ | ----------------- | ------------------------------ |
| 1.3.6.1.2.1.25.1.1.0     | hrSystemUptime    | 호스트 시스템 가동 시간        |
| 1.3.6.1.2.1.25.1.5.0     | hrSystemNumUsers  | 사용자 수                      |
| 1.3.6.1.2.1.25.1.6.0     | hrSystemProcesses | 실행 중인 프로세스 수          |
| 1.3.6.1.2.1.25.3.3.1.2.X | hrProcessorLoad   | CPU 부하 (X = 프로세서 인덱스) |

### 고급 설정

| 필드     | 설명                      | 기본값 |
| -------- | ------------------------- | ------ |
| 타임아웃 | 응답을 기다리는 시간 (ms) | 5000   |
| 재시도   | 실패 시 재시도 횟수       | 3      |

## 모니터링 기준

SNMP 응답을 확인하고 알림이나 인시던트를 트리거하기 위한 기준을 설정할 수 있습니다.

### 사용 가능한 확인 유형

| 확인 유형             | 설명                                     |
| --------------------- | ---------------------------------------- |
| SNMP 장치 온라인 여부 | 장치가 SNMP 쿼리에 응답하는지 확인       |
| SNMP 응답 시간        | 밀리초 단위의 쿼리 응답 시간 확인        |
| SNMP OID 값           | 특정 OID에서 반환된 값 확인              |
| SNMP OID 존재 여부    | OID가 값을 반환하는지 확인 (null이 아닌) |

### 기준 예시

#### 장치가 온라인인지 확인

- **확인 대상**: SNMP 장치 온라인 여부
- **필터 유형**: True

#### 응답 시간이 임계값을 초과하는 경우 알림

- **확인 대상**: SNMP 응답 시간 (ms)
- **필터 유형**: 초과
- **값**: 1000

#### 인터페이스 상태 확인

- **확인 대상**: SNMP OID 값
- **OID**: 1.3.6.1.2.1.2.2.1.8.1
- **필터 유형**: 동일
- **값**: 1 (1 = up, 2 = down)

#### CPU 부하 임계값 확인

- **확인 대상**: SNMP OID 값
- **OID**: 1.3.6.1.2.1.25.3.3.1.2.1
- **필터 유형**: 초과
- **값**: 80

## 모니터 시크릿 사용

보안을 위해 커뮤니티 문자열 및 SNMPv3 자격 증명과 같은 민감한 정보를 시크릿으로 저장할 수 있습니다.

### 시크릿 추가

1. **프로젝트 설정** -> **모니터 시크릿** -> **모니터 시크릿 생성**으로 이동합니다
2. 시크릿을 추가합니다 (예: 커뮤니티 문자열 또는 SNMPv3 비밀번호)
3. 이 시크릿에 액세스해야 하는 SNMP 모니터를 선택합니다

### SNMP 구성에서 시크릿 사용

민감한 필드에서 `{{monitorSecrets.SECRET_NAME}}` 구문을 사용합니다:

- **커뮤니티 문자열**: `{{monitorSecrets.SnmpCommunity}}`
- **SNMPv3 인증 키**: `{{monitorSecrets.SnmpAuthKey}}`
- **SNMPv3 개인 정보 보호 키**: `{{monitorSecrets.SnmpPrivKey}}`

## 알림을 위한 템플릿 변수

인시던트 또는 알림 템플릿을 생성할 때 다음 변수를 사용할 수 있습니다:

| 변수                   | 설명                                         |
| ---------------------- | -------------------------------------------- |
| `{{isOnline}}`         | 장치가 온라인인지 여부 (true/false)          |
| `{{responseTimeInMs}}` | 밀리초 단위의 쿼리 응답 시간                 |
| `{{failureCause}}`     | 쿼리가 실패한 경우 오류 메시지               |
| `{{oidResponses}}`     | OID 응답 객체 배열                           |
| `{{OID_NAME}}`         | 이름으로 특정 OID의 값 (예: `{{sysUpTime}}`) |

## 문제 해결

### 일반적인 문제

#### 장치가 응답하지 않는 경우

- 장치 IP/호스트 이름이 올바른지 확인합니다
- 장치에서 SNMP가 활성화되어 있는지 확인합니다
- 방화벽 규칙이 UDP 포트 161을 허용하는지 확인합니다
- 커뮤니티 문자열이 올바른지 확인합니다

#### 인증 실패 (v3)

- 사용자 이름, 인증 프로토콜 및 인증 키를 확인합니다
- 보안 수준이 장치 구성과 일치하는지 확인합니다
- authPriv 수준의 경우 개인 정보 보호 프로토콜과 키가 올바른지 확인합니다

#### OID를 찾을 수 없는 경우

- OID가 장치에서 지원되는지 확인합니다
- OID에 로드할 특정 MIB가 필요한지 확인합니다
- snmpget/snmpwalk 도구를 사용하여 OID를 직접 쿼리해 봅니다

### SNMP 연결 테스트

모니터링을 설정하기 전에 명령줄 도구를 사용하여 SNMP 연결을 테스트할 수 있습니다:

```bash
# SNMP v2c
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# SNMP v3 (authPriv)
snmpget -v3 -u username -l authPriv -a SHA -A authpassword -x AES -X privpassword 192.168.1.1 1.3.6.1.2.1.1.1.0
```

## 모범 사례

1. **가능하면 SNMPv3 사용** - 더 나은 보안을 위한 인증 및 암호화를 제공합니다
2. **자격 증명을 시크릿으로 저장** - 커뮤니티 문자열이나 비밀번호를 하드코딩하지 마십시오
3. **필수 OID만 모니터링** - 네트워크 오버헤드를 줄이기 위해 필요한 것만 쿼리합니다
4. **적절한 타임아웃 설정** - 네트워크 장치의 응답 시간이 다를 수 있습니다
5. **설명적인 OID 이름 사용** - 알림 메시지를 이해하기 쉽게 만듭니다
6. **배포 전 테스트** - 모니터를 생성하기 전에 SNMP 연결을 확인합니다
