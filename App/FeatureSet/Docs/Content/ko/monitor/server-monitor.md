# 서버 / VM 모니터

서버 및 VM 모니터링을 통해 시스템 메트릭을 OneUptime에 보고하는 경량 에이전트를 설치하여 서버, 가상 머신 및 기타 인프라의 상태와 성능을 모니터링할 수 있습니다.

## 개요

서버 모니터는 서버에 설치된 인프라 에이전트를 사용하여 시스템 메트릭을 수집하고 보고합니다. 이를 통해 다음이 가능합니다:

- 서버 업타임 및 가용성 모니터링
- CPU, 메모리 및 디스크 사용량 추적
- 실행 중인 프로세스 모니터링
- 리소스 사용 임계값을 기반으로 알림 설정
- 서비스에 영향을 미치기 전에 인프라 문제 감지

## 서버 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **서버 / VM**을 선택합니다
4. 이 모니터에 대한 **비밀 키**가 생성됩니다 — 에이전트를 구성하는 데 필요합니다
5. 서버에서 에이전트를 설정하기 위한 설치 지침을 따릅니다

## 인프라 에이전트 설치

OneUptime 인프라 에이전트는 시스템 메트릭을 수집하고 30초마다 OneUptime으로 전송하는 경량 Go 기반 데몬입니다. Linux, macOS 및 Windows를 지원합니다.

### Linux / macOS

```bash
# 에이전트 설치
curl -sSL https://oneuptime.com/docs/static/scripts/infrastructure-agent/install.sh | sudo bash

# 에이전트 구성
sudo oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com

# 에이전트 시작
sudo oneuptime-infrastructure-agent start
```

`YOUR_SECRET_KEY`를 모니터 설정에 표시된 비밀 키로 교체하고, 자체 호스팅하는 경우 `https://oneuptime.com`을 OneUptime 인스턴스 URL로 교체합니다.

### Windows

1. [GitHub Releases](https://github.com/OneUptime/oneuptime/releases/latest)에서 최신 에이전트를 다운로드합니다
   - x64 시스템의 경우 `oneuptime-infrastructure-agent_windows_amd64.zip`
   - ARM64 시스템의 경우 `oneuptime-infrastructure-agent_windows_arm64.zip`
2. zip 파일을 압축 해제합니다
3. 관리자 권한으로 명령 프롬프트를 열고 실행합니다:

```bash
# 에이전트 구성
oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com

# 에이전트 시작
oneuptime-infrastructure-agent start
```

### 프록시 지원

서버가 프록시를 통해 인터넷에 연결하는 경우 프록시를 사용하도록 에이전트를 구성할 수 있습니다:

```bash
sudo oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com --proxy-url=http://proxy.example.com:8080
```

## 에이전트 명령

인프라 에이전트는 다음 명령을 지원합니다:

| 명령 | 설명 |
|---------|-------------|
| `configure` | 비밀 키와 OneUptime URL로 에이전트 구성 |
| `start` | 에이전트 서비스 시작 |
| `stop` | 에이전트 서비스 중지 |
| `restart` | 에이전트 서비스 재시작 |
| `status` | 현재 서비스 상태 표시 |
| `logs` | 에이전트 로그 보기 (줄 수에는 `-n`, 팔로우에는 `-f` 사용) |
| `uninstall` | 에이전트 서비스 제거 |

## 수집된 메트릭

에이전트는 서버에서 다음 메트릭을 수집합니다:

### CPU

- **CPU 사용률** — 백분율로 표시된 전체 CPU 사용률
- **CPU 코어** — CPU 코어 수

### 메모리

- **총 메모리** — 사용 가능한 총 메모리
- **사용된 메모리** — 현재 사용 중인 메모리
- **여유 메모리** — 사용 가능한 여유 메모리
- **메모리 사용률** — 백분율로 표시된 메모리 사용률

### 디스크

마운트된 각 디스크/볼륨에 대해:

- **총 디스크 공간** — 디스크의 총 용량
- **사용된 디스크 공간** — 현재 사용 중인 공간
- **여유 디스크 공간** — 사용 가능한 여유 공간
- **디스크 사용률** — 백분율로 표시된 디스크 사용률
- **디스크 경로** — 디스크의 마운트 경로

### 프로세스

- **프로세스 이름** — 실행 중인 프로세스의 이름
- **프로세스 ID (PID)** — 프로세스 식별자
- **프로세스 명령** — 프로세스를 시작하는 데 사용된 전체 명령

## 모니터링 기준

서버가 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다.

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| 온라인 여부 | 서버 에이전트가 보고하는지 여부 (하트비트 기반) |
| CPU 사용률 | 현재 CPU 사용률 백분율 |
| 메모리 사용률 | 현재 메모리 사용률 백분율 |
| 디스크 사용률 | 현재 디스크 사용률 백분율 (특정 디스크 경로용) |
| 서버 프로세스 이름 | 특정 이름의 프로세스가 실행 중인지 확인 |
| 서버 프로세스 명령 | 특정 명령의 프로세스가 실행 중인지 확인 |
| 서버 프로세스 PID | 특정 PID의 프로세스가 실행 중인지 확인 |

### 필터 유형

수치 메트릭 (CPU, 메모리, 디스크)의 경우:

- **초과** — 값이 임계값을 초과함
- **미만** — 값이 임계값보다 낮음
- **이상** — 값이 임계값 이상임
- **이하** — 값이 임계값 이하임
- **시간에 따른 평가** — 시간 창에 걸쳐 집계 (평균, 합계, 최대, 최소, 모든 값, 임의 값)를 사용하여 평가

프로세스 확인의 경우:

- **실행 중** — 프로세스가 현재 실행 중임
- **실행 중이지 않음** — 프로세스가 실행 중이지 않음

### 기준 예시

#### 에이전트가 보고를 중지하면 서버를 오프라인으로 표시

- **확인 대상**: 온라인 여부
- **필터 유형**: False

#### CPU 사용량이 90%를 초과하면 알림

- **확인 대상**: CPU 사용률
- **필터 유형**: 초과
- **값**: 90

#### 디스크 사용량이 85%를 초과하면 알림

- **확인 대상**: 디스크 사용률
- **디스크 경로**: `/`
- **필터 유형**: 초과
- **값**: 85

#### 메모리 사용량이 80%를 초과하면 알림

- **확인 대상**: 메모리 사용률
- **필터 유형**: 초과
- **값**: 80

#### 중요 프로세스가 중지되면 알림

- **확인 대상**: 서버 프로세스 이름
- **필터 유형**: 실행 중이지 않음
- **값**: `nginx`

## 문제 해결

### 에이전트가 보고하지 않는 경우

- 에이전트가 실행 중인지 확인합니다: `sudo oneuptime-infrastructure-agent status`
- 에이전트 로그를 확인합니다: `sudo oneuptime-infrastructure-agent logs -n 50`
- 비밀 키가 올바른지 확인합니다
- 서버가 OneUptime 인스턴스 URL에 도달할 수 있는지 확인합니다
- 방화벽 규칙이 아웃바운드 HTTPS 연결을 허용하는지 확인합니다

### 에이전트의 높은 리소스 사용량

에이전트는 경량으로 설계되었습니다. 높은 리소스 사용량이 발견되면:
- 에이전트를 재시작합니다: `sudo oneuptime-infrastructure-agent restart`
- 오류에 대한 에이전트 로그를 확인합니다

### 프록시 문제

- 프록시 URL과 포트가 올바른지 확인합니다
- 프록시가 OneUptime 인스턴스로의 연결을 허용하는지 확인합니다
- 다음으로 재구성합니다: `sudo oneuptime-infrastructure-agent configure --proxy-url=http://proxy:port --secret-key=YOUR_KEY --oneuptime-url=YOUR_URL`

## 모범 사례

1. **의미 있는 임계값 설정** — 서버의 정상 운영 범위와 일치하는 저하 및 오프라인 기준 구성
2. **중요 프로세스 모니터링** — 프로세스 모니터링을 사용하여 웹 서버 및 데이터베이스와 같은 필수 서비스가 항상 실행되도록 확인
3. **디스크 사용량 사전 모니터링** — 디스크 공간 문제는 애플리케이션 장애로 이어질 수 있으므로 디스크가 가득 차기 전에 알림 설정
4. **"시간에 따른 평가" 사용** — 잠시 급증할 수 있는 CPU와 같은 메트릭의 경우 오경보를 방지하기 위해 시간 기반 집계 사용
5. **에이전트를 최신 상태로 유지** — 최신 개선 사항 및 수정 사항을 받기 위해 인프라 에이전트를 주기적으로 업데이트
