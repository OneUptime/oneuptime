# Ping 모니터

Ping 모니터링을 통해 호스트 또는 IP 주소의 가용성과 응답성을 모니터링할 수 있습니다. OneUptime은 주기적으로 대상에 ping 요청을 전송하고 올바르게 응답하는지 확인합니다.

## 개요

Ping 모니터는 ICMP ping 요청을 호스트에 전송하여 기본 네트워크 연결을 테스트합니다. 이를 통해 다음이 가능합니다:

- 호스트 업타임 및 가용성 모니터링
- 네트워크 레이턴시 및 응답 시간 추적
- 서비스에 영향을 미치기 전에 연결 문제 감지
- 서버 및 네트워크 장치에 도달 가능한지 확인

## Ping 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **Ping**을 선택합니다
4. 모니터링할 호스트 이름 또는 IP 주소를 입력합니다
5. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### Ping 호스트 이름 또는 IP 주소

모니터링할 대상의 호스트 이름 또는 IP 주소를 입력합니다 (예: `example.com` 또는 `192.168.1.1`). 호스트 이름과 IP 주소 모두 허용됩니다.

## 모니터링 기준

다음을 기반으로 호스트가 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다:

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| 온라인 여부 | 호스트가 ping 요청에 응답하는지 여부 |
| 응답 시간 (ms) | 밀리초 단위의 ping 요청 왕복 시간 |
| 요청 타임아웃 여부 | ping 요청이 타임아웃되었는지 여부 |

### 필터 유형

**온라인 여부** 및 **요청 타임아웃 여부**:

- **True** — 조건이 참임
- **False** — 조건이 거짓임

**응답 시간**:

- **초과** — 응답 시간이 임계값을 초과함
- **미만** — 응답 시간이 임계값보다 낮음
- **이상** — 응답 시간이 임계값 이상임
- **이하** — 응답 시간이 임계값 이하임
- **동일** — 응답 시간이 정확히 일치함
- **다름** — 응답 시간이 일치하지 않음
- **시간에 따른 평가** — 시간 창에 걸쳐 집계 (평균, 합계, 최대, 최소, 모든 값, 임의 값)를 사용하여 평가

### 기준 예시

#### 호스트에 도달할 수 없는 경우 오프라인으로 표시

- **확인 대상**: 온라인 여부
- **필터 유형**: False

#### 응답 시간이 200ms를 초과하는 경우 알림

- **확인 대상**: 응답 시간 (ms)
- **필터 유형**: 초과
- **값**: 200
