# DNS 모니터

DNS 모니터링을 통해 도메인의 DNS 확인 상태와 정확성을 모니터링할 수 있습니다. OneUptime은 주기적으로 DNS 레코드를 쿼리하고 구성된 기준에 따라 응답을 검증합니다.

## 개요

DNS 모니터는 특정 레코드 유형에 대해 DNS 서버를 쿼리하고 결과를 평가합니다. 이를 통해 다음이 가능합니다:

- DNS 서비스 가용성 모니터링
- DNS 레코드가 올바른 값을 반환하는지 확인
- DNS 확인 응답 시간 추적
- DNSSEC 구성 유효성 검사
- DNS 전파 문제 또는 하이재킹 감지

## DNS 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **DNS**를 선택합니다
4. 쿼리할 도메인 이름과 레코드 유형을 입력합니다
5. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### 기본 설정

| 필드        | 설명                                                                  | 필수 여부 |
| ----------- | --------------------------------------------------------------------- | --------- |
| 도메인 이름 | 쿼리할 도메인 (예: `example.com`)                                     | 예        |
| 레코드 유형 | 쿼리할 DNS 레코드 유형                                                | 예        |
| DNS 서버    | 사용할 커스텀 DNS 서버 (예: `8.8.8.8`). 시스템 기본값으로 비워 둡니다 | 아니요    |

### 지원되는 레코드 유형

| 레코드 유형 | 설명                         |
| ----------- | ---------------------------- |
| A           | IPv4 주소 레코드             |
| AAAA        | IPv6 주소 레코드             |
| CNAME       | 정규 이름(별칭) 레코드       |
| MX          | 메일 교환 레코드             |
| NS          | 네임서버 레코드              |
| TXT         | 텍스트 레코드 (SPF, DKIM 등) |
| SOA         | 권한 시작 레코드             |
| PTR         | 포인터 레코드 (역방향 DNS)   |
| SRV         | 서비스 로케이터 레코드       |
| CAA         | 인증 기관 권한 부여 레코드   |

### 고급 설정

| 필드          | 설명                 | 기본값 |
| ------------- | -------------------- | ------ |
| 포트          | DNS 포트 번호        | 53     |
| 타임아웃 (ms) | 응답을 기다리는 시간 | 5000   |
| 재시도        | 실패 시 재시도 횟수  | 3      |

## 모니터링 기준

다음을 기반으로 DNS가 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다:

### 사용 가능한 확인 유형

| 확인 유형            | 설명                                     |
| -------------------- | ---------------------------------------- |
| DNS 온라인 여부      | DNS 서버가 쿼리에 응답하는지 여부        |
| DNS 응답 시간 (ms)   | 밀리초 단위의 쿼리 응답 시간             |
| DNS 레코드 존재 여부 | 쿼리에 대한 DNS 레코드가 존재하는지 여부 |
| DNS 레코드 값        | DNS 레코드에서 반환된 값                 |
| DNSSEC 유효 여부     | DNSSEC 유효성 검사가 통과하는지 여부     |

### 필터 유형

**DNS 온라인 여부**, **DNS 레코드 존재 여부** 및 **DNSSEC 유효 여부**:

- **True** — 조건이 참임
- **False** — 조건이 거짓임

**DNS 응답 시간**:

- **초과**, **미만**, **이상**, **이하**, **동일**, **다름**

**DNS 레코드 값**:

- **포함** — 레코드 값에 지정된 텍스트가 포함됨
- **포함하지 않음** — 레코드 값에 지정된 텍스트가 포함되지 않음
- **시작** — 레코드 값이 지정된 텍스트로 시작함
- **끝** — 레코드 값이 지정된 텍스트로 끝남
- **동일** — 레코드 값이 정확히 일치함
- **다름** — 레코드 값이 일치하지 않음

### 기준 예시

#### DNS가 확인되고 있는지 확인

- **확인 대상**: DNS 온라인 여부
- **필터 유형**: True

#### A 레코드가 올바른 IP를 가리키는지 확인

- **확인 대상**: DNS 레코드 값
- **필터 유형**: 동일
- **값**: `93.184.216.34`

#### DNS 응답이 느린 경우 알림

- **확인 대상**: DNS 응답 시간 (ms)
- **필터 유형**: 초과
- **값**: 500

#### DNSSEC 유효성 확인

- **확인 대상**: DNSSEC 유효 여부
- **필터 유형**: True
