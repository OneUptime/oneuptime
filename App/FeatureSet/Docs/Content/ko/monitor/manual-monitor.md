# 수동 모니터

수동 모니터링을 통해 상태가 완전히 수동으로 또는 API를 통해 관리되는 모니터를 생성할 수 있습니다. OneUptime은 자동화된 확인을 수행하지 않으며 모니터 상태를 직접 제어합니다.

## 개요

수동 모니터는 직접 업데이트하는 플레이스홀더입니다. 다음의 경우에 유용합니다:

- 모니터 상태를 OneUptime API를 통해 업데이트하는 외부 모니터링 도구와의 통합
- 자동으로 모니터링할 수 없는 서비스나 시스템 추적
- 자동화된 상태 확인 없이 구성 요소에 대한 인시던트 관리
- 수동으로 추적하는 상태의 타사 종속성 표현

## 수동 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **수동**을 선택합니다
4. 모니터의 이름과 설명을 입력합니다

## 작동 방식

수동 모니터에는 모니터링 간격, 프로브 또는 자동화된 기준 평가가 없습니다. 모니터 상태는 변경할 때까지 설정한 대로 유지됩니다.

### 상태 업데이트

수동 모니터의 상태를 두 가지 방법으로 업데이트할 수 있습니다:

- **대시보드** — OneUptime 대시보드에서 직접 모니터 상태 변경
- **API** — OneUptime API를 사용하여 프로그래밍 방식으로 모니터 상태 업데이트

### 인시던트 및 알림

수동 모니터에 대해 다른 모니터 유형과 마찬가지로 인시던트와 알림을 생성할 수 있습니다. 이를 통해 다음이 가능합니다:

- 외부적으로 모니터링되는 서비스의 다운타임 추적
- 문제가 보고되면 인시던트를 수동으로 생성
- 상태 페이지에서 사용자에게 상태를 알리기 위해 수동 모니터 사용

## 수동 모니터를 사용하는 경우

| 사용 사례                | 설명                                                               |
| ------------------------ | ------------------------------------------------------------------ |
| 타사 서비스              | 직접 모니터링할 수 없지만 의존하는 외부 서비스의 상태 추적         |
| 물리적 인프라            | 네트워크 모니터링 없이 하드웨어나 물리적 시스템 표현               |
| 비즈니스 프로세스        | 서비스 상태에 영향을 미치는 비기술적 프로세스 추적                 |
| API 기반 상태            | 외부 도구가 OneUptime API를 통해 모니터 상태를 업데이트하도록 허용 |
| 상태 페이지 플레이스홀더 | OneUptime 외부에서 관리되는 구성 요소를 상태 페이지에 표시         |
