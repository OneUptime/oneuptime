# 트레이스 모니터

트레이스 모니터링을 통해 애플리케이션의 분산 트레이스를 모니터링하고 스팬 패턴, 카운트 및 상태를 기반으로 알림을 트리거할 수 있습니다. OneUptime은 시간 창에 걸쳐 텔레메트리 서비스의 트레이스 데이터를 평가합니다.

## 개요

트레이스 모니터는 특정 필터와 일치하는 스팬을 검색하고 카운트합니다. 이를 통해 다음이 가능합니다:

- 서비스의 오류 스팬 급증에 대한 알림
- 특정 작업 및 엔드포인트 모니터링
- 스팬 볼륨 및 패턴 추적
- 스팬 상태, 이름 및 커스텀 속성으로 필터링
- 트레이스 데이터에서 성능 및 안정성 문제 감지

## 트레이스 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **트레이스**를 선택합니다
4. 모니터링할 텔레메트리 서비스를 선택합니다
5. 필요에 따라 스팬 필터 및 기준을 구성합니다

## 구성 옵션

### 텔레메트리 서비스

트레이스를 모니터링할 하나 이상의 서비스를 선택합니다. 서비스는 OpenTelemetry를 통해 OneUptime으로 트레이스를 전송해야 합니다.

### 스팬 필터

| 필터      | 설명                                                              | 필수 여부 |
| --------- | ----------------------------------------------------------------- | --------- |
| 스팬 상태 | 스팬 상태 코드로 필터링 (OK, ERROR, UNSET)                        | 아니요    |
| 스팬 이름 | 특정 스팬 이름에 대한 텍스트 검색 (예: 작업 또는 엔드포인트 이름) | 아니요    |
| 속성      | 커스텀 스팬 속성에 대해 필터링할 키-값 쌍                         | 아니요    |
| 시간 창   | 스팬을 검색할 범위 (초 단위, 기본값: 60)                          | 아니요    |

### 스팬 상태 코드

- **OK** — 작업이 성공적으로 완료됨
- **ERROR** — 작업에서 오류가 발생함
- **UNSET** — 상태가 명시적으로 설정되지 않음

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형   | 설명                                |
| ----------- | ----------------------------------- |
| 스팬 카운트 | 시간 창에서 필터와 일치하는 스팬 수 |

### 필터 유형

- **초과** — 스팬 카운트가 임계값을 초과함
- **미만** — 스팬 카운트가 임계값보다 낮음
- **이상** — 스팬 카운트가 임계값 이상임
- **이하** — 스팬 카운트가 임계값 이하임
- **동일** — 스팬 카운트가 정확히 일치함
- **다름** — 스팬 카운트가 일치하지 않음

### 기준 예시

#### 60초 내에 50개 이상의 오류 스팬이 발생하는 경우 알림

- **스팬 상태**: ERROR
- **시간 창**: 60초
- **확인 대상**: 스팬 카운트
- **필터 유형**: 초과
- **값**: 50

#### 특정 엔드포인트의 오류에 대한 알림

- **스팬 이름**: `POST /api/checkout`
- **스팬 상태**: ERROR
- **시간 창**: 120초
- **확인 대상**: 스팬 카운트
- **필터 유형**: 초과
- **값**: 0

## 설정 요구 사항

트레이스 모니터링을 위해서는 애플리케이션이 OpenTelemetry를 통해 OneUptime으로 분산 트레이스를 전송해야 합니다. 설정 지침은 [OpenTelemetry](/docs/telemetry/open-telemetry) 문서를 참조하십시오.
