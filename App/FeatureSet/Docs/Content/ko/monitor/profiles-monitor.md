# 프로파일 모니터

프로파일 모니터링을 통해 애플리케이션의 연속 프로파일링 데이터를 모니터링하고 프로파일 수 및 패턴을 기반으로 알림을 트리거할 수 있습니다. OneUptime은 시간 창에 걸쳐 텔레메트리 서비스의 프로파일 데이터를 평가합니다.

## 개요

프로파일 모니터는 특정 기준에 일치하는 프로파일링 데이터를 카운트하고 필터링합니다. 이를 통해 다음이 가능합니다:

- 애플리케이션의 연속 프로파일링 데이터 모니터링
- 유형별 프로파일 필터링 (CPU, 메모리, goroutine 등)
- 프로파일 볼륨 및 패턴 추적
- 프로파일링 이상 감지 시 알림
- 커스텀 프로파일 속성으로 필터링

## 프로파일 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **프로파일**을 선택합니다
4. 모니터링할 텔레메트리 서비스를 선택합니다
5. 필요에 따라 프로파일 필터 및 기준을 구성합니다

## 구성 옵션

### 텔레메트리 서비스

프로파일을 모니터링할 하나 이상의 서비스를 선택합니다. 서비스는 OpenTelemetry를 통해 OneUptime으로 연속 프로파일링 데이터를 전송해야 합니다.

### 프로파일 필터

| 필터          | 설명                                                        | 필수 여부 |
| ------------- | ----------------------------------------------------------- | --------- |
| 프로파일 유형 | 프로파일 유형 이름으로 필터링 (예: CPU, memory, goroutines) | 아니요    |
| 속성          | 커스텀 프로파일 속성에 대해 필터링할 키-값 쌍               | 아니요    |
| 시간 창       | 프로파일을 검색할 범위 (초 단위, 기본값: 60)                | 아니요    |

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형       | 설명                                    |
| --------------- | --------------------------------------- |
| 프로파일 카운트 | 시간 창에서 필터와 일치하는 프로파일 수 |

### 필터 유형

- **초과** — 프로파일 카운트가 임계값을 초과함
- **미만** — 프로파일 카운트가 임계값보다 낮음
- **이상** — 프로파일 카운트가 임계값 이상임
- **이하** — 프로파일 카운트가 임계값 이하임
- **동일** — 프로파일 카운트가 정확히 일치함
- **다름** — 프로파일 카운트가 일치하지 않음

### 기준 예시

#### 5분 내에 프로파일이 수신되지 않는 경우 알림

- **시간 창**: 300초
- **확인 대상**: 프로파일 카운트
- **필터 유형**: 동일
- **값**: 0

## 설정 요구 사항

프로파일 모니터링을 위해서는 애플리케이션이 OpenTelemetry를 통해 OneUptime으로 연속 프로파일링 데이터를 전송해야 합니다. 설정 지침은 [OpenTelemetry](/docs/telemetry/open-telemetry) 문서를 참조하십시오.
