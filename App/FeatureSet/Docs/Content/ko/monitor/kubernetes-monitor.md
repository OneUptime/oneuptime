# Kubernetes 모니터

Kubernetes 모니터링을 통해 노드, 파드, 워크로드, 컨트롤 플레인 구성 요소를 포함한 Kubernetes 클러스터의 상태와 성능을 모니터링할 수 있습니다. OneUptime은 클러스터에서 메트릭을 수집하고 구성된 기준에 따라 평가합니다.

## 개요

Kubernetes 모니터는 클러스터의 메트릭을 사용하여 인프라에 대한 심층적인 가시성을 제공합니다. 이를 통해 다음이 가능합니다:

- 클러스터, 네임스페이스, 워크로드, 노드, 파드 상태 모니터링
- 리소스 전반에 걸친 CPU, 메모리, 디스크, 네트워크 사용량 추적
- 파드 충돌, 재시작, 스케줄링 실패 감지
- Deployment 복제본 가용성 모니터링
- 컨트롤 플레인 문제 알림(etcd, API 서버, 스케줄러)
- 리소스 요청 및 제한 추적

## Kubernetes 모니터 생성

1. OneUptime 대시보드에서 **Monitors**로 이동합니다
2. **Create Monitor**를 클릭합니다
3. 모니터 유형으로 **Kubernetes**를 선택합니다
4. 모니터링할 클러스터와 리소스 범위를 선택합니다
5. 리소스 필터와 메트릭 쿼리를 구성합니다
6. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### 클러스터

모니터링할 Kubernetes 클러스터를 선택합니다. 클러스터는 OpenTelemetry를 통해 OneUptime과 통합되어야 합니다.

### 리소스 범위

리소스를 모니터링할 수준을 선택합니다:

| 범위         | 설명                                                               |
| ------------ | ------------------------------------------------------------------ |
| 클러스터     | 전체 클러스터 모니터링                                             |
| 네임스페이스 | 특정 네임스페이스 내의 리소스 모니터링                             |
| 워크로드     | 특정 deployment, statefulset, daemonset, job 또는 cronjob 모니터링 |
| 노드         | 특정 클러스터 노드 모니터링                                        |
| 파드         | 특정 파드 모니터링                                                 |

### 리소스 필터

선택적 필터로 범위를 좁힙니다:

| 필터          | 설명                                             | 적용 가능한 범위             |
| ------------- | ------------------------------------------------ | ---------------------------- |
| 네임스페이스  | Kubernetes 네임스페이스                          | 네임스페이스, 워크로드, 파드 |
| 워크로드 유형 | deployment, statefulset, daemonset, job, cronjob | 워크로드                     |
| 워크로드 이름 | 워크로드의 이름                                  | 워크로드                     |
| 노드 이름     | 노드의 이름                                      | 노드                         |
| 파드 이름     | 파드의 이름                                      | 파드                         |

### 메트릭 쿼리

평가할 하나 이상의 메트릭 쿼리를 구성합니다. 각 쿼리는 다음을 지정합니다:

- **메트릭 이름** — 쿼리할 Kubernetes 메트릭
- **집계** — 메트릭 값을 집계하는 방법
- **필터** — 추가 속성 기반 필터링

수학적 표현식을 사용하여 여러 메트릭 쿼리를 결합하는 **수식(formula)**을 생성할 수도 있습니다.

### 롤링 시간 윈도우

메트릭 평가를 위한 시간 윈도우를 선택합니다:

- 지난 1분
- 지난 5분
- 지난 10분
- 지난 15분
- 지난 30분
- 지난 60분

## 일반적인 Kubernetes 메트릭

### 파드 메트릭

| 메트릭                  | 설명                                        |
| ----------------------- | ------------------------------------------- |
| 파드 CPU 사용량         | 파드의 CPU 소비량                           |
| 파드 메모리 사용량      | 파드의 메모리 소비량                        |
| 파드 파일시스템 사용량  | 파드의 디스크 사용량                        |
| 파드 네트워크 수신/송신 | 네트워크 트래픽                             |
| 파드 단계(Phase)        | 현재 파드 단계(Running, Pending, Failed 등) |

### 노드 메트릭

| 메트릭                 | 설명                 |
| ---------------------- | -------------------- |
| 노드 CPU 사용량        | 노드별 CPU 사용률    |
| 노드 메모리 사용량     | 노드별 메모리 사용률 |
| 노드 파일시스템 사용량 | 노드별 디스크 사용량 |
| 노드 디스크 I/O        | 읽기/쓰기 작업       |
| 노드 Ready 조건        | 노드 준비 상태 여부  |

### 컨테이너 메트릭

| 메트릭                             | 설명                    |
| ---------------------------------- | ----------------------- |
| 컨테이너 재시작                    | 컨테이너 재시작 횟수    |
| 컨테이너 CPU/메모리 제한(Limits)   | 리소스 제한             |
| 컨테이너 CPU/메모리 요청(Requests) | 리소스 요청             |
| 컨테이너 Ready 상태                | 컨테이너 준비 상태 여부 |

### 워크로드 메트릭

| 메트릭                         | 설명                 |
| ------------------------------ | -------------------- |
| Deployment 가용/비가용 복제본  | 복제본 수            |
| DaemonSet 잘못 스케줄링된 노드 | 스케줄링 문제        |
| StatefulSet Ready 복제본       | Ready 상태 복제본 수 |
| Job 활성/실패/성공 파드        | Job 상태             |

## 모니터링 기준

### 사용 가능한 검사 유형

| 검사 유형 | 설명                              |
| --------- | --------------------------------- |
| 메트릭 값 | 구성된 메트릭 쿼리 또는 수식의 값 |

### 집계 유형

| 집계    | 설명                                |
| ------- | ----------------------------------- |
| 평균    | 시간 윈도우에 대한 평균값           |
| 합계    | 모든 값의 합계                      |
| 최대값  | 시간 윈도우에서 가장 높은 값        |
| 최소값  | 시간 윈도우에서 가장 낮은 값        |
| 모든 값 | 모든 값이 기준을 충족해야 함        |
| 임의 값 | 하나 이상의 값이 기준을 충족해야 함 |

### 필터 유형

- **초과**, **미만**, **이상**, **이하**, **같음**, **같지 않음**

## 사전 구축된 알림 템플릿

OneUptime은 일반적인 Kubernetes 모니터링 시나리오에 대한 템플릿을 제공합니다:

| 템플릿                   | 설명                    | 임계값          |
| ------------------------ | ----------------------- | --------------- |
| CrashLoopBackOff 감지    | 컨테이너 재시작 횟수    | > 5회 재시작    |
| Pending 상태에 멈춘 파드 | Pending 단계의 파드     | > 0개 파드      |
| Node Not Ready           | 노드 Ready 조건         | = 0 (not ready) |
| 높은 노드 CPU            | 노드 CPU 사용률         | > 90%           |
| 높은 노드 메모리         | 노드 메모리 사용률      | > 85%           |
| Deployment 복제본 불일치 | 비가용 복제본           | > 0개 복제본    |
| Job 실패                 | Job의 실패한 파드       | > 0회 실패      |
| etcd 리더 없음           | etcd 클러스터 리더 누락 | = 0 (리더 없음) |
| API 서버 스로틀링        | 거부된 API 요청         | > 0개 요청      |
| 스케줄러 백로그          | 스케줄러의 Pending 파드 | > 0개 파드      |
| 높은 노드 디스크 사용량  | 노드 파일시스템 사용량  | > 90%           |
| DaemonSet 비가용         | 잘못 스케줄링된 노드    | > 0개 노드      |

## 설정 요구 사항

Kubernetes 모니터링을 사용하려면 클러스터에 OneUptime Kubernetes 에이전트를 설치해야 합니다. 에이전트는 클러스터 메트릭, 이벤트, 파드 로그와 — 기본적으로 — **eBPF를 통해 캡처된 애플리케이션 트레이스 및 HTTP RED 메트릭**을 OTLP로 OneUptime에 전송합니다. 서비스 레벨 트래픽을 확인하는 데 코드 변경이나 앱별 SDK가 필요하지 않습니다.

[Kubernetes 에이전트 설치](/docs/monitor/kubernetes-agent) 가이드를 참조하십시오 — 단일 명령 Helm 설치, 클러스터에 적합한 구성을 선택하는 `preset` 옵션(standard, GKE Autopilot, EKS Fargate), 그리고 개별 시그널 패밀리(HTTP RED 메트릭, 서비스 그래프, 네트워크 플로우, TCP 통계)를 위한 `ebpf.features.*` 토글을 다룹니다.
