# Kubernetes 모니터

Kubernetes 모니터링을 통해 노드, 파드, 워크로드 및 컨트롤 플레인 구성 요소를 포함한 Kubernetes 클러스터의 상태 및 성능을 모니터링할 수 있습니다. OneUptime은 클러스터에서 메트릭을 수집하고 구성된 기준에 따라 평가합니다.

## 개요

Kubernetes 모니터는 클러스터의 메트릭을 사용하여 인프라에 대한 심층적인 가시성을 제공합니다. 이를 통해 다음이 가능합니다:

- 클러스터, 네임스페이스, 워크로드, 노드 및 파드 상태 모니터링
- 리소스 전반의 CPU, 메모리, 디스크 및 네트워크 사용량 추적
- 파드 충돌, 재시작 및 예약 실패 감지
- 배포 복제본 가용성 모니터링
- 컨트롤 플레인 문제 알림 (etcd, API 서버, 스케줄러)
- 리소스 요청 및 제한 추적

## Kubernetes 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **Kubernetes**를 선택합니다
4. 모니터링할 클러스터와 리소스 범위를 선택합니다
5. 리소스 필터 및 메트릭 쿼리를 구성합니다
6. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### 클러스터

모니터링할 Kubernetes 클러스터를 선택합니다. 클러스터는 OpenTelemetry를 통해 OneUptime과 통합되어야 합니다.

### 리소스 범위

리소스를 모니터링할 수준을 선택합니다:

| 범위 | 설명 |
|-------|-------------|
| 클러스터 | 전체 클러스터 모니터링 |
| 네임스페이스 | 특정 네임스페이스 내의 리소스 모니터링 |
| 워크로드 | 특정 배포, 스테이트풀셋, 데몬셋, 작업 또는 크론 작업 모니터링 |
| 노드 | 특정 클러스터 노드 모니터링 |
| 파드 | 특정 파드 모니터링 |

### 리소스 필터

선택적 필터로 범위를 좁힙니다:

| 필터 | 설명 | 적용 가능한 범위 |
|--------|-------------|-------------------|
| 네임스페이스 | Kubernetes 네임스페이스 | 네임스페이스, 워크로드, 파드 |
| 워크로드 유형 | deployment, statefulset, daemonset, job, cronjob | 워크로드 |
| 워크로드 이름 | 워크로드의 이름 | 워크로드 |
| 노드 이름 | 노드의 이름 | 노드 |
| 파드 이름 | 파드의 이름 | 파드 |

### 메트릭 쿼리

평가할 하나 이상의 메트릭 쿼리를 구성합니다. 각 쿼리는 다음을 지정합니다:

- **메트릭 이름** — 쿼리할 Kubernetes 메트릭
- **집계** — 메트릭 값을 집계하는 방법
- **필터** — 추가 속성 기반 필터링

여러 메트릭 쿼리를 수학적 표현식을 사용하여 결합하는 **수식**을 생성할 수도 있습니다.

### 롤링 시간 창

메트릭 평가를 위한 시간 창을 선택합니다:

- 과거 1분
- 과거 5분
- 과거 10분
- 과거 15분
- 과거 30분
- 과거 60분

## 일반적인 Kubernetes 메트릭

### 파드 메트릭

| 메트릭 | 설명 |
|--------|-------------|
| 파드 CPU 사용량 | 파드의 CPU 소비량 |
| 파드 메모리 사용량 | 파드의 메모리 소비량 |
| 파드 파일시스템 사용량 | 파드의 디스크 사용량 |
| 파드 네트워크 수신/전송 | 네트워크 트래픽 |
| 파드 단계 | 현재 파드 단계 (Running, Pending, Failed 등) |

### 노드 메트릭

| 메트릭 | 설명 |
|--------|-------------|
| 노드 CPU 사용량 | 노드당 CPU 사용률 |
| 노드 메모리 사용량 | 노드당 메모리 사용률 |
| 노드 파일시스템 사용량 | 노드당 디스크 사용량 |
| 노드 디스크 I/O | 읽기/쓰기 작업 |
| 노드 준비 상태 | 노드가 준비 상태인지 여부 |

### 컨테이너 메트릭

| 메트릭 | 설명 |
|--------|-------------|
| 컨테이너 재시작 | 컨테이너 재시작 횟수 |
| 컨테이너 CPU/메모리 제한 | 리소스 제한 |
| 컨테이너 CPU/메모리 요청 | 리소스 요청 |
| 컨테이너 준비 상태 | 컨테이너가 준비 상태인지 여부 |

### 워크로드 메트릭

| 메트릭 | 설명 |
|--------|-------------|
| 배포 가용/비가용 복제본 | 복제본 수 |
| DaemonSet 잘못 예약된 노드 | 예약 문제 |
| StatefulSet 준비 복제본 | 준비된 복제본 수 |
| 작업 활성/실패/성공 파드 | 작업 상태 |

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| 메트릭 값 | 구성된 메트릭 쿼리 또는 수식의 값 |

### 집계 유형

| 집계 | 설명 |
|-------------|-------------|
| 평균 | 시간 창에 대한 평균 값 |
| 합계 | 모든 값의 합계 |
| 최대값 | 시간 창에서 가장 높은 값 |
| 최솟값 | 시간 창에서 가장 낮은 값 |
| 모든 값 | 모든 값이 기준을 충족해야 함 |
| 임의 값 | 하나 이상의 값이 기준을 충족해야 함 |

### 필터 유형

- **초과**, **미만**, **이상**, **이하**, **동일**, **다름**

## 사전 구축된 알림 템플릿

OneUptime은 일반적인 Kubernetes 모니터링 시나리오에 대한 템플릿을 제공합니다:

| 템플릿 | 설명 | 임계값 |
|----------|-------------|-----------|
| CrashLoopBackOff 감지 | 컨테이너 재시작 횟수 | > 5회 재시작 |
| 대기 중인 파드 | 대기 단계의 파드 | > 0개 파드 |
| 준비되지 않은 노드 | 노드 준비 상태 | = 0 (준비 안 됨) |
| 높은 노드 CPU | 노드 CPU 사용률 | > 90% |
| 높은 노드 메모리 | 노드 메모리 사용률 | > 85% |
| 배포 복제본 불일치 | 비가용 복제본 | > 0개 복제본 |
| 작업 실패 | 작업의 실패한 파드 | > 0개 실패 |
| etcd 리더 없음 | etcd 클러스터 리더 없음 | = 0 (리더 없음) |
| API 서버 스로틀링 | 삭제된 API 요청 | > 0개 요청 |
| 스케줄러 백로그 | 스케줄러에서 대기 중인 파드 | > 0개 파드 |
| 높은 노드 디스크 사용량 | 노드 파일시스템 사용량 | > 90% |
| DaemonSet 비가용 | 잘못 예약된 노드 | > 0개 노드 |

## 설정 요구 사항

Kubernetes 모니터링을 사용하려면 클러스터에 OneUptime Kubernetes 에이전트를 설치해야 합니다. 에이전트는 OTLP를 통해 클러스터 메트릭, 이벤트 및 파드 로그를 OneUptime으로 전송합니다.

[Kubernetes 에이전트 설치](/docs/monitor/kubernetes-agent) 가이드를 참조하십시오 — 원 커맨드 Helm 설치와 클러스터에 맞는 올바른 구성을 선택하기 위한 `preset` 옵션 (standard, GKE Autopilot, EKS Fargate)을 다룹니다.
