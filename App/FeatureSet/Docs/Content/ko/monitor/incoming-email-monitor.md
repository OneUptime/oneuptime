# 수신 이메일 모니터

수신 이메일 모니터를 통해 고유한 모니터별 이메일 주소로 전송된 이메일을 기반으로 알림을 생성하고 해결할 수 있습니다. 레거시 시스템, 타사 알림 도구 또는 이메일 알림을 전송할 수 있는 모든 서비스와 통합하는 데 유용합니다.

## 작동 방식

1. 수신 이메일 모니터를 생성하면 OneUptime이 해당 모니터에 대한 고유한 이메일 주소를 생성합니다
2. 해당 주소로 전송된 이메일이 수신되고 구성된 기준에 따라 평가됩니다
3. 기준에 따라 OneUptime은 새 알림을 생성하거나 기존 알림을 해결할 수 있습니다

이것은 이메일 기반 알림 시스템을 OneUptime의 인시던트 관리 워크플로와 통합하는 강력한 방법입니다.

## 수신 이메일 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **수신 이메일**을 선택합니다
4. 모니터 설정을 구성합니다:
   - **이름:** 모니터의 설명적인 이름
   - **설명:** 이 모니터의 용도
5. **알림 생성 기준** (알림을 생성하는 조건)을 설정합니다
6. **알림 해결 기준** (알림을 해결하는 조건)을 설정합니다
7. **생성**을 클릭합니다

생성 후 모니터 세부 정보 페이지에 이 모니터의 고유한 이메일 주소가 표시됩니다.

## 이메일 주소 형식

각 수신 이메일 모니터는 다음 형식의 고유한 이메일 주소를 받습니다:

```
monitor-{secret-key}@{inbound-domain}
```

예시: `monitor-abc123def456@inbound.yourdomain.com`

모니터 세부 정보 페이지에서 이 주소를 복사하고 해당 주소로 이메일을 전송하도록 외부 시스템을 구성할 수 있습니다.

## 사용 가능한 기준 필드

다음 이메일 필드를 기반으로 기준을 생성할 수 있습니다:

| 필드              | 설명                             |
| ----------------- | -------------------------------- |
| **이메일 제목**   | 수신 이메일의 제목 줄            |
| **이메일 발신자** | 발신자의 이메일 주소             |
| **이메일 본문**   | 이메일 본문의 일반 텍스트 내용   |
| **이메일 수신자** | 수신자 이메일 주소               |
| **이메일 수신**   | 이메일이 수신되는 시간 기반 기준 |

## 사용 가능한 필터 유형

### 문자열 필터 (제목, 발신자, 본문, 수신자)

| 필터               | 설명                                 | 예시                                 |
| ------------------ | ------------------------------------ | ------------------------------------ |
| **포함**           | 필드에 지정된 텍스트가 포함됨        | 제목에 "CRITICAL" 포함               |
| **포함하지 않음**  | 필드에 지정된 텍스트가 포함되지 않음 | 제목에 "TEST" 미포함                 |
| **동일**           | 필드가 지정된 텍스트와 정확히 일치함 | 발신자가 "alerts@service.com"과 동일 |
| **다름**           | 필드가 지정된 텍스트와 일치하지 않음 | 제목이 "OK"와 다름                   |
| **시작**           | 필드가 지정된 텍스트로 시작함        | 제목이 "[ALERT]"로 시작              |
| **끝**             | 필드가 지정된 텍스트로 끝남          | 제목이 "- Production"으로 끝남       |
| **비어 있음**      | 필드가 비어 있거나 공백임            | 본문이 비어 있음                     |
| **비어 있지 않음** | 필드에 내용이 있음                   | 제목이 비어 있지 않음                |

### 시간 기반 필터 (이메일 수신)

| 필터             | 설명                              | 예시                    |
| ---------------- | --------------------------------- | ----------------------- |
| **분 내 수신**   | 이메일이 X분 이내에 수신됨        | 30분 이내 이메일 수신   |
| **분 내 미수신** | X분 이내에 이메일이 수신되지 않음 | 60분 이내 이메일 미수신 |

## 구성 예시

### 예시 1: 중요 이메일에 대한 알림 생성

**알림 생성 기준:**

- 이메일 제목 **포함** "CRITICAL"
- 또는 이메일 제목 **포함** "ALERT"
- 또는 이메일 제목 **포함** "ERROR"

**알림 해결 기준:**

- 이메일 제목 **포함** "RESOLVED"
- 또는 이메일 제목 **포함** "OK"
- 또는 이메일 제목 **포함** "RECOVERED"

### 예시 2: 특정 발신자 모니터링

**알림 생성 기준:**

- 이메일 발신자 **동일** "monitoring@legacy-system.com"
- AND 이메일 제목 **포함** "Failed"

**알림 해결 기준:**

- 이메일 발신자 **동일** "monitoring@legacy-system.com"
- AND 이메일 제목 **포함** "Success"

### 예시 3: 하트비트 모니터 (이메일 없음 = 알림)

**알림 생성 기준:**

- 이메일 수신 **분 내 미수신** 값 `60`

이 설정은 60분 동안 이메일이 수신되지 않으면 알림을 생성합니다 — 완료 이메일을 전송해야 하는 예약된 작업이나 배치 프로세스를 모니터링하는 데 유용합니다.

**알림 해결 기준:**

- 이메일 수신 **분 내 수신** 값 `5`

이 설정은 이메일이 수신되면 알림을 해결합니다.

## 사용 사례

### 레거시 시스템 통합

많은 구형 시스템은 이메일 기반 알림만 지원합니다. 수신 이메일 모니터를 사용하여:

- 이메일 알림을 OneUptime 인시던트로 변환
- 복구 이메일이 도착하면 인시던트를 자동으로 해결
- 여러 레거시 시스템의 알림 중앙화

### 타사 서비스 모니터링

이메일 알림을 전송하는 서비스와 통합:

- 클라우드 공급자 알림 (AWS, GCP, Azure)
- 보안 스캐닝 도구
- 백업 완료 알림
- SSL 인증서 만료 경고

### 예약 작업 모니터링

배치 작업 및 예약 작업 모니터링:

- 완료 이메일이 제때 수신되지 않으면 알림 생성
- 오류 알림 이메일을 통해 작업 실패 추적
- 데이터 파이프라인 완료 모니터링

### 다중 공급업체 알림 집계

여러 모니터링 도구의 알림 통합:

- 이메일을 통해 Nagios, Zabbix 또는 기타 도구의 알림 수신
- OneUptime에서 인시던트 관리 통합
- 모든 알림에 대한 단일 정보 소스 유지

## 템플릿 변수

인시던트 템플릿을 구성할 때 수신 이메일의 다음 변수를 사용할 수 있습니다:

| 변수                  | 설명                      |
| --------------------- | ------------------------- |
| `{{emailSubject}}`    | 수신 이메일의 제목        |
| `{{emailFrom}}`       | 발신자의 이메일 주소      |
| `{{emailTo}}`         | 수신자 이메일 주소        |
| `{{emailBody}}`       | 이메일의 일반 텍스트 본문 |
| `{{emailReceivedAt}}` | 이메일이 수신된 시간      |

## 모니터 요약 보기

모니터 요약에는 다음이 표시됩니다:

- **마지막 이메일 수신 시간:** 가장 최근 이메일이 수신된 시간
- **발신자:** 마지막 이메일의 발신자
- **제목:** 마지막 이메일의 제목 줄
- **이메일 헤더:** 마지막 이메일의 전체 헤더 (확장 가능)
- **이메일 본문:** 마지막 이메일의 내용 (확장 가능)

## 자체 호스팅 설정

OneUptime을 자체 호스팅하는 경우 인바운드 이메일 공급자를 구성해야 합니다. 현재 지원:

- **SendGrid Inbound Parse** - 설정 지침은 [SendGrid 인바운드 이메일 통합](/docs/self-hosted/sendgrid-inbound-email)을 참조하십시오

## 고려해야 할 사항

- **이메일 주소 보안:** 모니터 이메일 주소에는 비밀 키가 포함되어 있습니다. 비밀번호처럼 취급하고 공개적으로 공유하지 마십시오.
- **이메일 크기:** 매우 큰 이메일 (대용량 첨부 파일 포함)은 이메일 공급자에 의해 잘리거나 거부될 수 있습니다.
- **처리 시간:** 이메일은 비동기적으로 처리됩니다. 이메일 전송과 알림 생성 사이에 몇 초 지연이 있을 수 있습니다.
- **대소문자 구분 없음:** 모든 문자열 비교 (포함, 동일 등)는 대소문자를 구분하지 않습니다.
- **일반 텍스트:** 이메일 본문 기준은 이메일의 일반 텍스트 버전을 사용합니다. HTML 형식이 제거됩니다.

## 문제 해결

### 이메일이 수신되지 않는 경우

1. 이메일 주소가 올바른지 확인합니다 (오타 확인)
2. 이메일이 스팸 필터에 의해 차단되지 않는지 확인합니다
3. 인바운드 이메일 공급자가 올바르게 구성되어 있는지 확인합니다
4. 오류 메시지에 대한 OneUptime 로그를 확인합니다

### 알림이 생성되지 않는 경우

1. 기준이 이메일 내용과 일치하는지 확인합니다
2. 모니터가 비활성화되어 있지 않은지 확인합니다
3. 모니터 세부 정보의 평가 로그를 검토합니다
4. 패턴 매칭을 사용하기 전에 정확한 문자열 일치로 테스트합니다

### 알림이 해결되지 않는 경우

1. 해결 기준이 복구 이메일과 일치하는지 확인합니다
2. 해결할 활성 알림이 있는지 확인합니다
3. 해결 이메일이 동일한 모니터 주소로 전송되는지 확인합니다
