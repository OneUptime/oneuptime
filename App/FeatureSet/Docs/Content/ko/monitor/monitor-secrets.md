# 모니터 시크릿

모니터링 확인에 사용하려는 민감한 정보를 저장하기 위해 시크릿을 사용할 수 있습니다. 시크릿은 암호화되어 안전하게 저장됩니다.

### 시크릿 추가

시크릿을 추가하려면 OneUptime 대시보드 -> 프로젝트 설정 -> 모니터 시크릿 -> 모니터 시크릿 생성으로 이동하십시오.

![시크릿 생성](/docs/static/images/CreateMonitorSecret.png)

어느 모니터가 시크릿에 액세스할 수 있는지 선택할 수 있습니다. 이 경우 `ApiKey` 시크릿을 추가하고 액세스할 모니터를 선택했습니다.

**참고**: 시크릿은 암호화되어 안전하게 저장됩니다. 시크릿을 분실하면 새 시크릿을 생성해야 합니다. 저장 후에는 시크릿을 보거나 업데이트할 수 없습니다.

### 시크릿 사용

다음 모니터링 유형에서 시크릿을 사용할 수 있습니다:

- API (요청 헤더, 요청 본문 및 URL에서)
- 웹사이트, IP, 포트, Ping, SSL 인증서 (URL에서)
- 합성 모니터, 커스텀 코드 모니터 (코드에서)
- SNMP 모니터 (커뮤니티 문자열, SNMPv3 인증 키 및 개인 키에서)

![시크릿 사용](/docs/static/images/UsingMonitorSecret.png)

시크릿을 사용하려면 시크릿을 사용하려는 필드에 `{{monitorSecrets.SECRET_NAME}}`을 추가합니다. 예를 들어, 이 경우 요청 헤더 필드에 `{{monitorSecrets.ApiKey}}`를 추가했습니다.

시크릿은 합성 또는 커스텀 코드 모니터 스크립트가 실행되기 전에 프로브에 주입되므로 실행 중인 스크립트 내에서 `{{monitorSecrets.ApiKey}}`와 같은 참조가 복호화된 값으로 확인됩니다.

### 모니터 시크릿 권한

어느 모니터가 시크릿에 액세스할 수 있는지 선택할 수 있습니다. 언제든지 권한을 업데이트할 수도 있습니다. 따라서 새 모니터가 시크릿에 액세스하도록 추가하려면 권한을 업데이트하면 됩니다.
