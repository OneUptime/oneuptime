# DNSSEC 모니터

DNSSEC 모니터링을 통해 영역(zone)에 대한 DNS 응답의 암호화 무결성을 검증할 수 있습니다. OneUptime은 주기적으로 전체 DNSSEC 검증을 수행하여 DNSKEY 레코드, 상위 영역의 DS 위임, RRSIG 서명 유효성, 리졸버 간 AD flag 합의, 권한 있는 네임서버 간의 일관성을 확인합니다.

## 개요

DNSSEC 모니터는 루트 영역부터 사용자의 도메인까지 이어지는 전체 신뢰 체인을 검증합니다. 이를 통해 다음이 가능합니다:

- 리졸버가 사용자에게 SERVFAIL을 반환하기 전에 손상된 DNSSEC 체인 감지
- 영역 서명 키가 만료되기 전에 경고 수신
- DS 레코드가 상위 영역에 올바르게 게시되었는지 확인
- 권한 있는 네임서버 간의 불일치(프라이머리/세컨더리 동기화 어긋남) 포착
- 검증 리졸버가 실제로 영역에 대해 AD flag를 설정하는지 확인

## DNSSEC 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **DNSSEC**를 선택합니다
4. 검증할 영역(도메인)을 입력합니다
5. 필요에 따라 리졸버 및 모니터링 기준을 구성합니다

## 구성 옵션

### 기본 설정

| 필드 | 설명 | 필수 여부 |
|-------|-------------|----------|
| 영역 (도메인 이름) | DNSSEC를 통해 검증할 영역 (예: `example.com`) | 예 |
| 리졸버 | 쿼리할 검증 리졸버의 쉼표로 구분된 목록 (예: `1.1.1.1, 8.8.8.8, 9.9.9.9`) | 예 |
| 네임서버 일관성 확인 | 각 권한 있는 네임서버에 직접 쿼리하여 동일한 SOA serial을 반환하는지 확인 | 아니요 |

### 고급 설정

| 필드 | 설명 | 기본값 |
|-------|-------------|---------|
| 서명 만료 경고 (일) | RRSIG 만료 필터의 기본 임계값 | 7 |
| 타임아웃 (ms) | 각 DNS 쿼리에 대한 대기 시간 | 10000 |
| 재시도 | 실패 시 재시도 횟수 | 3 |

## 모니터링 기준

다음을 기반으로 영역이 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다:

### 사용 가능한 확인 유형

| 확인 유형 | 설명 |
|------------|-------------|
| DNSSEC 체인 유효 여부 | 전체 검증 체인(루트 → TLD → 영역)이 올바르게 확인됨 |
| DNSSEC DNSKEY 레코드 존재 여부 | 영역이 하나 이상의 DNSKEY 레코드를 게시함 |
| 상위에 DNSSEC DS 레코드 존재 여부 | 상위 영역이 영역의 KSK와 일치하는 DS 레코드를 게시함 |
| DNSSEC 서명 만료까지 남은 일수 | 가장 빨리 만료되는 RRSIG 서명까지 남은 일수 |
| DNSSEC 리졸버 합의 (AD Flag) | 쿼리한 모든 리졸버가 AD(Authenticated Data) flag를 반환함 |
| DNSSEC 네임서버 일관성 여부 | 모든 권한 있는 네임서버가 동일한 SOA serial을 반환함 |
| DNSSEC 유효 여부 | 모든 검증 확인에 걸친 종합 통과/실패 |

### 필터 유형

**DNSSEC 체인 유효 여부**, **DNSSEC DNSKEY 레코드 존재 여부**, **상위에 DNSSEC DS 레코드 존재 여부**, **DNSSEC 리졸버 합의 (AD Flag)**, **DNSSEC 네임서버 일관성 여부**, **DNSSEC 유효 여부**:

- **True** — 조건이 참임
- **False** — 조건이 거짓임

**DNSSEC 서명 만료까지 남은 일수**:

- **초과**, **미만**, **이상**, **이하**, **동일**, **다름**

### 기준 예시

#### DNSSEC 체인이 손상된 경우 알림

- **확인 대상**: DNSSEC 체인 유효 여부
- **필터 유형**: False

#### 서명 만료 전 경고

- **확인 대상**: DNSSEC 서명 만료까지 남은 일수
- **필터 유형**: 미만
- **값**: 7

#### 상위의 DS 누락(위임 손상) 포착

- **확인 대상**: 상위에 DNSSEC DS 레코드 존재 여부
- **필터 유형**: False

#### 리졸버 간 불일치 감지

- **확인 대상**: DNSSEC 리졸버 합의 (AD Flag)
- **필터 유형**: False

#### 네임서버 간 데이터 불일치 포착

- **확인 대상**: DNSSEC 네임서버 일관성 여부
- **필터 유형**: False

## 모범 사례

1. **여러 공용 리졸버 사용** — `1.1.1.1`, `8.8.8.8`, `9.9.9.9`를 기본값으로 사용하여 단일 리졸버의 장애로 인한 오탐을 방지하세요
2. **만료 훨씬 전에 경고** — 서명 만료 7일 전에 저하 알림을, 2일 전에 오프라인 알림을 구성하세요. 키 롤오버는 조용히 실패할 수 있습니다
3. **서명된 모든 영역 모니터링** — apex 도메인, 서명된 서브도메인, 그리고 다른 운영자에게 위임된 모든 영역을 포함하세요
4. **네임서버 일관성 확인 활성화** — 네트워크가 임의의 IP로의 아웃바운드 DNS를 차단하지 않는 한, DNSSEC 검증만으로는 놓칠 수 있는 프라이머리/세컨더리 동기화 문제를 포착합니다
