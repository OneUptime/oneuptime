# 예외 모니터

예외 모니터링을 통해 애플리케이션 예외 및 오류를 모니터링하고, 예외 수가 구성된 임계값을 초과하면 알림을 트리거할 수 있습니다. OneUptime은 시간 창에 걸쳐 텔레메트리 서비스의 예외 데이터를 평가합니다.

## 개요

예외 모니터는 특정 기준에 일치하는 예외를 카운트하고 필터링합니다. 이를 통해 다음이 가능합니다:

- 애플리케이션의 예외 급증에 대한 알림
- 특정 예외 유형 모니터링
- 오류 메시지로 예외 검색
- 해결된 예외와 활성 예외를 별도로 추적
- 오류 패턴에서 애플리케이션 안정성 문제 감지

## 예외 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **예외**를 선택합니다
4. 모니터링할 텔레메트리 서비스를 선택합니다
5. 필요에 따라 예외 필터 및 기준을 구성합니다

## 구성 옵션

### 텔레메트리 서비스

예외를 모니터링할 하나 이상의 서비스를 선택합니다. 서비스는 OpenTelemetry를 통해 OneUptime으로 예외 데이터를 전송해야 합니다.

### 예외 필터

| 필터             | 설명                                                                | 필수 여부 |
| ---------------- | ------------------------------------------------------------------- | --------- |
| 예외 유형        | 예외 유형 이름으로 필터링 (예: `NullPointerException`, `TypeError`) | 아니요    |
| 메시지           | 예외 메시지 내 텍스트 검색                                          | 아니요    |
| 해결된 항목 포함 | 해결됨으로 표시된 예외 포함 여부 (기본값: false)                    | 아니요    |
| 보관된 항목 포함 | 보관된 예외 포함 여부 (기본값: false)                               | 아니요    |
| 시간 창          | 예외를 검색할 범위 (초 단위, 기본값: 60)                            | 아니요    |

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형   | 설명                                |
| ----------- | ----------------------------------- |
| 예외 카운트 | 시간 창에서 필터와 일치하는 예외 수 |

### 필터 유형

- **초과** — 예외 수가 임계값을 초과함
- **미만** — 예외 수가 임계값보다 낮음
- **이상** — 예외 수가 임계값 이상임
- **이하** — 예외 수가 임계값 이하임
- **동일** — 예외 수가 정확히 일치함
- **다름** — 예외 수가 일치하지 않음

### 기준 예시

#### 60초 내에 10개 이상의 예외가 발생하는 경우 알림

- **시간 창**: 60초
- **확인 대상**: 예외 카운트
- **필터 유형**: 초과
- **값**: 10

#### NullPointerException에 대한 알림

- **예외 유형**: `NullPointerException`
- **시간 창**: 60초
- **확인 대상**: 예외 카운트
- **필터 유형**: 초과
- **값**: 0

#### 특정 메시지가 포함된 예외 모니터링

- **메시지**: `out of memory`
- **시간 창**: 300초
- **확인 대상**: 예외 카운트
- **필터 유형**: 초과
- **값**: 0

## 설정 요구 사항

예외 모니터링을 위해서는 애플리케이션이 OpenTelemetry를 통해 OneUptime으로 예외 데이터를 전송해야 합니다. 설정 지침은 [OpenTelemetry](/docs/telemetry/open-telemetry) 문서를 참조하십시오.
