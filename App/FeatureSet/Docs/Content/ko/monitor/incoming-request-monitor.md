# 수신 요청 모니터

수신 요청 모니터링 (하트비트 모니터링이라고도 함)을 통해 서비스가 OneUptime으로 주기적인 HTTP 요청을 전송하도록 하여 서비스를 모니터링할 수 있습니다. OneUptime이 서비스에 접근하는 대신, 서비스가 실행 중임을 확인하기 위해 OneUptime으로 ping을 전송합니다.

## 개요

수신 요청 모니터는 서비스가 일정에 따라 호출하는 고유한 웹훅 URL을 제공합니다. 이를 통해 다음이 가능합니다:

- cron 작업 및 예약 작업 모니터링
- 백그라운드 워커가 실행 중인지 확인
- 외부에서 도달할 수 없는 방화벽 뒤의 서비스 모니터링
- 타사 모니터링 도구와 통합
- HTTP 기능이 있는 모든 시스템의 하트비트 신호 추적

## 수신 요청 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **수신 요청**을 선택합니다
4. 이 모니터에 대한 **비밀 키**와 하트비트 URL이 생성됩니다
5. 하트비트 URL로 요청을 전송하도록 서비스를 구성합니다
6. 필요에 따라 모니터링 기준을 구성합니다

## 하트비트 URL

생성 후 모니터는 다음 형식의 고유한 하트비트 URL을 가집니다:

```
https://oneuptime.com/heartbeat/YOUR_SECRET_KEY
```

서비스는 정기적인 간격으로 이 URL에 HTTP **GET** 또는 **POST** 요청을 전송해야 합니다.

### 하트비트 전송

#### curl 사용

```bash
# 간단한 GET 요청
curl https://oneuptime.com/heartbeat/YOUR_SECRET_KEY

# 커스텀 본문이 있는 POST 요청
curl -X POST https://oneuptime.com/heartbeat/YOUR_SECRET_KEY \
  -H "Content-Type: application/json" \
  -d '{"status": "healthy", "version": "1.2.3"}'
```

#### cron 작업에서

```bash
# 5분마다 하트비트를 전송하기 위해 crontab에 추가
*/5 * * * * curl -s https://oneuptime.com/heartbeat/YOUR_SECRET_KEY > /dev/null
```

#### 애플리케이션 코드에서

```javascript
// Node.js 예시
const https = require("https");
https.get("https://oneuptime.com/heartbeat/YOUR_SECRET_KEY");
```

```python
# Python 예시
import requests
requests.get('https://oneuptime.com/heartbeat/YOUR_SECRET_KEY')
```

자체 호스팅하는 경우 `https://oneuptime.com`을 OneUptime 인스턴스 URL로 교체합니다.

## 모니터링 기준

다음을 기반으로 서비스가 온라인, 저하 또는 오프라인으로 간주되는 시점을 결정하는 기준을 구성할 수 있습니다:

### 사용 가능한 확인 유형

| 확인 유형    | 설명                                      |
| ------------ | ----------------------------------------- |
| 수신 요청    | 시간 창 내에 하트비트가 수신되었는지 여부 |
| 요청 본문    | 하트비트와 함께 전송된 요청 본문의 내용   |
| 요청 헤더    | 특정 요청 헤더의 이름                     |
| 요청 헤더 값 | 특정 요청 헤더의 값                       |

### 필터 유형

**수신 요청**:

- **분 내 수신** — 지정된 분 수 이내에 하트비트가 수신됨
- **분 내 미수신** — 지정된 분 수 이내에 하트비트가 수신되지 않음

**요청 본문**, **요청 헤더** 및 **요청 헤더 값**:

- **포함** — 값에 지정된 텍스트가 포함됨
- **포함하지 않음** — 값에 지정된 텍스트가 포함되지 않음

### 기준 예시

#### 10분 이내에 하트비트가 없으면 오프라인으로 표시

- **확인 대상**: 수신 요청
- **필터 유형**: 분 내 미수신
- **값**: 10

#### 요청 본문 내용에 따라 저하로 표시

- **확인 대상**: 요청 본문
- **필터 유형**: 포함
- **값**: `"status": "degraded"`

## 모범 사례

1. **시간 창을 적절하게 설정** — cron 작업이 5분마다 실행되는 경우 가끔 지연을 허용하기 위해 "분 내 미수신" 임계값을 10-15분으로 설정합니다
2. **의미 있는 데이터 포함** — 세분화된 기준을 설정할 수 있도록 요청 본문에 상태 정보를 전송합니다
3. **풍부한 데이터에는 POST 사용** — 상세한 상태 정보를 전송해야 할 때는 JSON 본문이 있는 POST 요청을 사용합니다
4. **모니터 모니터링** — 하트비트를 전송하는 서비스에 적절한 오류 처리가 있는지 확인하여 실패한 하트비트 요청이 눈에 띄지 않게 되는 일이 없도록 합니다
