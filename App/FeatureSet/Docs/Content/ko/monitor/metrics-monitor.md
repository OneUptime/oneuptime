# 메트릭 모니터

메트릭 모니터링을 통해 OpenTelemetry를 통해 수집된 커스텀 애플리케이션 및 인프라 메트릭을 모니터링할 수 있습니다. OneUptime은 시간 창에 걸쳐 메트릭 값을 평가하고 구성된 기준에 따라 알림을 트리거합니다.

## 개요

메트릭 모니터는 텔레메트리 서비스의 수치 메트릭을 쿼리하고 평가합니다. 이를 통해 다음이 가능합니다:

- 커스텀 애플리케이션 메트릭 모니터링 (요청 속도, 큐 깊이, 오류율 등)
- 인프라 메트릭 추적 (CPU, 메모리, 디스크, 네트워크)
- 필터 및 집계를 사용한 복잡한 메트릭 쿼리 생성
- 수학적 수식을 사용하여 여러 메트릭 결합
- 메트릭 임계값을 기반으로 알림 설정

## 메트릭 모니터 생성

1. OneUptime 대시보드의 **모니터**로 이동합니다
2. **모니터 생성**을 클릭합니다
3. 모니터 유형으로 **메트릭**을 선택합니다
4. 메트릭 쿼리 및 선택적 수식을 구성합니다
5. 집계 전략을 선택합니다
6. 필요에 따라 모니터링 기준을 구성합니다

## 구성 옵션

### 메트릭 쿼리

하나 이상의 메트릭 쿼리를 정의합니다. 각 쿼리에는 다음이 포함됩니다:

| 필드        | 설명                                                       | 필수 여부 |
| ----------- | ---------------------------------------------------------- | --------- |
| 메트릭 이름 | 쿼리할 메트릭의 이름                                       | 예        |
| 집계 유형   | 원시 메트릭 값을 집계하는 방법 (sum, avg, min, max, count) | 예        |
| 속성        | 메트릭 데이터를 좁히기 위한 키-값 필터                     | 아니요    |
| 집계 기준   | 메트릭을 그룹화할 차원                                     | 아니요    |

각 쿼리에는 수식에서 사용할 수 있는 별칭 (예: `a`, `b`, `c`)이 할당됩니다.

### 수식

수학적 표현식을 사용하여 여러 메트릭 쿼리를 결합합니다. 예:

- `a / b * 100` — 두 쿼리에서 백분율 계산
- `a + b` — 두 메트릭의 합계
- `a - b` — 메트릭 간의 차이

### 롤링 시간 창

메트릭 평가를 위한 시간 창을 선택합니다:

- 과거 1분
- 과거 5분
- 과거 10분
- 과거 15분
- 과거 30분
- 과거 60분

### 집계 전략

평가를 위해 메트릭 값을 집계하는 방법을 선택합니다:

| 전략    | 설명                                |
| ------- | ----------------------------------- |
| 평균    | 시간 창에 대한 평균 값              |
| 합계    | 모든 값의 합계                      |
| 최대값  | 시간 창에서 가장 높은 값            |
| 최솟값  | 시간 창에서 가장 낮은 값            |
| 모든 값 | 모든 값이 기준을 충족해야 함        |
| 임의 값 | 하나 이상의 값이 기준을 충족해야 함 |

## 모니터링 기준

### 사용 가능한 확인 유형

| 확인 유형 | 설명                                     |
| --------- | ---------------------------------------- |
| 메트릭 값 | 구성된 메트릭 쿼리 또는 수식의 집계된 값 |

### 필터 유형

- **초과** — 메트릭 값이 임계값을 초과함
- **미만** — 메트릭 값이 임계값보다 낮음
- **이상** — 메트릭 값이 임계값 이상임
- **이하** — 메트릭 값이 임계값 이하임
- **동일** — 메트릭 값이 정확히 일치함
- **다름** — 메트릭 값이 일치하지 않음

### 기준 예시

#### 오류율이 5%를 초과하는 경우 알림

- **쿼리 a**: `http_requests_total` `status=5xx` 필터 적용
- **쿼리 b**: `http_requests_total`
- **수식**: `a / b * 100`
- **확인 대상**: 메트릭 값
- **필터 유형**: 초과
- **값**: 5

#### 요청 큐 깊이가 높은 경우 알림

- **쿼리**: `request_queue_size`, 집계: 최대값
- **확인 대상**: 메트릭 값
- **필터 유형**: 초과
- **값**: 1000

## 설정 요구 사항

메트릭 모니터링을 위해서는 애플리케이션이나 인프라가 OpenTelemetry를 통해 OneUptime으로 메트릭을 전송해야 합니다. 설정 지침은 [OpenTelemetry](/docs/telemetry/open-telemetry) 문서를 참조하십시오.
