# FAQ 및 문제 해결

OneUptime 모바일 및 데스크탑 앱(PWA)에 대한 자주 묻는 질문과 해결책입니다.

## 일반 FAQ

### 프로그레시브 웹 앱(PWA)이란?

프로그레시브 웹 앱은 앱과 같은 경험을 제공하기 위해 현대적인 웹 기술을 사용하는 웹 애플리케이션입니다. PWA는 앱 스토어 없이 브라우저에서 직접 설치할 수 있고, 오프라인에서 작동하며, 푸시 알림을 전송하고, 기기의 운영 체제와 통합될 수 있습니다.

### 왜 OneUptime은 전통적인 앱 스토어를 사용하지 않나요?

OneUptime은 여러 가지 이점을 제공하기 때문에 PWA 기술을 사용합니다:
- **즉각적인 업데이트**: 앱 스토어 승인이나 수동 업데이트를 기다릴 필요 없음
- **크로스 플랫폼**: 단일 코드베이스가 모든 기기에서 작동
- **다운로드 크기 제한 없음**: 크기 제한 없이 모든 기능 사용 가능
- **직접 배포**: OneUptime 인스턴스에서 직접 설치
- **항상 최신 버전**: 사용자는 항상 최신 버전을 사용
- **보안**: 웹 애플리케이션과 동일한 보안 이점

### OneUptime PWA는 얼마나 많은 저장 공간을 사용하나요?

- **초기 설치**: 10-20MB
- **캐시 증가**: 정기적인 사용으로 50-100MB
- **최대 캐시**: 일반적으로 브라우저에서 200MB로 제한됨
- **자동 정리**: 브라우저가 자동으로 저장 공간을 관리함

### OneUptime PWA가 푸시 알림을 지원하나요?

예, OneUptime PWA는 풍부한 푸시 알림을 지원합니다:
- **인시던트 알림**: 실시간 인시던트 알림
- **상태 업데이트**: 모니터 상태 변경 알림
- **커스텀 트리거**: 알림 규칙 구성
- **풍부한 내용**: 이미지, 작업 및 상세 정보
- **배지 업데이트**: 앱 아이콘의 읽지 않은 카운트

## 설치 FAQ

### "설치" 버튼이 보이지 않는 이유는 무엇인가요?

일반적인 이유와 해결책:
1. **브라우저 호환성**: Chrome, Edge 또는 Safari를 사용합니다
2. **HTTPS 필요**: OneUptime 인스턴스가 HTTPS를 사용하는지 확인합니다
3. **PWA 요구 사항**: 서버가 PWA 매니페스트 요구 사항을 충족해야 합니다
4. **캐시 문제**: 브라우저 캐시를 지우고 다시 로드합니다
5. **이미 설치됨**: 앱이 이미 설치되어 있을 수 있습니다
6. **대기 시간**: 일부 브라우저는 페이지에서 30초 이상이 필요합니다

### 여러 기기에 설치할 수 있나요?

예! OneUptime PWA를 다음에 설치할 수 있습니다:
- 사용자당 무제한 기기
- 동일한 기기의 여러 브라우저
- 다양한 운영 체제
- 공유/가족 기기 (별도 계정으로)

### 설치된 앱을 어떻게 업데이트하나요?

OneUptime PWA는 자동으로 업데이트됩니다:
- **자동 업데이트**: 온라인 상태에서 방문할 때 앱이 업데이트됩니다
- **백그라운드 업데이트**: 업데이트가 백그라운드에서 다운로드됩니다
- **즉시 사용 가능**: 새 기능을 즉시 사용할 수 있습니다
- **사용자 작업 없음**: 스토어 앱과 달리 수동 업데이트가 필요하지 않습니다

### 설치 중에 앱 이름을 커스터마이징할 수 있나요?

예, 설치 중에 다음을 할 수 있습니다:
- 앱 이름 변경 (기본값: "OneUptime")
- 조직 이름 추가
- 커스텀 명명 규칙 사용
- 아이콘 레이블 수정 (플랫폼에 따라 다름)

### OneUptime PWA를 어떻게 제거하나요?

제거 방법은 플랫폼에 따라 다릅니다:

**Android:**
- 앱 아이콘 길게 누르기 → 제거
- 설정 → 앱 → OneUptime → 제거

**iOS:**
- 앱 아이콘 길게 누르기 → 앱 제거 → 앱 삭제

**Windows:**
- 설정 → 앱 → OneUptime → 제거
- 시작 메뉴 항목 오른쪽 클릭 → 제거

**macOS:**
- 응용 프로그램에서 휴지통으로 드래그
- Dock 아이콘 오른쪽 클릭 → 제거

**Linux:**
- 애플리케이션 런처에서 제거
- .desktop 파일 삭제


## 알림 FAQ

### 알림을 받지 못하는 이유는 무엇인가요?

일반적인 알림 문제 및 수정 사항:

**권한 확인:**
```
1. 브라우저 알림 권한이 활성화되어 있는지 확인
2. 운영 체제 알림 권한 확인
3. OneUptime 알림 설정이 구성되어 있는지 확인
4. 방해 금지 모드가 비활성화되어 있는지 확인
```

**플랫폼별:**
- **Android**: 배터리 최적화 설정 확인
- **iOS**: 설정 앱에서 알림 설정 확인
- **Windows**: 집중 지원 설정 확인
- **macOS**: 알림 센터 권한 확인
- **Linux**: 알림 데몬 상태 확인

### 알림 소리를 커스터마이징할 수 있나요?

알림 커스터마이징 옵션:
- **시스템 소리**: OS 알림 소리 설정 사용
- **브라우저 설정**: 브라우저 알림 설정에서 구성
- **OneUptime 설정**: 대시보드에서 알림 설정 지정
- **우선 순위 수준**: 심각도 수준에 따라 다른 소리 구성

### 알림을 일시적으로 비활성화하려면 어떻게 하나요?

알림 일시적 비활성화:
- **방해 금지**: 시스템 DND 모드 활성화
- **브라우저 설정**: 사이트 알림을 일시적으로 비활성화
- **OneUptime 대시보드**: 설정에서 알림 일시 정지
- **집중 모드**: OS 집중/집중력 모드 사용

## 보안 FAQ

### OneUptime PWA는 안전한가요?

보안 기능 및 고려 사항:
- **HTTPS 암호화**: 모든 데이터가 안전하게 전송됨
- **동일 출처 정책**: 브라우저 보안 제한이 적용됨
- **샌드박스 환경**: 브라우저 보안 샌드박스에서 실행됨
- **정기적인 업데이트**: 보안 패치가 자동으로 적용됨
- **루트 액세스 없음**: 네이티브 앱에 비해 제한된 시스템 액세스

*참고: 민감한 데이터는 암호화되어 브라우저 보안 표준을 따릅니다.*

### 기업 네트워크에서 OneUptime PWA를 사용할 수 있나요?

기업 네트워크 고려 사항:
- **방화벽 규칙**: HTTPS (포트 443) 액세스를 확인합니다
- **프록시 구성**: 브라우저 프록시 설정을 구성합니다
- **인증서 신뢰**: 필요한 경우 기업 인증서를 설치합니다
- **VPN 액세스**: 원격 액세스에 VPN을 사용합니다
- **보안 정책**: IT 보안 요구 사항을 준수합니다

## 문제 해결

### 설치 문제

**문제**: 설치 버튼이 나타나지 않음
```
해결책:
1. OneUptime 페이지에서 30초 이상 기다립니다
2. 페이지를 새로 고침하고 다시 기다립니다
3. 브라우저 캐시와 쿠키를 지웁니다
4. 다른 브라우저를 시도합니다 (Chrome/Edge 권장)
5. HTTPS 연결을 확인합니다 (자물쇠 아이콘 확인)
6. 이미 설치되었는지 확인합니다
```

**문제**: 설치가 실패하거나 충돌함
```
해결책:
1. 충분한 저장 공간을 확보합니다 (100MB 이상)
2. 다른 브라우저 탭과 애플리케이션을 닫습니다
3. 브라우저를 최신 버전으로 업데이트합니다
4. 브라우저 확장 프로그램을 일시적으로 비활성화합니다
5. 비공개/시크릿 모드에서 설치를 시도합니다
6. 브라우저를 다시 시작하고 다시 시도합니다
```

**문제**: 앱이 설치되었지만 나타나지 않음
```
해결책:
1. 모든 앱 런처 위치를 확인합니다
2. 기기 검색에서 "OneUptime"을 검색합니다
3. 브라우저의 앱 관리 섹션을 확인합니다
4. 시스템이 새로 고침될 때까지 1-2분 기다립니다
5. 기기를 다시 시작하고 다시 확인합니다
```

**문제**: 앱이 자주 충돌함
```
해결책:
1. 브라우저를 최신 버전으로 업데이트합니다
2. OneUptime의 모든 브라우저 데이터를 지웁니다
3. 브라우저 확장 프로그램을 비활성화합니다
4. 사용 가능한 저장 공간을 확인합니다
5. 운영 체제를 다시 시작합니다
6. OneUptime PWA를 재설치합니다
```

**문제**: 푸시 알림이 작동하지 않음
```
해결책:
1. 브라우저에서 알림 권한을 확인합니다
2. 시스템 알림 설정을 확인합니다
3. 먼저 간단한 알림으로 테스트합니다
4. 알림 데이터를 지우고 권한을 다시 부여합니다
5. 방해 금지/집중 모드 설정을 확인합니다
6. OneUptime 알림 구성을 확인합니다
```

**문제**: 앱이 최신 데이터를 동기화하지 않음
```
해결책:
1. 아래로 당겨 새로 고침합니다 (모바일)
2. Ctrl+F5 (Windows/Linux) 또는 Cmd+R (Mac)을 누릅니다
3. 앱을 닫고 다시 엽니다
4. 앱 캐시를 지우고 다시 로드합니다
5. 네트워크 연결을 확인합니다
```

### 플랫폼별 문제

**Android 문제:**
```
문제: 앱 서랍에 앱이 나타나지 않음
해결책: "최근 추가된" 앱 섹션을 확인하고 앱 서랍에서 검색합니다

문제: 알림이 지연됨
해결책: 브라우저 앱의 배터리 최적화를 비활성화합니다

문제: 시작 시 앱이 충돌함
해결책: Chrome 앱 데이터를 지우고 기기를 다시 시작합니다
```

**iOS 문제:**
```
문제: 홈 화면에 추가할 수 없음
해결책: Safari 브라우저를 사용하고 iOS 11.3 이상인지 확인합니다

문제: 앱 아이콘이 없음
해결책: 모든 홈 화면 페이지와 앱 라이브러리를 확인합니다

문제: Face ID가 작동하지 않음
해결책: 설정에서 Safari에 Face ID를 활성화합니다
```

**Windows 문제:**
```
문제: 시작 메뉴에 앱이 나타나지 않음
해결책: 앱 이름으로 검색하고 설치된 앱 목록을 확인합니다

문제: 알림이 표시되지 않음
해결책: Windows 알림 설정을 확인하고 브라우저에 활성화합니다

문제: 창 크기 조정 문제
해결책: 수동으로 크기를 조정하면 앱이 크기를 기억합니다
```

**macOS 문제:**
```
문제: Safari를 통해 설치할 수 없음
해결책: macOS Sonoma 이상으로 업데이트하고 파일 → Dock에 추가를 사용합니다

문제: 응용 프로그램 폴더에 앱이 없음
해결책: Launchpad를 확인하고 Spotlight 검색을 사용합니다

문제: 알림이 작동하지 않음
해결책: 시스템 환경설정 → 알림을 확인합니다
```

**Linux 문제:**
```
문제: PWA 설치 옵션이 없음
해결책: Chrome/Chromium을 사용하고 데스크탑 환경 지원을 확인합니다

문제: 런처에 아이콘이 나타나지 않음
해결책: 데스크탑 데이터베이스를 업데이트하고 .desktop 파일을 확인합니다

문제: 오디오 알림이 작동하지 않음
해결책: PulseAudio를 확인하고 브라우저 오디오 권한을 확인합니다
```

### 오류 메시지

**"이 사이트를 설치할 수 없습니다"**
```
원인:
- OneUptime 인스턴스가 PWA 요구 사항을 충족하지 않음
- 웹 앱 매니페스트가 누락되거나 잘못됨
- HTTPS가 올바르게 구성되지 않음
- 브라우저가 PWA 설치를 지원하지 않음

해결책:
- 관리자에게 PWA 설정을 확인하도록 요청합니다
- 다른 브라우저를 시도합니다
- 자세한 오류에 대한 브라우저 콘솔을 확인합니다
```
