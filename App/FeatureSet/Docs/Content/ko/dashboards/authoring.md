# 대시보드 작성

**대시보드 → Create Dashboard**에서 대시보드를 만들고, 이름을 부여한 다음 엽니다. 캔버스는 위젯을 받을 준비가 된 **Edit** 모드로 열립니다.

## 캔버스

대시보드는 그리드입니다. 기본 캔버스는 **너비 12 대시보드 단위** × **높이 60 단위**입니다 — 아래쪽에 행을 추가해 높이를 늘릴 수 있습니다. 각 단위는 뷰포트에 따라 확장되는 정사각형입니다: 데스크톱에서는 폰보다 넓지만, 모든 위젯은 비율을 유지합니다.

위젯은 단위의 사각형을 차지합니다. 위치(왼쪽 위 모서리, 캔버스의 왼쪽 위에서 단위로 측정)와 크기(단위로 측정된 너비와 높이) 모두를 결정합니다. 최소 차원은 작은 위젯도 여전히 읽을 수 있도록 강제합니다.

## 편집(Edit) vs. 보기(View)

페이지 헤더의 토글은 두 모드 사이를 전환합니다:

- **Edit** — 위젯 팔레트가 열려 있고, 위젯은 끌어다 놓고 크기 조절이 가능하며, 모든 위젯은 설정 톱니바퀴를 가집니다. 만들 때 사용합니다.
- **View** — 대시보드가 읽기 전용으로 렌더됩니다. 보기 전용 접근을 가진 사람(또는 공개 방문자)이 보는 그대로입니다. 공유 전에 결과를 확인할 때 사용합니다.

같은 대시보드가 두 모드에서 보입니다 — 별도의 "publish" 단계는 없습니다. 편집 저장은 모든 뷰어에게 즉시 효과가 있습니다.

## 위젯 추가

1. 위젯 팔레트를 엽니다(Edit 모드의 **+** 버튼).
2. 위젯 타입을 고릅니다. 카탈로그는 [위젯](/docs/dashboards/widgets)을 참고하세요.
3. 위젯은 기본 크기로 다음 비어 있는 위치에 캔버스에 떨어집니다.
4. 위젯의 톱니바퀴를 클릭해 설정 패널을 엽니다.
5. 데이터 소스(메트릭 쿼리, 목록 필터, 텍스트 본문 등)와 표시 옵션(임계값, 단위, 축, 컬럼)을 설정합니다.
6. 위젯을 끌어 위치 지정합니다. 모서리를 끌어 크기를 조절합니다.

반복합니다. 그리드는 위젯을 정수 단위 경계에 스냅합니다.

## 데이터 소스 설정

대부분의 위젯은 세 곳 중 하나에서 읽습니다:

- **메트릭** — ClickHouse 기반 메트릭 쿼리. 위젯은 `metricQueryConfig`(단일 시리즈) 또는 `metricQueryConfigs`(여러 시리즈를 쌓거나 겹치기)를 만듭니다. 선택적 `transformAsRate`는 OpenTelemetry 누적 카운터를 변화율로 변환합니다. 선택적 `formula`는 두 쿼리를 결합하게 해줍니다(예: 오류 수 / 총 수).
- **라이브 리소스 목록** — 인시던트, 알람, 모니터, Kubernetes 리소스, Docker 리소스, 호스트. 각 목록 위젯은 필터(예: 라벨, 상태, 네임스페이스)를 받아 일치하는 행을 라이브로 표시합니다.
- **정적 콘텐츠** — **Text** 위젯은 Markdown 본문을 받습니다. 헤더, 구분선, runbook 링크, "이 대시보드가 무엇인가?" 같은 주석에 사용하세요.

메트릭 위젯의 경우, 설정은 OneUptime의 다른 곳에서 보는 인라인 쿼리 빌더를 미러링합니다 — 메트릭을 고르고, 집계를 고르고, `WHERE` 필터를 추가하고, 시간 그룹화를 선택합니다. 쿼리는 프로젝트의 텔레메트리 데이터에 대해 실행됩니다.

## 임계값과 형식

단일 숫자를 표시하는 위젯(**Value**, **Gauge**)은 선택적 임계값을 받습니다:

- **Warning threshold** — 값이 이를 넘으면 노란색으로 렌더.
- **Critical threshold** — 값이 이를 넘으면 빨간색으로 렌더.

차트는 Y축 단위, 범례 위치, 시리즈를 쌓을지 여부를 설정할 수 있습니다. 테이블은 표시할 컬럼과 행 한도를 고를 수 있습니다.

## 시간 범위와 새로고침

대시보드 헤더는 모든 메트릭 위젯에 영향을 주는 두 가지 글로벌 컨트롤을 가집니다:

- **시간 범위** — 프리셋(과거 1시간, 24시간, 7일, 30일) 또는 커스텀 범위를 고릅니다. 모든 메트릭 위젯이 이 창에 대해 쿼리합니다.
- **새로고침 간격** — Off, 5s, 10s, 30s, 1m, 5m, 15m. 선택된 주기로 모든 위젯의 쿼리를 다시 실행합니다. 네이티브로 웹소켓을 지원하는 목록 위젯은 선택된 간격과 관계없이 푸시로 업데이트합니다.

글로벌 시간 범위를 무시하는 위젯(예: 텍스트 블록)의 경우 컨트롤은 무동작입니다.

## 저장

캔버스는 편집 중 자동 저장합니다. 헤더의 작은 표시기가 가장 최근 변경이 영구 저장되었을 때 알려줍니다. "publish"는 없습니다 — 모든 편집은 저장되는 순간 라이브입니다. 위험한 변경을 한다면, 대시보드를 먼저 복제하세요.

## 잘 먹히는 패턴

- **대시보드당 하나의 주제.** "우리가 모니터링하는 모든 것"을 한 페이지에 두려는 유혹에 저항하세요. `oncall-checkout`, `oncall-payments`, `oncall-search`로 라벨된 세 개의 대시보드가 하나의 메가 대시보드보다 더 잘 늙습니다.
- **페이지 상단을 가장 중요한 위젯으로 닻을 내리세요.** 사람들은 위에서 스캔합니다 — 그들이 보는 첫 번째 것이 "이 시스템이 건강한가?"에 대한 답이 되도록 하세요.
- **섹션을 라벨링하기 위해 Text 위젯을 사용하세요.** 몇 행마다 짧은 헤딩("Latency" / "Errors" / "Capacity")이 방 건너편에서도 대시보드를 스캔 가능하게 만듭니다.
- **중복 대신 변수를 사용하세요.** 두 서비스에 대해 같은 대시보드를 두 번 만들고 있다면, `service` 변수가 필요합니다. [변수 및 필터](/docs/dashboards/variables)를 참고하세요.
