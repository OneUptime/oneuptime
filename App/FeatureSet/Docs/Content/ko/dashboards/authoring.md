# 대시보드 작성하기

대시보드를 만들려면 **Dashboards → Create Dashboard**를 열고 이름을 입력한 후 엽니다. 캔버스가 **Edit** 모드로 열리며 위젯을 추가할 준비가 됩니다.

## 캔버스

대시보드는 그리드입니다. 위젯이 제자리에 배치되며, 각 위젯의 위치와 크기를 결정할 수 있습니다. 행을 추가하면서 페이지를 아래로 확장할 수 있습니다. 모든 위젯은 더 크거나 작은 화면에서도 비율을 유지합니다.

## Edit와 View

헤더의 토글로 두 모드를 전환할 수 있습니다.

- **Edit** — 위젯 팔레트가 열려 있고, 위젯을 끌어 이동, 크기 조정, 클릭으로 설정 변경이 가능합니다.
- **View** — 대시보드가 읽기 전용입니다. 방문자와 다른 팀원이 보는 것과 동일한 화면입니다. 공유 전에 결과를 확인하기에 적합합니다.

두 모드에서 동일한 대시보드입니다. 별도의 "게시" 단계는 없습니다. 모든 편집은 저장되는 순간 즉시 반영됩니다.

## 위젯 추가하기

1. **+** 버튼을 클릭하여 위젯 팔레트를 엽니다.
2. 위젯 유형을 선택합니다. 카탈로그는 [위젯](/docs/dashboards/widgets)을 참조하시기 바랍니다.
3. 위젯이 캔버스에 표시됩니다.
4. 위젯의 톱니바퀴 아이콘을 클릭하여 설정을 엽니다.
5. 데이터 소스(메트릭, 목록 필터, 텍스트 단락 등)와 표시 옵션을 선택합니다.
6. 위젯을 끌어 이동합니다. 모서리를 끌어 크기를 조정합니다.

## 데이터의 출처

대부분의 위젯은 세 가지 곳 중 하나에서 데이터를 읽습니다.

- **메트릭** — 메트릭과 집계(평균, 최대, 카운트, 백분위수)를 선택합니다. 필터를 추가합니다. 결과 그룹화 방식을 선택합니다. 이것은 OneUptime 다른 곳에서 사용하는 것과 동일한 쿼리 빌더입니다.
- **실시간 목록** — 인시던트, 알림, 모니터, Kubernetes 파드, Docker 컨테이너, 호스트. 각 목록 위젯은 필터를 받고 일치하는 항목을 실시간으로 업데이트하여 보여줍니다.
- **정적 콘텐츠** — **Text** 위젯은 Markdown 블록을 받습니다. 제목, 컨텍스트, 런북 링크, 또는 인시던트 중 임시 메모에 사용하시기 바랍니다.

## 임계값과 서식

단일 값 위젯(**Value**, **Gauge**)에서는 다음을 설정할 수 있습니다.

- **경고 임계값** — 값이 이를 넘으면 노란색으로 바뀝니다.
- **심각 임계값** — 값이 이를 넘으면 빨간색으로 바뀝니다.

차트에서는 Y축 단위를 설정하고, 범례 위치를 선택하며, 시리즈를 서로 위에 쌓을지 겹쳐서 표시할지 선택할 수 있습니다. 테이블에서는 표시할 열과 행 개수를 선택할 수 있습니다.

## 시간 범위 및 새로고침

대시보드 상단에 모든 메트릭 위젯에 영향을 미치는 두 개의 컨트롤이 있습니다.

- **시간 범위** — 프리셋(지난 1시간, 24시간, 7일, 30일) 또는 사용자 지정 범위. 모든 차트와 숫자가 이 시간 창을 사용합니다.
- **새로고침** — 위젯이 다시 쿼리하는 주기. 끄기, 5초, 10초, 30초, 1분, 5분, 15분. 실시간 목록은 이 설정과 관계없이 자체적으로 업데이트됩니다.

시간 범위를 사용하지 않는 위젯(예: Text 위젯)은 두 컨트롤을 모두 무시합니다.

## 저장

캔버스는 작업하는 동안 자동으로 저장됩니다. 헤더의 작은 표시기로 최신 변경 사항의 저장 여부를 알 수 있습니다. 큰 변경 작업을 하는 경우, 먼저 대시보드를 복제하여 안전한 사본을 만들어 두시기 바랍니다.

## 오랫동안 잘 활용되는 대시보드를 위한 팁

- **대시보드 하나당 하나의 주제.** "우리가 모니터링하는 모든 것"을 한 페이지에 넣지 마시기 바랍니다. 집중된 대시보드 여러 개가 거대한 페이지 하나보다 낫습니다.
- **가장 중요한 위젯을 맨 위에.** 사람은 위에서 아래로 훑어봅니다. 가장 먼저 보이는 것이 "이 시스템은 정상인가?"에 대한 답이 되도록 하시기 바랍니다.
- **Text 위젯으로 섹션에 라벨 표시.** 몇 행마다 짧은 제목("Latency", "Errors", "Capacity")을 두면 멀리서도 페이지를 훑어볼 수 있습니다.
- **복제 대신 변수 사용.** 두 번째 서비스에 같은 대시보드를 빌드하려 한다면, 대신 `service` 변수가 있는 하나의 대시보드를 빌드하시기 바랍니다. [변수 및 필터](/docs/dashboards/variables)를 참조하세요.
