# 변수 및 필터

변수는 단일 대시보드를 템플릿으로 바꿉니다. `service` 변수를 정의하면 같은 차트가 `checkout`, `payments`, `search`에 대해 다시 렌더됩니다 — 세 개의 거의 동일한 대시보드를 만드는 대신 상단의 드롭다운에서 고릅니다.

이 페이지는 네 가지 변수 타입, 그 값이 위젯 쿼리에 주입되는 방법, 그리고 그 옆에 자리한 글로벌 시간 범위와 새로고침 컨트롤을 다룹니다.

## 변수 타입

**대시보드 → Settings → Variables**에서 변수를 추가합니다. 각각은 이름(위젯 쿼리에서 `{{name}}`으로 참조), 선택적 라벨, 타입을 가집니다.

### Custom List

정적 드롭다운. 콤마로 구분된 값 목록을 제공하면 뷰어가 하나를 고릅니다.

사용 시점: 선택지 집합이 작고, 고정되어 있으며, 팀에만 의미 있을 때. `prod, staging, dev` 값을 가진 `environment`. `us-east-1, eu-west-1, ap-south-1` 값을 가진 `region`.

### Query

드롭다운의 옵션은 렌더 시점에 ClickHouse 쿼리로 계산됩니다.

사용 시점: 선택지가 동적이고 텔레메트리에 살 때. `SELECT DISTINCT customer_id FROM ...`을 통해 "과거 24시간에 로그인한 모든 고객 ID". 쿼리는 프로젝트의 데이터에 대해 실행됩니다; 자신의 데이터일지라도 결과를 신뢰할 수 없는 입력으로 취급하세요.

### Text Input

자유 텍스트 필드. 뷰어가 입력하는 모든 것이 주입됩니다.

사용 시점: 대시보드가 검색 도구처럼 작동하길 원할 때. "IP로 필터" 또는 "요청 ID로 필터" 대시보드.

### Telemetry Attribute

옵션은 대시보드의 시간 범위에 걸친 프로젝트 텔레메트리의 OpenTelemetry 속성 키의 고유 값입니다.

**속성 키**(예: `k8s.cluster.name`, `service.name`, `host.name`)를 설정합니다. 위젯은 로그/메트릭/트레이스에서 고유 값을 가져와 드롭다운으로 제공합니다.

사용 시점: 선택지가 정확히 텔레메트리에 이미 태그한 엔티티일 때. 클러스터 이름, 서비스 이름, 리전, 고객 ID, 배포 환경 — OpenTelemetry 리소스나 스팬 속성으로 이미 보내는 모든 것.

이것은 서비스 지향 대시보드에서 가장 흔한 변수 타입입니다. 자동 업데이트되기 때문이죠: `service.name = inventory`로 태그된 새 서비스를 출시하면, 그 값이 누가 대시보드를 편집하지 않아도 드롭다운에 나타납니다.

## 다중 선택

각 변수는 **다중 선택**으로 설정할 수 있습니다. 켜져 있으면, 뷰어는 하나 이상의 값을 고르고; 대시보드는 `value = ...` 대신 `value IN (...)`로 필터합니다.

다중 선택 사용 시점: 대시보드를 떠나지 않고 "checkout + payments를 함께" 보고 싶을 때. 선택된 값 전반에 차트 수학이 맞지 않을 때 — 예: 평균의 평균 — 피하세요.

## 기본값

모든 변수는 선택적 기본값을 받습니다. 뷰어가 드롭다운을 바꿀 때까지 대시보드는 기본값으로 렌더됩니다. 공개 대시보드의 경우 기본값은 방문자가 도착하는 것입니다.

## 보간이 작동하는 방식

위젯 쿼리가 문자열 필터를 받는 어디서든 — 메트릭 쿼리의 `WHERE` 절, 목록 위젯의 필터, 로그 스트림의 속성 일치 — `{{variable_name}}`을 참조할 수 있습니다.

예를 들어, Chart의 메트릭 쿼리는 다음과 같을 수 있습니다:

```
SELECT avg(latency_ms) FROM spans WHERE service.name = '{{service}}'
```

`service`가 `checkout`으로 설정되면, 쿼리는 `service.name = 'checkout'`로 실행됩니다. 뷰어가 `payments`로 바꾸면, 쿼리는 `service.name = 'payments'`로 다시 실행됩니다.

**Telemetry Attribute** 변수의 경우 특별히, OneUptime이 속성 키를 알고 있어 같은 속성을 언급하는 모든 위젯에 필터를 주입합니다 — 변수가 바뀔 때 각 위젯의 쿼리를 손으로 편집할 필요가 없습니다. 이것이 서비스 템플릿 대시보드가 기본으로 작동하게 하는 마법입니다.

## 시간 범위

대시보드 헤더는 글로벌 **시간 범위** 선택기를 가집니다. 모든 메트릭 위젯이 이 창에 대해 쿼리합니다. 선택지:

- **프리셋** — 과거 1시간, 24시간, 7일, 30일, 90일(보존에 따라).
- **커스텀 범위** — 시작과 끝 타임스탬프를 고릅니다.

시간 범위는 대시보드의 URL의 일부입니다 — URL 공유는 창을 공유합니다. 이는 인시던트 동안 편리합니다: 시간 범위를 "오늘 UTC 10:00–10:30"으로 고정하고 인시던트 채널에 링크를 공유하세요.

## 새로고침 간격

시간 범위 옆에서 위젯이 다시 쿼리하는 빈도를 고릅니다:

- **Off** — 위젯은 로드 시 한 번 쿼리합니다.
- **5s / 10s / 30s / 1m / 5m / 15m** — 자동 새로고침.

자동 새로고침은 벽걸이 화면과 현재 인시던트 뷰에 편리합니다. 임시 조사를 위해서는 꺼두어 스크롤하는 동안 뷰가 안정되도록 하세요.

## 통합 사용

서비스 템플릿 대시보드는 일반적으로 다음을 가집니다:

1. `service.name`에 바인딩된 **Telemetry Attribute** 타입의 `service` 변수. 기본값: 가장 자주 보는 서비스. 다중 선택: 끔(차트가 항상 한 번에 한 서비스를 보여주도록).
2. **Custom List** 타입의 `environment` 변수. 기본값: `prod`.
3. `k8s.cluster.name`에 바인딩된 **Telemetry Attribute** 타입의 `cluster` 변수. 다중 선택: 켬(클러스터 전반에 걸쳐 롤업할 수 있도록).
4. 대시보드의 위젯이 필터에서 이 변수를 참조합니다.

결과: 하나의 대시보드, 전체 플릿의 커버리지, 상단의 몇 개 드롭다운.
