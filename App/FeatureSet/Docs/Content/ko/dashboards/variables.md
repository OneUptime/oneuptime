# 변수 및 필터

변수는 하나의 대시보드를 템플릿으로 만들어 줍니다. 대시보드에 `service` 변수를 추가하면 같은 차트가 `checkout`, `payments`, `search`에 대해 다시 렌더링됩니다. 거의 동일한 세 개의 대시보드를 빌드하는 대신, 보는 사람이 상단의 드롭다운에서 고르도록 할 수 있습니다.

## 변수 유형

**Dashboard → Settings → Variables**에서 변수를 추가합니다. 각 변수에는 이름(위젯에서 `{{name}}`으로 사용), 선택적 라벨, 그리고 유형이 있습니다.

### Custom List

정적인 드롭다운입니다. 옵션을 직접 입력하시면 됩니다.

용도: 선택지가 작고 고정된 경우. `prod, staging, dev` 값을 가진 `environment`. `us-east-1, eu-west-1, ap-south-1` 값을 가진 `region` 등에 사용합니다.

### Query

옵션이 데이터에 대한 쿼리에서 나옵니다.

용도: 선택지가 시간이 지남에 따라 변하고 드롭다운이 이를 따라잡길 원할 때. "지난 24시간 동안 본 모든 고객 ID." 쿼리가 프로젝트의 데이터에 대해 실행되며 결과가 드롭다운이 됩니다.

### Text Input

자유 텍스트 필드입니다. 보는 사람이 입력한 모든 내용이 사용됩니다.

용도: 대시보드가 검색 도구처럼 동작하길 원할 때. IP 주소, 요청 ID 또는 다른 자유 형식 값으로 필터링할 수 있습니다.

### Telemetry Attribute

옵션은 대시보드의 시간 범위 동안 텔레메트리에서 본 속성의 고유 값입니다.

**attribute key**(예: `service.name`, `host.name`, `k8s.cluster.name`)를 구성합니다. 드롭다운은 로그, 메트릭, 트레이스에서 본 모든 고유 값으로 채워집니다.

용도: 선택지가 텔레메트리와 함께 이미 보내는 태그와 일치할 때. 새 서비스를 `service.name = inventory`로 태그하여 배포하면 대시보드를 편집하지 않아도 그 이름이 드롭다운에 나타나기 때문에, 가장 일반적으로 사용되는 유형입니다.

## 다중 선택

각 변수는 여러 선택을 허용할 수 있습니다. 켜져 있으면 보는 사람이 하나 이상의 값을 선택할 수 있으며, 대시보드는 그중 하나라도 일치하는 것을 필터링합니다.

용도: 대시보드를 벗어나지 않고 "checkout과 payments를 함께" 비교하고 싶을 때. 선택한 값들에 걸쳐 계산이 의미가 없는 경우(예: 평균의 평균)에는 피하시기 바랍니다.

## 기본값

모든 변수는 기본값을 가질 수 있습니다. 보는 사람이 변경할 때까지 대시보드는 기본값으로 렌더링됩니다. 공개 대시보드의 경우 방문자가 가장 먼저 보는 것이 기본값입니다.

## 위젯에서 변수 사용하기

위젯이 필터를 받는 곳(메트릭의 `WHERE`, 목록의 필터, 로그 스트림의 속성 일치)이라면 어디든 `{{variable_name}}`을 사용할 수 있습니다.

예를 들어, 서비스로 필터링된 차트의 예시는 다음과 같습니다.

```
service.name = '{{service}}'
```

드롭다운이 `checkout`으로 설정되면 차트는 결제 서비스로 필터링됩니다. 보는 사람이 `payments`로 전환하면 차트가 결제용으로 다시 렌더링됩니다.

**Telemetry Attribute** 변수의 경우 OneUptime은 변수가 어떤 속성에 매핑되는지 알고 있어서, 동일한 속성을 사용하는 모든 위젯에 필터를 자동으로 적용합니다. 위젯을 하나씩 손으로 편집할 필요가 없습니다.

## 시간 범위

대시보드 헤더에는 전역 시간 범위가 있습니다. 모든 메트릭 위젯이 이 시간 창에 대해 쿼리합니다. 옵션은 다음과 같습니다.

- **프리셋** — 지난 1시간, 24시간, 7일, 30일, 90일(데이터 보관 기간에 따라 다름).
- **사용자 지정** — 시작 시간과 종료 시간을 선택합니다.

시간 범위는 대시보드의 URL의 일부입니다. URL을 공유하면 시간 창도 공유됩니다. 인시던트 중 유용합니다. 시간 범위를 "오늘 10:00-10:30 UTC"로 고정하고 인시던트 채널에 링크를 붙여넣을 수 있습니다.

## 새로고침 간격

시간 범위 옆에서 위젯이 다시 쿼리하는 주기를 선택합니다.

- **Off** — 페이지가 로드될 때 한 번 쿼리합니다.
- **5초 / 10초 / 30초 / 1분 / 5분 / 15분** — 자동 새로고침.

자동 새로고침은 벽걸이 화면이나 실시간 인시던트 뷰에 적합합니다. 조사 중일 때는 보고 있는 뷰가 그대로 유지되도록 끄시기 바랍니다.

## 종합

서비스 템플릿 대시보드는 보통 다음과 같이 구성됩니다.

1. `service.name`에 대한 **Telemetry Attribute** 유형의 `service` 변수. 기본값: 가장 많이 관찰하는 서비스. 다중 선택 꺼짐(차트가 항상 한 번에 하나만 표시되도록).
2. **Custom List** 유형의 `environment` 변수. 기본값: `prod`.
3. `k8s.cluster.name`에 대한 **Telemetry Attribute** 유형의 `cluster` 변수. 다중 선택 켜짐(클러스터 간 비교 가능).
4. 필터에서 이 변수들을 참조하는 위젯들.

결과: 하나의 대시보드, 모든 서비스 커버, 상단에 세 개의 드롭다운.
