# 위젯

위젯은 대시보드의 한 타일입니다. 각 위젯은 타입(차트, 값, 목록…), 위치, 크기, 설정을 가집니다. 이 페이지는 카탈로그입니다 — 각 위젯이 보여주는 것, 입력으로 받는 것, 손에 닿는 시점.

캔버스 동작은 [대시보드 작성](/docs/dashboards/authoring)을 참고하세요.

## 시계열 위젯

### Chart

대시보드의 시간 범위에 걸친 하나 이상의 메트릭 시리즈의 선/막대/영역 차트.

**설정**:

- 하나 이상의 메트릭 쿼리(단일 시리즈는 `metricQueryConfig`, 여러 개는 `metricQueryConfigs`).
- 여러 쿼리를 결합하는 선택적 **formula** (예: `errors / total * 100`).
- OpenTelemetry 누적 카운터(예: `system.disk.io`)를 위한 선택적 **transformAsRate** — 위젯이 버킷당 `(value - previousValue) / Δt`을 계산합니다.
- 표시: 시리즈를 쌓을지 겹칠지, Y축 단위, 범례 켜기/끄기, 차트 타입.

사용 시점: 추세가 중요할 때. 요청 지연, 시간 경과에 따른 오류 수, 큐 깊이, 곡선의 모양이 무언가를 말해 주는 모든 것.

### Value

선택적 임계값과 선택적 스파크라인을 가진 큰 단일 숫자.

**설정**:

- 메트릭 쿼리(단일 값 — 보통 시간 범위에 대한 `last`, `avg`, `max`).
- 선택적 **warning threshold**(이상이면 노란색).
- 선택적 **critical threshold**(이상이면 빨간색).
- 표시: 숫자 형식, 단위 접미사.

사용 시점: 단일 숫자가 질문에 답할 때. 현재 오류율, 지금의 P95 지연, 열린 인시던트 수.

### Gauge

최소, 최대, 경고 밴드, 위험 밴드를 가진 원형 게이지.

**설정**: 메트릭 쿼리와 네 가지 경계(최소, 최대, 경고, 위험).

사용 시점: 값이 알려진 범위 안에 있을 때. CPU 사용률(0–100%), 디스크 채움, 큐 용량.

### Table

메트릭 쿼리 결과의 테이블 형태 표시, 그룹당 한 행.

**설정**: 메트릭 쿼리(보통 `host.name` 또는 `service.name` 같은 라벨로 그룹화됨), 보여줄 컬럼, 행 한도.

사용 시점: 추세보다 분해를 원할 때. 가장 소란스러운 호스트 상위 10개, 서비스당 오류 수, 엔드포인트당 요청률.

## 주석 위젯

### Text

Markdown의 정적 블록.

**설정**: Markdown 본문. 헤딩, 목록, 링크, 강조, 코드 스팬, 펜스드 코드 블록이 모두 렌더됩니다.

사용 시점: 섹션 헤더, 컨텍스트 단락("이 대시보드는 checkout 서비스를 다룹니다"), runbook이나 관련 대시보드로의 링크 목록, 또는 인시던트 동안의 임시 배너를 원할 때.

## 로그와 트레이스

### LogStream

필터에 일치하는 로그 줄의 라이브 꼬리.

**설정**: 로그 필터(서비스, 심각도, 속성 일치), 보여줄 컬럼.

사용 시점: 페이지를 떠나 로그 탐색기를 열지 않고 애플리케이션이 *지금* 무슨 말을 하고 있는지 대시보드에서 보고 싶을 때.

### TraceList

필터에 일치하는 최근 트레이스의 목록으로, 소요 시간, 상태, 서비스 이름과 함께.

**설정**: 트레이스 필터(서비스, 상태, 속성 일치).

사용 시점: 차트가 아닌 최근 활동의 페이지네이션된 뷰를 원할 때. 흔한 짝지음: 상단의 지연 Chart, 아래에 느린 트레이스의 TraceList.

## 운영 목록

### IncidentList

필터에 일치하는 인시던트의 라이브 목록.

**설정**: 상태, 심각도, 라벨, 모니터, 배정된 팀으로 필터링.

사용 시점: 대시보드가 "지금 무엇이 깨졌는가?"에 답하기 위한 것일 때.

### AlertList

필터에 일치하는 알람의 라이브 목록.

**설정**: 상태, 심각도, 라벨로 필터링.

사용 시점: 알람 주도 워크플로우용 대시보드(예: 자기 서비스의 알람을 보는 dev-팀 대시보드).

### MonitorList

필터에 일치하는 모니터의 라이브 목록, 각 모니터의 현재 상태를 보여줍니다.

**설정**: 모니터 타입, 라벨, 현재 상태로 필터링.

사용 시점: 플릿 수준의 "모든 웹사이트가 작동하는가?" 뷰 또는 모니터링되는 엔드포인트의 팀별 목록을 원할 때.

## Kubernetes 리소스 목록

[Kubernetes Agent](/docs/monitor/kubernetes-agent)가 설치된 프로젝트의 경우, 다음 라이브 리소스 위젯을 사용할 수 있습니다. 각각은 `cluster`, `namespace`, 라벨에 대한 선택적 필터를 받습니다.

- **KubernetesPodList** — 페이즈, 재시작, 노드 할당이 있는 파드.
- **KubernetesNodeList** — 컨디션, 용량, 할당이 있는 노드.
- **KubernetesNamespaceList** — 네임스페이스와 워크로드 수.
- **KubernetesDeploymentList** — desired 대 ready 레플리카가 있는 디플로이먼트.
- **KubernetesStatefulSetList** — ready 레플리카가 있는 statefulset.
- **KubernetesDaemonSetList** — desired 대 ready가 있는 daemonset.
- **KubernetesJobList** — 완료 상태가 있는 잡.
- **KubernetesCronJobList** — 스케줄과 마지막 실행이 있는 cronjob.

사용 시점: Kubernetes 리소스 상태와 그 워크로드의 텔레메트리를 혼합한 단일 대시보드를 원할 때.

## Docker 리소스 목록

Docker 모니터가 설치된 프로젝트의 경우:

- **DockerHostList** — 컨테이너 수와 함께 Docker를 실행하는 호스트.
- **DockerContainerList** — 상태, 이미지, 호스트, 업타임이 있는 컨테이너.
- **DockerImageList** — 이미지와 크기.
- **DockerNetworkList** — Docker 네트워크와 연결된 컨테이너 수.
- **DockerVolumeList** — Docker 볼륨과 사용량.

## 인프라

### HostList

OneUptime 서버 모니터가 모니터링하는 호스트 — 현재 상태, CPU, 메모리, 업타임과 함께.

**설정**: 라벨이나 현재 헬스 상태로 필터링.

## 알맞은 위젯 고르기

빠른 판단 가이드:

- **시간 경과에 따른 추세?** Chart.
- **지금 중요한 하나의 숫자?** Value(자연스러운 범위가 있다면 Gauge).
- **많은 것 사이의 분해?** Table.
- **시스템에서 지금 무슨 일이 일어나고 있는가?** LogStream, TraceList, IncidentList.
- **특정 리소스 플릿의 상태?** 그에 맞는 리소스 목록 위젯.
- **헤딩, 단락, 또는 링크?** Text.

대부분의 대시보드는 혼합을 사용합니다 — 상단에 Chart, 옆에 Value 한두 개, Text 구분선, 그 아래에 한두 개의 목록.
