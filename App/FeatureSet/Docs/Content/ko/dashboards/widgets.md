# 위젯

위젯은 대시보드의 한 타일입니다. 이 페이지는 추가할 수 있는 모든 위젯, 각 위젯이 보여주는 내용, 그리고 어떤 상황에서 사용할지 정리한 카탈로그입니다.

위젯을 캔버스에서 끌어다 놓는 방법은 [대시보드 작성하기](/docs/dashboards/authoring)를 참조하시기 바랍니다.

## 차트와 숫자

### Chart

대시보드의 시간 범위에 걸쳐 하나 이상의 메트릭 시리즈를 선, 막대, 또는 영역 차트로 표시합니다.

**설정**:

- 하나 이상의 메트릭 쿼리.
- 두 쿼리를 결합하는 선택적 공식(예: 오류율을 얻기 위한 `errors / total * 100`).
- 리셋 없이 증가하는 누적 카운터를 위한 "show as rate" 옵션.
- 표시 옵션: 누적 또는 겹침, Y축 단위, 범례 위치, 차트 유형.

용도: 추세가 중요한 경우. 시간 경과에 따른 지연, 오류 수, 큐 깊이 등 선의 형태가 이야기를 전달하는 경우에 사용합니다.

### Value

선택적인 색상 임계값이 있는 단일 큰 숫자입니다.

**설정**:

- 하나의 숫자를 반환하는 메트릭 쿼리(마지막 값, 평균, 또는 시간 범위 내 최대값).
- 선택적인 **경고** 임계값(이상에서 노란색).
- 선택적인 **심각** 임계값(이상에서 빨간색).
- 숫자 형식과 단위.

용도: 하나의 숫자가 질문에 답할 때. 현재 오류율, 지금 P95 지연, 열린 인시던트 수 등.

### Gauge

최솟값, 최댓값, 경고 구간, 심각 구간이 있는 원형 게이지입니다.

**설정**: 메트릭 쿼리와 네 가지 경계.

용도: 값이 알려진 범위 안에 들어가는 경우. CPU 백분율(0~100%), 디스크 사용량, 큐 용량 등에 사용합니다.

### Table

메트릭 결과를 그룹당 한 행씩 표시하는 테이블입니다.

**설정**: 메트릭 쿼리(보통 호스트나 서비스 같은 라벨로 그룹화), 표시할 열, 행 제한.

용도: 추세 대신 분석을 원할 때. 가장 시끄러운 호스트 10개, 서비스당 오류 수, 엔드포인트당 요청 수 등에 사용합니다.

## Text

정적인 Markdown 블록입니다.

**설정**: Markdown 본문. 제목, 목록, 링크, 강조, 코드 블록이 모두 렌더링됩니다.

용도: 섹션 제목, 컨텍스트 단락, 런북 링크 목록, 또는 인시던트 중 임시 배너 등에 사용합니다.

## 로그와 트레이스

### Log Stream

필터와 일치하는 로그 줄의 실시간 테일입니다.

**설정**: 로그 필터(서비스, 심각도, 속성)와 표시할 열.

용도: 대시보드를 벗어나지 않고 애플리케이션이 지금 무엇을 말하고 있는지 보고 싶을 때.

### Trace List

필터와 일치하는 최근 트레이스 목록입니다. 소요 시간, 상태, 서비스가 표시됩니다.

**설정**: 트레이스 필터(서비스, 상태, 속성).

용도: 차트 대신 최근 활동 목록을 원할 때. 일반적인 패턴은 상단에 지연 차트를 두고 아래에 느린 트레이스 목록을 두는 것입니다.

## 실시간 목록

### Incident List

필터와 일치하는 인시던트의 실시간 목록입니다.

**설정**: 상태, 심각도, 라벨, 모니터 또는 팀별 필터.

용도: 대시보드가 "지금 무엇이 망가졌나?"에 답할 때.

### Alert List

필터와 일치하는 알림의 실시간 목록입니다.

**설정**: 상태, 심각도, 라벨별 필터.

용도: 팀 대시보드가 해당 서비스의 알림을 추적할 때.

### Monitor List

모니터와 현재 상태의 실시간 목록입니다.

**설정**: 모니터 유형, 라벨, 또는 현재 상태별 필터.

용도: "모든 사이트가 정상인가?" 같은 전체 보기를 원할 때.

## Kubernetes 리소스 목록

[Kubernetes 에이전트](/docs/monitor/kubernetes-agent)가 설치된 프로젝트용입니다. 각각 클러스터, 네임스페이스, 라벨에 대한 선택적 필터를 받습니다.

- **Kubernetes Pod List** — 단계, 재시작, 노드와 함께 파드를 표시합니다.
- **Kubernetes Node List** — 조건과 용량과 함께 노드를 표시합니다.
- **Kubernetes Namespace List** — 네임스페이스와 워크로드 수.
- **Kubernetes Deployment List** — 원하는 레플리카와 준비된 레플리카가 있는 배포.
- **Kubernetes StatefulSet List** — 준비된 레플리카가 있는 스테이트풀 셋.
- **Kubernetes DaemonSet List** — 원하는 것과 준비된 것이 있는 데몬 셋.
- **Kubernetes Job List** — 작업과 완료 상태.
- **Kubernetes CronJob List** — 스케줄과 마지막 실행이 있는 cron 작업.

용도: 해당 워크로드의 텔레메트리와 Kubernetes 상태를 함께 보여주는 단일 대시보드를 원할 때.

## Docker 리소스 목록

Docker 모니터링이 설정된 프로젝트용입니다.

- **Docker Host List** — Docker를 실행하는 호스트와 컨테이너 수.
- **Docker Container List** — 상태, 이미지, 호스트, 가동 시간이 있는 컨테이너.
- **Docker Image List** — 이미지와 크기.
- **Docker Network List** — Docker 네트워크와 연결된 컨테이너.
- **Docker Volume List** — Docker 볼륨과 사용량.

## 인프라

### Host List

OneUptime의 서버 모니터가 모니터링하는 호스트를 상태, CPU, 메모리, 가동 시간과 함께 표시합니다.

**설정**: 라벨이나 현재 상태별 필터.

## 어떤 위젯을 사용해야 할까요?

몇 가지 간단한 규칙은 다음과 같습니다.

- **시간 경과에 따른 추세?** Chart.
- **지금 중요한 하나의 숫자?** Value(명확한 최솟값/최댓값이 있다면 Gauge).
- **여러 항목에 걸친 분석?** Table.
- **지금 시스템에서 무슨 일이 벌어지고 있는지?** Log Stream, Trace List, Incident List.
- **특정 리소스 그룹의 상태?** 일치하는 목록 위젯.
- **제목, 단락, 또는 링크?** Text.

대부분의 대시보드는 몇 가지를 섞어 사용합니다. 상단에 차트, 옆에 값 한두 개, 텍스트 구분자, 그리고 아래에 목록 한두 개 같은 식으로 구성할 수 있습니다.
