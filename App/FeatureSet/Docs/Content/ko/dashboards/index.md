# 대시보드 개요

대시보드는 OneUptime이 이미 수집하고 있는 데이터 — 메트릭, 로그, 트레이스, 인시던트, 모니터, Kubernetes 리소스, 호스트 — 를 한눈에 보고 무슨 일이 벌어지고 있는지 이해할 수 있는 단일 페이지로 만들어 줍니다.

요청 지연 차트를 열린 인시던트 목록 옆에 놓고, 그 옆에 CPU 게이지를, 그 옆에 상황 설명 단락을 두시면 됩니다. 저장하시고 링크를 공유하세요.

## 대시보드의 활용 사례

- **"모든 것이 정상인가요?" 페이지** — 온콜, 팀 스탠드업, 또는 벽걸이 TV용으로 활용할 수 있습니다.
- **연관성 파악** — CPU 급증, 지연 증가, 열린 인시던트가 동시에 발생하는 것은 세 개의 탭을 오가는 것보다 한 페이지에서 훨씬 쉽게 확인할 수 있습니다.
- **조사** — 디버깅 시, 즉석에서 빌드한 대시보드는 쿼리 10개를 하나씩 실행하는 것보다 효율적입니다.
- **외부 공유** — 고객용 성능 페이지, 파트너 상태 페이지, 오픈소스 프로젝트의 공개 대시보드 등에 활용할 수 있습니다.

## 대시보드에 넣을 수 있는 항목

- **차트** — 지연, 오류, 처리량 등 시간 경과에 따른 추세를 표시합니다.
- **단일 값 타일과 게이지** — 현재 오류율, CPU, 열린 인시던트 수를 표시합니다.
- **테이블** — 가장 시끄러운 호스트 10개, 서비스별 오류 수 등 분석 자료를 보여줍니다.
- **텍스트 블록** — 제목, 컨텍스트, 런북 링크 등에 사용합니다.
- **실시간 목록** — 인시던트, 알림, 모니터, 로그, 트레이스, Kubernetes 리소스, Docker 리소스, 호스트를 보여줍니다.

전체 목록과 각각의 표시 내용은 [위젯](/docs/dashboards/widgets)을 참조하시기 바랍니다.

## 핵심 용어

| 용어 | 의미 |
| --- | --- |
| **대시보드** | 페이지 전체 — 이름, 위젯 그리드, 시간 범위 컨트롤, 변수 목록. |
| **위젯** | 페이지 위의 한 타일 — 차트, 숫자, 목록, 단락. |
| **변수** | 상단의 드롭다운으로 모든 위젯을 한 번에 필터링합니다(클러스터, 서비스, 고객, 환경). |
| **시간 범위** | 모든 차트와 숫자가 사용하는 시간 창입니다. 페이지 상단에서 한 번만 설정하면 됩니다. |
| **새로고침** | 위젯이 데이터를 다시 쿼리하는 주기입니다. 끄기, 몇 초마다, 몇 분마다 등을 설정할 수 있습니다. |
| **모드** | **Edit**(위젯을 끌어 배치) 또는 **View**(읽기 전용, 방문자가 보는 그대로). |

## 대시보드 위치

왼쪽 내비게이션에서 **Dashboards**를 엽니다.

| 페이지 | 할 수 있는 일 |
| --- | --- |
| **Dashboards** | 대시보드 목록입니다. 새로 만들거나, 검색하거나, 라벨로 필터링합니다. |
| **Dashboard → View** | 캔버스입니다. 헤더에서 **Edit**와 **View**를 전환합니다. |
| **Dashboard → Overview** | 설명, 소유자, 라벨. |
| **Dashboard → Settings** | 공개 공유, 비밀번호, IP 허용 목록, 커스텀 도메인, 브랜딩. |
| **Dashboard → Owners** | 명시적인 접근 권한이 있는 사용자와 팀. |
| **Dashboard → Delete** | 대시보드를 제거합니다. |

## 대시보드 빌드하기

1. **만들기** — 이름을 선택합니다. 캔버스가 빈 상태로 열립니다.
2. **위젯 추가** — 위젯 유형을 선택하고, 데이터를 구성한 후 원하는 곳으로 끌어다 놓습니다.
3. **(선택) 변수 추가** — 예를 들어, 모든 서비스에 동일한 대시보드를 사용할 수 있도록 `service` 드롭다운을 추가합니다.
4. **시간 범위 설정** — 기본값으로도 충분합니다. 나중에 조정하시면 됩니다.
5. **(선택) 공개 공유** — Settings에서 스위치를 켜고 필요하면 비밀번호나 IP 허용 목록을 추가합니다.
6. **(선택) 커스텀 도메인** — `status.your-domain.com`에서 대시보드를 호스팅합니다.

## 간단한 예시

목표: 결제 서비스에 대한 지연, 오류율, 열린 인시던트, 실시간 로그 테일을 보여주는 온콜 페이지.

1. "Checkout on-call"이라는 대시보드를 만듭니다.
2. `service` 변수를 추가합니다. 기본값을 `checkout`으로 설정합니다.
3. P95 지연을 보여주는 **Chart** 위젯을 추가하고, `service` 변수로 필터링합니다.
4. 옆에 1%에서 경고, 5%에서 심각으로 설정된 오류율용 **Value** 위젯을 추가합니다.
5. 아래에 `checkout`으로 태그된 인시던트의 **Incident List** 위젯을 추가합니다.
6. 그 아래에는 같은 서비스의 로그를 표시하는 **Log Stream** 위젯을 추가합니다.
7. 저장합니다. 드롭다운을 `payments`로 전환합니다. 동일한 대시보드가 이제 결제 서비스를 표시합니다.

## 대시보드와 OneUptime의 다른 기능 간의 관계

- **모니터와 텔레메트리**는 데이터의 소스입니다. 수집하는 모든 메트릭, 로그, 트레이스를 위젯에서 쿼리할 수 있습니다.
- **인시던트와 알림**은 **Incident List**와 **Alert List** 위젯에 표시됩니다. 대시보드는 이들에 대해 읽기 전용입니다. 생성과 업데이트는 다른 곳에서 수행하시기 바랍니다.
- **상태 페이지**는 고객 대상 커뮤니케이션입니다("시스템이 정상인가요?"). 대시보드는 시스템이 어떻게 동작하는지 자세히 보는 용도입니다. 두 가지는 함께 사용하며, 서로 대체하지 않습니다.
- **워크플로**는 OneUptime이 작업을 수행하는 방식입니다. 대시보드는 무슨 일이 벌어지고 있는지 보는 방식입니다.
