# 대시보드 개요

대시보드는 OneUptime이 이미 수집하고 있는 텔레메트리 — 메트릭, 로그, 트레이스, 인시던트, 모니터, Kubernetes와 Docker 리소스 — 를 누군가 한눈에 보고 시스템 상태를 이해할 수 있는 단일 페이지로 바꾸는 방법입니다.

요청 지연 차트를 열린 인시던트 목록 옆에 놓고, 그 옆에 CPU 사용률 게이지를, 그 옆에 평이한 한국어로 된 상태 문장을 둡니다. 저장합니다. 링크를 공유합니다.

## 한눈에 보기

- OneUptime 대시보드의 **대시보드** 아래에 있는 **최상위 기능**.
- **그리드 기반 캔버스** — 기본 12 단위 너비 × 60 단위 높이. 위젯을 끌어다 놓고, 크기를 조절하고, 그리드에 스냅.
- **20개 이상의 위젯 타입** — 차트, 단일 값, 게이지, 테이블, 텍스트 블록, 로그 스트림, 트레이스 목록, 인시던트·알람·모니터·Kubernetes(파드, 노드, 디플로이먼트…)·Docker·호스트의 라이브 리소스 목록.
- **변수와 필터** — 단일 대시보드를 모든 클러스터, 서비스, 고객, 환경에 재사용하는 템플릿 뷰로 바꿉니다.
- **공개 공유** — 스위치를 켜면 대시보드가 공개 URL에서 접근 가능하며, 선택적 비밀번호 보호와 IP 허용 목록을 가집니다.
- **커스텀 도메인** — OneUptime 대신 `status.your-domain.com`에 공개 대시보드를 호스팅합니다.

## 왜 대시보드를 쓰는가?

대시보드는 다음 중 하나가 참일 때 가치를 발휘합니다:

- **"모든 것이 괜찮은가?" 페이지가 필요** — 온콜 로테이션, 팀 스탠드업, 또는 벽걸이 TV를 지나가는 CEO를 위해.
- **신호를 상관시켜야 함** — 트레이스 지연 증가와 열린 인시던트가 같은 분에 일어난 CPU 스파이크는 세 개의 탭에 흩어진 것보다 하나의 대시보드에서 훨씬 명백합니다.
- **조사 중** — 디버깅 세션 동안 만드는 자유 형식 대시보드는 열 번의 쿼리를 손으로 실행하는 것보다 빠릅니다.
- **외부에 게시 중** — 고객용 성능 대시보드, 파트너용 롤업, 오픈소스 서비스를 위한 공개 상태 보드.

## 핵심 개념

| 용어 | 의미 |
| --- | --- |
| **대시보드** | 캔버스. 위젯 목록, 시간 범위 컨트롤, 변수 집합을 포함하는 이름 붙은 재사용 가능한 뷰. |
| **위젯** | 캔버스 위의 한 구성 요소 — 차트, 값, 테이블, 텍스트 블록, 목록. 각각은 타입과 JSON 스타일 설정을 가진다. |
| **대시보드 단위** | 그리드 칸. 위젯은 대시보드 단위로 크기가 정해진다(예: "너비 4 × 높이 6"). 단위는 뷰포트에 따라 픽셀로 환산된다. |
| **변수** | 뷰어가 드롭다운에서 고르거나 입력하는 이름 붙은 값으로, 대시보드는 이를 모든 위젯의 쿼리에 주입한다. 클러스터, 서비스, 고객, 환경 — 필터링할 만한 무엇이든. |
| **시간 범위** | 모든 위젯이 쿼리하는 시간 창. 프리셋("과거 24시간") 또는 커스텀 범위를 선택. |
| **새로고침 간격** | **View** 모드에서 위젯이 다시 쿼리하는 빈도. Off, 5s, 10s, 30s, 1m, 5m, 15m. |
| **모드** | `Edit`(드래그, 크기 조절, 설정) 또는 `View`(읽기 전용). 둘은 같은 캔버스를 공유한다. |

## 위젯 카탈로그

대시보드에 둘 수 있는 것의 비포괄적 지도:

| 분류 | 위젯 |
| --- | --- |
| **시계열** | Chart |
| **단일 숫자** | Value, Gauge |
| **테이블 형태** | Table |
| **주석** | Text |
| **로그와 트레이스** | LogStream, TraceList |
| **운영 목록** | IncidentList, AlertList, MonitorList |
| **Kubernetes** | KubernetesPodList, KubernetesNodeList, KubernetesNamespaceList, KubernetesDeploymentList, KubernetesStatefulSetList, KubernetesDaemonSetList, KubernetesJobList, KubernetesCronJobList |
| **Docker** | DockerHostList, DockerContainerList, DockerImageList, DockerNetworkList, DockerVolumeList |
| **인프라** | HostList |

각각의 인자와 사용 시점은 [위젯](/docs/dashboards/widgets)을 참고하세요.

## 대시보드가 대시보드의 어디에 있는가

| 페이지 | 거기서 하는 일 |
| --- | --- |
| **대시보드** | 대시보드를 둘러보고, 만들고, 검색하고, 라벨을 붙인다. |
| 대시보드 → **View** | 캔버스 — 작성자에게는 Edit 모드, 다른 모두에게는 View 모드. 헤더에서 둘 사이를 전환. |
| 대시보드 → **Overview** | 설명, 소유권, 라벨. |
| 대시보드 → **Settings** | 공개 공유, 마스터 비밀번호, IP 허용 목록, 커스텀 도메인, 브랜딩(페이지 제목, 설명, 로고, 파비콘). |
| 대시보드 → **Owners** | 명시적 소유권을 가진 사용자와 팀. |
| 대시보드 → **Delete** | 대시보드 제거(되돌릴 수 없음). |

## 대시보드의 라이프사이클

1. **생성** — **대시보드 → Create Dashboard**에서 이름을 부여합니다. 캔버스가 비어 있는 상태로 열립니다.
2. **위젯 놓기** — 위젯 팔레트에서 타입을 고르고, 소스(메트릭 쿼리, 목록 필터, 자유 텍스트 본문)를 설정합니다. 위치 지정과 크기 조절.
3. **(선택) 변수 추가** — `cluster`나 `service` 같은 드롭다운을 정의해 같은 대시보드가 각 값에 대해 렌더되도록 합니다.
4. **시간 범위와 새로고침 간격 설정** — 기본값으로 충분합니다; 나중에 조정하세요.
5. **(선택) 공개 공유** — **Settings**에서 **Public Dashboard**를 켭니다. 게이트를 원하면 마스터 비밀번호를 추가하거나 IP로 제한합니다.
6. **(선택) 커스텀 도메인** — `dashboard.your-domain.com` 레코드를 추가하고 DNS를 검증한 다음, 자체 URL에서 대시보드를 제공합니다.

## 작동하는 예제

목표: 지연, 오류율, 열린 인시던트, 최근 로그 꼬리표를 가진 checkout 서비스용 온콜 페이지.

1. "Checkout oncall" 대시보드를 만듭니다.
2. 속성 키 `service.name`에 바인딩된 **Telemetry Attribute** 타입의 `service` 변수를 추가합니다. 기본값 `checkout`.
3. **Chart** 위젯을 추가: `service.name = {{service}}`로 필터링된 APM 메트릭의 P95 지연. 시간 범위는 대시보드를 따릅니다.
4. 그 옆에 **Value** 위젯을 추가: 경고 임계값 1%, 위험 임계값 5%의 오류율 백분율.
5. 아래에 `checkout`을 포함하는 라벨로 필터링된 **IncidentList** 위젯을 추가합니다.
6. 그 아래에 `service.name = {{service}}`로 필터링된 **LogStream** 위젯을 추가합니다.
7. 저장합니다. 변수 드롭다운을 `payments`로 바꾸면 — 전체 대시보드가 payments 서비스에 대해 다시 렌더됩니다. 같은 템플릿, 다른 필터.

## 대시보드가 OneUptime의 나머지와 어떻게 맞물리는가

- **모니터와 텔레메트리**가 원시 데이터로 대시보드를 공급합니다 — 설정한 모든 메트릭, 수집한 모든 로그 줄, 모든 트레이스 스팬이 위젯에서 쿼리 가능합니다.
- **인시던트와 알람**이 **IncidentList**와 **AlertList** 위젯에 나타납니다 — 대시보드는 그것들에 대한 읽기 전용 뷰입니다; 그 엔티티는 다른 곳에서 만들고 편집하세요.
- **상태 페이지**는 고객용 커뮤니케이션 도구("시스템이 지금 작동하는가?")입니다. 대시보드는 분석 도구("시스템이 자세히 어떻게 동작하는가?")입니다. 둘은 보완적이며 대체재가 아닙니다.
- **워크플로우**는 OneUptime의 쓰기 측면입니다 — 대시보드는 읽기 측면입니다.
