# 공유 및 공개 대시보드

대부분의 대시보드는 프로젝트에 사적입니다 — 로그인한 프로젝트 멤버만 볼 수 있습니다. 하지만 OneUptime은 공개 URL에 대시보드를 게시하고, 선택적으로 비밀번호로 보호하고, IP로 제한하고, 커스텀 도메인에 호스팅하게도 해줍니다. 이 페이지는 그 네 가지를 모두 다룹니다.

## 사적 대시보드(기본값)

기본적으로 대시보드는 프로젝트 멤버인 로그인한 사용자만 접근할 수 있습니다. URL은 `https://oneuptime.com/dashboards/<id>/view` 같이 생겼습니다. 직접 접근은 인증과 대시보드에 대한 적절한 읽기 권한을 요구합니다.

프로젝트 내에서, 소유권과 라벨이 누가 무엇을 보는지 제어합니다 — [설정 및 권한](/docs/dashboards/configuration)을 참고하세요.

## 공개 대시보드

**대시보드 → Settings**에서 **Public Dashboard**를 켭니다. 이제 대시보드는 로그인을 요구하지 않는 두 번째 URL을 가집니다. 벤더, 파트너, 고객과 공유하거나 공개 README에 붙여 넣으세요.

공개 대시보드:

- **View** 모드에서만 렌더됩니다. 공개 방문자는 편집할 수 없고, 시간 범위 URL을 제외하면 변경할 수 없으며, 위젯 팔레트를 볼 수 없습니다.
- 정의한 변수를 포함합니다 — 방문자는 내부 사용자처럼 드롭다운에서 고를 수 있습니다.
- Settings에서 설정한 **브랜딩**을 가집니다: 페이지 제목, 페이지 설명, 로고 파일, 파비콘. 이것이 브라우저 탭과 소셜 미리보기에 나타나는 것입니다.

**Public Dashboard** 활성화를 웹페이지 게시처럼 취급하세요. 대시보드의 모든 위젯이 이제 전 세계에서 읽을 수 있습니다. 스위치를 켜기 전에 캔버스에 무엇이 있는지 감사하세요.

## 마스터 비밀번호

공개 대시보드를 완전히 열어두는 대신 비밀번호로 게이트하려면:

1. **Public Dashboard**를 활성화합니다.
2. **Master Password**를 활성화합니다.
3. 비밀번호를 설정합니다.

방문자는 대시보드가 렌더되기 전에 비밀번호 프롬프트를 만납니다. 비밀번호는 저장 시 해시되며 해시만 저장됩니다.

마스터 비밀번호 사용 시점:

- 파트너나 고객과 공유하고 싶지만 URL이 유출되면 유효하지 않기를 원할 때.
- 대시보드가 "준공개"일 때 — 모든 뷰어에 대한 OneUptime 계정을 원하지 않을 만큼 열려 있지만, 공개 인터넷에 두기에는 충분히 열리지 않은.

더 높은 가치 게이팅(뷰어별 계정, 누가 무엇을 보았는지의 감사 흔적)의 경우, 대시보드를 사적으로 유지하고 뷰어를 프로젝트에 읽기 전용 멤버로 초대하세요.

## IP 허용 목록

**Scale** 플랜에서는 공개 대시보드를 소스 IP나 CIDR 범위 목록으로 제한할 수 있습니다. **대시보드 → Settings → IP Whitelist**에서 목록을 설정합니다.

IP 허용 목록 사용 시점:

- 대시보드가 사무실이나 VPN에서만 접근 가능해야 할 때.
- 벤더 포털이 그들의 게시된 이그레스 IP에서만 접근 가능해야 할 때.
- 마스터 비밀번호 위에 심층 방어를 원할 때.

다른 IP에서의 요청은 403을 받습니다.

## 커스텀 도메인

기본적으로 공개 대시보드는 `oneuptime.com`에서 제공됩니다. 자체 서브도메인(예: `dashboard.acme.com`)에 호스팅하려면:

1. 서브도메인을 OneUptime의 게시된 대상으로 가리키는 CNAME 레코드를 DNS에 추가합니다.
2. **대시보드 → Settings → Custom Domains**에서 도메인을 추가합니다.
3. DNS 레코드를 검증합니다(OneUptime이 대신 확인합니다).
4. 검증되면 대시보드는 OneUptime URL과 커스텀 도메인 모두에서 접근 가능합니다.

커스텀 도메인은 다음에 유용합니다:

- 자체 브랜드의 고객용 대시보드.
- 공동 브랜드 파트너 대시보드.
- 공개 헬스 페이지의 SEO.

같은 콘텐츠를 여러 청중에 제공한다면 하나의 대시보드에 여러 커스텀 도메인을 부착할 수 있습니다.

## 공개 대시보드용 브랜딩

**대시보드 → Settings**에서 다음을 설정합니다:

- **Page title** — `<title>` 태그와 방문자가 보는 헤딩.
- **Page description** — 검색 엔진과 소셜 미리보기가 사용하는 메타 설명.
- **Logo file** — PNG/SVG 업로드; 대시보드 헤더에 표시됩니다.
- **Favicon** — 업로드; 브라우저 탭에 표시됩니다.

브랜딩은 공개 모드 렌더링에만 적용됩니다. 내부 뷰어는 항상 OneUptime 브랜딩을 봅니다.

## 임베딩

자체 사이트의 `<iframe>`에 공개 대시보드를 임베드할 수 있습니다:

```html
<iframe src="https://dashboard.acme.com/view"
        width="100%" height="800"
        frameborder="0"></iframe>
```

마스터 비밀번호로 보호된 대시보드를 임베드하면, 방문자는 iframe 안에서 여전히 비밀번호 프롬프트를 봅니다.

## 변수 상태가 있는 공유 가능 URL

대시보드 URL은 현재 변수 선택과 시간 범위를 쿼리 파라미터로 인코딩합니다. 드롭다운을 조정하고, URL을 복사하고, 채팅에 붙여 넣으면 — 수신자는 보고 있던 시간 범위를 포함해 정확히 같은 뷰로 대시보드를 봅니다.

이것이 "인시던트가 시작된 시간의 대시보드"로 팀원을 가리키는 가장 빠른 방법입니다 — 시간 범위를 고정, 복사, 붙여넣기.
