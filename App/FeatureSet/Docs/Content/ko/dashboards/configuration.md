# 설정 및 권한

이 페이지는 실제로 유지하고 싶은 대시보드가 생긴 뒤 알아둘 만한 설정과 접근 제어 노브를 모은 것입니다.

## 소유권

대시보드의 **owners**는 그에 대한 명시적 권한을 부여받은 사용자와 팀입니다(프로젝트 단위 역할과는 별개).

**대시보드 → Owners**에서:

- **user owner**를 추가해 특정 사람에게 이 대시보드에 대한 추가 접근을 부여합니다.
- **team owner**를 추가해 팀의 모든 멤버에게 동일한 것을 부여합니다.

소유권 사용 시점: 프로젝트 단위 읽기 역할이 너무 광범위할 때 — 예: 고객 성공 팀에만 보여야 하는 민감한 고객 수준 세부 정보를 가진 대시보드.

## 라벨

라벨은 대시보드를 정리하기 위한 다대다 태그입니다. **대시보드 → Overview**에서 적용합니다.

흔한 라벨 패턴:

- **팀별**: `team:platform`, `team:checkout`, `team:growth`.
- **환경별**: `env:prod`, `env:staging`.
- **목적별**: `purpose:oncall`, `purpose:exec`, `purpose:investigation`.

**대시보드** 목록은 라벨로 필터링하게 해주는데, 이는 수십 개가 쌓인 프로젝트에서 대시보드를 찾는 가장 빠른 방법입니다.

## 권한

대시보드는 OneUptime의 역할 기반 접근 제어에서 일급 리소스입니다. 관련 권한:

| 권한 | 허용 |
| --- | --- |
| `CreateDashboard` | 프로젝트에 새 대시보드 생성. |
| `ReadDashboard` | 대시보드 보기(사적 모드에서). |
| `EditDashboard` | 대시보드의 위젯, 변수, 설정 수정. |
| `DeleteDashboard` | 대시보드 삭제. |

지원 엔티티에 대해 일치하는 권한이 있습니다: 대시보드 소유자(사용자/팀)와 커스텀 도메인은 자체 create/read/edit/delete 쌍을 가지므로 "대시보드 자체를 편집" 권한 부여 없이 "소유자 관리"를 부여할 수 있습니다.

**Project Settings → Teams & Roles**에서 프로젝트 역할에 이를 할당합니다.

## 공개 모드 접근 제어

공개 모드 접근([공유 및 공개 대시보드](/docs/dashboards/sharing) 참고)은 세 가지 계층으로 순서대로 관리됩니다:

1. **Public Dashboard** 토글 — 꺼져 있으면 공개 URL은 404를 반환합니다.
2. **Master Password** — 설정되어 있으면 방문자는 대시보드가 렌더되기 전에 입력해야 합니다.
3. **IP Whitelist**(Scale 플랜) — 설정되어 있으면 목록에 없는 IP의 요청은 403을 받습니다.

대시보드는 어떤 조합이든 가질 수 있습니다. 가장 방어적인 설정은 "Public 켬, 비밀번호 설정, IP 허용 목록 활성" — 세 가지 모두 원하는 파트너 포털에 유용합니다.

## 보존

대시보드 자체는 만료되지 않습니다. 대시보드가 표시하는 데이터는 프로젝트의 텔레메트리 보존을 따릅니다 — 메트릭, 로그, 트레이스는 플랜이 보존하는 동안 쿼리 가능합니다. 30일 보존 플랜에서 "과거 90일"을 가리키는 위젯은 저장소에 여전히 있는 것을 렌더합니다.

## 대시보드 복제

기존 대시보드를 복제하려면, 그것을 열고 대시보드 목록에서 **Duplicate** 액션을 사용합니다. 복사본은 공개 모드 설정(항상 끔으로 시작 — 복사본에서 다시 활성화할지 결정합니다)을 제외하고 모든 위젯, 변수, 설정을 포함합니다.

이것이 템플릿("우리의 온콜 대시보드")을 서비스별 버전으로 포크하고 싶을 때 올바른 패턴입니다.

## 대시보드 삭제

**대시보드 → Delete**. 이것은 되돌릴 수 없습니다 — 캔버스 설정과 모든 커스텀 도메인 바인딩이 제거됩니다. 텔레메트리 데이터는 영향받지 않습니다(메트릭/로그/트레이스 저장소에 있지 대시보드에 있지 않음).

대시보드가 커스텀 도메인으로 공개적으로 게시되어 있다면, 삭제하는 순간 공개 URL은 해석되지 않습니다. 다시 가리켜야 한다면 먼저 도메인을 떼어내세요.

## 마이그레이션과 백업

셀프 호스팅 설치의 경우: 대시보드의 전체 설정(위젯, 변수, 설정)은 Postgres의 `Dashboard` 테이블에 있습니다. 정기적인 데이터베이스 백업으로 충분합니다 — 별도의 대시보드 익스포트 형식은 없습니다.

OneUptime Cloud의 경우: 정기적인 백업이 대신 처리됩니다. 대시보드 설정의 로컬 복사본을 원한다면, [OneUptime API](/docs/api-reference/api-reference)를 사용해 `Dashboard` 레코드를 읽으세요.
