# Runbook 작성

**Runbooks → Runbook 생성**에서 Runbook을 만들고, 열어서 **단계** 탭으로 이동합니다.

## 단계의 구성

각 단계에는 다음이 있습니다:

| 필드 | 용도 |
| --- | --- |
| **제목** | 체크리스트 UI에 보이는 짧은 라벨. 필수. |
| **설명** | 대응자에게 줄 선택적 컨텍스트. 마크다운 텍스트. |
| **실패 시 계속** | 켜져 있으면 실패한 단계가 실행을 멈추지 않고 다음 단계가 그래도 실행됩니다. |
| **타입별 설정** | 스크립트, URL 등 — 아래 참고. |

단계는 **순서대로** 실행됩니다. 단계 편집기의 위/아래 화살표로 순서를 바꿉니다.

## 단계 타입

### Manual

대응자가 체크하는 체크박스. Manual 단계에 도달하면 실행이 일시 정지되며, 누군가 완료 처리(또는 건너뛰기) 할 때까지 `WaitingForManualStep` 상태로 남습니다.

사람만 확인할 수 있는 항목에 사용합니다: "로드 밸런서 대시보드에서 보조 리전으로 트래픽이 옮겨졌음 — 확인."

### JavaScript

샌드박스 `isolated-vm`에서 실행되는 JavaScript 스니펫(파일 시스템 없음, API를 직접 가져오지 않는 한 네트워크 없음).

```js
const start = Date.now();
// ... 로직 ...
return { durationMs: Date.now() - start };
```

반환 값은 단계 실행에 기록됩니다. `console.log` 출력은 로그 라인으로 캡처됩니다. 기본 타임아웃: 30초.

### HTTP 요청

외부로 나가는 HTTP 호출. 메서드(GET/POST/PUT/PATCH/DELETE/HEAD), URL, 선택적 JSON 헤더, 선택적 본문을 설정합니다. 응답 상태, 헤더, 본문이 캡처됩니다(전체 50KB 한도).

PagerDuty 인시던트 시작, Slack 게시, 자체 관리 API 호출 등에 유용합니다.

### Bash

[Runbook 에이전트](/docs/runbooks/agents)에서 실행되는 bash 스크립트 — 자신의 인프라 안 호스트에 설치하는 작은 프로세스입니다. Bash 단계는 OneUptime Worker에서 절대 실행되지 않습니다.

Bash 단계에서 두 가지를 설정하세요:

- **Agent Tag** — 이 단계를 실행할 에이전트를 식별하는 태그. 프로젝트에서 그 태그를 가진 임의의 건강한 에이전트가 작업을 차지하고 실행합니다.
- **스크립트** — 실행할 bash. 출력(stdout + stderr)은 50KB까지 캡처되며 타임아웃 시 프로세스가 종료됩니다.

Runbook이 이 단계에 도달했을 때 선택한 태그를 가진 에이전트가 하나도 온라인이 아니면, 단계는 **claim 타임아웃**(기본 2분)까지 대기한 뒤 실패합니다. Bash 단계에 의존하기 전에 **Runbooks → Agents**에서 에이전트를 추가하세요.

## 저장과 편집

**단계 저장**을 눌러 영속합니다. 이전 버전 Runbook의 진행 중 실행은 영향을 받지 않습니다 — 그들은 자기 스냅샷을 계속 사용합니다.

## 여러 단계와 실패 처리

기본적으로 실패한 단계는 실행을 멈추고 실행을 `Failed`로 표시합니다. 단계에 **실패 시 계속**을 설정하면 실패가 기록되지만 다음 단계가 실행됩니다. "이 세 가지를 시도하고 그 후에 알림" 같은 패턴에 유용합니다.

## 자세한 예시

"primary DB 도달 불가"에 대한 간단한 Runbook:

1. **JavaScript** — 설정 서비스에서 현재 primary 호스트를 가져와 로그.
2. **Manual** — "보조의 복제 지연이 5초 미만 — 확인."
3. **HTTP 요청** — 페일오버 오케스트레이터 API로 POST.
4. **Manual** — "쓰기가 새 primary로 가고 있음 — 확인."
5. **HTTP 요청** — Slack에 "정상화" 메시지 POST.

대응자는 자동 단계가 실행되는 것을 보고, Manual을 체크하고, 다음 자동 단계를 본 다음, 다음을 반복합니다. 각 단계의 출력은 포스트모템을 위해 보존됩니다.
