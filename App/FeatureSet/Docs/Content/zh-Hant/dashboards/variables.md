# 變量與篩選

變量把單張儀表板變成模板。給儀表板加一個 `service` 變量,同樣的圖表就會爲 `checkout`、`payments` 或 `search` 重新渲染——訪客從頂部下拉中選擇,而不是你去建三張幾乎相同的儀表板。

## 變量類型

在 **儀表板 → 設置 → 變量** 下添加變量。每個變量有一個名稱(在組件中以 `{{name}}` 引用)、一個可選的標籤和一個類型。

### 自定義列表

一個靜態下拉框。選項由你自己輸入。

適用於:選項少且固定時。`environment` 取值 `prod, staging, dev`。`region` 取值 `us-east-1, eu-west-1, ap-south-1`。

### 查詢

選項來自對你數據的查詢。

適用於:選項隨時間變化、你希望下拉框跟上時。"過去 24 小時內出現過的每個客戶 ID"。查詢會對你項目的數據運行,結果作爲下拉項。

### 文本輸入

一個自由文本字段。訪客輸入什麼就用什麼。

適用於:你想讓儀表板像搜索工具一樣使用時。按 IP 地址、請求 ID 或任何自由格式的值過濾。

### 遙測屬性

選項是你遙測數據在儀表板時間範圍內某個屬性的不同取值。

配置 **屬性鍵**(例如 `service.name`、`host.name`、`k8s.cluster.name`)。下拉框會填入你日誌、指標和追蹤中所見的每個不同取值。

適用於:選項與你的遙測中已經發送的標籤匹配時。這是最常用的類型,因爲它會自動更新——當你發佈一個打了 `service.name = inventory` 標籤的新服務,這個名字會出現在下拉里,你不必修改儀表板。

## 多選

每個變量都可以允許多選。開啓後,訪客可以選一個或多個值;儀表板會篩選爲它們中的任意一個。

什麼時候用多選:你想要"把 checkout 和 payments 放在一起比較"而不離開儀表板時。當跨所選值的算式不成立時(例如對平均值求平均)避免使用。

## 默認值

每個變量都可以有一個默認值。訪客修改之前,儀表板用默認值渲染。對公開儀表板來說,默認值就是訪客最初看到的內容。

## 如何在組件中使用變量

任何接受篩選條件的地方——指標的 `WHERE`、列表的篩選條件、日誌流的屬性匹配——你都可以用 `{{variable_name}}`。

例如,按服務過濾的圖表:

```
service.name = '{{service}}'
```

下拉設爲 `checkout` 時,圖表篩選到 checkout 服務。訪客切到 `payments` 時,圖表爲 payments 重新渲染。

對於 **遙測屬性** 變量,OneUptime 知道這個變量對應的是哪個屬性,並會把篩選應用到所有使用相同屬性的組件——你不必逐個手動修改組件。

## 時間範圍

儀表板頭部有一個全局時間範圍。每個指標類組件都對這個窗口進行查詢。選項:

- **預設**——過去 1 小時、24 小時、7 天、30 天、90 天(取決於你的數據保留期)。
- **自定義**——選擇開始和結束時間。

時間範圍是儀表板 URL 的一部分——分享 URL 就是在分享窗口。在事件中很有用:把時間範圍釘到"今天 UTC 10:00–10:30",然後把鏈接粘到事件頻道。

## 刷新間隔

時間範圍旁邊,選擇組件重新查詢的頻率:

- **關閉**——頁面加載時只查詢一次。
- **5s / 10s / 30s / 1m / 5m / 15m**——自動刷新。

自動刷新適合掛牆屏幕或實時事件視圖。在排查問題時關閉它,讓畫面在你查看的過程中保持不動。

## 組合起來

一張以服務爲模板的儀表板通常有:

1. 一個類型爲 **遙測屬性** 的 `service` 變量,對應 `service.name`。默認:你最關注的服務。多選關閉(這樣圖表每次只顯示一個)。
2. 一個類型爲 **自定義列表** 的 `environment` 變量。默認:`prod`。
3. 一個類型爲 **遙測屬性** 的 `cluster` 變量,對應 `k8s.cluster.name`。多選打開(這樣可以跨集羣比較)。
4. 在篩選條件裏引用這些變量的組件。

結果:一張儀表板覆蓋所有服務,頂部三個下拉。
