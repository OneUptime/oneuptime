# 分享與公開儀表板

默認情況下,儀表板對你的項目是私有的——只有已登錄的團隊成員才能查看。但 OneUptime 也允許你公開分享儀表板、用密碼保護它、限制到特定 IP,並把它託管在你自己的域名上。本頁涵蓋這四點。

## 私有儀表板(默認)

儀表板只對已登錄的項目成員可達。URL 形如 `https://oneuptime.com/dashboards/<id>/view`,需要登錄。

在項目內部,所有者和標籤控制誰能看到什麼——見 [配置與權限](/docs/dashboards/configuration)。

## 公開儀表板

在 **儀表板 → 設置** 下,切換 **公開儀表板** 爲開啓。儀表板現在有了第二個 URL,無需登錄。可以分享給供應商、合作伙伴、客戶,或者粘貼到公開的 README 裏。

公開儀表板:

- 始終在 **查看** 模式下打開。公開訪客不能編輯或看到組件面板。
- 包含你添加的變量。訪客從和團隊相同的下拉中選擇。
- 使用你在設置裏配置的 **品牌**——頁面標題、描述、Logo、Favicon。

啓用公開儀表板就像發佈一個網頁。它上面的每個組件都會變得全球可讀。在切換開關之前看看畫布上有什麼。

## 主密碼

要給公開儀表板加密碼:

1. 打開 **公開儀表板**。
2. 打開 **主密碼**。
3. 設置密碼。

訪客在儀表板出現之前會看到密碼提示。密碼以哈希形式儲存——我們永遠看不到實際密碼。

什麼時候用主密碼:

- 你想與合作伙伴或客戶分享,但又不希望 URL 泄露後被濫用。
- 儀表板是"半公開"的——足夠開放到不需要邀請每個訪客成爲團隊成員,但又不至於開放到放上公開互聯網。

如果需要更強的控制(每個訪客單獨賬戶、誰看過什麼的審計),保持儀表板私有,並把訪客作爲只讀團隊成員邀請進來。

## IP 白名單

在 **Scale** 套餐上,你可以把公開儀表板限制到一組 IP 地址或網段。在 **儀表板 → 設置 → IP 白名單** 下配置。

什麼時候用:

- 儀表板應該只能從你的辦公室或 VPN 訪問。
- 一個供應商門戶應該只能從他們已知的 IP 訪問。
- 你想在主密碼之上再加一層保護。

來自其他 IP 的請求會被拒絕。

## 自定義域名

開箱即用時,公開儀表板部署在 `oneuptime.com` 上。要把它託管在你自己的子域名比如 `dashboard.acme.com` 上:

1. 在你的 DNS 上添加一條 CNAME 記錄,把子域名指向 OneUptime 的目標。
2. 在 **儀表板 → 設置 → 自定義域名** 下,添加這個域名。
3. 驗證它。OneUptime 會替你檢查 DNS 記錄。
4. 驗證後,儀表板在你的自定義域名和原始 URL 上都可達。

自定義域名適用於:

- 用你自己的品牌做面向客戶的儀表板。
- 聯合品牌的合作伙伴儀表板。
- 擁有獨立 URL 的公開健康頁面。

如果你需要把同樣的內容服務給多個受衆,可以爲單個儀表板掛多個自定義域名。

## 品牌定製

在 **儀表板 → 設置** 下,你可以配置:

- **頁面標題**——瀏覽器標籤和頁面頂部顯示的標題。
- **頁面描述**——搜索引擎和社交預覽使用的描述。
- **Logo**——上傳 PNG 或 SVG 在頭部顯示。
- **Favicon**——瀏覽器標籤的小圖標。

品牌定製只在儀表板被公開查看時應用。內部查看者總是看到 OneUptime 的品牌。

## 嵌入

你可以用 iframe 把公開儀表板嵌入到自己的網站:

```html
<iframe src="https://dashboard.acme.com/view"
        width="100%" height="800"
        frameborder="0"></iframe>
```

如果儀表板有主密碼,訪客會在 iframe 內看到密碼提示。

## 可分享的 URL

儀表板 URL 把當前的變量選擇和時間範圍作爲查詢參數包含進去。調整下拉、複製 URL、粘貼到聊天裏——打開鏈接的人看到的是完全一樣的視圖。

這是把同事指向"事件發生那一刻的儀表板"的最快方法。釘住時間範圍、複製、粘貼。
