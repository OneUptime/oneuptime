# 配置與權限

本頁涵蓋當你有了一張想長期保留的儀表板時,值得了解的設置與訪問控制。

## 所有者

儀表板的 **所有者** 是被你賦予顯式訪問權限的用戶和團隊(在他們項目範圍角色之上)。

在 **儀表板 → 所有者** 下:

- 添加一個 **用戶所有者** 來給某個人對該儀表板的額外訪問權限。
- 添加一個 **團隊所有者** 來給某個團隊的每位成員同樣的權限。

當項目範圍的讀取角色過於寬泛時,使用所有者——例如,一張涉及客戶級細節的儀表板,只應當對客戶成功團隊可見。

## 標籤

標籤是用於整理儀表板的標記。在 **儀表板 → 概覽** 下應用它們。

常見模式:

- **按團隊**:`team:platform`、`team:checkout`、`team:growth`。
- **按環境**:`env:prod`、`env:staging`。
- **按用途**:`purpose:oncall`、`purpose:exec`、`purpose:investigation`。

**儀表板** 列表允許按標籤篩選,這是在儀表板較多的項目裏找到目標儀表板最快的方式。

## 權限

儀表板與項目的基於角色的訪問控制配合。相關權限:

| 權限 | 允許的操作 |
| --- | --- |
| **創建 Dashboard** | 創建新儀表板。 |
| **讀取 Dashboard** | 查看儀表板(在私有模式下)。 |
| **編輯 Dashboard** | 修改組件、變量和設置。 |
| **刪除 Dashboard** | 刪除儀表板。 |

儀表板所有者和自定義域名也有對應的權限,因此你可以授予"管理所有者"而不授予"編輯儀表板"。

在 **項目設置 → 團隊與角色** 下爲項目角色分配這些權限。

## 公開儀表板的訪問

當你把儀表板設爲公開(見 [分享與公開儀表板](/docs/dashboards/sharing)),三個設置控制誰能看到它:

1. **公開儀表板** 開關——關閉時,公開 URL 返回 404。
2. **主密碼**——若設置,訪客先輸入密碼才能看到儀表板。
3. **IP 白名單**(Scale 套餐)——若設置,來自其他 IP 的請求會被拒絕。

你可以任意組合這些。最嚴格的組合是"公開打開、密碼設置、IP 白名單啓用"——適合需要三層防護的合作伙伴門戶。

## 數據保留

儀表板本身不會過期。它們顯示的數據遵循你項目的保留設置——指標、日誌和追蹤只要你的套餐保留它們就可查詢。一個指向"過去 90 天"的組件,在只保留 30 天的套餐上,會顯示當前還存的內容。

## 複製儀表板

要複製現有的儀表板,打開儀表板列表並選擇 **複製**。副本會包含每個組件、變量和設置,但不包括公開分享——分享始終從關閉開始,這樣你可以決定是否再打開它。

當你想把一個模板(比如"我們的值班儀表板")分叉成針對某個服務的副本時,這是正確的做法。

## 刪除儀表板

在 **儀表板 → 刪除** 下。這無法撤銷——儀表板的佈局以及掛在它上面的任何自定義域名都會被移除。你的遙測數據不受影響。

如果儀表板是公開的並且使用了自定義域名,刪除後 URL 立即停止解析。如果想讓 URL 繼續工作,請先把域名移動到另一個儀表板。

## 備份

如果你自託管 OneUptime,定期的數據庫備份就夠了——儀表板配置和項目的其他內容一起儲存。

在 OneUptime Cloud 上,備份由我們處理。如果你想要自己的副本,可以通過 [OneUptime API](/docs/api-reference/api-reference) 讀取儀表板。
