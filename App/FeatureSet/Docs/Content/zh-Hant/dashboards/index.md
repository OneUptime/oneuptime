# 儀表板概覽

儀表板把 OneUptime 已經在採集的數據——指標、日誌、追蹤、事件、監控、Kubernetes 資源、主機——變成一個讓人一眼就能看清現狀的頁面。

把一個請求延遲的圖表放在一個未解決事件列表旁邊、再加一個 CPU 儀表、再附一段說明。保存。分享鏈接。

## 儀表板適合做什麼

- **一個"一切都正常嗎?"頁面**——給值班、給團隊站會,或者掛在牆上的電視上。
- **發現關聯**——CPU 飆升的同時延遲在漲、還伴隨着一個未解決的事件,這種聯繫在一張頁面上比在三個標籤頁之間切換容易看多了。
- **排查問題**——調試時,一張你即興搭的儀表板比一個個跑十次查詢要快得多。
- **對外分享**——給客戶的性能頁面、給合作伙伴的狀態頁面、爲開源項目搭的公共儀表板。

## 儀表板上可以放什麼

- **圖表** 用於時序趨勢——延遲、錯誤、吞吐量。
- **單值瓦片和儀表盤**——當前錯誤率、CPU、未解決事件數。
- **表格** 用於分組對比——最吵的 10 臺主機、各服務的錯誤數。
- **文本塊** 用於標題、說明,以及指向 Runbook 的鏈接。
- **實時列表** 包含事件、警報、監控、日誌、追蹤、Kubernetes 資源、Docker 資源,以及主機。

完整清單和每個組件的展示內容見 [組件](/docs/dashboards/widgets)。

## 關鍵術語

| 術語 | 含義 |
| --- | --- |
| **儀表板** | 整個頁面——一個名稱、一組網格化的組件、時間範圍控件,以及變量列表。 |
| **組件** | 頁面上的一塊瓦片——一個圖表、一個數字、一個列表、一段文字。 |
| **變量** | 頂部的下拉框,可以一次性篩選所有組件(集羣、服務、客戶、環境)。 |
| **時間範圍** | 所有圖表和數字使用的時間窗口。在頁面頂部設置一次。 |
| **刷新** | 組件重新查詢數據的頻率。可以關閉、幾秒一次、幾分鐘一次。 |
| **模式** | 要麼是 **編輯**(拖動組件),要麼是 **查看**(只讀,訪客看到的樣子)。 |

## 在哪裏找到儀表板

在左側導航打開 **儀表板**。

| 頁面 | 你在那裏可以做什麼 |
| --- | --- |
| **儀表板** | 你的儀表板列表。新建一個、搜索,或按標籤篩選。 |
| **儀表板 → 查看** | 畫布。在頭部切換 **編輯** 與 **查看**。 |
| **儀表板 → 概覽** | 描述、所有者和標籤。 |
| **儀表板 → 設置** | 公開分享、密碼、IP 白名單、自定義域名、品牌定製。 |
| **儀表板 → 所有者** | 擁有顯式訪問權限的用戶和團隊。 |
| **儀表板 → 刪除** | 刪除儀表板。 |

## 構建儀表板

1. **創建**——起一個名稱。畫布打開時是空的。
2. **添加組件**——選擇組件類型,配置數據,然後拖到你想要的位置。
3. **(可選)添加變量**——例如,加一個 `service` 下拉框,讓同一張儀表板適用於每個服務。
4. **設置時間範圍**——默認值夠用;以後再調整。
5. **(可選)公開分享**——在設置中切換開關,如有需要再加密碼或 IP 白名單。
6. **(可選)自定義域名**——把儀表板部署到 `status.your-domain.com`。

## 一個簡單的例子

目標:爲 checkout 服務做一個值班頁面,包含延遲、錯誤率、未解決事件,以及實時日誌尾部。

1. 創建一個名爲 "Checkout 值班" 的儀表板。
2. 添加一個 `service` 變量。默認設爲 `checkout`。
3. 添加一個 **圖表** 組件,顯示 P95 延遲,按 `service` 變量過濾。
4. 在它旁邊添加一個 **單值** 組件顯示錯誤率,警告閾值 1%、嚴重閾值 5%。
5. 下面添加一個 **事件列表** 組件,展示打了 `checkout` 標籤的事件。
6. 再下面加一個 **日誌流** 組件,顯示同一服務的日誌。
7. 保存。把下拉切換到 `payments`——同一張儀表板現在展示的是 payments 服務。

## 儀表板與 OneUptime 其他部分的關係

- **監控與遙測** 是數據來源。你採集的每個指標、日誌和追蹤都可以在組件上查詢。
- **事件與警報** 會顯示在 **事件列表** 和 **警報列表** 組件中。儀表板對這些是隻讀的——創建和更新在別處進行。
- **狀態頁** 是面向客戶的溝通("系統在線嗎?")。儀表板用於詳細查看系統的運行情況。兩者協同工作,不互相替代。
- **工作流** 是 OneUptime 採取行動的方式。儀表板是你查看現狀的方式。
