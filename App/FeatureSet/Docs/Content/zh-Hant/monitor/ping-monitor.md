# Ping 監控器

Ping 監控允許您監控任何主機或 IP 地址的可用性和響應能力。OneUptime 定期向目標發送 Ping 請求，並檢查其是否正確響應。

## 概述

Ping 監控器通過向主機發送 ICMP Ping 請求來測試基本網絡連接性。這使您能夠：

- 監控主機正常運行時間和可用性
- 跟蹤網絡延遲和響應時間
- 在連接問題影響您的服務之前檢測到它們
- 驗證服務器和網絡設備是否可達

## 創建 Ping 監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **Ping** 作爲監控器類型
4. 輸入您要監控的主機名或 IP 地址
5. 根據需要配置監控標準

## 配置選項

### Ping 主機名或 IP 地址

輸入您要監控的目標主機名或 IP 地址（例如 `example.com` 或 `192.168.1.1`）。主機名和 IP 地址均可接受。

## 監控標準

您可以配置標準來判斷主機何時處於在線、降級或離線狀態，基於以下條件：

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 是否在線 | 主機是否響應 Ping 請求 |
| 響應時間（毫秒） | Ping 請求的往返時間（毫秒） |
| 請求是否超時 | Ping 請求是否超時 |

### 過濾類型

對於 **是否在線** 和 **請求是否超時**：

- **True** — 條件爲真
- **False** — 條件爲假

對於 **響應時間**：

- **大於** — 響應時間超過閾值
- **小於** — 響應時間低於閾值
- **大於或等於** — 響應時間等於或超過閾值
- **小於或等於** — 響應時間等於或低於閾值
- **等於** — 響應時間完全匹配
- **不等於** — 響應時間不匹配
- **隨時間評估** — 使用聚合（平均值、求和、最大值、最小值、所有值、任意值）在時間窗口內評估

### 示例標準

#### 如果主機不可達則標記爲離線

- **檢查項**：是否在線
- **過濾類型**：False

#### 如果響應時間超過 200ms 則發出警報

- **檢查項**：響應時間（毫秒）
- **過濾類型**：大於
- **值**：200
