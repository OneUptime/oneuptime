# 指標監控器

指標監控允許您監控通過 OpenTelemetry 收集的自定義應用程序和基礎設施指標。OneUptime 在時間窗口內評估指標值，並根據您配置的標準觸發警報。

## 概述

指標監控器查詢並評估來自遙測服務的數值指標。這使您能夠：

- 監控自定義應用程序指標（請求率、隊列深度、錯誤率等）
- 跟蹤基礎設施指標（CPU、內存、磁盤、網絡）
- 使用過濾器和聚合創建複雜的指標查詢
- 使用數學公式組合多個指標
- 基於指標閾值設置警報

## 創建指標監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **指標** 作爲監控器類型
4. 配置指標查詢和可選公式
5. 選擇聚合策略
6. 根據需要配置監控標準

## 配置選項

### 指標查詢

定義一個或多個指標查詢。每個查詢包含：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 指標名稱 | 要查詢的指標名稱 | 是 |
| 聚合類型 | 如何聚合原始指標值（sum、avg、min、max、count） | 是 |
| 屬性 | 用於縮小指標數據範圍的鍵值過濾器 | 否 |
| 分組依據 | 用於分組指標的維度 | 否 |

每個查詢都會分配一個別名（例如 `a`、`b`、`c`），用於公式中。

### 公式

使用數學表達式組合多個指標查詢。例如：

- `a / b * 100` — 從兩個查詢計算百分比
- `a + b` — 對兩個指標求和
- `a - b` — 兩個指標之間的差值

### 滾動時間窗口

選擇指標評估的時間窗口：

- 過去 1 分鐘
- 過去 5 分鐘
- 過去 10 分鐘
- 過去 15 分鐘
- 過去 30 分鐘
- 過去 60 分鐘

### 聚合策略

選擇如何聚合指標值進行評估：

| 策略 | 描述 |
|------|------|
| 平均值 | 時間窗口內的平均值 |
| 求和 | 所有值的總和 |
| 最大值 | 時間窗口內的最高值 |
| 最小值 | 時間窗口內的最低值 |
| 所有值 | 所有值必須滿足標準 |
| 任意值 | 至少一個值必須滿足標準 |

## 監控標準

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 指標值 | 配置的指標查詢或公式的聚合值 |

### 過濾類型

- **大於** — 指標值超過閾值
- **小於** — 指標值低於閾值
- **大於或等於** — 指標值等於或超過閾值
- **小於或等於** — 指標值等於或低於閾值
- **等於** — 指標值完全匹配
- **不等於** — 指標值不匹配

### 示例標準

#### 當錯誤率超過 5% 時發出警報

- **查詢 a**：`http_requests_total` 按 `status=5xx` 過濾
- **查詢 b**：`http_requests_total`
- **公式**：`a / b * 100`
- **檢查項**：指標值
- **過濾類型**：大於
- **值**：5

#### 當請求隊列深度過高時發出警報

- **查詢**：`request_queue_size`，聚合：最大值
- **檢查項**：指標值
- **過濾類型**：大於
- **值**：1000

## 設置要求

指標監控需要您的應用程序或基礎設施通過 OpenTelemetry 向 OneUptime 發送指標。設置說明請參見 [OpenTelemetry](/docs/telemetry/open-telemetry) 文檔。
