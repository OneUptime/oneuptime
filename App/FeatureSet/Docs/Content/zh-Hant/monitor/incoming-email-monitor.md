# 傳入郵件監控器

傳入郵件監控器允許您根據發送到唯一監控器特定郵件地址的郵件來創建和解決警報。這對於與舊系統、第三方警報工具或任何可以發送郵件通知的服務集成非常有用。

## 工作原理

1. 當您創建傳入郵件監控器時，OneUptime 會爲該監控器生成唯一的郵件地址
2. 發送到該地址的任何郵件都會被接收，並根據您配置的標準進行評估
3. 根據標準，OneUptime 可以創建新警報或解決現有警報

這是將基於郵件的警報系統與 OneUptime 事件管理工作流集成的強大方式。

## 創建傳入郵件監控器

1. 在 OneUptime 控制台中導航至 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **傳入郵件** 作爲監控器類型
4. 配置監控器設置：
   - **名稱：** 監控器的描述性名稱
   - **描述：** 此監控器的用途
5. 設置您的 **警報創建標準**（創建警報的條件）
6. 設置您的 **警報解決標準**（解決警報的條件）
7. 點擊 **創建**

創建後，您將在監控器詳情頁面上看到此監控器的唯一郵件地址。

## 郵件地址格式

每個傳入郵件監控器都會獲得一個格式如下的唯一郵件地址：

```
monitor-{secret-key}@{inbound-domain}
```

例如：`monitor-abc123def456@inbound.yourdomain.com`

您可以從監控器詳情頁面複製此地址，並配置您的外部系統向其發送郵件。

## 可用標準字段

您可以根據以下郵件字段創建標準：

| 字段 | 描述 |
|------|------|
| **郵件主題** | 傳入郵件的主題行 |
| **郵件發件人** | 發件人的郵件地址 |
| **郵件正文** | 郵件正文的純文本內容 |
| **郵件收件人** | 收件人郵件地址 |
| **郵件接收** | 基於郵件接收時間的時間條件 |

## 可用過濾類型

### 字符串過濾器（主題、發件人、正文、收件人）

| 過濾器 | 描述 | 示例 |
|--------|------|------|
| **包含** | 字段包含指定文本 | 主題包含"CRITICAL" |
| **不包含** | 字段不包含指定文本 | 主題不包含"TEST" |
| **等於** | 字段與指定文本完全匹配 | 發件人等於"alerts@service.com" |
| **不等於** | 字段與指定文本不匹配 | 主題不等於"OK" |
| **以...開頭** | 字段以指定文本開頭 | 主題以"[ALERT]"開頭 |
| **以...結尾** | 字段以指定文本結尾 | 主題以"- Production"結尾 |
| **爲空** | 字段爲空或空白 | 正文爲空 |
| **不爲空** | 字段有內容 | 主題不爲空 |

### 時間過濾器（郵件接收）

| 過濾器 | 描述 | 示例 |
|--------|------|------|
| **在 X 分鐘內收到** | 郵件在 X 分鐘內被接收 | 郵件在 30 分鐘內收到 |
| **X 分鐘內未收到** | X 分鐘內未收到郵件 | 郵件 60 分鐘內未收到 |

## 配置示例

### 示例 1：對關鍵郵件創建警報

**警報創建標準：**
- 郵件主題 **包含** "CRITICAL"
- 或郵件主題 **包含** "ALERT"
- 或郵件主題 **包含** "ERROR"

**警報解決標準：**
- 郵件主題 **包含** "RESOLVED"
- 或郵件主題 **包含** "OK"
- 或郵件主題 **包含** "RECOVERED"

### 示例 2：監控特定發件人

**警報創建標準：**
- 郵件發件人 **等於** "monitoring@legacy-system.com"
- 且郵件主題 **包含** "Failed"

**警報解決標準：**
- 郵件發件人 **等於** "monitoring@legacy-system.com"
- 且郵件主題 **包含** "Success"

### 示例 3：心跳監控（無郵件 = 警報）

**警報創建標準：**
- 郵件接收 **60 分鐘內未收到**

如果 60 分鐘內未收到郵件，則創建警報——適用於監控應發送完成郵件的計劃任務或批處理。

**警報解決標準：**
- 郵件接收 **在 5 分鐘內收到**

收到郵件時解決警報。

## 使用場景

### 舊系統集成

許多舊系統只支持基於郵件的警報。使用傳入郵件監控器可以：
- 將郵件警報轉換爲 OneUptime 事件
- 當收到恢復郵件時自動解決事件
- 集中管理來自多箇舊系統的警報

### 第三方服務監控

與發送郵件通知的服務集成：
- 雲提供商警報（AWS、GCP、Azure）
- 安全掃描工具
- 備份完成通知
- SSL 證書過期警告

### 計劃任務監控

監控批處理和計劃任務：
- 如果未及時收到完成郵件，創建警報
- 通過錯誤通知郵件跟蹤任務失敗
- 監控數據管道完成情況

### 多供應商警報聚合

整合來自多個監控工具的警報：
- 通過郵件接收來自 Nagios、Zabbix 或其他工具的警報
- 在 OneUptime 中統一事件管理
- 爲所有警報維護單一的事實來源

## 模板變量

配置事件模板時，您可以使用這些來自傳入郵件的變量：

| 變量 | 描述 |
|------|------|
| `{{emailSubject}}` | 接收到的郵件主題 |
| `{{emailFrom}}` | 發件人的郵件地址 |
| `{{emailTo}}` | 收件人郵件地址 |
| `{{emailBody}}` | 郵件的純文本正文 |
| `{{emailReceivedAt}}` | 郵件收到的時間 |

## 監控器摘要視圖

監控器摘要顯示：
- **最後收到郵件時間：** 最近一封郵件的接收時間
- **發件人：** 最後一封郵件的發件人
- **主題：** 最後一封郵件的主題行
- **郵件頭：** 最後一封郵件的完整頭信息（可展開）
- **郵件正文：** 最後一封郵件的內容（可展開）

## 自託管設置

如果您是自託管 OneUptime，需要配置入站郵件提供商。目前支持：

- **SendGrid Inbound Parse** — 設置說明請參見 [SendGrid 入站郵件集成](/docs/self-hosted/sendgrid-inbound-email)

## 注意事項

- **郵件地址安全性：** 監控器郵件地址包含密鑰。請將其視爲密碼，不要公開分享。
- **郵件大小：** 非常大的郵件（帶有大附件）可能會被郵件提供商截斷或拒絕。
- **處理時間：** 郵件是異步處理的。發送郵件和創建警報之間可能有幾秒延遲。
- **大小寫不敏感：** 所有字符串比較（包含、等於等）均不區分大小寫。
- **純文本：** 郵件正文標準使用郵件的純文本版本。HTML 格式會被去除。

## 故障排查

### 郵件未被接收

1. 驗證郵件地址是否正確（檢查是否有拼寫錯誤）
2. 檢查郵件是否被垃圾郵件過濾器攔截
3. 驗證您的入站郵件提供商是否正確配置
4. 檢查 OneUptime 日誌中的任何錯誤消息

### 未創建警報

1. 驗證您的標準是否與郵件內容匹配
2. 檢查監控器是否未被禁用
3. 查看監控器詳情中的評估日誌
4. 在使用模式匹配之前，先用精確字符串匹配進行測試

### 未解決警報

1. 驗證您的解決標準是否與恢復郵件匹配
2. 確保有活躍警報需要解決
3. 檢查解決郵件是否發送到同一監控器地址
