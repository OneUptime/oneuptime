# 性能分析監控器

性能分析監控允許您監控來自應用程序的持續分析數據，並根據分析數量和模式觸發警報。OneUptime 在時間窗口內評估來自遙測服務的分析數據。

## 概述

性能分析監控器統計並過濾符合特定條件的分析數據。這使您能夠：

- 監控來自應用程序的持續分析數據
- 按類型過濾分析（CPU、內存、goroutine 等）
- 跟蹤分析量和模式
- 對分析異常發出警報
- 按自定義分析屬性過濾

## 創建性能分析監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **性能分析** 作爲監控器類型
4. 選擇要監控的遙測服務
5. 根據需要配置分析過濾器和標準

## 配置選項

### 遙測服務

選擇一個或多個服務來監控其分析數據。服務必須通過 OpenTelemetry 向 OneUptime 發送持續分析數據。

### 分析過濾器

| 過濾器 | 描述 | 是否必填 |
|--------|------|---------|
| 分析類型 | 按分析類型名稱過濾（例如 CPU、內存、goroutine） | 否 |
| 屬性 | 用於過濾自定義分析屬性的鍵值對 | 否 |
| 時間窗口 | 搜索分析數據的時間範圍（秒，默認：60） | 否 |

## 監控標準

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 分析數量 | 時間窗口內符合過濾條件的分析數量 |

### 過濾類型

- **大於** — 分析數量超過閾值
- **小於** — 分析數量低於閾值
- **大於或等於** — 分析數量等於或超過閾值
- **小於或等於** — 分析數量等於或低於閾值
- **等於** — 分析數量完全匹配
- **不等於** — 分析數量不匹配

### 示例標準

#### 當 5 分鐘內未收到分析數據時發出警報

- **時間窗口**：300 秒
- **檢查項**：分析數量
- **過濾類型**：等於
- **值**：0

## 設置要求

性能分析監控需要您的應用程序通過 OpenTelemetry 向 OneUptime 發送持續分析數據。設置說明請參見 [OpenTelemetry](/docs/telemetry/open-telemetry) 文檔。
