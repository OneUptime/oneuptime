# SNMP 監控器

SNMP（簡單網絡管理協議）監控允許您通過查詢 SNMP OID（對象標識符）來監控交換機、路由器、防火牆和其他網絡基礎設施等網絡設備。

## 概述

SNMP 監控器使用 OID 查詢網絡設備的特定管理信息。這使您能夠：

- 監控設備可用性和健康狀況
- 跟蹤接口統計信息（流量、錯誤、狀態）
- 監控系統指標（CPU、內存、運行時間）
- 檢查自定義廠商特定的 OID
- 基於 OID 值設置警報

## 創建 SNMP 監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **SNMP** 作爲監控器類型
4. 按照下面的說明配置 SNMP 設置

## 配置選項

### 基本設置

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| SNMP 版本 | 協議版本：v1、v2c 或 v3 | 是 |
| 主機名/IP | SNMP 設備的主機名或 IP 地址 | 是 |
| 端口 | SNMP 端口（默認：161） | 是 |

### 認證

#### SNMP v1/v2c

對於 SNMP v1 和 v2c，您只需提供 community string：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| Community String | SNMP community string（例如"public"） | 是 |

#### SNMP v3

SNMPv3 提供增強的安全性，包括認證和加密：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 安全級別 | noAuthNoPriv、authNoPriv 或 authPriv | 是 |
| 用戶名 | SNMPv3 用戶名 | 是 |
| 認證協議 | MD5、SHA、SHA256 或 SHA512 | 如果是 authNoPriv 或 authPriv |
| 認證密鑰 | 認證密碼 | 如果是 authNoPriv 或 authPriv |
| 加密協議 | DES、AES 或 AES256 | 如果是 authPriv |
| 加密密鑰 | 隱私/加密密碼 | 如果是 authPriv |

### 要監控的 OID

添加您要從設備查詢的 OID。對於每個 OID，您可以指定：

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| OID | 數字 OID（例如 1.3.6.1.2.1.1.1.0） | 是 |
| 名稱 | OID 的友好名稱（例如 sysDescr） | 否 |
| 描述 | 此 OID 代表內容的描述 | 否 |

### 常用 OID 模板

OneUptime 爲常用的 OID 提供模板：

#### System MIB

| OID | 名稱 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.1.1.0 | sysDescr | 系統描述 |
| 1.3.6.1.2.1.1.3.0 | sysUpTime | 系統運行時間（時鐘滴答數） |
| 1.3.6.1.2.1.1.5.0 | sysName | 系統名稱 |
| 1.3.6.1.2.1.1.6.0 | sysLocation | 系統位置 |
| 1.3.6.1.2.1.1.4.0 | sysContact | 系統聯繫人 |

#### Interface MIB

| OID | 名稱 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.2.1.0 | ifNumber | 網絡接口數量 |
| 1.3.6.1.2.1.2.2.1.8.X | ifOperStatus | 接口操作狀態（X = 接口索引） |
| 1.3.6.1.2.1.2.2.1.10.X | ifInOctets | 輸入字節數（X = 接口索引） |
| 1.3.6.1.2.1.2.2.1.16.X | ifOutOctets | 輸出字節數（X = 接口索引） |

#### Host Resources MIB

| OID | 名稱 | 描述 |
|-----|------|------|
| 1.3.6.1.2.1.25.1.1.0 | hrSystemUptime | 主機系統運行時間 |
| 1.3.6.1.2.1.25.1.5.0 | hrSystemNumUsers | 用戶數量 |
| 1.3.6.1.2.1.25.1.6.0 | hrSystemProcesses | 運行進程數量 |
| 1.3.6.1.2.1.25.3.3.1.2.X | hrProcessorLoad | CPU 負載（X = 處理器索引） |

### 高級設置

| 字段 | 描述 | 默認值 |
|------|------|--------|
| 超時 | 等待響應的時間（毫秒） | 5000 |
| 重試次數 | 失敗時的重試次數 | 3 |

## 監控標準

您可以設置標準來檢查 SNMP 響應並觸發警報或事件。

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| SNMP 設備是否在線 | 檢查設備是否響應 SNMP 查詢 |
| SNMP 響應時間 | 檢查查詢響應時間（毫秒） |
| SNMP OID 值 | 檢查特定 OID 返回的值 |
| SNMP OID 是否存在 | 檢查 OID 是否返回值（非空） |

### 示例標準

#### 檢查設備是否在線
- **檢查項**：SNMP 設備是否在線
- **過濾類型**：True

#### 當響應時間超過閾值時發出警報
- **檢查項**：SNMP 響應時間（毫秒）
- **過濾類型**：大於
- **值**：1000

#### 檢查接口狀態
- **檢查項**：SNMP OID 值
- **OID**：1.3.6.1.2.1.2.2.1.8.1
- **過濾類型**：等於
- **值**：1（1 = 正常，2 = 故障）

#### 檢查 CPU 負載閾值
- **檢查項**：SNMP OID 值
- **OID**：1.3.6.1.2.1.25.3.3.1.2.1
- **過濾類型**：大於
- **值**：80

## 使用監控器密鑰

出於安全原因，您可以將 community string 和 SNMPv3 憑據等敏感信息儲存爲密鑰。

### 添加密鑰

1. 前往 **項目設置** -> **監控器密鑰** -> **創建監控器密鑰**
2. 添加您的密鑰（例如 community string 或 SNMPv3 密碼）
3. 選擇應有權訪問此密鑰的 SNMP 監控器

### 在 SNMP 配置中使用密鑰

在任何敏感字段中使用 `{{monitorSecrets.SECRET_NAME}}` 語法：

- **Community String**：`{{monitorSecrets.SnmpCommunity}}`
- **SNMPv3 認證密鑰**：`{{monitorSecrets.SnmpAuthKey}}`
- **SNMPv3 加密密鑰**：`{{monitorSecrets.SnmpPrivKey}}`

## 警報模板變量

創建事件或警報模板時，您可以使用以下變量：

| 變量 | 描述 |
|------|------|
| `{{isOnline}}` | 設備是否在線（true/false） |
| `{{responseTimeInMs}}` | 查詢響應時間（毫秒） |
| `{{failureCause}}` | 查詢失敗時的錯誤消息 |
| `{{oidResponses}}` | OID 響應對象數組 |
| `{{OID_NAME}}` | 按名稱獲取特定 OID 的值（例如 `{{sysUpTime}}`） |

## 故障排查

### 常見問題

#### 設備不響應
- 驗證設備 IP/主機名是否正確
- 檢查設備上是否啓用了 SNMP
- 驗證防火牆規則是否允許 UDP 端口 161
- 確認 community string 是否正確

#### 認證失敗（v3）
- 驗證用戶名、認證協議和認證密鑰
- 確保安全級別與設備配置匹配
- 檢查 authPriv 級別的加密協議和密鑰是否正確

#### OID 未找到
- 驗證 OID 是否受您的設備支持
- 檢查 OID 是否需要加載特定的 MIB
- 嘗試使用 snmpget/snmpwalk 工具直接查詢 OID

### 測試 SNMP 連接性

設置監控之前，您可以使用命令行工具測試 SNMP 連接性：

```bash
# SNMP v2c
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# SNMP v3 (authPriv)
snmpget -v3 -u username -l authPriv -a SHA -A authpassword -x AES -X privpassword 192.168.1.1 1.3.6.1.2.1.1.1.0
```

## 最佳實踐

1. **儘可能使用 SNMPv3** — 它提供認證和加密以增強安全性
2. **將憑據儲存爲密鑰** — 切勿硬編碼 community string 或密碼
3. **僅監控必要的 OID** — 只查詢所需內容以減少網絡開銷
4. **設置適當的超時** — 網絡設備的響應時間可能各不相同
5. **使用描述性 OID 名稱** — 使警報消息更易於理解
6. **部署前先測試** — 在創建監控器之前驗證 SNMP 連接性
