# 異常監控器

異常監控允許您監控應用程序異常和錯誤，當異常數量超過配置的閾值時觸發警報。OneUptime 在一個時間窗口內評估來自您遙測服務的異常數據。

## 概述

異常監控器統計並過濾符合特定條件的異常。這使您能夠：

- 對應用程序中的異常峯值發出警報
- 監控特定的異常類型
- 按錯誤消息搜索異常
- 分別跟蹤已解決和活躍的異常
- 從錯誤模式中檢測應用程序穩定性問題

## 創建異常監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **異常** 作爲監控器類型
4. 選擇要監控的遙測服務
5. 根據需要配置異常過濾器和標準

## 配置選項

### 遙測服務

選擇一個或多個服務來監控其異常。服務必須通過 OpenTelemetry 向 OneUptime 發送異常數據。

### 異常過濾器

| 過濾器 | 描述 | 是否必填 |
|--------|------|---------|
| 異常類型 | 按異常類型名稱過濾（例如 `NullPointerException`、`TypeError`） | 否 |
| 消息 | 在異常消息中進行文本搜索 | 否 |
| 包含已解決 | 包含已標記爲已解決的異常（默認：false） | 否 |
| 包含已歸檔 | 包含已歸檔的異常（默認：false） | 否 |
| 時間窗口 | 搜索異常的時間範圍（秒，默認：60） | 否 |

## 監控標準

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 異常數量 | 時間窗口內符合過濾條件的異常數量 |

### 過濾類型

- **大於** — 異常數量超過閾值
- **小於** — 異常數量低於閾值
- **大於或等於** — 異常數量等於或超過閾值
- **小於或等於** — 異常數量等於或低於閾值
- **等於** — 異常數量完全匹配
- **不等於** — 異常數量不匹配

### 示例標準

#### 當 60 秒內超過 10 個異常時發出警報

- **時間窗口**：60 秒
- **檢查項**：異常數量
- **過濾類型**：大於
- **值**：10

#### 對任何 NullPointerException 發出警報

- **異常類型**：`NullPointerException`
- **時間窗口**：60 秒
- **檢查項**：異常數量
- **過濾類型**：大於
- **值**：0

#### 監控包含特定消息的異常

- **消息**：`out of memory`
- **時間窗口**：300 秒
- **檢查項**：異常數量
- **過濾類型**：大於
- **值**：0

## 設置要求

異常監控需要您的應用程序通過 OpenTelemetry 向 OneUptime 發送異常數據。設置說明請參見 [OpenTelemetry](/docs/telemetry/open-telemetry) 文檔。
