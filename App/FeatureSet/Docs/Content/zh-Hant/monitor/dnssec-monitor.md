# DNSSEC 監控器

DNSSEC 監控允許您驗證區域 DNS 響應的加密完整性。OneUptime 定期執行完整的 DNSSEC 驗證 — 檢查 DNSKEY 記錄、父區域的 DS 委派、RRSIG 簽名有效性、解析器對 AD flag 的共識，以及權威名稱服務器之間的一致性。

## 概述

DNSSEC 監控器驗證從根區域到您的域名的整個信任鏈。這使您能夠：

- 在解析器開始向用戶返回 SERVFAIL 之前檢測損壞的 DNSSEC 鏈
- 在區域簽名密鑰過期之前收到警告
- 驗證您的 DS 記錄已在父區域正確發佈
- 捕獲權威名稱服務器之間的差異（主/從不同步）
- 確認驗證解析器確實爲您的區域設置了 AD flag

## 創建 DNSSEC 監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **DNSSEC** 作爲監控器類型
4. 輸入要驗證的區域（域名）
5. 根據需要配置解析器和監控標準

## 配置選項

### 基本設置

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 區域（域名） | 通過 DNSSEC 驗證的區域（例如 `example.com`） | 是 |
| 解析器 | 要查詢的驗證解析器的逗號分隔列表（例如 `1.1.1.1, 8.8.8.8, 9.9.9.9`） | 是 |
| 檢查名稱服務器一致性 | 直接查詢每個權威名稱服務器，並驗證它們返回相同的 SOA serial | 否 |

### 高級設置

| 字段 | 描述 | 默認值 |
|------|------|--------|
| 簽名過期警告（天） | RRSIG 過期過濾器的默認閾值 | 7 |
| 超時（毫秒） | 等待每個 DNS 查詢的時長 | 10000 |
| 重試次數 | 失敗時的重試次數 | 3 |

## 監控標準

您可以配置標準來判斷區域何時處於在線、降級或離線狀態，基於以下條件：

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| DNSSEC 鏈是否有效 | 整個驗證鏈（根 → TLD → 區域）正確解析 |
| DNSSEC DNSKEY 記錄是否存在 | 區域至少發佈了一條 DNSKEY 記錄 |
| 父區域是否存在 DNSSEC DS 記錄 | 父區域發佈的 DS 記錄與該區域的 KSK 匹配 |
| DNSSEC 簽名剩餘有效天數 | 距離最早過期的 RRSIG 簽名的天數 |
| DNSSEC 解析器共識（AD Flag） | 每個查詢的解析器都返回 AD（Authenticated Data）flag |
| DNSSEC 名稱服務器是否一致 | 所有權威名稱服務器返回相同的 SOA serial |
| DNSSEC 是否有效 | 所有驗證檢查的綜合通過/失敗 |

### 過濾類型

對於 **DNSSEC 鏈是否有效**、**DNSSEC DNSKEY 記錄是否存在**、**父區域是否存在 DNSSEC DS 記錄**、**DNSSEC 解析器共識（AD Flag）**、**DNSSEC 名稱服務器是否一致** 和 **DNSSEC 是否有效**：

- **True** — 條件爲真
- **False** — 條件爲假

對於 **DNSSEC 簽名剩餘有效天數**：

- **大於**、**小於**、**大於或等於**、**小於或等於**、**等於**、**不等於**

### 示例標準

#### 當 DNSSEC 鏈損壞時警報

- **檢查項**：DNSSEC 鏈是否有效
- **過濾類型**：False

#### 在簽名過期前發出警告

- **檢查項**：DNSSEC 簽名剩餘有效天數
- **過濾類型**：小於
- **值**：7

#### 捕獲父區域缺失的 DS（委派損壞）

- **檢查項**：父區域是否存在 DNSSEC DS 記錄
- **過濾類型**：False

#### 檢測解析器之間的分歧

- **檢查項**：DNSSEC 解析器共識（AD Flag）
- **過濾類型**：False

#### 捕獲名稱服務器數據不一致

- **檢查項**：DNSSEC 名稱服務器是否一致
- **過濾類型**：False

## 最佳實踐

1. **使用多個公共解析器** — 默認使用 `1.1.1.1`、`8.8.8.8` 和 `9.9.9.9`，避免單個解析器故障導致誤報
2. **在過期之前充分提前警報** — 在簽名過期前 7 天配置降級警報，在 2 天前配置離線警報；密鑰滾動可能會靜默失敗
3. **監控每個已簽名的區域** — 包括 apex 域名、已簽名的子域名以及委派給不同運營商的任何區域
4. **啓用名稱服務器一致性檢查** — 除非您的網絡阻止對任意 IP 的出站 DNS，否則可捕獲 DNSSEC 驗證本身無法發現的主/從同步問題
