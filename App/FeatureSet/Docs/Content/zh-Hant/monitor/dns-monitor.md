# DNS 監控器

DNS 監控允許您監控域名的 DNS 解析健康狀況和正確性。OneUptime 定期查詢 DNS 記錄並根據您配置的標準驗證響應。

## 概述

DNS 監控器查詢 DNS 服務器以獲取特定記錄類型並評估結果。這使您能夠：

- 監控 DNS 服務可用性
- 驗證 DNS 記錄返回正確的值
- 跟蹤 DNS 解析響應時間
- 驗證 DNSSEC 配置
- 檢測 DNS 傳播問題或劫持

## 創建 DNS 監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **DNS** 作爲監控器類型
4. 輸入要查詢的域名和記錄類型
5. 根據需要配置監控標準

## 配置選項

### 基本設置

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 域名 | 要查詢的域（例如 `example.com`） | 是 |
| 記錄類型 | 要查詢的 DNS 記錄類型 | 是 |
| DNS 服務器 | 要使用的自定義 DNS 服務器（例如 `8.8.8.8`）。留空使用系統默認值 | 否 |

### 支持的記錄類型

| 記錄類型 | 描述 |
|---------|------|
| A | IPv4 地址記錄 |
| AAAA | IPv6 地址記錄 |
| CNAME | 規範名稱（別名）記錄 |
| MX | 郵件交換記錄 |
| NS | 名稱服務器記錄 |
| TXT | 文本記錄（SPF、DKIM 等） |
| SOA | 授權起始記錄 |
| PTR | 指針記錄（反向 DNS） |
| SRV | 服務定位器記錄 |
| CAA | 證書授權機構授權記錄 |

### 高級設置

| 字段 | 描述 | 默認值 |
|------|------|--------|
| 端口 | DNS 端口號 | 53 |
| 超時（毫秒） | 等待響應的時間 | 5000 |
| 重試次數 | 失敗時的重試次數 | 3 |

## 監控標準

您可以配置標準來判斷 DNS 何時處於在線、降級或離線狀態，基於以下條件：

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| DNS 是否在線 | DNS 服務器是否響應查詢 |
| DNS 響應時間（毫秒） | 查詢響應時間（毫秒） |
| DNS 記錄是否存在 | 查詢的 DNS 記錄是否存在 |
| DNS 記錄值 | DNS 記錄返回的值 |
| DNSSEC 是否有效 | DNSSEC 驗證是否通過 |

### 過濾類型

對於 **DNS 是否在線**、**DNS 記錄是否存在** 和 **DNSSEC 是否有效**：

- **True** — 條件爲真
- **False** — 條件爲假

對於 **DNS 響應時間**：

- **大於**、**小於**、**大於或等於**、**小於或等於**、**等於**、**不等於**

對於 **DNS 記錄值**：

- **包含** — 記錄值包含指定文本
- **不包含** — 記錄值不包含指定文本
- **以...開頭** — 記錄值以指定文本開頭
- **以...結尾** — 記錄值以指定文本結尾
- **等於** — 記錄值完全匹配
- **不等於** — 記錄值不匹配

### 示例標準

#### 檢查 DNS 是否正常解析

- **檢查項**：DNS 是否在線
- **過濾類型**：True

#### 驗證 A 記錄指向正確的 IP

- **檢查項**：DNS 記錄值
- **過濾類型**：等於
- **值**：`93.184.216.34`

#### 當 DNS 響應緩慢時警報

- **檢查項**：DNS 響應時間（毫秒）
- **過濾類型**：大於
- **值**：500

#### 驗證 DNSSEC 是否有效

- **檢查項**：DNSSEC 是否有效
- **過濾類型**：True
