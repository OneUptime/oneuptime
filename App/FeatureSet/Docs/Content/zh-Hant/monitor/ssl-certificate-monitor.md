# SSL 證書監控器

SSL 證書監控允許您監控網站和服務上 SSL/TLS 證書的有效性和過期情況。OneUptime 定期檢查您的證書，並在證書過期之前或檢測到任何問題時提醒您。

## 概述

SSL 證書監控器連接到您的 HTTPS 端點並檢查 SSL/TLS 證書。這使您能夠：

- 監控證書過期日期
- 檢測已過期或即將過期的證書
- 識別自簽名證書
- 驗證證書有效性
- 防止因證書過期導致的服務中斷

## 創建 SSL 證書監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **SSL 證書** 作爲監控器類型
4. 輸入要檢查的 HTTPS 端點的 URL
5. 根據需要配置監控標準

## 配置選項

### URL

輸入您要監控其 SSL 證書的端點的完整 HTTPS URL（例如 `https://example.com` 或 `https://example.com:8443`）。

## 監控標準

您可以配置標準來判斷證書狀態何時處於在線、降級或離線，基於以下條件：

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 是否在線 | 服務器是否可達 |
| 是否有效證書 | 證書是否有效（未過期、非自簽名） |
| 是否爲自簽名證書 | 證書是否爲自簽名 |
| 證書是否已過期 | 證書是否已過期 |
| 是否爲無效證書 | 證書是否無效 |
| 在 X 小時內過期 | 證書過期前的小時數 |
| 在 X 天內過期 | 證書過期前的天數 |
| 請求是否超時 | 連接是否超時 |

### 過濾類型

對於 **是否在線**、**是否有效證書**、**是否爲自簽名證書**、**證書是否已過期**、**是否爲無效證書** 和 **請求是否超時**：

- **True** — 條件爲真
- **False** — 條件爲假

對於 **在 X 小時內過期** 和 **在 X 天內過期**：

- **大於** — 過期時間距離超過指定值
- **小於** — 過期時間距離少於指定值
- **大於或等於** — 過期時間距離等於或超過指定值
- **小於或等於** — 過期時間距離等於或少於指定值
- **等於** — 過期時間完全匹配
- **不等於** — 過期時間不匹配

### 示例標準

#### 當證書在 30 天內過期時標記爲降級

- **檢查項**：在 X 天內過期
- **過濾類型**：小於
- **值**：30

#### 當證書已過期時標記爲離線

- **檢查項**：證書是否已過期
- **過濾類型**：True

#### 當證書爲自簽名時發出警報

- **檢查項**：是否爲自簽名證書
- **過濾類型**：True

#### 當證書無效時標記爲離線

- **檢查項**：是否爲無效證書
- **過濾類型**：True

## 最佳實踐

1. **設置多個閾值** — 在過期前 30 天使用降級狀態，在過期前 7 天使用離線狀態，給自己足夠的續簽時間
2. **監控所有端點** — 如果您有多個域名或子域名，爲每個創建一個監控器
3. **包含非標準端口** — 不要忘記在非標準端口上運行 HTTPS 的服務
4. **續簽後監控** — 證書續簽後，驗證監控器確認其有效
