# 手動監控器

手動監控允許您創建狀態完全由手動或通過 API 管理的監控器。OneUptime 不執行任何自動檢查——您直接控制監控器狀態。

## 概述

手動監控器是您自行更新的佔位符。這在以下情況非常有用：

- 與通過 OneUptime API 更新狀態的外部監控工具集成
- 跟蹤無法自動監控的服務或系統
- 管理沒有自動健康檢查的組件的事件
- 表示您手動跟蹤狀態的第三方依賴項

## 創建手動監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **手動** 作爲監控器類型
4. 輸入監控器的名稱和描述

## 工作原理

手動監控器沒有監控間隔、探針或自動標準評估。監控器狀態保持您設置的狀態，直到您更改它。

### 更新狀態

您可以通過兩種方式更新手動監控器的狀態：

- **控制台** — 直接從 OneUptime 控制台更改監控器狀態
- **API** — 使用 OneUptime API 以程序化方式更新監控器狀態

### 事件和警報

您可以像對任何其他監控器類型一樣，針對手動監控器創建事件和警報。這允許您：

- 跟蹤外部監控服務的停機時間
- 在報告問題時手動創建事件
- 在狀態頁面上使用手動監控器向用戶傳達狀態

## 何時使用手動監控器

| 使用場景 | 描述 |
|---------|------|
| 第三方服務 | 跟蹤您依賴但無法直接監控的外部服務狀態 |
| 物理基礎設施 | 表示沒有網絡監控的硬件或物理系統 |
| 業務流程 | 跟蹤影響服務狀態的非技術流程 |
| API 驅動的狀態 | 讓外部工具通過 OneUptime API 更新監控器狀態 |
| 狀態頁面佔位符 | 在狀態頁面上顯示 OneUptime 外部管理的組件 |
