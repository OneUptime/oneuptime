# 監控器密鑰

您可以使用密鑰儲存您在監控檢查中想要使用的敏感信息。密鑰經過加密並安全儲存。

### 添加密鑰

要添加密鑰，請前往 OneUptime 控制台 -> 項目設置 -> 監控器密鑰 -> 創建監控器密鑰。

![創建密鑰](/docs/static/images/CreateMonitorSecret.png)

您可以選擇哪些監控器有權訪問該密鑰。在本例中，我們添加了 `ApiKey` 密鑰並選擇了有權訪問它的監控器。

**請注意**：密鑰經過加密並安全儲存。如果您丟失了密鑰，需要創建新密鑰。保存後無法查看或更新密鑰。

### 使用密鑰

您可以在以下監控類型中使用密鑰：

- API（在請求頭、請求體和 URL 中）
- 網站、IP、端口、Ping、SSL 證書（在 URL 中）
- 合成監控器、自定義代碼監控器（在代碼中）
- SNMP 監控器（在 community string、SNMPv3 認證密鑰和加密密鑰中）


![使用密鑰](/docs/static/images/UsingMonitorSecret.png)

要使用密鑰，請在您想使用密鑰的字段中添加 `{{monitorSecrets.SECRET_NAME}}`。例如，在本例中，我們在請求頭字段中添加了 `{{monitorSecrets.ApiKey}}`。

密鑰在探針上注入，在合成監控器或自定義代碼監控器腳本執行之前，`{{monitorSecrets.ApiKey}}` 等引用將在運行中的腳本內解析爲解密後的值。


### 監控器密鑰權限

您可以選擇哪些監控器有權訪問密鑰。您也可以隨時更新權限。因此，如果您想讓新的監控器訪問密鑰，可以通過更新權限來實現。
