# 日誌監控器

日誌監控允許您監控應用程序日誌，並根據日誌模式、數量和嚴重程度觸發警報。OneUptime 評估來自遙測服務的日誌，並根據您配置的標準進行檢查。

## 概述

日誌監控器在時間窗口內搜索並統計符合特定過濾器的日誌。這使您能夠：

- 對錯誤日誌峯值發出警報
- 監控特定的日誌模式或消息
- 按嚴重程度跟蹤日誌量
- 按服務、屬性和內容過濾日誌
- 從日誌模式中檢測應用程序問題

## 創建日誌監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **日誌** 作爲監控器類型
4. 選擇要監控的遙測服務
5. 根據需要配置日誌過濾器和標準

## 配置選項

### 遙測服務

選擇一個或多個服務來監控其日誌。服務必須通過 OpenTelemetry 向 OneUptime 發送日誌。

### 日誌過濾器

| 過濾器 | 描述 | 是否必填 |
|--------|------|---------|
| 嚴重程度 | 按日誌嚴重程度過濾（ERROR、WARN、INFO、DEBUG 等） | 否 |
| 正文 | 在日誌消息正文中進行文本搜索 | 否 |
| 屬性 | 用於過濾自定義日誌屬性的鍵值對 | 否 |
| 時間窗口 | 搜索日誌的時間範圍（秒，默認：60） | 否 |

### 嚴重程度

按一個或多個嚴重程度過濾日誌：

- **FATAL** / **EMERGENCY** / **CRITICAL**
- **ERROR**
- **WARN** / **WARNING**
- **INFO** / **INFORMATIONAL**
- **DEBUG**
- **TRACE**
- **UNSPECIFIED**

## 監控標準

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 日誌數量 | 時間窗口內符合過濾條件的日誌數量 |

### 過濾類型

- **大於** — 日誌數量超過閾值
- **小於** — 日誌數量低於閾值
- **大於或等於** — 日誌數量等於或超過閾值
- **小於或等於** — 日誌數量等於或低於閾值
- **等於** — 日誌數量完全匹配
- **不等於** — 日誌數量不匹配

### 示例標準

#### 當 60 秒內超過 100 條錯誤日誌時發出警報

- **嚴重程度**：ERROR
- **時間窗口**：60 秒
- **檢查項**：日誌數量
- **過濾類型**：大於
- **值**：100

#### 當出現任何致命日誌時發出警報

- **嚴重程度**：FATAL
- **時間窗口**：60 秒
- **檢查項**：日誌數量
- **過濾類型**：大於
- **值**：0

#### 監控包含特定錯誤消息的日誌

- **正文**：`database connection timeout`
- **時間窗口**：300 秒
- **檢查項**：日誌數量
- **過濾類型**：大於
- **值**：5

## 設置要求

日誌監控需要您的應用程序通過 OpenTelemetry 向 OneUptime 發送日誌。設置說明請參見 [OpenTelemetry](/docs/telemetry/open-telemetry) 文檔。
