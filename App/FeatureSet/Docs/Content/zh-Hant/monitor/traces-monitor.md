# 追蹤監控器

追蹤監控允許您監控來自應用程序的分佈式追蹤，並根據 Span 模式、數量和狀態觸發警報。OneUptime 在時間窗口內評估來自遙測服務的追蹤數據。

## 概述

追蹤監控器搜索並統計符合特定過濾器的 Span。這使您能夠：

- 對服務中的錯誤 Span 峯值發出警報
- 監控特定操作和端點
- 跟蹤 Span 量和模式
- 按 Span 狀態、名稱和自定義屬性過濾
- 從追蹤數據中檢測性能和可靠性問題

## 創建追蹤監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **追蹤** 作爲監控器類型
4. 選擇要監控的遙測服務
5. 根據需要配置 Span 過濾器和標準

## 配置選項

### 遙測服務

選擇一個或多個服務來監控其追蹤。服務必須通過 OpenTelemetry 向 OneUptime 發送追蹤數據。

### Span 過濾器

| 過濾器 | 描述 | 是否必填 |
|--------|------|---------|
| Span 狀態 | 按 Span 狀態碼過濾（OK、ERROR、UNSET） | 否 |
| Span 名稱 | 搜索特定 Span 名稱（例如操作或端點名稱）的文本 | 否 |
| 屬性 | 用於過濾自定義 Span 屬性的鍵值對 | 否 |
| 時間窗口 | 搜索 Span 的時間範圍（秒，默認：60） | 否 |

### Span 狀態碼

- **OK** — 操作成功完成
- **ERROR** — 操作遇到錯誤
- **UNSET** — 狀態未明確設置

## 監控標準

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| Span 數量 | 時間窗口內符合過濾條件的 Span 數量 |

### 過濾類型

- **大於** — Span 數量超過閾值
- **小於** — Span 數量低於閾值
- **大於或等於** — Span 數量等於或超過閾值
- **小於或等於** — Span 數量等於或低於閾值
- **等於** — Span 數量完全匹配
- **不等於** — Span 數量不匹配

### 示例標準

#### 當 60 秒內超過 50 個錯誤 Span 時發出警報

- **Span 狀態**：ERROR
- **時間窗口**：60 秒
- **檢查項**：Span 數量
- **過濾類型**：大於
- **值**：50

#### 對特定端點的錯誤發出警報

- **Span 名稱**：`POST /api/checkout`
- **Span 狀態**：ERROR
- **時間窗口**：120 秒
- **檢查項**：Span 數量
- **過濾類型**：大於
- **值**：0

## 設置要求

追蹤監控需要您的應用程序通過 OpenTelemetry 向 OneUptime 發送分佈式追蹤數據。設置說明請參見 [OpenTelemetry](/docs/telemetry/open-telemetry) 文檔。
