# 端口監控器

端口監控允許您監控主機上特定 TCP 或 UDP 端口的可用性。OneUptime 定期嘗試連接到指定端口，並檢查其是否開放且響應正常。

## 概述

端口監控器測試特定網絡端口是否接受連接。這使您能夠：

- 監控特定端口上的服務可用性
- 跟蹤端口響應時間
- 驗證數據庫、郵件服務器和應用服務器等服務是否正在運行
- 在服務中斷影響用戶之前檢測到它們

## 創建端口監控器

1. 在 OneUptime 控制台中轉到 **監控器**
2. 點擊 **創建監控器**
3. 選擇 **端口** 作爲監控器類型
4. 輸入主機名或 IP 地址和端口號
5. 根據需要配置監控標準

## 配置選項

### 主機名或 IP 地址

輸入目標主機的主機名或 IP 地址（例如 `example.com` 或 `192.168.1.1`）。

### 端口

輸入要監控的端口號（1–65535）。常見示例：

| 端口 | 服務 |
|------|------|
| 22 | SSH |
| 25 | SMTP |
| 80 | HTTP |
| 443 | HTTPS |
| 3306 | MySQL |
| 5432 | PostgreSQL |
| 6379 | Redis |
| 27017 | MongoDB |

## 監控標準

您可以配置標準來判斷端口何時處於在線、降級或離線狀態，基於以下條件：

### 可用檢查類型

| 檢查類型 | 描述 |
|---------|------|
| 是否在線 | 端口是否開放且接受連接 |
| 響應時間（毫秒） | 建立連接的時間（毫秒） |
| 請求是否超時 | 連接嘗試是否超時 |

### 過濾類型

對於 **是否在線** 和 **請求是否超時**：

- **True** — 條件爲真
- **False** — 條件爲假

對於 **響應時間**：

- **大於** — 響應時間超過閾值
- **小於** — 響應時間低於閾值
- **大於或等於** — 響應時間等於或超過閾值
- **小於或等於** — 響應時間等於或低於閾值
- **等於** — 響應時間完全匹配
- **不等於** — 響應時間不匹配
- **隨時間評估** — 使用聚合（平均值、求和、最大值、最小值、所有值、任意值）在時間窗口內評估

### 示例標準

#### 如果端口關閉則標記爲離線

- **檢查項**：是否在線
- **過濾類型**：False

#### 如果連接時間超過 500ms 則發出警報

- **檢查項**：響應時間（毫秒）
- **過濾類型**：大於
- **值**：500

#### 如果連接緩慢則標記爲降級

- **檢查項**：響應時間（毫秒）
- **過濾類型**：大於
- **值**：200
