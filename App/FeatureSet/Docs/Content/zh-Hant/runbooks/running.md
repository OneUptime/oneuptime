# 運行 Runbook

Runbook 執行有三種生成方式：

1. **通過規則自動觸發** — 見 [Runbook 規則](/docs/runbooks/rules)。
2. **從 Runbook 頁面手動觸發** — 在 Runbook 概覽頁點 **立即運行**。不會附屬任何事件、警報或計劃維護。
3. **從實體動態手動觸發** — 在事件、警報或計劃維護事件上點 **運行 Runbook**。執行會被附屬到該實體。

## 執行視圖

打開任意一次執行查看其清單 UI。每個步驟顯示：

- **狀態徽標** — Pending、Running、等你處理、Done、Skipped、Failed。
- **標題與描述** — 在執行創建時從 Runbook 拷貝過來。
- **輸出**（可摺疊） — stdout、返回值、HTTP 響應。
- 步驟失敗時的 **錯誤信息**。
- 對於 `WaitingForUser` 狀態的 Manual 步驟：**標記完成** 與 **跳過** 按鈕。

執行未進入終態期間，頁面每 3 秒輪詢一次，所以你能近實時看到自動步驟完成。

## 手動與自動步驟交織

經典流程：

1. **腳本步驟**：捕獲系統狀態，寫到 S3。
2. **手動步驟**："通過狀態頁橫幅通知客戶。" 響應者勾選。
3. **HTTP 步驟**：通過 PagerDuty 呼叫 DBA。
4. **手動步驟**："確認副本 DB 現在是主庫。" 響應者勾選。
5. **腳本步驟**：發送恢復 Slack 消息。

步驟 2 與 4 會讓執行暫停直到被勾選。步驟 1、3、5 自動運行。整個運行是一次執行、一條時間線、一份事實來源。

## 取消運行

在執行頁點 **取消執行**。當前步驟（如果有）會跑完；後續步驟不再啓動。狀態變爲 `Cancelled`。

## 輸出保留

每步輸出上限爲 **50KB**，以防失控的腳本把數據庫撐爆。如果需要更大產物，從腳本寫到 S3 或日誌器，把 URL 放在返回值裏。

## 重新運行一個 Runbook

Runbook 的執行是一次性的、不可變的記錄。要重新運行，再點一次 **立即運行** — 那會創建一次新的執行，並附上當前步驟的新快照。原來的執行會原樣保留，作爲審計軌跡。

## 查找過往執行

每個 Runbook 都有 **執行** 標籤，列出它的所有運行，可按狀態、日期範圍和來源實體過濾。從事件、警報或計劃維護事件頁面，**Runbooks** 標籤會顯示附屬到該實體的運行。
