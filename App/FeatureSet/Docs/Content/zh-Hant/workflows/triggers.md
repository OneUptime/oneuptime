# 觸發器

觸發器是工作流的第一個模塊——它決定工作流何時運行。每個工作流恰好有一個觸發器。你可以從四種類型中選擇。

## 手動

通過在工作流頁面上點擊 **手動運行** 按需運行工作流。你可以粘貼一段 JSON 負載,工作流的其餘部分可以讀取它。

適合用於:你想要一鍵自動化的場景,比如"輪換這個密鑰"或"發一個測試警報"。

**輸出**:你粘貼的 JSON,如果沒粘貼則是一個空對象。

## 定時

使用 cron 表達式按重複的計劃運行工作流。

適合用於:夜間清理、每小時同步、每週報表。

**設置**:一個 cron 表達式。幾個常見的:

- `0 * * * *`——每小時整點。
- `*/5 * * * *`——每 5 分鐘。
- `0 9 * * 1`——每週一上午 9:00。

如果系統短時間不可用,運行會在恢復後立即被領取——短時中斷不必擔心錯過的觸發點。

## Webhook

OneUptime 會創建一個專屬 URL。任何對該 URL 的訪問都會啓動工作流。請求的頭、查詢參數和請求體都會被傳入。

適合用於:從其他工具接收數據到 OneUptime——CI/CD 回調、來自其他監控的警報、你 CRM 的註冊事件。

**輸出**:

- **Request Headers**——入站請求的所有頭部。
- **Request Query Params**——解析後的查詢字符串。
- **Request Body**——解析後的請求體(如果不是 JSON,則爲原始文本)。

該 URL 同時接受 `GET` 和 `POST`。調用方會立即收到確認——工作流本身在後臺運行。

把 URL 當作密碼對待。任何拿到它的人都可以啓動你的工作流。

## OneUptime 事件觸發器

OneUptime 中幾乎所有對象——監控、事件、警報、計劃維護、狀態頁、值班策略、團隊——都可以觸發工作流。每種對象都提供三種事件:

- **創建時**——當新建一個時觸發。
- **更新時**——當某個被更改時觸發。
- **刪除時**——當某個被刪除時觸發。

這就是你構建"當 OneUptime 中發生 X 時,做 Y"的方式,不需要循環輪詢。

完整記錄會傳遞給下一個模塊。例如,**事件 → 創建時** 觸發器會傳入新建的事件,下一個模塊就可以讀取它的標題、描述、嚴重程度以及任何其他字段。

### 團隊最常用的事件

- **事件**——當事件被打開、更新(已確認、已解決)或刪除時響應。
- **警報**——警報的同樣三種事件。
- **監控**——當監控被添加、編輯或移除時響應。
- **計劃維護**——在計劃好後自動公告維護窗口。
- **狀態頁訂閱者**——歡迎訂閱狀態頁的人。
- **值班策略**——把排班變更同步到另一個排班系統。

按名稱搜索觸發器面板找到你想要的那個。

## 我該用哪個觸發器?

| 如果你想… | 選擇 |
| --- | --- |
| 點一個按鈕運行工作流 | **手動** |
| 按重複計劃運行 | **定時** |
| 讓另一個系統把數據推進來 | **Webhook** |
| 響應 OneUptime 內部的事件 | **OneUptime 事件** |

一個工作流只能有一個觸發器。如果你需要兩種方式啓動同一個自動化,把共享邏輯放在一個工作流中,然後用 **執行工作流** 組件從兩個輕量的"包裝"工作流中調用它。
