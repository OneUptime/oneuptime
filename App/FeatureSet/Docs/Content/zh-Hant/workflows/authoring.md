# 創作工作流

要創建工作流,打開 **工作流 → 創建工作流**,給它起一個名稱,然後點擊進入 **構建器** 標籤。你會看到一張空白畫布,接下來就在上面構建自動化。

## 畫布

構建器是一張拖拽式畫布。你從側邊的面板裏添加模塊,用線把它們連接起來,然後點擊每個模塊來配置它要做的事。改動會自動保存——保存後頂部會顯示一個指示。

每個工作流的開頭都是一個 **觸發器**。其餘的都是 **組件**——具體執行操作的部分。

## 模塊上有什麼

| 字段 | 作用 |
| --- | --- |
| **標題** | 在畫布上顯示的名稱。重命名它能讓複雜工作流更易讀。 |
| **設置** | 模塊完成工作所需的內容——URL、Slack 頻道、消息正文等。必填字段會帶星號標記。 |
| **輸入** | 模塊左側的小圓點,來自前面模塊的連線在這裏匯入。 |
| **輸出** | 模塊右側的小圓點,從這裏連線指向下一個模塊。許多模塊有獨立的 **成功** 和 **錯誤** 輸出,這樣你可以分別處理兩種情況。 |

## 連接模塊

從一個模塊的輸出點拖到下一個模塊的輸入點。你畫的這條線決定了接下來執行什麼。

- 如果從 **成功** 輸出連出,只有前一個模塊執行成功時,下一個模塊纔會運行。
- 如果從 **錯誤** 輸出連出,只有前一個模塊執行失敗時,下一個模塊纔會運行。
- 如果某個輸出沒有連接,那條路徑就停在那裏。

你可以把一個輸出連接到多個模塊。它們會從那一點開始同時運行。

## 配置模塊

點擊模塊,側邊會打開它的設置。每個設置都有合適的輸入方式——文本框、下拉菜單、代碼編輯器、開關等等。

大多數文本字段都接受變量——這就是數據在模塊之間流動的方式。具體語法見 [變量](/docs/workflows/variables)。

## 你的第一個工作流

熟悉畫布最快的方式:

1. 拖一個 **手動** 觸發器到畫布上。
2. 在它旁邊拖一個 **日誌** 組件(在 **工具** 下)。把觸發器連接到日誌組件。
3. 在日誌模塊的消息字段裏,輸入 `Hello from {{Manual.JSON.name}}`。
4. 保存並啓用工作流。
5. 點擊 **手動運行**,粘貼 `{ "name": "Ada" }` 作爲輸入,然後提交。
6. 打開 **日誌** 標籤。最近一次運行會顯示 `Hello from Ada`。

這個循環——拖入、連接、配置、運行、查看日誌——就是你構建每個工作流的方式。

## 保存與啓用

畫布會在你工作時自動保存。沒有單獨的"發佈"步驟。

但工作流只有在 **設置** 中的 **已啓用** 開關打開時才真正運行。新工作流默認是停用的。把這個開關當作你的安全網——先構建、用 **手動運行** 測試、檢查日誌,然後再打開。

要暫停工作流而不刪除它,把 **已啓用** 關掉。正在進行中的運行會跑完;但不會開始新的運行。

## 整理畫布

- 拖動模塊來移動它們。佈局會被保存,下次別人看到的還是同樣的排列。
- 右鍵單擊連線刪除它。右鍵單擊模塊可以刪除或複製它。
- 對於較寬的工作流,從左到右排列,這樣閱讀方向就和執行方向一致。
