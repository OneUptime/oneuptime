# 工作流概覽

工作流讓你無需寫代碼就能自動化 OneUptime 裏的各種任務。把幾個模塊拖到畫布上、連起來,就有了一個會在事件發生時運行的自動化——比如打開一個事件、定時觸發,或者其他工具向 OneUptime 推送數據。

可以把工作流看作項目的後臺幫手:它們響應事件、與其他工具對話,在你專注於工作時悄悄保持各處同步。

## 工作流能做什麼

- **把 OneUptime 接入其他工具**——把事件發送到 Slack、在 Jira 中創建工單、向你技術棧中的 webhook 發起調用。
- **響應 OneUptime 中發生的事情**——當一個嚴重事件被創建時,通知值班團隊並自動開工單。
- **按計劃運行任務**——每五分鐘、每晚、每週一早上。
- **從外部接收數據**——通過專屬 URL 讓其他系統把數據推送到 OneUptime。
- **複用常用自動化**——構建一次,從其他任何工作流裏調用。

## 工作流的工作原理

每個工作流都有三個部分:

1. **一個觸發器**——決定工作流何時啓動。可以是手動按鈕、定時計劃、入站 webhook,或者 OneUptime 內部的事件(比如新建事件)。
2. **一個或多個組件**——工作流要做的事。發送一條消息、發起一次 HTTP 調用、運行一次快速檢查、按條件分支。
3. **它們之間的連接**——你從一個模塊畫線到下一個,決定執行順序。

所有這些都在畫布上以可視化方式構建。大多數工作流不需要編碼,不過當你確實需要時,也可以加入一段 JavaScript。

## 關鍵術語

| 術語 | 含義 |
| --- | --- |
| **工作流** | 整個自動化——一個名稱、一張畫布,以及一個開關來啓用或停用它。 |
| **觸發器** | 第一個模塊。決定工作流何時運行。每個工作流恰好有一個觸發器。 |
| **組件** | 動作模塊——發送消息、發起請求、檢查條件。 |
| **運行** | 工作流的一次執行。保存了時間戳和每個模塊的輸出。 |
| **全局變量** | 一次保存便可在任意工作流中複用的值(比如 API 密鑰)。 |

## 在 OneUptime 中哪裏找工作流

在左側導航中打開 **工作流**。在那裏:

- **工作流**——你的工作流列表。新建一個或打開已有的。
- **構建器標籤**——你設計工作流的畫布。
- **日誌標籤**——這個工作流的每一次運行,帶有詳情。
- **設置標籤**——名稱、描述、所有者、標籤、啓用/停用。
- **全局變量**——所有工作流之間共享的值。
- **運行與日誌**——項目中所有工作流的執行歷史。

## 構建你的第一個工作流

1. **創建**——給工作流起一個名稱和簡短描述。
2. **選擇觸發器**——手動、定時、webhook,或來自 OneUptime 的事件。
3. **添加組件**——把動作拖到畫布上並連接起來。
4. **測試**——點擊 **手動運行** 並在日誌中查看結果。
5. **打開它**——準備好後在設置中切換 **已啓用** 開關。

## 一個簡單的例子

假設你想在創建嚴重事件時在 Slack 中發佈消息:

1. 創建一個名爲 "嚴重事件推送到 Slack" 的工作流。
2. 選擇 **事件 → 創建時** 觸發器。
3. 添加一個 **條件** 模塊。設置它檢查事件標題是否包含 "Sev 1"。
4. 從 **是** 分支添加一個 **Slack** 模塊。選擇頻道並寫好消息。
5. 啓用工作流。

下次有人打開標題包含 "Sev 1" 的事件時,Slack 就會亮起來。

## 工作流如何與 OneUptime 的其他部分配合

- **監控** 發現問題。**事件** 記錄它。**工作流** 響應它。
- **Runbook** 是給人用的分步指南。工作流是無人值守的自動化。當需要人來做決定時使用 Runbook;當步驟完全自動時使用工作流。
- **工作區連接**(Slack、Teams)是工作流發送消息的地方。
