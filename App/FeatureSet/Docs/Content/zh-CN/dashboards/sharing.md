# 分享与公开仪表板

默认情况下,仪表板对你的项目是私有的——只有已登录的团队成员才能查看。但 OneUptime 也允许你公开分享仪表板、用密码保护它、限制到特定 IP,并把它托管在你自己的域名上。本页涵盖这四点。

## 私有仪表板(默认)

仪表板只对已登录的项目成员可达。URL 形如 `https://oneuptime.com/dashboards/<id>/view`,需要登录。

在项目内部,所有者和标签控制谁能看到什么——见 [配置与权限](/docs/dashboards/configuration)。

## 公开仪表板

在 **仪表板 → 设置** 下,切换 **公开仪表板** 为开启。仪表板现在有了第二个 URL,无需登录。可以分享给供应商、合作伙伴、客户,或者粘贴到公开的 README 里。

公开仪表板:

- 始终在 **查看** 模式下打开。公开访客不能编辑或看到组件面板。
- 包含你添加的变量。访客从和团队相同的下拉中选择。
- 使用你在设置里配置的 **品牌**——页面标题、描述、Logo、Favicon。

启用公开仪表板就像发布一个网页。它上面的每个组件都会变得全球可读。在切换开关之前看看画布上有什么。

## 主密码

要给公开仪表板加密码:

1. 打开 **公开仪表板**。
2. 打开 **主密码**。
3. 设置密码。

访客在仪表板出现之前会看到密码提示。密码以哈希形式存储——我们永远看不到实际密码。

什么时候用主密码:

- 你想与合作伙伴或客户分享,但又不希望 URL 泄露后被滥用。
- 仪表板是"半公开"的——足够开放到不需要邀请每个访客成为团队成员,但又不至于开放到放上公开互联网。

如果需要更强的控制(每个访客单独账户、谁看过什么的审计),保持仪表板私有,并把访客作为只读团队成员邀请进来。

## IP 白名单

在 **Scale** 套餐上,你可以把公开仪表板限制到一组 IP 地址或网段。在 **仪表板 → 设置 → IP 白名单** 下配置。

什么时候用:

- 仪表板应该只能从你的办公室或 VPN 访问。
- 一个供应商门户应该只能从他们已知的 IP 访问。
- 你想在主密码之上再加一层保护。

来自其他 IP 的请求会被拒绝。

## 自定义域名

开箱即用时,公开仪表板部署在 `oneuptime.com` 上。要把它托管在你自己的子域名比如 `dashboard.acme.com` 上:

1. 在你的 DNS 上添加一条 CNAME 记录,把子域名指向 OneUptime 的目标。
2. 在 **仪表板 → 设置 → 自定义域名** 下,添加这个域名。
3. 验证它。OneUptime 会替你检查 DNS 记录。
4. 验证后,仪表板在你的自定义域名和原始 URL 上都可达。

自定义域名适用于:

- 用你自己的品牌做面向客户的仪表板。
- 联合品牌的合作伙伴仪表板。
- 拥有独立 URL 的公开健康页面。

如果你需要把同样的内容服务给多个受众,可以为单个仪表板挂多个自定义域名。

## 品牌定制

在 **仪表板 → 设置** 下,你可以配置:

- **页面标题**——浏览器标签和页面顶部显示的标题。
- **页面描述**——搜索引擎和社交预览使用的描述。
- **Logo**——上传 PNG 或 SVG 在头部显示。
- **Favicon**——浏览器标签的小图标。

品牌定制只在仪表板被公开查看时应用。内部查看者总是看到 OneUptime 的品牌。

## 嵌入

你可以用 iframe 把公开仪表板嵌入到自己的网站:

```html
<iframe
  src="https://dashboard.acme.com/view"
  width="100%"
  height="800"
  frameborder="0"
></iframe>
```

如果仪表板有主密码,访客会在 iframe 内看到密码提示。

## 可分享的 URL

仪表板 URL 把当前的变量选择和时间范围作为查询参数包含进去。调整下拉、复制 URL、粘贴到聊天里——打开链接的人看到的是完全一样的视图。

这是把同事指向"事件发生那一刻的仪表板"的最快方法。钉住时间范围、复制、粘贴。
