# 配置与权限

本页涵盖当你有了一张想长期保留的仪表板时,值得了解的设置与访问控制。

## 所有者

仪表板的 **所有者** 是被你赋予显式访问权限的用户和团队(在他们项目范围角色之上)。

在 **仪表板 → 所有者** 下:

- 添加一个 **用户所有者** 来给某个人对该仪表板的额外访问权限。
- 添加一个 **团队所有者** 来给某个团队的每位成员同样的权限。

当项目范围的读取角色过于宽泛时,使用所有者——例如,一张涉及客户级细节的仪表板,只应当对客户成功团队可见。

## 标签

标签是用于整理仪表板的标记。在 **仪表板 → 概览** 下应用它们。

常见模式:

- **按团队**:`team:platform`、`team:checkout`、`team:growth`。
- **按环境**:`env:prod`、`env:staging`。
- **按用途**:`purpose:oncall`、`purpose:exec`、`purpose:investigation`。

**仪表板** 列表允许按标签筛选,这是在仪表板较多的项目里找到目标仪表板最快的方式。

## 权限

仪表板与项目的基于角色的访问控制配合。相关权限:

| 权限 | 允许的操作 |
| --- | --- |
| **创建 Dashboard** | 创建新仪表板。 |
| **读取 Dashboard** | 查看仪表板(在私有模式下)。 |
| **编辑 Dashboard** | 修改组件、变量和设置。 |
| **删除 Dashboard** | 删除仪表板。 |

仪表板所有者和自定义域名也有对应的权限,因此你可以授予"管理所有者"而不授予"编辑仪表板"。

在 **项目设置 → 团队与角色** 下为项目角色分配这些权限。

## 公开仪表板的访问

当你把仪表板设为公开(见 [分享与公开仪表板](/docs/dashboards/sharing)),三个设置控制谁能看到它:

1. **公开仪表板** 开关——关闭时,公开 URL 返回 404。
2. **主密码**——若设置,访客先输入密码才能看到仪表板。
3. **IP 白名单**(Scale 套餐)——若设置,来自其他 IP 的请求会被拒绝。

你可以任意组合这些。最严格的组合是"公开打开、密码设置、IP 白名单启用"——适合需要三层防护的合作伙伴门户。

## 数据保留

仪表板本身不会过期。它们显示的数据遵循你项目的保留设置——指标、日志和追踪只要你的套餐保留它们就可查询。一个指向"过去 90 天"的组件,在只保留 30 天的套餐上,会显示当前还存的内容。

## 复制仪表板

要复制现有的仪表板,打开仪表板列表并选择 **复制**。副本会包含每个组件、变量和设置,但不包括公开分享——分享始终从关闭开始,这样你可以决定是否再打开它。

当你想把一个模板(比如"我们的值班仪表板")分叉成针对某个服务的副本时,这是正确的做法。

## 删除仪表板

在 **仪表板 → 删除** 下。这无法撤销——仪表板的布局以及挂在它上面的任何自定义域名都会被移除。你的遥测数据不受影响。

如果仪表板是公开的并且使用了自定义域名,删除后 URL 立即停止解析。如果想让 URL 继续工作,请先把域名移动到另一个仪表板。

## 备份

如果你自托管 OneUptime,定期的数据库备份就够了——仪表板配置和项目的其他内容一起存储。

在 OneUptime Cloud 上,备份由我们处理。如果你想要自己的副本,可以通过 [OneUptime API](/docs/api-reference/api-reference) 读取仪表板。
