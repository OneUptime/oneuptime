# 创作工作流

要创建工作流,打开 **工作流 → 创建工作流**,给它起一个名称,然后点击进入 **构建器** 标签。你会看到一张空白画布,接下来就在上面构建自动化。

## 画布

构建器是一张拖拽式画布。你从侧边的面板里添加模块,用线把它们连接起来,然后点击每个模块来配置它要做的事。改动会自动保存——保存后顶部会显示一个指示。

每个工作流的开头都是一个 **触发器**。其余的都是 **组件**——具体执行操作的部分。

## 模块上有什么

| 字段     | 作用                                                                                                               |
| -------- | ------------------------------------------------------------------------------------------------------------------ |
| **标题** | 在画布上显示的名称。重命名它能让复杂工作流更易读。                                                                 |
| **设置** | 模块完成工作所需的内容——URL、Slack 频道、消息正文等。必填字段会带星号标记。                                        |
| **输入** | 模块左侧的小圆点,来自前面模块的连线在这里汇入。                                                                    |
| **输出** | 模块右侧的小圆点,从这里连线指向下一个模块。许多模块有独立的 **成功** 和 **错误** 输出,这样你可以分别处理两种情况。 |

## 连接模块

从一个模块的输出点拖到下一个模块的输入点。你画的这条线决定了接下来执行什么。

- 如果从 **成功** 输出连出,只有前一个模块执行成功时,下一个模块才会运行。
- 如果从 **错误** 输出连出,只有前一个模块执行失败时,下一个模块才会运行。
- 如果某个输出没有连接,那条路径就停在那里。

你可以把一个输出连接到多个模块。它们会从那一点开始同时运行。

## 配置模块

点击模块,侧边会打开它的设置。每个设置都有合适的输入方式——文本框、下拉菜单、代码编辑器、开关等等。

大多数文本字段都接受变量——这就是数据在模块之间流动的方式。具体语法见 [变量](/docs/workflows/variables)。

## 你的第一个工作流

熟悉画布最快的方式:

1. 拖一个 **手动** 触发器到画布上。
2. 在它旁边拖一个 **日志** 组件(在 **工具** 下)。把触发器连接到日志组件。
3. 在日志模块的消息字段里,输入 `Hello from {{Manual.JSON.name}}`。
4. 保存并启用工作流。
5. 点击 **手动运行**,粘贴 `{ "name": "Ada" }` 作为输入,然后提交。
6. 打开 **日志** 标签。最近一次运行会显示 `Hello from Ada`。

这个循环——拖入、连接、配置、运行、查看日志——就是你构建每个工作流的方式。

## 保存与启用

画布会在你工作时自动保存。没有单独的"发布"步骤。

但工作流只有在 **设置** 中的 **已启用** 开关打开时才真正运行。新工作流默认是停用的。把这个开关当作你的安全网——先构建、用 **手动运行** 测试、检查日志,然后再打开。

要暂停工作流而不删除它,把 **已启用** 关掉。正在进行中的运行会跑完;但不会开始新的运行。

## 整理画布

- 拖动模块来移动它们。布局会被保存,下次别人看到的还是同样的排列。
- 右键单击连线删除它。右键单击模块可以删除或复制它。
- 对于较宽的工作流,从左到右排列,这样阅读方向就和执行方向一致。
