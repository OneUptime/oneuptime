# 传入邮件监控器

传入邮件监控器允许您根据发送到唯一监控器特定邮件地址的邮件来创建和解决告警。这对于与旧系统、第三方告警工具或任何可以发送邮件通知的服务集成非常有用。

## 工作原理

1. 当您创建传入邮件监控器时，OneUptime 会为该监控器生成唯一的邮件地址
2. 发送到该地址的任何邮件都会被接收，并根据您配置的标准进行评估
3. 根据标准，OneUptime 可以创建新告警或解决现有告警

这是将基于邮件的告警系统与 OneUptime 事件管理工作流集成的强大方式。

## 创建传入邮件监控器

1. 在 OneUptime 控制台中导航至 **监控器**
2. 点击 **创建监控器**
3. 选择 **传入邮件** 作为监控器类型
4. 配置监控器设置：
   - **名称：** 监控器的描述性名称
   - **描述：** 此监控器的用途
5. 设置您的 **告警创建标准**（创建告警的条件）
6. 设置您的 **告警解决标准**（解决告警的条件）
7. 点击 **创建**

创建后，您将在监控器详情页面上看到此监控器的唯一邮件地址。

## 邮件地址格式

每个传入邮件监控器都会获得一个格式如下的唯一邮件地址：

```
monitor-{secret-key}@{inbound-domain}
```

例如：`monitor-abc123def456@inbound.yourdomain.com`

您可以从监控器详情页面复制此地址，并配置您的外部系统向其发送邮件。

## 可用标准字段

您可以根据以下邮件字段创建标准：

| 字段           | 描述                       |
| -------------- | -------------------------- |
| **邮件主题**   | 传入邮件的主题行           |
| **邮件发件人** | 发件人的邮件地址           |
| **邮件正文**   | 邮件正文的纯文本内容       |
| **邮件收件人** | 收件人邮件地址             |
| **邮件接收**   | 基于邮件接收时间的时间条件 |

## 可用过滤类型

### 字符串过滤器（主题、发件人、正文、收件人）

| 过滤器        | 描述                   | 示例                           |
| ------------- | ---------------------- | ------------------------------ |
| **包含**      | 字段包含指定文本       | 主题包含"CRITICAL"             |
| **不包含**    | 字段不包含指定文本     | 主题不包含"TEST"               |
| **等于**      | 字段与指定文本完全匹配 | 发件人等于"alerts@service.com" |
| **不等于**    | 字段与指定文本不匹配   | 主题不等于"OK"                 |
| **以...开头** | 字段以指定文本开头     | 主题以"[ALERT]"开头            |
| **以...结尾** | 字段以指定文本结尾     | 主题以"- Production"结尾       |
| **为空**      | 字段为空或空白         | 正文为空                       |
| **不为空**    | 字段有内容             | 主题不为空                     |

### 时间过滤器（邮件接收）

| 过滤器              | 描述                  | 示例                 |
| ------------------- | --------------------- | -------------------- |
| **在 X 分钟内收到** | 邮件在 X 分钟内被接收 | 邮件在 30 分钟内收到 |
| **X 分钟内未收到**  | X 分钟内未收到邮件    | 邮件 60 分钟内未收到 |

## 配置示例

### 示例 1：对关键邮件创建告警

**告警创建标准：**

- 邮件主题 **包含** "CRITICAL"
- 或邮件主题 **包含** "ALERT"
- 或邮件主题 **包含** "ERROR"

**告警解决标准：**

- 邮件主题 **包含** "RESOLVED"
- 或邮件主题 **包含** "OK"
- 或邮件主题 **包含** "RECOVERED"

### 示例 2：监控特定发件人

**告警创建标准：**

- 邮件发件人 **等于** "monitoring@legacy-system.com"
- 且邮件主题 **包含** "Failed"

**告警解决标准：**

- 邮件发件人 **等于** "monitoring@legacy-system.com"
- 且邮件主题 **包含** "Success"

### 示例 3：心跳监控（无邮件 = 告警）

**告警创建标准：**

- 邮件接收 **60 分钟内未收到**

如果 60 分钟内未收到邮件，则创建告警——适用于监控应发送完成邮件的计划任务或批处理。

**告警解决标准：**

- 邮件接收 **在 5 分钟内收到**

收到邮件时解决告警。

## 使用场景

### 旧系统集成

许多旧系统只支持基于邮件的告警。使用传入邮件监控器可以：

- 将邮件告警转换为 OneUptime 事件
- 当收到恢复邮件时自动解决事件
- 集中管理来自多个旧系统的告警

### 第三方服务监控

与发送邮件通知的服务集成：

- 云提供商告警（AWS、GCP、Azure）
- 安全扫描工具
- 备份完成通知
- SSL 证书过期警告

### 计划任务监控

监控批处理和计划任务：

- 如果未及时收到完成邮件，创建告警
- 通过错误通知邮件跟踪任务失败
- 监控数据管道完成情况

### 多供应商告警聚合

整合来自多个监控工具的告警：

- 通过邮件接收来自 Nagios、Zabbix 或其他工具的告警
- 在 OneUptime 中统一事件管理
- 为所有告警维护单一的事实来源

## 模板变量

配置事件模板时，您可以使用这些来自传入邮件的变量：

| 变量                  | 描述             |
| --------------------- | ---------------- |
| `{{emailSubject}}`    | 接收到的邮件主题 |
| `{{emailFrom}}`       | 发件人的邮件地址 |
| `{{emailTo}}`         | 收件人邮件地址   |
| `{{emailBody}}`       | 邮件的纯文本正文 |
| `{{emailReceivedAt}}` | 邮件收到的时间   |

## 监控器摘要视图

监控器摘要显示：

- **最后收到邮件时间：** 最近一封邮件的接收时间
- **发件人：** 最后一封邮件的发件人
- **主题：** 最后一封邮件的主题行
- **邮件头：** 最后一封邮件的完整头信息（可展开）
- **邮件正文：** 最后一封邮件的内容（可展开）

如果您是自托管 OneUptime，需要配置入站邮件提供商。目前支持：

- **SendGrid Inbound Parse** — 设置说明请参见 [SendGrid 入站邮件集成](/docs/self-hosted/sendgrid-inbound-email)

## 注意事项

- **邮件地址安全性：** 监控器邮件地址包含密钥。请将其视为密码，不要公开分享。
- **邮件大小：** 非常大的邮件（带有大附件）可能会被邮件提供商截断或拒绝。
- **处理时间：** 邮件是异步处理的。发送邮件和创建告警之间可能有几秒延迟。
- **大小写不敏感：** 所有字符串比较（包含、等于等）均不区分大小写。
- **纯文本：** 邮件正文标准使用邮件的纯文本版本。HTML 格式会被去除。

## 故障排查

### 邮件未被接收

1. 验证邮件地址是否正确（检查是否有拼写错误）
2. 检查邮件是否被垃圾邮件过滤器拦截
3. 验证您的入站邮件提供商是否正确配置
4. 检查 OneUptime 日志中的任何错误消息

### 未创建告警

1. 验证您的标准是否与邮件内容匹配
2. 检查监控器是否未被禁用
3. 查看监控器详情中的评估日志
4. 在使用模式匹配之前，先用精确字符串匹配进行测试

### 未解决告警

1. 验证您的解决标准是否与恢复邮件匹配
2. 确保有活跃告警需要解决
3. 检查解决邮件是否发送到同一监控器地址
