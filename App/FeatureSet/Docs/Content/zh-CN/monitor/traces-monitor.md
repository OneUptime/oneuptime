# 追踪监控器

追踪监控允许您监控来自应用程序的分布式追踪，并根据 Span 模式、数量和状态触发告警。OneUptime 在时间窗口内评估来自遥测服务的追踪数据。

## 概述

追踪监控器搜索并统计符合特定过滤器的 Span。这使您能够：

- 对服务中的错误 Span 峰值发出告警
- 监控特定操作和端点
- 跟踪 Span 量和模式
- 按 Span 状态、名称和自定义属性过滤
- 从追踪数据中检测性能和可靠性问题

## 创建追踪监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **追踪** 作为监控器类型
4. 选择要监控的遥测服务
5. 根据需要配置 Span 过滤器和标准

## 配置选项

### 遥测服务

选择一个或多个服务来监控其追踪。服务必须通过 OpenTelemetry 向 OneUptime 发送追踪数据。

### Span 过滤器

| 过滤器    | 描述                                           | 是否必填 |
| --------- | ---------------------------------------------- | -------- |
| Span 状态 | 按 Span 状态码过滤（OK、ERROR、UNSET）         | 否       |
| Span 名称 | 搜索特定 Span 名称（例如操作或端点名称）的文本 | 否       |
| 属性      | 用于过滤自定义 Span 属性的键值对               | 否       |
| 时间窗口  | 搜索 Span 的时间范围（秒，默认：60）           | 否       |

### Span 状态码

- **OK** — 操作成功完成
- **ERROR** — 操作遇到错误
- **UNSET** — 状态未明确设置

## 监控标准

### 可用检查类型

| 检查类型  | 描述                               |
| --------- | ---------------------------------- |
| Span 数量 | 时间窗口内符合过滤条件的 Span 数量 |

### 过滤类型

- **大于** — Span 数量超过阈值
- **小于** — Span 数量低于阈值
- **大于或等于** — Span 数量等于或超过阈值
- **小于或等于** — Span 数量等于或低于阈值
- **等于** — Span 数量完全匹配
- **不等于** — Span 数量不匹配

### 示例标准

#### 当 60 秒内超过 50 个错误 Span 时发出告警

- **Span 状态**：ERROR
- **时间窗口**：60 秒
- **检查项**：Span 数量
- **过滤类型**：大于
- **值**：50

#### 对特定端点的错误发出告警

- **Span 名称**：`POST /api/checkout`
- **Span 状态**：ERROR
- **时间窗口**：120 秒
- **检查项**：Span 数量
- **过滤类型**：大于
- **值**：0

## 设置要求

追踪监控需要您的应用程序通过 OpenTelemetry 向 OneUptime 发送分布式追踪数据。设置说明请参见 [OpenTelemetry](/docs/telemetry/open-telemetry) 文档。
