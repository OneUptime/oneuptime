# 手动监控器

手动监控允许您创建状态完全由手动或通过 API 管理的监控器。OneUptime 不执行任何自动检查——您直接控制监控器状态。

## 概述

手动监控器是您自行更新的占位符。这在以下情况非常有用：

- 与通过 OneUptime API 更新状态的外部监控工具集成
- 跟踪无法自动监控的服务或系统
- 管理没有自动健康检查的组件的事件
- 表示您手动跟踪状态的第三方依赖项

## 创建手动监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **手动** 作为监控器类型
4. 输入监控器的名称和描述

## 工作原理

手动监控器没有监控间隔、探针或自动标准评估。监控器状态保持您设置的状态，直到您更改它。

### 更新状态

您可以通过两种方式更新手动监控器的状态：

- **控制台** — 直接从 OneUptime 控制台更改监控器状态
- **API** — 使用 OneUptime API 以程序化方式更新监控器状态

### 事件和告警

您可以像对任何其他监控器类型一样，针对手动监控器创建事件和告警。这允许您：

- 跟踪外部监控服务的停机时间
- 在报告问题时手动创建事件
- 在状态页面上使用手动监控器向用户传达状态

## 何时使用手动监控器

| 使用场景       | 描述                                        |
| -------------- | ------------------------------------------- |
| 第三方服务     | 跟踪您依赖但无法直接监控的外部服务状态      |
| 物理基础设施   | 表示没有网络监控的硬件或物理系统            |
| 业务流程       | 跟踪影响服务状态的非技术流程                |
| API 驱动的状态 | 让外部工具通过 OneUptime API 更新监控器状态 |
| 状态页面占位符 | 在状态页面上显示 OneUptime 外部管理的组件   |
