# 指标监控器

指标监控允许您监控通过 OpenTelemetry 收集的自定义应用程序和基础设施指标。OneUptime 在时间窗口内评估指标值，并根据您配置的标准触发告警。

## 概述

指标监控器查询并评估来自遥测服务的数值指标。这使您能够：

- 监控自定义应用程序指标（请求率、队列深度、错误率等）
- 跟踪基础设施指标（CPU、内存、磁盘、网络）
- 使用过滤器和聚合创建复杂的指标查询
- 使用数学公式组合多个指标
- 基于指标阈值设置告警

## 创建指标监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **指标** 作为监控器类型
4. 配置指标查询和可选公式
5. 选择聚合策略
6. 根据需要配置监控标准

## 配置选项

### 指标查询

定义一个或多个指标查询。每个查询包含：

| 字段     | 描述                                            | 是否必填 |
| -------- | ----------------------------------------------- | -------- |
| 指标名称 | 要查询的指标名称                                | 是       |
| 聚合类型 | 如何聚合原始指标值（sum、avg、min、max、count） | 是       |
| 属性     | 用于缩小指标数据范围的键值过滤器                | 否       |
| 分组依据 | 用于分组指标的维度                              | 否       |

每个查询都会分配一个别名（例如 `a`、`b`、`c`），用于公式中。

### 公式

使用数学表达式组合多个指标查询。例如：

- `a / b * 100` — 从两个查询计算百分比
- `a + b` — 对两个指标求和
- `a - b` — 两个指标之间的差值

### 滚动时间窗口

选择指标评估的时间窗口：

- 过去 1 分钟
- 过去 5 分钟
- 过去 10 分钟
- 过去 15 分钟
- 过去 30 分钟
- 过去 60 分钟

### 聚合策略

选择如何聚合指标值进行评估：

| 策略   | 描述                   |
| ------ | ---------------------- |
| 平均值 | 时间窗口内的平均值     |
| 求和   | 所有值的总和           |
| 最大值 | 时间窗口内的最高值     |
| 最小值 | 时间窗口内的最低值     |
| 所有值 | 所有值必须满足标准     |
| 任意值 | 至少一个值必须满足标准 |

## 监控标准

### 可用检查类型

| 检查类型 | 描述                         |
| -------- | ---------------------------- |
| 指标值   | 配置的指标查询或公式的聚合值 |

### 过滤类型

- **大于** — 指标值超过阈值
- **小于** — 指标值低于阈值
- **大于或等于** — 指标值等于或超过阈值
- **小于或等于** — 指标值等于或低于阈值
- **等于** — 指标值完全匹配
- **不等于** — 指标值不匹配

### 示例标准

#### 当错误率超过 5% 时发出告警

- **查询 a**：`http_requests_total` 按 `status=5xx` 过滤
- **查询 b**：`http_requests_total`
- **公式**：`a / b * 100`
- **检查项**：指标值
- **过滤类型**：大于
- **值**：5

#### 当请求队列深度过高时发出告警

- **查询**：`request_queue_size`，聚合：最大值
- **检查项**：指标值
- **过滤类型**：大于
- **值**：1000

## 设置要求

指标监控需要您的应用程序或基础设施通过 OpenTelemetry 向 OneUptime 发送指标。设置说明请参见 [OpenTelemetry](/docs/telemetry/open-telemetry) 文档。
