# Ping 监控器

Ping 监控允许您监控任何主机或 IP 地址的可用性和响应能力。OneUptime 定期向目标发送 Ping 请求，并检查其是否正确响应。

## 概述

Ping 监控器通过向主机发送 ICMP Ping 请求来测试基本网络连接性。这使您能够：

- 监控主机正常运行时间和可用性
- 跟踪网络延迟和响应时间
- 在连接问题影响您的服务之前检测到它们
- 验证服务器和网络设备是否可达

## 创建 Ping 监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **Ping** 作为监控器类型
4. 输入您要监控的主机名或 IP 地址
5. 根据需要配置监控标准

## 配置选项

### Ping 主机名或 IP 地址

输入您要监控的目标主机名或 IP 地址（例如 `example.com` 或 `192.168.1.1`）。主机名和 IP 地址均可接受。

## 监控标准

您可以配置标准来判断主机何时处于在线、降级或离线状态，基于以下条件：

### 可用检查类型

| 检查类型         | 描述                        |
| ---------------- | --------------------------- |
| 是否在线         | 主机是否响应 Ping 请求      |
| 响应时间（毫秒） | Ping 请求的往返时间（毫秒） |
| 请求是否超时     | Ping 请求是否超时           |

### 过滤类型

对于 **是否在线** 和 **请求是否超时**：

- **True** — 条件为真
- **False** — 条件为假

对于 **响应时间**：

- **大于** — 响应时间超过阈值
- **小于** — 响应时间低于阈值
- **大于或等于** — 响应时间等于或超过阈值
- **小于或等于** — 响应时间等于或低于阈值
- **等于** — 响应时间完全匹配
- **不等于** — 响应时间不匹配
- **随时间评估** — 使用聚合（平均值、求和、最大值、最小值、所有值、任意值）在时间窗口内评估

### 示例标准

#### 如果主机不可达则标记为离线

- **检查项**：是否在线
- **过滤类型**：False

#### 如果响应时间超过 200ms 则发出告警

- **检查项**：响应时间（毫秒）
- **过滤类型**：大于
- **值**：200
