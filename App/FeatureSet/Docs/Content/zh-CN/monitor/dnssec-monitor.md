# DNSSEC 监控器

DNSSEC 监控允许您验证区域 DNS 响应的加密完整性。OneUptime 定期执行完整的 DNSSEC 验证 — 检查 DNSKEY 记录、父区域的 DS 委派、RRSIG 签名有效性、解析器对 AD flag 的共识，以及权威名称服务器之间的一致性。

## 概述

DNSSEC 监控器验证从根区域到您的域名的整个信任链。这使您能够：

- 在解析器开始向用户返回 SERVFAIL 之前检测损坏的 DNSSEC 链
- 在区域签名密钥过期之前收到警告
- 验证您的 DS 记录已在父区域正确发布
- 捕获权威名称服务器之间的差异（主/从不同步）
- 确认验证解析器确实为您的区域设置了 AD flag

## 创建 DNSSEC 监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **DNSSEC** 作为监控器类型
4. 输入要验证的区域（域名）
5. 根据需要配置解析器和监控标准

## 配置选项

### 基本设置

| 字段                 | 描述                                                                 | 是否必填 |
| -------------------- | -------------------------------------------------------------------- | -------- |
| 区域（域名）         | 通过 DNSSEC 验证的区域（例如 `example.com`）                         | 是       |
| 解析器               | 要查询的验证解析器的逗号分隔列表（例如 `1.1.1.1, 8.8.8.8, 9.9.9.9`） | 是       |
| 检查名称服务器一致性 | 直接查询每个权威名称服务器，并验证它们返回相同的 SOA serial          | 否       |

### 高级设置

| 字段               | 描述                       | 默认值 |
| ------------------ | -------------------------- | ------ |
| 签名过期警告（天） | RRSIG 过期过滤器的默认阈值 | 7      |
| 超时（毫秒）       | 等待每个 DNS 查询的时长    | 10000  |
| 重试次数           | 失败时的重试次数           | 3      |

## 监控标准

您可以配置标准来判断区域何时处于在线、降级或离线状态，基于以下条件：

### 可用检查类型

| 检查类型                      | 描述                                                |
| ----------------------------- | --------------------------------------------------- |
| DNSSEC 链是否有效             | 整个验证链（根 → TLD → 区域）正确解析               |
| DNSSEC DNSKEY 记录是否存在    | 区域至少发布了一条 DNSKEY 记录                      |
| 父区域是否存在 DNSSEC DS 记录 | 父区域发布的 DS 记录与该区域的 KSK 匹配             |
| DNSSEC 签名剩余有效天数       | 距离最早过期的 RRSIG 签名的天数                     |
| DNSSEC 解析器共识（AD Flag）  | 每个查询的解析器都返回 AD（Authenticated Data）flag |
| DNSSEC 名称服务器是否一致     | 所有权威名称服务器返回相同的 SOA serial             |
| DNSSEC 是否有效               | 所有验证检查的综合通过/失败                         |

### 过滤类型

对于 **DNSSEC 链是否有效**、**DNSSEC DNSKEY 记录是否存在**、**父区域是否存在 DNSSEC DS 记录**、**DNSSEC 解析器共识（AD Flag）**、**DNSSEC 名称服务器是否一致** 和 **DNSSEC 是否有效**：

- **True** — 条件为真
- **False** — 条件为假

对于 **DNSSEC 签名剩余有效天数**：

- **大于**、**小于**、**大于或等于**、**小于或等于**、**等于**、**不等于**

### 示例标准

#### 当 DNSSEC 链损坏时告警

- **检查项**：DNSSEC 链是否有效
- **过滤类型**：False

#### 在签名过期前发出警告

- **检查项**：DNSSEC 签名剩余有效天数
- **过滤类型**：小于
- **值**：7

#### 捕获父区域缺失的 DS（委派损坏）

- **检查项**：父区域是否存在 DNSSEC DS 记录
- **过滤类型**：False

#### 检测解析器之间的分歧

- **检查项**：DNSSEC 解析器共识（AD Flag）
- **过滤类型**：False

#### 捕获名称服务器数据不一致

- **检查项**：DNSSEC 名称服务器是否一致
- **过滤类型**：False

## 最佳实践

1. **使用多个公共解析器** — 默认使用 `1.1.1.1`、`8.8.8.8` 和 `9.9.9.9`，避免单个解析器故障导致误报
2. **在过期之前充分提前告警** — 在签名过期前 7 天配置降级告警，在 2 天前配置离线告警；密钥滚动可能会静默失败
3. **监控每个已签名的区域** — 包括 apex 域名、已签名的子域名以及委派给不同运营商的任何区域
4. **启用名称服务器一致性检查** — 除非您的网络阻止对任意 IP 的出站 DNS，否则可捕获 DNSSEC 验证本身无法发现的主/从同步问题
