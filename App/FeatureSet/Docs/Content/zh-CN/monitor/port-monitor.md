# 端口监控器

端口监控允许您监控主机上特定 TCP 或 UDP 端口的可用性。OneUptime 定期尝试连接到指定端口，并检查其是否开放且响应正常。

## 概述

端口监控器测试特定网络端口是否接受连接。这使您能够：

- 监控特定端口上的服务可用性
- 跟踪端口响应时间
- 验证数据库、邮件服务器和应用服务器等服务是否正在运行
- 在服务中断影响用户之前检测到它们

## 创建端口监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **端口** 作为监控器类型
4. 输入主机名或 IP 地址和端口号
5. 根据需要配置监控标准

## 配置选项

### 主机名或 IP 地址

输入目标主机的主机名或 IP 地址（例如 `example.com` 或 `192.168.1.1`）。

### 端口

输入要监控的端口号（1–65535）。常见示例：

| 端口  | 服务       |
| ----- | ---------- |
| 22    | SSH        |
| 25    | SMTP       |
| 80    | HTTP       |
| 443   | HTTPS      |
| 3306  | MySQL      |
| 5432  | PostgreSQL |
| 6379  | Redis      |
| 27017 | MongoDB    |

## 监控标准

您可以配置标准来判断端口何时处于在线、降级或离线状态，基于以下条件：

### 可用检查类型

| 检查类型         | 描述                   |
| ---------------- | ---------------------- |
| 是否在线         | 端口是否开放且接受连接 |
| 响应时间（毫秒） | 建立连接的时间（毫秒） |
| 请求是否超时     | 连接尝试是否超时       |

### 过滤类型

对于 **是否在线** 和 **请求是否超时**：

- **True** — 条件为真
- **False** — 条件为假

对于 **响应时间**：

- **大于** — 响应时间超过阈值
- **小于** — 响应时间低于阈值
- **大于或等于** — 响应时间等于或超过阈值
- **小于或等于** — 响应时间等于或低于阈值
- **等于** — 响应时间完全匹配
- **不等于** — 响应时间不匹配
- **随时间评估** — 使用聚合（平均值、求和、最大值、最小值、所有值、任意值）在时间窗口内评估

### 示例标准

#### 如果端口关闭则标记为离线

- **检查项**：是否在线
- **过滤类型**：False

#### 如果连接时间超过 500ms 则发出告警

- **检查项**：响应时间（毫秒）
- **过滤类型**：大于
- **值**：500

#### 如果连接缓慢则标记为降级

- **检查项**：响应时间（毫秒）
- **过滤类型**：大于
- **值**：200
