# ダッシュボード 概要

ダッシュボードは、OneUptime がすでに収集しているデータ — メトリクス、ログ、トレース、インシデント、モニター、Kubernetes リソース、ホスト — を、誰かが一目見て状況を把握できる 1 枚のページにまとめます。

リクエストレイテンシーのチャートを、開いているインシデントのリストの隣に、CPU のゲージの隣に、文脈を説明する段落の隣に並べます。保存します。リンクを共有します。

## ダッシュボードの用途

- **「すべて順調か?」ページ** — オンコール、チームのスタンドアップ、または壁掛けテレビ用。
- **関係性に気づく** — CPU のスパイク、レイテンシーの増加、開いているインシデントが同時に起きていることは、3 つのタブを行き来するより 1 ページで見たほうが格段にわかりやすくなります。
- **調査** — デバッグ中、その場で組み立てたダッシュボードは、10 個のクエリを 1 つずつ実行するより効果的です。
- **外部への共有** — 顧客向けのパフォーマンスページ、パートナー向けのステータスページ、オープンソースプロジェクト用の公開ダッシュボード。

## ダッシュボードに置けるもの

- **チャート** — 時系列のトレンド (レイテンシー、エラー、スループット)。
- **シングル値のタイルとゲージ** — 現在のエラー率、CPU、開いているインシデント数。
- **テーブル** — 内訳 (最も騒がしいホスト Top 10、サービスごとのエラー数)。
- **テキストブロック** — 見出し、文脈、Runbook へのリンク。
- **ライブリスト** — インシデント、アラート、モニター、ログ、トレース、Kubernetes リソース、Docker リソース、ホスト。

完全なリストと各ウィジェットの表示内容については [ウィジェット](/docs/dashboards/widgets) を参照してください。

## キー概念

| 用語 | 意味 |
| --- | --- |
| **ダッシュボード** | ページ全体 — 名前、ウィジェットのグリッド、時間範囲のコントロール、変数のリスト。 |
| **ウィジェット** | ページ上の 1 つのタイル — チャート、数値、リスト、段落。 |
| **変数** | 上部のドロップダウンで、すべてのウィジェットを一度にフィルターします (クラスター、サービス、顧客、環境)。 |
| **時間範囲** | すべてのチャートと数値が使う時間ウィンドウ。ページ上部で一度設定します。 |
| **更新間隔** | ウィジェットがデータを再クエリする頻度。オフ、数秒ごと、数分ごとから選択。 |
| **モード** | **Edit** (ウィジェットをドラッグして配置) または **View** (読み取り専用で訪問者と同じ見え方)。 |

## ダッシュボードの場所

左ナビゲーションの **Dashboards** を開きます。

| ページ | できること |
| --- | --- |
| **Dashboards** | ダッシュボードの一覧。新規作成、検索、ラベルでのフィルター。 |
| **Dashboard → View** | キャンバス。ヘッダーで **Edit** と **View** を切り替えます。 |
| **Dashboard → Overview** | 説明、オーナー、ラベル。 |
| **Dashboard → Settings** | 公開共有、パスワード、IP 許可リスト、カスタムドメイン、ブランディング。 |
| **Dashboard → Owners** | 明示的にアクセス権を持つユーザーとチーム。 |
| **Dashboard → Delete** | ダッシュボードを削除。 |

## ダッシュボードの構築

1. **作成** — 名前を選びます。キャンバスは空の状態で開きます。
2. **ウィジェットを追加** — ウィジェットの種類を選び、データを設定して、置きたい場所にドラッグします。
3. **(任意) 変数を追加** — 例えば `service` ドロップダウンを追加すれば、同じダッシュボードがあらゆるサービスで使えるようになります。
4. **時間範囲を設定** — デフォルトのままで構いません。後で調整します。
5. **(任意) 公開で共有** — Settings でスイッチを切り替え、必要に応じてパスワードや IP 許可リストを追加します。
6. **(任意) カスタムドメイン** — `status.your-domain.com` でダッシュボードをホスト。

## 簡単な例

目標: チェックアウトサービスのオンコールページに、レイテンシー、エラー率、開いているインシデント、ライブログのテイルを表示。

1. 「Checkout on-call」という名前のダッシュボードを作成。
2. `service` 変数を追加し、デフォルトを `checkout` に。
3. `service` 変数でフィルターした P95 レイテンシーの **Chart** ウィジェットを追加。
4. その隣に、警告 1%、クリティカル 5% のしきい値を持つエラー率の **Value** ウィジェットを追加。
5. 下に、`checkout` タグの付いたインシデントの **Incident List** ウィジェットを追加。
6. その下に、同じサービスのログを表示する **Log Stream** ウィジェットを追加。
7. 保存。ドロップダウンを `payments` に切り替えると、同じダッシュボードが payments サービスを表示します。

## ダッシュボードと OneUptime の他の機能との関係

- **モニターとテレメトリー** はデータのソースです。収集しているすべてのメトリクス、ログ、トレースは、ウィジェットでクエリできます。
- **インシデントとアラート** は **Incident List** と **Alert List** ウィジェットに表示されます。ダッシュボードはこれらに対しては読み取り専用で、作成と更新は他の場所で行います。
- **ステータスページ** は顧客向けのコミュニケーション (「システムは稼働しているか?」) です。ダッシュボードはシステムの詳細な挙動を見るためのものです。両者は補完的で、置き換え可能ではありません。
- **ワークフロー** は OneUptime がアクションを取る方法です。ダッシュボードは何が起きているかを読み取る方法です。
