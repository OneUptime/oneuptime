# ダッシュボードの作成

ダッシュボードを作成するには、**Dashboards → Create Dashboard** を開き、名前をつけて開きます。キャンバスは **Edit** モードで開き、ウィジェットを追加できる状態になっています。

## キャンバス

ダッシュボードはグリッドです。ウィジェットは決まった位置にスナップします — 各ウィジェットを置く場所とサイズを決めます。行を追加してページを下に伸ばせます。すべてのウィジェットは、大きい画面でも小さい画面でも比率を保ちます。

## Edit と View

ヘッダーのトグルで 2 つのモードを切り替えます:

- **Edit** — ウィジェットパレットが開き、ウィジェットをドラッグして移動・リサイズしたり、ウィジェットをクリックして設定を変更したりできます。
- **View** — ダッシュボードは読み取り専用になり、訪問者やチームメンバーが見るのと同じ見え方になります。共有前に結果を確認するために使います。

両モードとも同じダッシュボードです。別途「公開」ステップはなく、編集はすべて保存された瞬間に反映されます。

## ウィジェットの追加

1. **+** ボタンをクリックしてウィジェットパレットを開きます。
2. ウィジェットの種類を選びます。カタログは [ウィジェット](/docs/dashboards/widgets) を参照してください。
3. ウィジェットがキャンバスに表示されます。
4. ウィジェットの歯車アイコンをクリックして設定を開きます。
5. データソース (メトリクス、リストフィルター、テキストの段落など) と表示オプションを選びます。
6. ウィジェットをドラッグして移動。コーナーをドラッグしてリサイズ。

## データの出どころ

ほとんどのウィジェットは次の 3 つのいずれかからデータを読み取ります:

- **メトリクス** — メトリクスと集計方法 (平均、最大、カウント、パーセンタイル) を選びます。フィルターを追加し、結果のグループ化を選びます。OneUptime の他の場所で使うのと同じクエリビルダーです。
- **ライブリスト** — インシデント、アラート、モニター、Kubernetes Pod、Docker コンテナ、ホスト。各リストウィジェットはフィルターを受け取り、一致するアイテムをライブで表示します。
- **静的コンテンツ** — **Text** ウィジェットは Markdown のブロックを受け取ります。見出し、文脈、Runbook へのリンク、インシデント中の一時的なメモなどに使います。

## しきい値と書式

シングル値ウィジェット (**Value**、**Gauge**) では次を設定できます:

- **警告しきい値** — 値が超えると黄色になります。
- **クリティカルしきい値** — 値が超えると赤になります。

チャートでは Y 軸の単位、凡例の位置、系列をスタックするかオーバーレイするかを選べます。テーブルでは表示する列と行数を選べます。

## 時間範囲と更新

ダッシュボードの上部にある 2 つのコントロールが、すべてのメトリクスウィジェットに影響します:

- **時間範囲** — プリセット (過去 1 時間、24 時間、7 日、30 日) またはカスタム範囲。すべてのチャートと数値がこのウィンドウを使います。
- **更新間隔** — ウィジェットが再クエリする頻度。オフ、5 秒、10 秒、30 秒、1 分、5 分、15 分。ライブリストはこの設定に関係なく独自に更新されます。

時間範囲を使わないウィジェット (Text ウィジェットなど) は両方のコントロールを無視します。

## 保存

キャンバスは作業中に自動保存されます。ヘッダーの小さなインジケーターが、最新の変更が保存されたかどうかを示します。大きな変更を加える場合は、安全なコピーが残るように先にダッシュボードを複製しておくとよいでしょう。

## 長く使えるダッシュボードのコツ

- **1 つのダッシュボードに 1 つのトピック。** 「監視している全部」を 1 ページに詰め込まないでください。フォーカスを絞った数枚のダッシュボードのほうが、巨大な 1 ページより優れています。
- **最も重要なウィジェットを上に。** 人は上から下へ目を走らせます — 「このシステムは健全か?」の答えを最初に見えるようにしてください。
- **Text ウィジェットでセクションにラベルをつける。** 数行ごとに短い見出し (「Latency」「Errors」「Capacity」) を入れると、離れた場所からでもページが読みやすくなります。
- **複製ではなく変数を使う。** 同じダッシュボードを 2 つ目のサービス向けに作りそうになったら、代わりに `service` 変数を持つ 1 つのダッシュボードを作りましょう。[変数とフィルター](/docs/dashboards/variables) を参照してください。
