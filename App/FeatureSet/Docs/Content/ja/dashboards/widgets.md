# ウィジェット

ウィジェットはダッシュボード上の 1 つのタイルです。このページでは、追加できるすべてのウィジェット、それぞれの表示内容、いつ使うかをまとめます。

キャンバス上でウィジェットをドラッグして配置する方法については、[ダッシュボードの作成](/docs/dashboards/authoring) を参照してください。

## チャートと数値

### Chart

ダッシュボードの時間範囲に対する 1 つ以上のメトリクス系列の折れ線、棒、エリアチャート。

**設定**:

- 1 つ以上のメトリクスクエリ。
- 2 つのクエリを組み合わせるオプションの数式 (例えば `errors / total * 100` でエラー率)。
- リセットされずに増え続けるカウンタ用の「レートで表示」オプション。
- 表示オプション: スタックかオーバーレイか、Y 軸の単位、凡例の位置、チャートタイプ。

向いている用途: トレンドが重要なとき。時系列のレイテンシー、エラー数、キュー深度など、線の形が物語を伝える場面。

### Value

オプションで色付きしきい値を持つ、1 つの大きな数字。

**設定**:

- 1 つの数値 (時間範囲内の最新値、平均、最大) を返すメトリクスクエリ。
- オプションの **警告** しきい値 (上回ると黄色)。
- オプションの **クリティカル** しきい値 (上回ると赤)。
- 数値のフォーマットと単位。

向いている用途: 1 つの数字が答えになるとき。現在のエラー率、今この瞬間の P95 レイテンシー、開いているインシデント数。

### Gauge

最小値、最大値、警告帯、クリティカル帯を持つ円形のゲージ。

**設定**: メトリクスクエリと 4 つの境界。

向いている用途: 値が既知の範囲内に収まるとき。CPU 使用率 (0–100%)、ディスク使用量、キュー容量。

### Table

メトリクス結果のテーブル。グループごとに 1 行。

**設定**: メトリクスクエリ (通常はホストやサービスなどのラベルでグループ化)、表示する列、行数の制限。

向いている用途: トレンドではなく内訳が欲しいとき。最も騒がしいホスト Top 10、サービスごとのエラー数、エンドポイントごとのリクエスト数。

## Text

Markdown の静的なブロック。

**設定**: Markdown 本文。見出し、リスト、リンク、強調、コードブロックがレンダリングされます。

向いている用途: セクション見出し、文脈の段落、Runbook へのリンク一覧、インシデント中の一時的なバナー。

## ログとトレース

### Log Stream

フィルターに一致するログ行のライブテイル。

**設定**: ログフィルター (サービス、重要度、属性) と表示する列。

向いている用途: ダッシュボードを離れずに、今この瞬間アプリケーションが何を言っているかを見たいとき。

### Trace List

フィルターに一致する最近のトレース一覧。所要時間、ステータス、サービスを表示。

**設定**: トレースフィルター (サービス、ステータス、属性)。

向いている用途: チャートではなく最近のアクティビティのリストが欲しいとき。よくあるパターンは、上にレイテンシーチャート、下に遅いトレースのリスト。

## ライブリスト

### Incident List

フィルターに一致するインシデントのライブリスト。

**設定**: 状態、重大度、ラベル、モニター、チームによるフィルター。

向いている用途: ダッシュボードが「今何が壊れているか?」に答えるとき。

### Alert List

フィルターに一致するアラートのライブリスト。

**設定**: 状態、重大度、ラベルによるフィルター。

向いている用途: チームのダッシュボードで自社サービスのアラートを追うとき。

### Monitor List

モニターと現在のステータスのライブリスト。

**設定**: モニタータイプ、ラベル、現在の状態によるフィルター。

向いている用途: フリート全体を見たいとき — 「すべてのサイトは稼働しているか?」

## Kubernetes リソースリスト

[Kubernetes エージェント](/docs/monitor/kubernetes-agent) がインストールされたプロジェクト用。それぞれにクラスター、ネームスペース、ラベルのオプションフィルターがあります。

- **Kubernetes Pod List** — Pod とそのフェーズ、再起動回数、ノード。
- **Kubernetes Node List** — ノードとその状態、キャパシティ。
- **Kubernetes Namespace List** — ネームスペースとワークロード数。
- **Kubernetes Deployment List** — Deployment と希望 / 準備完了レプリカ数。
- **Kubernetes StatefulSet List** — StatefulSet と準備完了レプリカ数。
- **Kubernetes DaemonSet List** — DaemonSet と希望 / 準備完了数。
- **Kubernetes Job List** — Job と完了ステータス。
- **Kubernetes CronJob List** — CronJob とスケジュール、最終実行。

向いている用途: Kubernetes の状態とそのワークロードのテレメトリーを 1 つのダッシュボードでまとめたいとき。

## Docker リソースリスト

Docker 監視を設定したプロジェクト用。

- **Docker Host List** — Docker を実行しているホストとコンテナ数。
- **Docker Container List** — コンテナと状態、イメージ、ホスト、稼働時間。
- **Docker Image List** — イメージとそのサイズ。
- **Docker Network List** — Docker ネットワークと接続されたコンテナ。
- **Docker Volume List** — Docker ボリュームと使用状況。

## インフラ

### Host List

OneUptime のサーバーモニターで監視しているホスト。ステータス、CPU、メモリ、稼働時間を表示します。

**設定**: ラベルや現在の状態によるフィルター。

## どのウィジェットを使うべきか?

簡単な指針:

- **時系列のトレンド?** Chart。
- **今この瞬間に重要な 1 つの数字?** Value (最小 / 最大が明確なら Gauge)。
- **多数のものを横並びで比較する内訳?** Table。
- **システムで今何が起きているか?** Log Stream、Trace List、Incident List。
- **特定のリソースグループの状態?** 対応するリストウィジェット。
- **見出し、段落、リンク?** Text。

ほとんどのダッシュボードはこれらを組み合わせています — 上にチャート、横に値が 1〜2 個、テキストの仕切り、下にリストが 1〜2 個、といった具合です。
