# ダッシュボード ウィジェット

ウィジェットはダッシュボード上の 1 つのタイルです。各ウィジェットにはタイプ (chart、value、list、…)、位置、サイズ、設定があります。このページはカタログです — 各ウィジェットが何を表示し、何を入力として取り、いつ手に取るかを示します。

キャンバスの仕組みについては[ダッシュボードを作成する](/docs/dashboards/authoring) を参照してください。

## 時系列ウィジェット

### Chart

ダッシュボードの時間範囲にわたる 1 つ以上のメトリクス系列の折れ線 / 棒 / エリアチャート。

**設定**:

- 1 つ以上のメトリクスクエリ (単一系列は `metricQueryConfig`、複数は `metricQueryConfigs`)。
- 複数クエリを組み合わせる任意の **formula** (例: `errors / total * 100`)。
- OpenTelemetry の累積カウンター (例: `system.disk.io`) 用の任意の **transformAsRate** — ウィジェットがバケットごとに `(value - previousValue) / Δt` を計算します。
- 表示: 積み上げか重ね合わせ、Y 軸の単位、凡例のオン/オフ、チャートタイプ。

これを手に取るのは: トレンドが重要なとき。リクエストレイテンシー、時間経過のエラー数、キューの深さ — 曲線の形が何かを語るもの。

### Value

任意の閾値とオプションのスパークラインを持つ大きな単一の数値。

**設定**:

- メトリクスクエリ (単一値 — 通常は時間範囲の `last`、`avg`、`max`)。
- 任意の**警告閾値** (これを超えると黄色)。
- 任意の**危険閾値** (これを超えると赤)。
- 表示: 数値フォーマット、単位サフィックス。

これを手に取るのは: 1 つの数値で問いに答えられるとき。現在のエラー率、今この瞬間の P95 レイテンシー、開いているインシデントの数。

### Gauge

最小値、最大値、警告バンド、危険バンドを持つ円形のゲージ。

**設定**: メトリクスクエリと 4 つの境界 (min、max、warning、critical)。

これを手に取るのは: 値が既知の範囲に収まるとき。CPU 利用率 (0–100%)、ディスクの埋まり具合、キューの容量。

### Table

メトリクスクエリ結果の表形式表示。グループごとに 1 行。

**設定**: メトリクスクエリ (通常は `host.name` や `service.name` のようなラベルでグループ化)、表示する列、行数の上限。

これを手に取るのは: トレンドではなく内訳が欲しいとき。最もうるさいホストトップ 10、サービスごとのエラー数、エンドポイントごとのリクエストレート。

## 注釈ウィジェット

### Text

Markdown の静的ブロック。

**設定**: Markdown 本文。見出し、リスト、リンク、強調、コードスパン、フェンス付きコードブロックがすべてレンダーされます。

これを手に取るのは: セクション見出し、コンテキストを示す段落 (「このダッシュボードはチェックアウトサービスをカバーします」)、Runbook や関連ダッシュボードへのリンクのリスト、インシデント中の一時的なバナーが欲しいとき。

## ログとトレース

### LogStream

フィルターに一致するログ行のライブテイル。

**設定**: ログフィルター (サービス、深刻度、属性マッチ)、表示する列。

これを手に取るのは: ログエクスプローラーを開きにページを離れずに、アプリケーションが*今*何を言っているかをダッシュボードで見たいとき。

### TraceList

フィルターに一致する最近のトレースのリスト。所要時間、ステータス、サービス名付き。

**設定**: トレースフィルター (サービス、ステータス、属性マッチ)。

これを手に取るのは: チャートではなく最近の活動のページング付きビューが欲しいとき。よくある組み合わせ: 上部にレイテンシー Chart、下に遅いトレースの TraceList。

## 運用リスト

### IncidentList

フィルターに一致するインシデントのライブリスト。

**設定**: 状態、深刻度、ラベル、モニター、割り当てチームでのフィルター。

これを手に取るのは: ダッシュボードが「今、何が壊れている?」に答えるためのものであるとき。

### AlertList

フィルターに一致するアラートのライブリスト。

**設定**: 状態、深刻度、ラベルでのフィルター。

これを手に取るのは: アラート駆動のワークフロー向けダッシュボード (例えば自チームのサービスのアラートを見守る dev-team ダッシュボード)。

### MonitorList

フィルターに一致するモニターのライブリスト。各モニターの現在のステータスを表示。

**設定**: モニターのタイプ、ラベル、現在の状態でのフィルター。

これを手に取るのは: フリート全体の「すべてのウェブサイトは動いていますか?」ビューや、チームごとの監視対象エンドポイントリストが欲しいとき。

## Kubernetes リソースリスト

[Kubernetes エージェント](/docs/monitor/kubernetes-agent) をインストールしているプロジェクトでは、次のライブリソースウィジェットが利用できます。各ウィジェットは `cluster`、`namespace`、ラベル用の任意のフィルターを取ります。

- **KubernetesPodList** — Pod とそのフェーズ、再起動回数、ノード割り当て。
- **KubernetesNodeList** — ノードとその状態、キャパシティ、割り当て。
- **KubernetesNamespaceList** — ネームスペースとそのワークロード数。
- **KubernetesDeploymentList** — デプロイメントの desired と ready のレプリカ。
- **KubernetesStatefulSetList** — StatefulSet の ready レプリカ。
- **KubernetesDaemonSetList** — DaemonSet の desired と ready。
- **KubernetesJobList** — Job の完了ステータス。
- **KubernetesCronJobList** — CronJob のスケジュールと最終実行。

これらを手に取るのは: Kubernetes リソースの状態と、それらのワークロードからのテレメトリーを混ぜた単一のダッシュボードが欲しいとき。

## Docker リソースリスト

Docker モニターをインストールしているプロジェクトでは:

- **DockerHostList** — Docker を実行しているホストとコンテナ数。
- **DockerContainerList** — コンテナの状態、イメージ、ホスト、稼働時間。
- **DockerImageList** — イメージとそのサイズ。
- **DockerNetworkList** — Docker ネットワークと接続コンテナ数。
- **DockerVolumeList** — Docker ボリュームとその使用状況。

## インフラ

### HostList

OneUptime のサーバーモニターが監視するホスト — 現在のステータス、CPU、メモリ、稼働時間付き。

**設定**: ラベルや現在のヘルス状態でのフィルター。

## 適切なウィジェットを選ぶ

経験則をいくつか:

- **時間にわたるトレンド?** Chart。
- **今この瞬間に重要な 1 つの数値?** Value (自然な範囲があれば Gauge)。
- **多数の内訳?** Table。
- **今、システムで何が起きている?** LogStream、TraceList、IncidentList。
- **特定のリソースフリートの状態?** マッチするリソースリストウィジェット。
- **見出し、段落、リンク?** Text。

ほとんどのダッシュボードは混合します — 上部に Chart、隣に Value をいくつか、Text の区切り、その下に 1 つか 2 つのリスト。
