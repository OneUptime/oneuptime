# 構成とアクセス権

このページでは、長く使うダッシュボードを手にしたときに知っておくべき設定とアクセス制御をまとめます。

## オーナー

ダッシュボードの **オーナー** は、(プロジェクト全体のロールに加えて) 明示的にアクセス権を付与したユーザーやチームです。

**Dashboard → Owners** で:

- **ユーザーオーナー** を追加すると、特定の 1 人にこのダッシュボードへの追加アクセス権を付与できます。
- **チームオーナー** を追加すると、チームの全メンバーに同じことができます。

プロジェクト全体の閲覧ロールが広すぎる場合にオーナーを使います — 例えば、カスタマーサクセスチームだけが見るべき顧客レベルの詳細を含むダッシュボードなど。

## ラベル

ラベルはダッシュボードを整理するためのタグです。**Dashboard → Overview** で適用します。

よくあるパターン:

- **チーム別**: `team:platform`、`team:checkout`、`team:growth`。
- **環境別**: `env:prod`、`env:staging`。
- **目的別**: `purpose:oncall`、`purpose:exec`、`purpose:investigation`。

**Dashboards** リストはラベルでフィルターできます。これは、ダッシュボードが多く溜まったプロジェクトで目的のものを見つける最速の方法です。

## アクセス権

ダッシュボードはプロジェクトのロールベースアクセス制御に従います。関連する権限:

| 権限 | できること |
| --- | --- |
| **Create Dashboard** | 新しいダッシュボードを作成。 |
| **Read Dashboard** | ダッシュボードを閲覧 (プライベートモード)。 |
| **Edit Dashboard** | ウィジェット、変数、設定を変更。 |
| **Delete Dashboard** | ダッシュボードを削除。 |

ダッシュボードのオーナーやカスタムドメインに対応する権限もあるので、「オーナーの管理」を「ダッシュボードの編集」と切り離して付与できます。

これらは **Project Settings → Teams & Roles** でプロジェクトのロールに割り当てます。

## 公開ダッシュボードのアクセス

ダッシュボードを公開にした (([共有と公開ダッシュボード](/docs/dashboards/sharing) を参照)) 場合、誰が見られるかを 3 つの設定がコントロールします:

1. **Public Dashboard** スイッチ — オフだと、公開 URL は 404 を返します。
2. **Master Password** — 設定されていると、訪問者はダッシュボードが表示される前にパスワードを入力します。
3. **IP Whitelist** (Scale プラン) — 設定されていると、他の IP からのリクエストは拒否されます。

これらを組み合わせて使えます。最も厳重なのは「公開オン、パスワード設定、IP 許可リスト有効」の組み合わせで、3 層の保護がすべて欲しいパートナーポータルに向いています。

## データ保持

ダッシュボード自体は期限切れになりません。表示されるデータはプロジェクトの保持設定に従います — メトリクス、ログ、トレースはプランの保持期間中はクエリ可能です。30 日保持のプランで「過去 90 日」を見るウィジェットを使うと、保存されている分だけが表示されます。

## ダッシュボードの複製

既存のダッシュボードをコピーするには、ダッシュボード一覧を開いて **Duplicate** を選びます。コピーには、公開共有を除く、すべてのウィジェット、変数、設定が含まれます — 公開共有は常にオフで開始するので、再度オンにするかどうかを判断できます。

これは、テンプレート (例: 「オンコールダッシュボード」) をサービス固有のコピーにフォークするときに最適な方法です。

## ダッシュボードの削除

**Dashboard → Delete** で削除します。これは元に戻せません — ダッシュボードのレイアウトとそれに付随するカスタムドメインが削除されます。テレメトリーデータには影響しません。

ダッシュボードがカスタムドメインで公開されている場合、削除した瞬間に URL は解決されなくなります。URL を稼働させ続けたい場合は、先にドメインを別のダッシュボードに移してください。

## バックアップ

OneUptime をセルフホストしている場合は、通常のデータベースバックアップで十分です — ダッシュボードの設定はプロジェクトの他のデータと一緒に保存されます。

OneUptime Cloud では、バックアップは自動で行われます。自分のコピーが欲しい場合は、[OneUptime API](/docs/api-reference/api-reference) からダッシュボードを読み取れます。
