# 共有と公開ダッシュボード

ほとんどのダッシュボードはプロジェクトに対してプライベートです — ログインしているプロジェクトメンバーだけが見られます。しかし OneUptime では、ダッシュボードを公開 URL で公開し、任意でパスワードで保護し、IP で制限し、カスタムドメインでホストすることもできます。このページではその 4 つすべてを扱います。

## プライベートダッシュボード (デフォルト)

デフォルトでは、ダッシュボードはプロジェクトメンバーのログインしているユーザーだけが到達できます。URL は `https://oneuptime.com/dashboards/<id>/view` のような形です。直接アクセスには認証とダッシュボードの適切な読み取り権限が必要です。

プロジェクト内では、所有者とラベルが誰が何を見るかを制御します — [設定とアクセス権](/docs/dashboards/configuration) を参照してください。

## 公開ダッシュボード

**Dashboard → Settings** で **Public Dashboard** をオンにします。ダッシュボードはログインを必要としない 2 つ目の URL を持ちます。ベンダー、パートナー、顧客と共有したり、公開 README に貼り付けたりします。

公開ダッシュボードは:

- **View** モードでのみレンダーされます。公開訪問者は編集したり、時間範囲 URL 以外を変更したり、ウィジェットパレットを見たりできません。
- 定義した変数を含みます — 訪問者は内部ユーザーと同様にドロップダウンから選べます。
- Settings で設定した**ブランディング**を引き継ぎます: ページタイトル、ページ説明、ロゴファイル、ファビコン。これらはブラウザタブやソーシャルプレビューに表示されます。

**Public Dashboard** を有効にすることは、ウェブページを公開することと同じだと考えてください。ダッシュボード上のすべてのウィジェットが世界中から読めるようになります。スイッチを入れる前にキャンバスにあるものを監査してください。

## マスターパスワード

公開ダッシュボードを完全にオープンにせず、パスワードでゲートするには:

1. **Public Dashboard** を有効化。
2. **Master Password** を有効化。
3. パスワードを設定。

訪問者はダッシュボードがレンダーされる前にパスワードプロンプトを受けます。パスワードは保存時にハッシュ化され、ハッシュだけが保存されます。

マスターパスワードを使うのは:

- パートナーや顧客と共有したいが、URL が漏れたときに有効でないようにしたいとき。
- ダッシュボードが「セミパブリック」 — 各閲覧者用に OneUptime アカウントを欲しくないほど開かれているが、オープンなインターネットに置くほどは開いていないとき。

より高価値のゲート (閲覧者ごとのアカウント、誰が何を見たかの監査証跡) が必要なら、ダッシュボードをプライベートに保ち、閲覧者を読み取り専用メンバーとしてプロジェクトに招待してください。

## IP アローリスト

**Scale** プランでは、公開ダッシュボードをソース IP または CIDR レンジのリストに制限できます。**Dashboard → Settings → IP Whitelist** でリストを設定します。

IP アローリストを使うのは:

- ダッシュボードが社内オフィスや VPN からだけ到達可能であるべきとき。
- ベンダーポータルが公開された送信元 IP からだけ到達可能であるべきとき。
- マスターパスワードの上に多層防御が欲しいとき。

それ以外の IP からのリクエストは 403 を受けます。

## カスタムドメイン

箱から出した状態では、公開ダッシュボードは `oneuptime.com` で提供されます。自社のサブドメイン (例: `dashboard.acme.com`) でホストするには:

1. DNS に CNAME レコードを追加し、サブドメインを OneUptime の公開ターゲットに向けます。
2. **Dashboard → Settings → Custom Domains** でドメインを追加。
3. DNS レコードを検証 (OneUptime がチェックします)。
4. 検証されると、ダッシュボードは OneUptime の URL とカスタムドメインの両方から到達可能になります。

カスタムドメインが便利なのは:

- 自社ブランドの顧客向けダッシュボード。
- 共同ブランドのパートナーダッシュボード。
- 公開ヘルスページの SEO。

複数のオーディエンスに同じコンテンツを提供する場合、1 つのダッシュボードに複数のカスタムドメインを付けられます。

## 公開ダッシュボードのブランディング

**Dashboard → Settings** で設定します:

- **Page title** — `<title>` タグと訪問者が見る見出し。
- **Page description** — 検索エンジンとソーシャルプレビューが使うメタ説明。
- **Logo file** — PNG/SVG をアップロード。ダッシュボードヘッダーに表示されます。
- **Favicon** — アップロード。ブラウザタブに表示されます。

ブランディングは公開モードのレンダリングにのみ適用されます。内部の閲覧者は常に OneUptime のブランディングを見ます。

## 埋め込み

公開ダッシュボードを自社サイトの `<iframe>` に埋め込めます:

```html
<iframe src="https://dashboard.acme.com/view"
        width="100%" height="800"
        frameborder="0"></iframe>
```

マスターパスワードで保護されたダッシュボードを埋め込むと、訪問者は iframe 内でパスワードプロンプトを見ることになります。

## 変数状態を含む共有 URL

ダッシュボード URL は現在の変数の選択と時間範囲をクエリパラメータとしてエンコードします。ドロップダウンを調整し、URL をコピーしてチャットに貼ると、受信者は時間範囲を含めてまったく同じビューでダッシュボードを見ます。

これは「インシデントが始まった時点のダッシュボード」にチームメイトを案内する最速の方法です — 時間範囲を固定し、コピーし、貼り付けます。
