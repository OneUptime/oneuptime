# 共有と公開ダッシュボード

デフォルトでは、ダッシュボードはプロジェクトに対してプライベートです — ログインしているチームメンバーだけが見られます。しかし OneUptime では、ダッシュボードを公開で共有し、パスワードで保護し、特定の IP に制限し、独自のドメインでホストすることもできます。このページでは、その 4 つすべてを扱います。

## プライベートダッシュボード (デフォルト)

ダッシュボードは、ログインしているプロジェクトメンバーのみがアクセスできます。URL は `https://oneuptime.com/dashboards/<id>/view` のような形で、ログインが必要です。

プロジェクト内では、オーナーとラベルが誰が何を見られるかをコントロールします — [構成とアクセス権](/docs/dashboards/configuration) を参照してください。

## 公開ダッシュボード

**Dashboard → Settings** で **Public Dashboard** をオンにします。ダッシュボードは、ログイン不要の 2 つ目の URL を持つようになります。ベンダー、パートナー、顧客と共有したり、公開 README に貼り付けたりできます。

公開ダッシュボード:

- 常に **View** モードで開きます。公開の訪問者は編集もウィジェットパレットの閲覧もできません。
- 追加した変数を含みます。訪問者はチームと同じドロップダウンから選べます。
- Settings で設定した **ブランディング** を使います — ページタイトル、説明、ロゴ、ファビコン。

公開ダッシュボードを有効にするのは、Web ページを公開するようなものです。ダッシュボード上のすべてのウィジェットが世界中から読める状態になります。スイッチを切り替える前に、キャンバスに何が載っているかを確認してください。

## マスターパスワード

公開ダッシュボードにパスワードをかけるには:

1. **Public Dashboard** をオンにします。
2. **Master Password** をオンにします。
3. パスワードを設定します。

訪問者はダッシュボードが表示される前にパスワードを求められます。パスワードはハッシュとして保存され、実際のパスワードを OneUptime が知ることはありません。

マスターパスワードが向いている用途:

- パートナーや顧客と共有したいが、URL が漏れた場合に役に立たないようにしたい。
- ダッシュボードが「半公開」 — チームメンバーとして招待するほどではないがオープンインターネットには出したくない場合。

より強固なゲーティング (閲覧者ごとの別アカウント、誰が何を見たかの監査ログ) が必要な場合は、ダッシュボードをプライベートに保ち、閲覧者を読み取り専用のチームメンバーとして招待してください。

## IP 許可リスト

**Scale** プランでは、公開ダッシュボードを IP アドレスや範囲のリストに制限できます。**Dashboard → Settings → IP Whitelist** で設定します。

向いている用途:

- ダッシュボードがオフィスや VPN からのみアクセス可能であるべき場合。
- ベンダーポータルがその既知の IP からのみアクセス可能であるべき場合。
- マスターパスワードの上に追加の保護がほしい場合。

他の IP からのリクエストは拒否されます。

## カスタムドメイン

デフォルトでは、公開ダッシュボードは `oneuptime.com` で提供されます。`dashboard.acme.com` のような独自のサブドメインでホストするには:

1. DNS に CNAME レコードを追加し、サブドメインを OneUptime のターゲットに向けます。
2. **Dashboard → Settings → Custom Domains** でドメインを追加します。
3. 検証します。OneUptime が DNS レコードを確認します。
4. 検証されると、ダッシュボードはカスタムドメインと元の URL の両方からアクセスできるようになります。

カスタムドメインが向いている用途:

- 自社ブランドでの顧客向けダッシュボード。
- 共同ブランドのパートナーダッシュボード。
- 独自 URL を持つ公開ヘルスページ。

複数のオーディエンスに同じ内容を提供する場合は、1 つのダッシュボードに複数のカスタムドメインを付けることもできます。

## ブランディング

**Dashboard → Settings** では、次を設定できます:

- **ページタイトル** — ブラウザタブとページ上部に表示されるもの。
- **ページ説明** — 検索エンジンとソーシャルプレビューで使われる説明。
- **ロゴ** — ヘッダーに表示する PNG または SVG をアップロード。
- **ファビコン** — ブラウザタブの小さなアイコン。

ブランディングは公開で閲覧したときのみ適用されます。内部の閲覧者には常に OneUptime のブランディングが表示されます。

## 埋め込み

公開ダッシュボードは iframe で自社サイトに埋め込めます:

```html
<iframe
  src="https://dashboard.acme.com/view"
  width="100%"
  height="800"
  frameborder="0"
></iframe>
```

ダッシュボードにマスターパスワードがある場合、訪問者は iframe 内でパスワード入力を求められます。

## 共有可能な URL

ダッシュボードの URL には、現在の変数の選択と時間範囲がクエリパラメータとして含まれます。ドロップダウンを調整し、URL をコピーし、チャットに貼り付ければ、リンクを開いた人はまったく同じビューでダッシュボードを見ることができます。

これは「インシデントが起きた時刻のダッシュボード」をチームメイトに示す最速の方法です。時間範囲を固定し、コピーし、貼り付けるだけです。
