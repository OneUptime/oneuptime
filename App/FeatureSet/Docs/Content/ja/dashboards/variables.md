# 変数とフィルター

変数は 1 つのダッシュボードをテンプレートに変えます。`service` 変数をダッシュボードに追加すれば、同じチャートが `checkout`、`payments`、`search` 用に再描画されます — ほぼ同じダッシュボードを 3 つ作る代わりに、閲覧者は上部のドロップダウンから選びます。

## 変数のタイプ

変数は **Dashboard → Settings → Variables** で追加します。各変数は、ウィジェットで `{{name}}` として使う名前、オプションのラベル、タイプを持ちます。

### Custom List

静的なドロップダウン。選択肢を自分で入力します。

向いている用途: 選択肢が少なく固定されているとき。`prod, staging, dev` を値とする `environment`、`us-east-1, eu-west-1, ap-south-1` を値とする `region` など。

### Query

選択肢がデータに対するクエリから取得されます。

向いている用途: 選択肢が時間とともに変化し、ドロップダウンを追従させたいとき。「過去 24 時間に確認されたすべての顧客 ID」など。クエリがプロジェクトのデータに対して実行され、結果がドロップダウンになります。

### Text Input

自由入力のフィールド。閲覧者が入力したものがそのまま使われます。

向いている用途: ダッシュボードを検索ツールのように動作させたいとき。IP アドレス、リクエスト ID、その他自由な値でフィルター。

### Telemetry Attribute

選択肢が、ダッシュボードの時間範囲内のテレメトリー内の属性の個別の値になります。

**属性キー** (例: `service.name`、`host.name`、`k8s.cluster.name`) を設定します。ドロップダウンには、ログ、メトリクス、トレースで確認されたすべての個別の値が入ります。

向いている用途: 選択肢がテレメトリーで送信しているタグと一致するとき。これは最もよく使われるタイプです。自動的に更新されるためです — `service.name = inventory` というタグの新しいサービスを送信すると、ダッシュボードを編集せずにその名前がドロップダウンに表示されます。

## 複数選択

各変数は複数選択を許可できます。オンにすると、閲覧者は 1 つ以上の値を選べ、ダッシュボードはいずれかに一致するものをフィルターします。

複数選択が向いている用途: 「checkout と payments を一緒に」比較したいとき、ダッシュボードを離れずに行えます。選択した値の間で計算が成り立たない場合 (例えば平均の平均) は避けてください。

## デフォルト値

各変数にはデフォルトを設定できます。閲覧者が変更するまで、ダッシュボードはデフォルトでレンダリングされます。公開ダッシュボードの場合、デフォルトが訪問者に最初に表示される値になります。

## ウィジェットでの変数の使い方

ウィジェットがフィルターを受け取る場所 — メトリクスの `WHERE`、リストのフィルター、ログストリームの属性一致 — どこでも `{{variable_name}}` を使えます。

たとえば、サービスでフィルターしたチャート:

```
service.name = '{{service}}'
```

ドロップダウンが `checkout` に設定されると、チャートは checkout サービスをフィルターします。閲覧者が `payments` に切り替えると、チャートは payments 用に再描画されます。

**Telemetry Attribute** 変数の場合、OneUptime は変数がどの属性にマップされるかを知っており、同じ属性を使うすべてのウィジェットに自動的にフィルターを適用します — ウィジェットを手作業で編集する必要はありません。

## 時間範囲

ダッシュボードのヘッダーにはグローバルな時間範囲があります。すべてのメトリクスウィジェットはこのウィンドウに対してクエリします。オプション:

- **プリセット** — 過去 1 時間、24 時間、7 日、30 日、90 日 (データの保持期間に依存)。
- **カスタム** — 開始と終了の時刻を選択。

時間範囲はダッシュボードの URL の一部です — URL を共有すると、ウィンドウも共有されます。インシデント中に便利です: 時間範囲を「今日の 10:00–10:30 UTC」に固定し、インシデントチャンネルにリンクを貼り付けます。

## 更新間隔

時間範囲の隣で、ウィジェットの再クエリ頻度を選びます:

- **Off** — ページの読み込み時に一度だけクエリ。
- **5s / 10s / 30s / 1m / 5m / 15m** — 自動更新。

自動更新は壁掛けスクリーンやライブのインシデントビューに向いています。調査中は、見ている間にビューが動かないようオフにしておきます。

## 組み合わせる

サービステンプレート化されたダッシュボードには通常、次の要素があります:

1. `service.name` の **Telemetry Attribute** タイプの `service` 変数。デフォルト: 最もよく見るサービス。複数選択オフ (チャートは常に 1 つずつ表示)。
2. **Custom List** タイプの `environment` 変数。デフォルト: `prod`。
3. `k8s.cluster.name` の **Telemetry Attribute** タイプの `cluster` 変数。複数選択オン (クラスター間で比較できるように)。
4. これらの変数をフィルターで参照するウィジェット。

結果: 1 つのダッシュボードで、すべてのサービスをカバーし、上部に 3 つのドロップダウン。
