# トリガー

トリガーはワークフローの最初のブロックで、ワークフローがいつ実行されるかを決定します。すべてのワークフローはちょうど 1 つのトリガーを持ちます。4 種類のトリガーから選択できます。

## Manual

ワークフローページの **Run Manually** をクリックすると、オンデマンドでワークフローを実行できます。ワークフローの残りの部分が読み取れる JSON ペイロードを貼り付けることもできます。

向いている用途: 「このキーをローテーションする」「テストアラートを送る」など、ボタン 1 つで実行したいワンクリックの自動化。

**出力**: 貼り付けた JSON、または何も貼り付けなかった場合は空のオブジェクト。

## Schedule

cron 式を使って、定期的なスケジュールでワークフローを実行します。

向いている用途: 夜間のクリーンアップ、毎時の同期、週次レポート。

**設定**: cron 式。よく使うものをいくつか挙げます:

- `0 * * * *` — 毎時 0 分。
- `*/5 * * * *` — 5 分ごと。
- `0 9 * * 1` — 毎週月曜日の午前 9:00。

システムが一時的に利用できない場合でも、復旧次第すぐに実行が拾い上げられます — 短い停止での実行の取りこぼしを気にする必要はありません。

## Webhook

OneUptime が固有の URL を作成します。その URL にアクセスがあるとワークフローが開始します。リクエストのヘッダー、クエリパラメータ、ボディが渡されます。

向いている用途: 他のツールから OneUptime にデータを受け取る — CI/CD のコールバック、他の監視からのアラート、CRM のサインアップなど。

**出力**:

- **Request Headers** — 受信リクエストのすべてのヘッダー。
- **Request Query Params** — パース済みのクエリ文字列。
- **Request Body** — パース済みのボディ (JSON でない場合は生のテキスト)。

URL は `GET` と `POST` の両方を受け付けます。呼び出し元には素早く確認応答が返されます — ワークフロー自体はバックグラウンドで実行されます。

URL はパスワードのように扱ってください。URL を持っている人は誰でもワークフローを開始できます。

## OneUptime のイベントトリガー

OneUptime のほぼすべてのもの — モニター、インシデント、アラート、スケジュールされたメンテナンス、ステータスページ、オンコールポリシー、チーム — がワークフローをトリガーできます。それぞれに 3 つのイベントが用意されています:

- **On Create** — 新しいものが追加されたときに発火。
- **On Update** — 変更されたときに発火。
- **On Delete** — 削除されたときに発火。

これが、ループでチェックすることなく「OneUptime で X が起きたら Y を実行する」を作る方法です。

完全なレコードが次のブロックに渡されます。たとえば **Incident → On Create** トリガーは新しいインシデントを渡すので、次のブロックはタイトル、説明、重大度、その他のあらゆるフィールドを読み取れます。

### チームがよく使うイベント

- **Incident** — インシデントが開かれた、更新された (確認、解決)、削除されたときに反応。
- **Alert** — アラートに対する同じ 3 つのイベント。
- **Monitor** — モニターが追加、編集、削除されたときに反応。
- **Scheduled Maintenance** — メンテナンス枠がスケジュールされたときに自動で告知。
- **Status Page Subscriber** — ステータスページに購読した人をウェルカム。
- **On-Call Duty Policy** — スケジュールの変更を別の名簿システムに同期。

トリガーのパレットを名前で検索すると、目的のものが見つかります。

## どのトリガーを使うべきか?

| やりたいこと                           | 選ぶもの                 |
| -------------------------------------- | ------------------------ |
| ボタンをクリックしてワークフローを実行 | **Manual**               |
| 定期的なスケジュールで実行             | **Schedule**             |
| 別のシステムからデータをプッシュさせる | **Webhook**              |
| OneUptime 内の何かに反応する           | **OneUptime のイベント** |

ワークフローはトリガーを 1 つだけ持てます。同じ自動化を 2 通りの方法で開始したい場合は、共通ロジックを 1 つのワークフローにまとめ、**Execute Workflow** コンポーネントを使って 2 つの薄い「ラッパー」ワークフローから呼び出してください。
