# ワークフロー 概要

ワークフローは OneUptime のビジュアル自動化ビルダーです。トリガーをキャンバスにドラッグし、それを一連のアクション — HTTP コール、Slack メッセージ、JavaScript スニペット、条件分岐、データベース参照 — につなげれば、OneUptime (または外部世界) のイベントが発火するたびに動く自動化のできあがりです。

Runbook がインシデント中に人が使うチェックリストだとすれば、ワークフローはプロジェクトのバックグラウンドジョブです — 誰も見ていない場所で動き、出来事に反応し、OneUptime と他のスタックを糊付けします。

## ひと目で

- OneUptime ダッシュボードの **Workflows** にある**トップレベル機能**。
- **3 種類のトリガー方式**: Manual、Schedule (cron)、Webhook — さらに、OneUptime のあらゆるエンティティ (インシデント、アラート、モニター、ステータスページなど) が作成・更新・削除されたときに発火する**モデルイベントトリガー**。
- **ビジュアルキャンバス**: コンポーネントパレットからノードをドラッグし、出力ポートを入力ポートにつないでいきます。
- **混合自動化**: HTTP リクエスト、Slack / Discord / Microsoft Teams / Telegram メッセージ、カスタム JavaScript、JSON パース、条件分岐、メール、サブワークフロー呼び出し、そして OneUptime モデルへの CRUD 操作。
- **グローバル変数**: プロジェクト全体のシークレットと設定。コピー&ペーストせずに任意のワークフローから参照できます。
- **実行とログ**: すべての実行がステータス、所要時間、ステップごとの出力とともに記録されます。

## なぜワークフローを使うのか?

多くのチームは次のような場面でワークフローに手を伸ばします:

- **OneUptime を別のシステムに繋ぐ** — インシデントを PagerDuty に投稿、アラートを Jira にミラーリング、自社スタックの Webhook を叩く。
- **OneUptime のイベントに反応する** — `Sev 1` のインシデントが開いたら、オンコールのマネージャーをページし、*かつ* Linear チケットを作成し、*かつ* フィーチャーフラグをロックする。
- **定期ジョブをスケジュール** — 5 分ごとに内部 API を問い合わせ、その結果を外部システムに書き込む。
- **OneUptime の外からデータを受け取る** — CI システムからの Webhook が OneUptime の更新チェーンを開始する。
- **小さな糊付けロジックを再利用する** — 1 つのワークフローが別のワークフローを呼び出し、共通パターンを 1 か所に集約する。

## キー概念

| 用語 | 意味 |
| --- | --- |
| **ワークフロー** | キャンバス。名前付きの再利用可能なトリガーとコンポーネントのグラフで、`isEnabled` フラグを持つ。 |
| **トリガー** | ワークフロー実行を開始するノード。Manual、Schedule、Webhook、またはモデルイベント。各ワークフローは必ず 1 つのトリガーを持つ。 |
| **コンポーネント** | 作業を行うノード — HTTP コール、Slack メッセージ、JavaScript スニペット、条件分岐など。 |
| **ポート** | ノード上の入力または出力ソケット。コンポーネントには `success` や `error` のような出力ポートがあり、ポートを次のノードの入力ポートにつなぎます。 |
| **実行 / ログ** | ワークフローの 1 回の実行。タイムスタンプ、ステータス (Running、Success、Failed、Timeout)、そして実行された全ノードのキャプチャ済み出力を保持します。 |
| **グローバル変数** | プロジェクトレベルで一度定義され、`{{variable.NAME}}` として任意のワークフローから参照される名前付きの値 (多くの場合シークレットや API キー)。 |
| **ローカル変数** | 1 回のワークフロー実行内にスコープを持つ値 — 通常は先行ノードの戻り値で、`{{ComponentId.portName}}` として参照されます。 |

## ダッシュボードのどこにワークフローがあるか

| ページ | そこで行うこと |
| --- | --- |
| **Workflows** | ワークフローテンプレートを閲覧・作成・検索する。 |
| ワークフローの **Builder** タブ | ドラッグ&ドロップのキャンバス。ノードを追加、ポートをつなぎ、引数を設定する。 |
| ワークフローの **Logs** タブ | このワークフローのすべての実行。ステータスと時間範囲でフィルタ可能。実行をクリックするとノードごとの出力が見られます。 |
| ワークフローの **Settings** タブ | リネーム、有効化/無効化、説明の変更、ラベルの管理、削除。 |
| **Workflows → Global Variables** | あらゆるワークフローから参照されるプロジェクト全体の値を定義。保存後に値を UI から隠すには、値をシークレットとしてマークする。 |
| **Workflows → Runs & Logs** | すべてのワークフローを横断したプロジェクト全体の実行履歴。 |

## ワークフローのライフサイクル

1. **作成** — ワークフローを作成し、キャンバスにトリガーをドロップし、必要なコンポーネントをドラッグして、つなぎ、それぞれを設定する。
2. **有効化** — ワークフローは無効状態で出荷されます。配線が正しいと確信したら、Settings のスイッチをオンにします。
3. **トリガー** — Manual: 任意の JSON ペイロードとともに **Run Manually** をクリック。Schedule: cron が発火。Webhook: 外部システムがワークフロー URL に `POST` する。モデルイベント: 誰か (または別のワークフロー) がモニター、インシデント、アラートなどを作成・更新・削除する。
4. **実行** — Workflow Worker がグラフを順に辿ります。各コンポーネントは引数を読み (リテラル値または補間された変数)、自分の仕事をし、戻り値を書き、出力ポートを選びます。次のノードが発火します。
5. **監査** — 実行は **Logs** に現れます。ステータス、合計所要時間、コンポーネントごとの出力、すべてのエラーがプロジェクトの存続期間中保持されます。

## 具体例

ゴール: タイトルに `Sev 1` を含むインシデントが作成されたら、Slack チャンネルに投稿し、社内管理ツールでチケットを開く。

**1. 「Sev 1 fan-out」というワークフローを作成。**

**2. トリガーをドロップ。** パレットから **Incident → On Create** トリガーを選びます。トリガーは新しいインシデントを戻り値として公開します。

**3. Conditional コンポーネントをドロップ。** トリガーの出力ポートをその入力につなぎます。条件を設定: `{{Incident.title}}` *contains* `Sev 1`。

**4. Conditional の `yes` ポートから Slack コンポーネントをドロップ。** Channel: `#incident-room`。Message body: `Sev 1 declared: {{Incident.title}} — {{Incident.dashboardUrl}}`。

**5. 同じ `yes` ポートから (並列で) API コンポーネントをドロップ。** `https://admin.internal/incidents` への `POST`。Body は、そのインシデントから組み立てた小さな JSON オブジェクト。

**6. ワークフローを有効化。** 別タブで「Sev 1 — checkout 500s」というタイトルのインシデントを開きます。数秒以内に Slack メッセージが届き、**Logs** に各ノードの出力が記録された新しい実行が現れます。

## OneUptime の他機能との関わり

- **モニター**が問題を検出し、**インシデント/アラート**がそれを記録し、**ワークフロー**がそれに反応します — メッセージを投稿、チケットを開く、自動化を起動。
- **Runbook** は人のための対応手順 (任意のスクリプトステップ付き) です。ワークフローは無人のバックグラウンド自動化です。両者は補完的 — Runbook のステップがワークフローの Webhook トリガーへ `POST` することもあります。
- **ワークスペース連携** (Slack、Microsoft Teams) はワークフロー通知の典型的な宛先です。
- **ダッシュボード**は読み取り専用のビュー、ワークフローは書き込み側です — OneUptime の状態を更新し、外部 API を呼び、データを動かします。
