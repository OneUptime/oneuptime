# ワークフローの作成

ワークフローを作成するには、**Workflows → Create Workflow** を開き、名前をつけて **Builder** タブをクリックします。自動化を構築する空のキャンバスが表示されます。

## キャンバス

Builder はドラッグ&ドロップで操作するキャンバスです。サイドのパレットからブロックを追加し、線でつないで、各ブロックをクリックして動作を設定します。変更は自動的に保存され、保存されると上部にインジケーターが表示されます。

すべてのワークフローは、先頭に 1 つの **トリガー** から始まります。それ以外はすべて何らかの動作を行う **コンポーネント** です。

## ブロックの構成要素

| 項目 | 役割 |
| --- | --- |
| **タイトル** | キャンバスに表示される名前。複雑なワークフローを読みやすくするためにリネームできます。 |
| **設定** | ブロックが仕事をするために必要なもの — URL、Slack チャンネル、メッセージ本文など。必須項目にはアスタリスクが付きます。 |
| **入力** | 左側の点で、前のブロックから線が入ってくる場所です。 |
| **出力** | 右側の点で、次のブロックへ線が出ていく場所です。多くのブロックには **success** と **error** の出力が分かれており、両方のケースを扱えます。 |

## ブロックの接続

あるブロックの出力点から、次のブロックの入力点へドラッグします。引いた線が次に何が実行されるかを決定します。

- **success** から接続した場合、前のブロックが成功したときのみ次のブロックが実行されます。
- **error** から接続した場合、前のブロックが失敗したときのみ次のブロックが実行されます。
- 出力を接続しなければ、そのパスはそこで終わります。

1 つの出力を複数のブロックに接続することもできます。それらはその時点から同時に実行されます。

## ブロックの設定

ブロックをクリックすると、サイドに設定が開きます。各設定にはふさわしい種類の入力 — テキストフィールド、ドロップダウン、コードエディタ、トグルなど — が用意されています。

ほとんどのテキストフィールドは変数を受け付けます — これが、あるブロックから次のブロックへデータを流す仕組みです。構文については [変数](/docs/workflows/variables) を参照してください。

## 最初のワークフロー

キャンバスの感覚をつかむ最も手早い方法:

1. **Manual** トリガーをキャンバスにドラッグします。
2. **Log** コンポーネント (**Utils** 配下) をその隣にドラッグします。トリガーから Log コンポーネントへ接続します。
3. Log ブロックのメッセージフィールドに `Hello from {{Manual.JSON.name}}` と入力します。
4. 保存してワークフローをオンにします。
5. **Run Manually** をクリックし、入力として `{ "name": "Ada" }` を貼り付けて送信します。
6. **Logs** タブを開きます。最新の実行に `Hello from Ada` と表示されます。

このサイクル — ドラッグ、接続、設定、実行、ログの確認 — がすべてのワークフロー構築のやり方です。

## 保存とオン化

キャンバスは作業中に自動保存されます。別途「公開」ステップはありません。

ただし、ワークフローが実際に実行されるのは、Settings で **Enabled** がオンになっているときだけです。新しいワークフローは無効状態で始まります。このスイッチを安全網として使ってください — 構築し、**Run Manually** でテストし、ログを確認してから、オンにします。

ワークフローを削除せずに一時停止するには、**Enabled** をオフにします。実行中のものは終了まで続き、新しい実行は開始されません。

## 整理

- ブロックをドラッグして移動します。レイアウトは保存され、次に開く人にも同じ配置で表示されます。
- 線を右クリックすると削除できます。ブロックを右クリックすると削除や複製ができます。
- 横に広いワークフローでは、実行方向に沿って読めるように左から右へレイアウトすると見やすくなります。
