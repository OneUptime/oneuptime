# ワークフローを作成する

**Workflows → Create Workflow** からワークフローを作成し、名前と任意の説明を付け、**Builder** タブを開いてキャンバスにノードをドロップし始めます。

## キャンバス

Builder はズームとパンができるグラフです。コンポーネントパレットからノードを追加し、エッジでつなぎ、各ノードの引数をサイドパネルで設定します。ヘッダーの保存インジケーターが、最新の編集が保存されたかどうかを示します。

ワークフローは必ず 1 つの**トリガー**ノードから始まります。トリガーには入力ポートがありません — そこから実行が開始されるからです。下流のすべては**コンポーネント**です。

## ノードの構造

すべてのノードには以下があります:

| 項目 | 用途 |
| --- | --- |
| **タイトル** | キャンバス上に表示されるラベル。デフォルトはコンポーネント名。複雑なワークフローを読みやすくするために上書きしてください。 |
| **引数** | コンポーネントが仕事をするのに必要な設定 — URL、Slack チャンネル、JavaScript スニペットなど。必須引数はアスタリスクで示されます。 |
| **入力ポート** | ノードの左側にあるソケットで、入ってくるエッジが着地します。コンポーネントには `in` という 1 つの入力ポートがあり、トリガーには入力ポートがありません。 |
| **出力ポート** | 右側のソケットで、出ていくエッジが始まります。コンポーネントは `success`、`error`、`yes`、`no` のようなポートを定義します。 |
| **戻り値** | ノードが生成するデータ — 出力ポートのペイロード。下流のノードは `{{NodeId.fieldName}}` として参照します。 |

## ノードを接続する

出力ポートから下流のノードの入力ポートへドラッグしてエッジを作成します。`success` からのエッジは上流ノードが成功したときだけそのブランチを実行し、`error` からのエッジは失敗したときだけ実行します。ポートをつながなければ、そのブランチは単に終わります。

ファンアウトもできます: 1 つの出力ポートが複数の下流ノードに供給でき、そこから並列で実行されます。

## 引数を設定する

ノードをクリックするとサイドパネルが開きます。各引数には型付きのエディターがあります:

- **Text / URL / Email / Number / Password** — 単一行入力。
- **JSON** — 構文ハイライトとバリデーション表示付きの JSON エディター。
- **JavaScript** — **Custom Code** コンポーネントで使うスニペット用のコードエディター。
- **Markdown / HTML** — メールやメッセージコンポーネント用のリッチテキスト本文。
- **CronTab** — スケジュール式 (Schedule トリガーで使用)。
- **Boolean** — トグル。
- **Select / Query** — 固定値の集合またはモデル形式のクエリを取るフィールドのドロップダウン。

任意のテキストフィールドは変数補間を受け入れます — ルールは[変数](/docs/workflows/variables) を参照してください。

## 最小の最初のワークフロー

キャンバスを最も手早く触ってみる方法:

1. **Manual** トリガーをドロップ。
2. **Log** コンポーネント (**Utils** の下) をドロップ。トリガーの出力ポートを Log コンポーネントの入力ポートにつなぐ。
3. Log コンポーネントの引数に `Hello from {{Manual.JSON.name}}` と入力。
4. 保存してワークフローを有効化。
5. **Run Manually** をクリック、入力として `{ "name": "Ada" }` を貼り付けて送信。
6. **Logs** タブを開く。最新の実行に Log ノードのキャプチャされた出力 `Hello from Ada` が表示されます。

この往復 — ドラッグ、配線、設定、実行、検査 — があらゆるワークフロー作成のリズムです。

## 保存、有効化、本番でのテスト

ワークフローは `Workflow.graph` カラムに JSON グラフとして保存されます。Builder は編集中に自動保存し、ヘッダーの保存インジケーターが最新の変更がサーバーに届いたタイミングを示します。別の「公開」ステップはありません。

ただし、ワークフローはトリガーを発火させるのは **isEnabled** がオンのときだけです。新しいワークフローは無効状態で出荷されます。そのフラグを「本番投入準備完了」スイッチとして扱ってください — 構築し、サンプルペイロードで **Run Manually** をクリックしてドライランし、**Logs** を確認し、それから初めて Enable をオンにします。

ワークフローを削除せずに一時停止する必要がある場合 (例えば無関係なインシデント中)、**Settings** で **isEnabled** をオフにします。既存の進行中の実行は続行し、新しい実行は開始されません。

## 並び替えと整理

- ノードをドラッグして位置を変えます。位置はグラフに保存されるので、次にキャンバスを開く人も同じレイアウトを見ます。
- エッジを右クリックして削除、ノードを右クリックして削除と複製のオプションを表示。
- 横長のワークフローは左から右にレイアウトすると、実行方向と読む方向が一致します。
