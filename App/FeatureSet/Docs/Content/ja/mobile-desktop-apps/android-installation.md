# Android インストールガイド

最高の監視体験のために、OneUptime を Android デバイスにネイティブアプリとしてインストールします。

## インストール方法

### 方法 1: Chrome ブラウザ（推奨）

1. **Chrome で OneUptime を開く**
   - Android デバイスで Google Chrome を起動します
   - OneUptime インスタンスの URL に移動します
   - ページが完全に読み込まれるまで待ちます

2. **インストールプロンプト**
   - 下部の「ホーム画面に追加」バナーを探します
   - 「インストール」または「ホーム画面に追加」をタップします
   - プロンプトが表示されない場合は、右上の三点メニュー（⋮）をタップします

3. **メニューからの手動インストール**
   - Chrome メニュー（三点）をタップします
   - 「ホーム画面に追加」または「アプリをインストール」を選択します
   - 必要に応じてアプリ名をカスタマイズします
   - 「追加」をタップして確認します

4. **アプリを起動する**
   - ホーム画面またはアプリドロワーで OneUptime アイコンを見つけます
   - タップしてフルスクリーンモードでアプリを起動します

### 方法 2: Samsung Internet

1. **OneUptime を開く**
   - Samsung Internet ブラウザを起動します
   - OneUptime インスタンスに移動します
   - ページが完全に読み込まれるまで待ちます

2. **ホーム画面に追加**
   - メニューボタン（三本線）をタップします
   - 「ページを次の場所に追加」→「ホーム画面」を選択します
   - アプリ名を入力して「追加」をタップします

3. **起動**
   - ホーム画面でアプリアイコンを見つけます
   - タップして OneUptime をアプリモードで開きます

### 方法 3: Firefox

1. **OneUptime を開く**
   - Firefox ブラウザを起動します
   - OneUptime の URL に移動します
   - ページが完全に読み込まれるまで待ちます

2. **インストール**
   - 三点メニューをタップします
   - 「インストール」を選択します（利用可能な場合）
   - または「ホーム画面に追加」を選択します
   - インストールを確認します

### カスタマイズオプション

### アプリ名
- インストール中にアプリ名をカスタマイズできます
- デフォルト: "OneUptime"
- 推奨: "OneUptime" のままにするか、会社名を追加します

### 通知設定
1. **権限を付与する**
   - プロンプトが表示されたら通知を許可します
   - 設定 → アプリ → OneUptime → 通知 に移動します
   - 最良のエクスペリエンスのためすべての通知カテゴリを有効にします

2. **アラートをカスタマイズする**
   - どのインシデントが通知をトリガーするかを設定します
   - 通知の優先度レベルを設定します
   - サウンドとバイブレーションの設定を選択します

## トラブルシューティング

### インストールの問題

**「ホーム画面に追加」が表示されない場合:**
```
1. ブラウザのキャッシュとクッキーをクリアします
2. HTTPS（セキュアな接続）を使用していることを確認します
3. プロンプトを探す前に 2〜3 分ページに留まります
4. OneUptime インスタンスで PWA 要件が満たされているか確認します
```

**インストールが失敗する場合:**
```
1. ストレージ容量を確保します（少なくとも 50MB 必要）
2. ブラウザを最新バージョンに更新します
3. ブラウザを再起動して再試行します
4. 別のブラウザを試します（Chrome 推奨）
```

**アプリアイコンが表示されない場合:**
```
1. ホーム画面とアプリドロワーを確認します
2. アプリセクションの「最近追加したもの」を確認します
3. アプリドロワーで「OneUptime」を検索します
4. 必要に応じて再インストールします
```

### 通知の問題

**通知が届かない場合:**
```
1. 通知権限を確認します:
   - 設定 → アプリ → OneUptime → 権限 → 通知
2. OneUptime ダッシュボードで通知が有効になっていることを確認します
3. 「マナーモード」設定を確認します
4. バッテリー最適化設定が OneUptime をブロックしていないか確認します
```

**通知が遅延する場合:**
```
1. OneUptime のバッテリー最適化を無効にします:
   - 設定 → アプリ → OneUptime → バッテリー → バッテリー使用量を最適化
2. バックグラウンドアクティビティを許可します
3. データセーバー設定を確認します
```

## アンインストール

### アプリを削除する
1. ホーム画面の OneUptime アイコンを**長押し**します
2. **「アンインストール」**を選択するか、ゴミ箱にドラッグします
3. 削除を確認します

### 別の方法
1. **設定 → アプリ** に移動します
2. **「OneUptime」**を見つけます
3. **「アンインストール」**をタップします
4. 削除を確認します

## 更新とメンテナンス

### 自動更新
OneUptime PWA は自動的に更新されます:
- **自動更新**: オンライン時に訪問するとアプリが更新されます
- **手動更新不要**: ストアアプリとは異なり、ユーザーの操作は不要です
- **即時更新**: 新機能が直ちに利用可能になります
- **ロールバック安全**: 問題のある更新は素早く元に戻すことができます

## 詳細設定

### 開発者オプション
PWA を詳しく検査したい上級ユーザー向け:
1. Android で開発者オプションを有効にします
2. ADB でコンピュータに接続します
3. リモートデバッグに Chrome DevTools を使用します

### ネットワーク設定
- 社内の OneUptime インスタンスにアクセスする場合は VPN を設定します
- 組織で必要な場合はプロキシ設定をします
- ファイアウォールが PWA リソースを許可することを確認します

## ベストプラクティス

### 最適なパフォーマンスのために
1. **初回起動**: 初期セットアップには常にオンライン状態で行います
2. **定期的な使用**: キャッシュを新鮮に保つためにアプリを定期的に開きます
3. **ストレージ管理**: 十分な空き容量を確保します
4. **ネットワーク**: 初期インストールと大きな更新には Wi-Fi を使用します

### セキュリティの推奨事項
1. **HTTPS のみ**: 安全な OneUptime インスタンスからのみインストールします
2. **公式 URL**: 組織の公式 OneUptime URL からインストールしていることを確認します
3. **権限**: 必要な権限のみを付与します
4. **更新**: Android OS とブラウザを常に最新の状態に保ちます
