# FAQ とトラブルシューティング

OneUptime モバイルおよびデスクトップアプリ（PWA）のよくある質問と解決策です。

## 一般的な FAQ

### プログレッシブウェブアプリ（PWA）とは何ですか？

プログレッシブウェブアプリは、モダンなウェブ技術を使用してアプリのような体験を提供するウェブアプリケーションです。PWA はアプリストアなしでブラウザから直接インストールでき、オフラインで動作し、プッシュ通知を送信し、デバイスのオペレーティングシステムと統合できます。

### なぜ OneUptime は従来のアプリストアを使用しないのですか？

OneUptime が PWA 技術を使用する理由は、いくつかの利点があるためです。
- **即時更新**: アプリストアの承認や手動更新を待つ必要がありません
- **クロスプラットフォーム**: 単一のコードベースがすべてのデバイスで動作します
- **ダウンロードサイズの制限なし**: サイズ制限なしにすべての機能が使用できます
- **直接配布**: OneUptime インスタンスから直接インストールできます
- **常に最新**: ユーザーは常に最新バージョンを持ちます
- **セキュリティ**: ウェブアプリケーションと同じセキュリティ上のメリット


### OneUptime PWA はどのくらいのストレージを使用しますか？

- **初期インストール**: 10〜20MB
- **キャッシュの増加**: 定期的な使用で 50〜100MB
- **最大キャッシュ**: ブラウザによって通常 200MB に制限されています
- **自動クリーンアップ**: ブラウザが自動的にストレージを管理します

### OneUptime PWA はプッシュ通知をサポートしていますか？

はい、OneUptime PWA はリッチなプッシュ通知をサポートしています。
- **インシデントアラート**: リアルタイムのインシデント通知
- **ステータス更新**: モニターのステータス変更アラート
- **カスタムトリガー**: 通知ルールの設定
- **リッチコンテンツ**: 画像、アクション、詳細情報
- **バッジ更新**: アプリアイコンの未読数

## インストール FAQ

### 「インストール」ボタンが表示されないのはなぜですか？

一般的な原因と解決策:
1. **ブラウザの互換性**: Chrome、Edge、または Safari を使用してください
2. **HTTPS が必要**: OneUptime インスタンスが HTTPS を使用していることを確認してください
3. **PWA 要件**: サーバーが PWA マニフェスト要件を満たしている必要があります
4. **キャッシュの問題**: ブラウザキャッシュをクリアして再読み込みします
5. **既にインストール済み**: アプリが既にインストールされている可能性があります
6. **待機時間**: 一部のブラウザはページに 30 秒以上留まる必要があります

### 複数のデバイスにインストールできますか？

はい！OneUptime PWA を以下にインストールできます:
- ユーザーごとに無制限のデバイス
- 同じデバイスの複数のブラウザ
- 異なるオペレーティングシステム
- 共有/家族のデバイス（別々のアカウントで）

### インストール済みアプリをどのように更新しますか？

OneUptime PWA は自動的に更新されます:
- **自動更新**: オンライン時に訪問するとアプリが更新されます
- **バックグラウンド更新**: 更新はバックグラウンドでダウンロードされます
- **即時利用可能**: 新機能が直ちに利用可能になります
- **ユーザーの操作不要**: ストアアプリとは異なり、手動更新は不要です

### インストール中にアプリ名をカスタマイズできますか？

はい、インストール中に以下が可能です:
- アプリ名の変更（デフォルト: "OneUptime"）
- 組織名の追加
- カスタム命名規則の使用
- アイコンラベルの変更（プラットフォームによる）

### OneUptime PWA のアンインストール方法は？

アンインストールはプラットフォームによって異なります:

**Android:**
- アプリアイコンを長押し → アンインストール
- 設定 → アプリ → OneUptime → アンインストール

**iOS:**
- アプリアイコンを長押し → アプリを削除 → アプリを削除

**Windows:**
- 設定 → アプリ → OneUptime → アンインストール
- スタートメニューのアイテムを右クリック → アンインストール

**macOS:**
- アプリケーションからゴミ箱にドラッグ
- Dock のアイコンを右クリック → 削除

**Linux:**
- アプリケーションランチャーから削除
- .desktop ファイルを削除


## 通知 FAQ

### 通知が届かないのはなぜですか？

一般的な通知の問題と修正方法:

**権限の確認:**
```
1. ブラウザの通知権限が有効になっていること
2. オペレーティングシステムの通知権限
3. OneUptime の通知設定が構成されていること
4. 「マナーモード」が無効になっていること
```

**プラットフォーム別:**
- **Android**: バッテリー最適化設定を確認します
- **iOS**: 設定アプリで通知設定を確認します
- **Windows**: フォーカスアシスト設定を確認します
- **macOS**: 通知センターの権限を確認します
- **Linux**: 通知デーモンのステータスを確認します

### 通知音をカスタマイズできますか？

通知のカスタマイズオプション:
- **システムサウンド**: OS の通知サウンド設定を使用します
- **ブラウザ設定**: ブラウザの通知設定で設定します
- **OneUptime 設定**: ダッシュボードで通知設定を行います
- **優先度レベル**: 重大度レベルごとに異なるサウンドを設定します

### 通知を一時的に無効にするには？

通知の一時無効化:
- **マナーモード**: システムの DND モードを有効にします
- **ブラウザ設定**: サイトの通知を一時的に無効にします
- **OneUptime ダッシュボード**: 設定で通知を一時停止します
- **フォーカスモード**: OS のフォーカス/集中モードを使用します

## セキュリティ FAQ

### OneUptime PWA は安全ですか？

セキュリティ機能と考慮事項:
- **HTTPS 暗号化**: すべてのデータが安全に転送されます
- **同一オリジンポリシー**: ブラウザのセキュリティ制限が適用されます
- **サンドボックス環境**: ブラウザのセキュリティサンドボックス内で動作します
- **定期的な更新**: セキュリティパッチが自動的に適用されます
- **ルートアクセスなし**: ネイティブアプリと比較してシステムアクセスが制限されています


*注意: 機密データは暗号化されており、ブラウザのセキュリティ標準に従います。*

### 企業ネットワークで OneUptime PWA を使用できますか？

企業ネットワークの考慮事項:
- **ファイアウォールルール**: HTTPS（ポート 443）へのアクセスを確保します
- **プロキシ設定**: ブラウザのプロキシ設定を構成します
- **証明書の信頼**: 必要に応じて企業証明書をインストールします
- **VPN アクセス**: リモートアクセスには VPN を使用します
- **セキュリティポリシー**: IT セキュリティ要件に準拠します

## トラブルシューティング

### インストールの問題

**問題**: インストールボタンが表示されない
```
解決策:
1. OneUptime ページで 30 秒以上待ちます
2. ページを更新して再度待ちます
3. ブラウザのキャッシュとクッキーをクリアします
4. 別のブラウザを試します（Chrome/Edge 推奨）
5. HTTPS 接続を確認します（鍵アイコンを確認）
6. 既にインストール済みかどうか確認します
```

**問題**: インストールが失敗またはクラッシュする
```
解決策:
1. 十分なストレージ容量を確保します（100MB 以上）
2. 他のブラウザタブとアプリケーションを閉じます
3. ブラウザを最新バージョンに更新します
4. ブラウザ拡張機能を一時的に無効にします
5. プライベート/シークレットモードでインストールを試みます
6. ブラウザを再起動して再試行します
```

**問題**: アプリがインストールされるが表示されない
```
解決策:
1. すべてのアプリランチャーの場所を確認します
2. デバイス検索で「OneUptime」を検索します
3. ブラウザのアプリ管理セクションを確認します
4. システムが更新されるまで 1〜2 分待ちます
5. デバイスを再起動して再確認します
```

**問題**: アプリが頻繁にクラッシュする
```
解決策:
1. ブラウザを最新バージョンに更新します
2. OneUptime のすべてのブラウザデータをクリアします
3. ブラウザ拡張機能を無効にします
4. 利用可能なストレージ容量を確認します
5. オペレーティングシステムを再起動します
6. OneUptime PWA を再インストールします
```

**問題**: プッシュ通知が機能しない
```
解決策:
1. ブラウザの通知権限を確認します
2. システムの通知設定を確認します
3. まずシンプルな通知でテストします
4. 通知データをクリアして権限を再付与します
5. マナーモード/フォーカスモードの設定を確認します
6. OneUptime の通知設定を確認します
```

**問題**: アプリが最新データを同期しない
```
解決策:
1. 下にスワイプして更新します（モバイル）
2. Ctrl+F5（Windows/Linux）または Cmd+R（Mac）を押します
3. アプリを閉じて再度開きます
4. アプリキャッシュをクリアして再読み込みします
5. ネットワーク接続を確認します
```

### プラットフォーム別の問題

**Android の問題:**
```
問題: アプリがアプリドロワーに表示されない
解決策: 「最近追加したもの」アプリセクションを確認し、アプリドロワーで検索します

問題: 通知が遅延する
解決策: ブラウザアプリのバッテリー最適化を無効にします

問題: 起動時にアプリがクラッシュする
解決策: Chrome アプリデータをクリアし、デバイスを再起動します
```

**iOS の問題:**
```
問題: ホーム画面に追加できない
解決策: Safari ブラウザを使用し、iOS 11.3+ であることを確認します

問題: アプリアイコンが見つからない
解決策: すべてのホーム画面ページとアプリライブラリを確認します

問題: Face ID が機能しない
解決策: 設定で Safari の Face ID を有効にします
```

**Windows の問題:**
```
問題: アプリがスタートメニューに表示されない
解決策: アプリ名を検索し、インストール済みアプリリストを確認します

問題: 通知が表示されない
解決策: Windows の通知設定を確認し、ブラウザで有効にします

問題: ウィンドウのサイズ調整の問題
解決策: 手動でサイズ変更します。アプリがサイズを記憶します
```

**macOS の問題:**
```
問題: Safari からインストールできない
解決策: macOS Sonoma 以降に更新し、ファイル → Dock に追加 を使用します

問題: アプリがアプリケーションフォルダーにない
解決策: Launchpad を確認し、Spotlight 検索を使用します

問題: 通知が機能しない
解決策: システム環境設定 → 通知 を確認します
```

**Linux の問題:**
```
問題: PWA インストールオプションが見つからない
解決策: Chrome/Chromium を使用し、デスクトップ環境のサポートを確認します

問題: ランチャーにアイコンが表示されない
解決策: デスクトップデータベースを更新し、.desktop ファイルを確認します

問題: オーディオ通知が機能しない
解決策: PulseAudio を確認し、ブラウザのオーディオ権限を確認します
```

### エラーメッセージ

**「このサイトはインストールできません」**
```
原因:
- OneUptime インスタンスが PWA 要件を満たしていない
- ウェブアプリマニフェストが見つからないか無効
- HTTPS が正しく設定されていない
- ブラウザが PWA インストールをサポートしていない

解決策:
- 管理者に PWA セットアップの確認を依頼します
- 別のブラウザを試します
- 詳細なエラーについてはブラウザコンソールを確認します
```
