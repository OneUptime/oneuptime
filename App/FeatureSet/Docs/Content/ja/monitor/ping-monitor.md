# Pingモニター

Pingモニタリングを使用すると、任意のホストまたはIPアドレスの可用性と応答性を監視できます。OneUptimeは定期的にターゲットにPingリクエストを送信し、正しく応答するかを確認します。

## 概要

PingモニターはICMP Pingリクエストをホストに送信することで、基本的なネットワーク接続をテストします。これにより、以下のことが可能になります。

- ホストの稼働時間と可用性の監視
- ネットワークレイテンシーとレスポンスタイムの追跡
- サービスへの影響が出る前の接続問題の検出
- サーバーやネットワーク機器への到達可能性の確認

## Pingモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **Ping** を選択します
4. 監視するホスト名またはIPアドレスを入力します
5. 必要に応じて監視条件を設定します

## 設定オプション

### Pingのホスト名またはIPアドレス

監視するターゲットのホスト名またはIPアドレスを入力します（例：`example.com` または `192.168.1.1`）。ホスト名とIPアドレスの両方を受け付けます。

## 監視条件

以下の条件に基づいて、ホストがオンライン、パフォーマンス低下、またはオフラインと判定されるタイミングを設定できます。

### 利用可能なチェックタイプ

| チェックタイプ             | 説明                                |
| -------------------------- | ----------------------------------- |
| オンラインか               | ホストがPingリクエストに応答するか  |
| レスポンスタイム（ミリ秒） | Pingリクエストの往復時間（ミリ秒）  |
| リクエストタイムアウトか   | PingリクエストがタイムアウトしたかZ |

### フィルタータイプ

**オンラインか** および **リクエストタイムアウトか** の場合：

- **True** — 条件が真
- **False** — 条件が偽

**レスポンスタイム** の場合：

- **より大きい** — レスポンスタイムがしきい値を超える
- **より小さい** — レスポンスタイムがしきい値を下回る
- **以上** — レスポンスタイムがしきい値以上
- **以下** — レスポンスタイムがしきい値以下
- **等しい** — レスポンスタイムが完全一致
- **等しくない** — レスポンスタイムが一致しない
- **時間経過で評価** — 時間ウィンドウで集計（平均、合計、最大、最小、すべての値、いずれかの値）を使って評価

### 条件の例

#### ホストに到達できない場合にオフラインとしてマークする

- **チェック対象**：オンラインか
- **フィルタータイプ**：False

#### レスポンスタイムが200ミリ秒を超えた場合にアラートを発する

- **チェック対象**：レスポンスタイム（ミリ秒）
- **フィルタータイプ**：より大きい
- **値**：200
