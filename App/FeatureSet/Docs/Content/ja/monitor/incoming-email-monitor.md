# 受信メールモニター

受信メールモニターを使用すると、モニター固有のメールアドレスに送信されたメールに基づいてアラートの作成と解決ができます。これはレガシーシステム、サードパーティのアラートツール、またはメール通知を送信できる任意のサービスとの統合に役立ちます。

## 仕組み

1. 受信メールモニターを作成すると、OneUptimeはそのモニター用の一意のメールアドレスを生成します
2. そのアドレスに送信されたメールが受信・評価されます
3. 設定した条件に基づいて、OneUptimeは新しいアラートを作成したり、既存のアラートを解決したりします

これは、メールベースのアラートシステムをOneUptimeのインシデント管理ワークフローと統合する強力な方法です。

## 受信メールモニターの作成

1. OneUptime ダッシュボードで **モニター** に移動します
2. **モニターの作成** をクリックします
3. モニタータイプとして **受信メール** を選択します
4. モニターの設定を構成します：
   - **名前：** モニターのわかりやすい名前
   - **説明：** このモニターの目的
5. **アラート作成条件**（アラートを作成する条件）を設定します
6. **アラート解決条件**（アラートを解決する条件）を設定します
7. **作成** をクリックします

作成後、モニターの詳細ページにこのモニター用の一意のメールアドレスが表示されます。

## メールアドレスの形式

各受信メールモニターは以下の形式で一意のメールアドレスを取得します。

```
monitor-{secret-key}@{inbound-domain}
```

例：`monitor-abc123def456@inbound.yourdomain.com`

このアドレスはモニターの詳細ページからコピーでき、外部システムがそこにメールを送信するように設定できます。

## 利用可能な条件フィールド

以下のメールフィールドに基づいて条件を作成できます。

| フィールド | 説明 |
|-------|-------------|
| **メール件名** | 受信メールの件名 |
| **メール送信元** | 送信者のメールアドレス |
| **メール本文** | メール本文のプレーンテキスト内容 |
| **メール宛先** | 受信者のメールアドレス |
| **メール受信日時** | メールが受信されたタイミングに基づく条件 |

## 利用可能なフィルタータイプ

### 文字列フィルター（件名、送信元、本文、宛先）

| フィルター | 説明 | 例 |
|--------|-------------|---------|
| **含む** | フィールドに指定テキストが含まれる | 件名に「CRITICAL」を含む |
| **含まない** | フィールドに指定テキストが含まれない | 件名に「TEST」を含まない |
| **等しい** | フィールドが指定テキストと完全一致 | 送信元が「alerts@service.com」と等しい |
| **等しくない** | フィールドが指定テキストと一致しない | 件名が「OK」と等しくない |
| **始まる** | フィールドが指定テキストで始まる | 件名が「[ALERT]」で始まる |
| **終わる** | フィールドが指定テキストで終わる | 件名が「- Production」で終わる |
| **空白** | フィールドが空または空白 | 本文が空白 |
| **空白でない** | フィールドに内容がある | 件名が空白でない |

### 時間ベースのフィルター（メール受信日時）

| フィルター | 説明 | 例 |
|--------|-------------|---------|
| **X分以内に受信** | X分以内にメールが受信された | メールを30分以内に受信 |
| **X分以内に未受信** | X分以内にメールが受信されなかった | メールを60分以内に受信しなかった |

## 設定例

### 例1：重要なメールでアラートを作成

**アラート作成条件：**
- メール件名が **「CRITICAL」を含む**
- または メール件名が **「ALERT」を含む**
- または メール件名が **「ERROR」を含む**

**アラート解決条件：**
- メール件名が **「RESOLVED」を含む**
- または メール件名が **「OK」を含む**
- または メール件名が **「RECOVERED」を含む**

### 例2：特定の送信者を監視

**アラート作成条件：**
- メール送信元が **「monitoring@legacy-system.com」と等しい**
- かつ メール件名が **「Failed」を含む**

**アラート解決条件：**
- メール送信元が **「monitoring@legacy-system.com」と等しい**
- かつ メール件名が **「Success」を含む**

### 例3：ハートビートモニター（メールなし = アラート）

**アラート作成条件：**
- メール受信日時が 値 `60` で **X分以内に未受信**

これにより60分間メールが受信されなかった場合にアラートが作成されます。定期的な完了メールを送信するバッチプロセスやスケジュールジョブの監視に役立ちます。

**アラート解決条件：**
- メール受信日時が 値 `5` で **X分以内に受信**

これにより、メールが受信されたときにアラートが解決されます。

## ユースケース

### レガシーシステムの統合

多くの旧システムはメールベースのアラートのみをサポートしています。受信メールモニターを使用して：
- メールアラートをOneUptimeのインシデントに変換する
- 回復メールが届いたときにインシデントを自動的に解決する
- 複数のレガシーシステムのアラートを一元管理する

### サードパーティサービスの監視

メール通知を送信するサービスとの統合：
- クラウドプロバイダーのアラート（AWS、GCP、Azure）
- セキュリティスキャンツール
- バックアップ完了通知
- SSL証明書の有効期限警告

### スケジュールジョブの監視

バッチジョブとスケジュールタスクを監視：
- 完了メールが時間内に受信されなかった場合のアラート作成
- エラー通知メールを通じたジョブ失敗の追跡
- データパイプラインの完了監視

### マルチベンダーアラートの集約

複数の監視ツールからのアラートを集約：
- Nagios、Zabbixなどのツールからメール経由でアラートを受信する
- インシデント管理をOneUptimeに統一する
- すべてのアラートの単一の信頼できる情報源を維持する

## テンプレート変数

インシデントテンプレートを設定する際、受信メールからこれらの変数を使用できます。

| 変数 | 説明 |
|----------|-------------|
| `{{emailSubject}}` | 受信メールの件名 |
| `{{emailFrom}}` | 送信者のメールアドレス |
| `{{emailTo}}` | 受信者のメールアドレス |
| `{{emailBody}}` | メールのプレーンテキスト本文 |
| `{{emailReceivedAt}}` | メールが受信された日時 |

## モニターサマリービュー

モニターのサマリーには以下が表示されます。
- **最終メール受信日時：** 最新メールが受信された日時
- **送信元：** 最後のメールの送信者
- **件名：** 最後のメールの件名
- **メールヘッダー：** 最後のメールの完全なヘッダー（展開可能）
- **メール本文：** 最後のメールの内容（展開可能）

## セルフホストのセットアップ

OneUptimeをセルフホストしている場合、インバウンドメールプロバイダーを設定する必要があります。現在サポートされているプロバイダー：

- **SendGrid Inbound Parse** — セットアップ手順については [SendGridインバウンドメール統合](/docs/self-hosted/sendgrid-inbound-email) を参照してください

## 注意事項

- **メールアドレスのセキュリティ：** モニターのメールアドレスにはシークレットキーが含まれています。パスワードと同様に扱い、公開しないでください。
- **メールサイズ：** 非常に大きなメール（大きな添付ファイルを含む）はメールプロバイダーによって切り捨てられたり拒否されたりする場合があります。
- **処理時間：** メールは非同期で処理されます。メール送信からアラート作成まで数秒かかる場合があります。
- **大文字小文字の区別なし：** すべての文字列比較（含む、等しいなど）は大文字小文字を区別しません。
- **プレーンテキスト：** メール本文の条件はメールのプレーンテキスト版を使用します。HTMLの書式は除去されます。

## トラブルシューティング

### メールが受信されない場合

1. メールアドレスが正しいか確認します（タイポがないか）
2. メールがスパムフィルターにブロックされていないか確認します
3. インバウンドメールプロバイダーが正しく設定されているか確認します
4. OneUptimeのログにエラーメッセージがないか確認します

### アラートが作成されない場合

1. 条件がメール内容と一致しているか確認します
2. モニターが無効になっていないか確認します
3. モニター詳細の評価ログを確認します
4. パターンマッチングを使用する前に完全一致の文字列でテストします

### アラートが解決されない場合

1. 解決条件が回復メールと一致しているか確認します
2. 解決すべきアクティブなアラートが存在することを確認します
3. 解決メールが同じモニターアドレスに送信されているか確認します
