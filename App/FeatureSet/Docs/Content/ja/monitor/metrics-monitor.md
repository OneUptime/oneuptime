# メトリクスモニター

メトリクスモニタリングを使用すると、OpenTelemetryを介して収集されたカスタムアプリケーションおよびインフラストラクチャのメトリクスを監視できます。OneUptimeは時間ウィンドウでメトリクス値を評価し、設定した条件に基づいてアラートをトリガーします。

## 概要

メトリクスモニターはテレメトリーサービスから数値メトリクスを照会・評価します。これにより、以下のことが可能になります。

- カスタムアプリケーションメトリクス（リクエスト率、キューの深さ、エラー率など）の監視
- インフラストラクチャメトリクス（CPU、メモリ、ディスク、ネットワーク）の追跡
- フィルターと集計を使った複雑なメトリクスクエリの作成
- 数式を使った複数のメトリクスの組み合わせ
- メトリクスのしきい値に基づくアラートの設定

## メトリクスモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **メトリクス** を選択します
4. メトリクスクエリとオプションの数式を設定します
5. 集計戦略を選択します
6. 必要に応じて監視条件を設定します

## 設定オプション

### メトリクスクエリ

1つ以上のメトリクスクエリを定義します。各クエリには以下が含まれます。

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| メトリクス名 | 照会するメトリクスの名前 | はい |
| 集計タイプ | 生のメトリクス値の集計方法（合計、平均、最小、最大、件数） | はい |
| 属性 | メトリクスデータを絞り込むキーと値のフィルター | いいえ |
| グループ化 | メトリクスをグループ化するディメンション | いいえ |

各クエリには数式で使用するエイリアス（例：`a`、`b`、`c`）が割り当てられます。

### 数式

数学的な式を使って複数のメトリクスクエリを組み合わせます。例：

- `a / b * 100` — 2つのクエリからパーセンテージを計算
- `a + b` — 2つのメトリクスの合計
- `a - b` — メトリクスの差分

### ローリング時間ウィンドウ

メトリクス評価の時間ウィンドウを選択します。

- 過去1分
- 過去5分
- 過去10分
- 過去15分
- 過去30分
- 過去60分

### 集計戦略

評価するメトリクス値の集計方法を選択します。

| 戦略 | 説明 |
|----------|-------------|
| 平均 | 時間ウィンドウ内の平均値 |
| 合計 | すべての値の合計 |
| 最大値 | 時間ウィンドウ内の最大値 |
| 最小値 | 時間ウィンドウ内の最小値 |
| すべての値 | すべての値が条件を満たす必要がある |
| いずれかの値 | 少なくとも1つの値が条件を満たす |

## 監視条件

### 利用可能なチェックタイプ

| チェックタイプ | 説明 |
|------------|-------------|
| メトリクス値 | 設定したメトリクスクエリまたは数式の集計値 |

### フィルタータイプ

- **より大きい** — メトリクス値がしきい値を超える
- **より小さい** — メトリクス値がしきい値を下回る
- **以上** — メトリクス値がしきい値以上
- **以下** — メトリクス値がしきい値以下
- **等しい** — メトリクス値が完全一致
- **等しくない** — メトリクス値が一致しない

### 条件の例

#### エラー率が5%を超えた場合にアラートを発する

- **クエリa**：`http_requests_total`（`status=5xx` でフィルタリング）
- **クエリb**：`http_requests_total`
- **数式**：`a / b * 100`
- **チェック対象**：メトリクス値
- **フィルタータイプ**：より大きい
- **値**：5

#### リクエストキューの深さが高い場合にアラートを発する

- **クエリ**：`request_queue_size`（集計：最大値）
- **チェック対象**：メトリクス値
- **フィルタータイプ**：より大きい
- **値**：1000

## セットアップ要件

メトリクスモニタリングには、OpenTelemetryを介してOneUptimeにメトリクスを送信するアプリケーションまたはインフラストラクチャが必要です。セットアップ手順については、[OpenTelemetry](/docs/telemetry/open-telemetry)のドキュメントを参照してください。
