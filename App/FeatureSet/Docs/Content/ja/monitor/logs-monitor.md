# ログモニター

ログモニタリングを使用すると、アプリケーションのログを監視し、ログパターン、件数、重大度レベルに基づいてアラートをトリガーできます。OneUptimeはテレメトリーサービスからのログを時間ウィンドウで評価し、設定した条件と照合します。

## 概要

ログモニターは特定のフィルターに一致するログを検索・カウントします。これにより、以下のことが可能になります。

- エラーログの急増に関するアラート
- 特定のログパターンやメッセージの監視
- 重大度レベルごとのログ量の追跡
- サービス、属性、内容によるログのフィルタリング
- ログパターンからアプリケーションの問題を検出

## ログモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **ログ** を選択します
4. 監視するテレメトリーサービスを選択します
5. 必要に応じてログフィルターと条件を設定します

## 設定オプション

### テレメトリーサービス

ログを監視する1つ以上のサービスを選択します。サービスはOpenTelemetryを介してOneUptimeにログを送信している必要があります。

### ログフィルター

| フィルター     | 説明                                                  | 必須   |
| -------------- | ----------------------------------------------------- | ------ |
| 重大度レベル   | ログの重大度でフィルタリング（ERROR、WARN、INFOなど） | いいえ |
| 本文           | ログメッセージ本文内のテキスト検索                    | いいえ |
| 属性           | カスタムログ属性でフィルタリングするキーと値のペア    | いいえ |
| 時間ウィンドウ | ログを検索する過去の期間（秒単位、デフォルト：60）    | いいえ |

### 重大度レベル

1つ以上の重大度レベルでログをフィルタリングします。

- **FATAL** / **EMERGENCY** / **CRITICAL**
- **ERROR**
- **WARN** / **WARNING**
- **INFO** / **INFORMATIONAL**
- **DEBUG**
- **TRACE**
- **UNSPECIFIED**

## 監視条件

### 利用可能なチェックタイプ

| チェックタイプ | 説明                                             |
| -------------- | ------------------------------------------------ |
| ログ件数       | 時間ウィンドウ内でフィルターに一致するログの件数 |

### フィルタータイプ

- **より大きい** — ログ件数がしきい値を超える
- **より小さい** — ログ件数がしきい値を下回る
- **以上** — ログ件数がしきい値以上
- **以下** — ログ件数がしきい値以下
- **等しい** — ログ件数が完全一致
- **等しくない** — ログ件数が一致しない

### 条件の例

#### 60秒間で100件以上のエラーログが発生した場合にアラートを発する

- **重大度レベル**：ERROR
- **時間ウィンドウ**：60秒
- **チェック対象**：ログ件数
- **フィルタータイプ**：より大きい
- **値**：100

#### FATALログが1件でも発生した場合にアラートを発する

- **重大度レベル**：FATAL
- **時間ウィンドウ**：60秒
- **チェック対象**：ログ件数
- **フィルタータイプ**：より大きい
- **値**：0

#### 特定のエラーメッセージを含むログを監視する

- **本文**：`database connection timeout`
- **時間ウィンドウ**：300秒
- **チェック対象**：ログ件数
- **フィルタータイプ**：より大きい
- **値**：5

## セットアップ要件

ログモニタリングには、OpenTelemetryを介してOneUptimeにログを送信するアプリケーションが必要です。セットアップ手順については、[OpenTelemetry](/docs/telemetry/open-telemetry)のドキュメントを参照してください。
