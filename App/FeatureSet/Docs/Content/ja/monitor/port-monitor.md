# ポートモニター

ポートモニタリングを使用すると、ホスト上の特定のTCPまたはUDPポートの可用性を監視できます。OneUptimeは定期的に指定されたポートへの接続を試み、開いていて応答しているかを確認します。

## 概要

ポートモニターは特定のネットワークポートが接続を受け入れているかをテストします。これにより、以下のことが可能になります。

- 特定のポートでのサービス可用性の監視
- ポートのレスポンスタイムの追跡
- データベース、メールサーバー、アプリケーションサーバーなどのサービスが稼働していることの確認
- ユーザーへの影響が出る前のサービス障害の検出

## ポートモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **ポート** を選択します
4. ホスト名またはIPアドレスとポート番号を入力します
5. 必要に応じて監視条件を設定します

## 設定オプション

### ホスト名またはIPアドレス

ターゲットホストのホスト名またはIPアドレスを入力します（例：`example.com` または `192.168.1.1`）。

### ポート

監視するポート番号を入力します（1〜65535）。一般的な例：

| ポート | サービス   |
| ------ | ---------- |
| 22     | SSH        |
| 25     | SMTP       |
| 80     | HTTP       |
| 443    | HTTPS      |
| 3306   | MySQL      |
| 5432   | PostgreSQL |
| 6379   | Redis      |
| 27017  | MongoDB    |

## 監視条件

以下の条件に基づいて、ポートがオンライン、パフォーマンス低下、またはオフラインと判定されるタイミングを設定できます。

### 利用可能なチェックタイプ

| チェックタイプ             | 説明                                     |
| -------------------------- | ---------------------------------------- |
| オンラインか               | ポートが開いていて接続を受け入れているか |
| レスポンスタイム（ミリ秒） | 接続確立までの時間（ミリ秒）             |
| リクエストタイムアウトか   | 接続試行がタイムアウトしたか             |

### フィルタータイプ

**オンラインか** および **リクエストタイムアウトか** の場合：

- **True** — 条件が真
- **False** — 条件が偽

**レスポンスタイム** の場合：

- **より大きい** — レスポンスタイムがしきい値を超える
- **より小さい** — レスポンスタイムがしきい値を下回る
- **以上** — レスポンスタイムがしきい値以上
- **以下** — レスポンスタイムがしきい値以下
- **等しい** — レスポンスタイムが完全一致
- **等しくない** — レスポンスタイムが一致しない
- **時間経過で評価** — 時間ウィンドウで集計（平均、合計、最大、最小、すべての値、いずれかの値）を使って評価

### 条件の例

#### ポートが閉じている場合にオフラインとしてマークする

- **チェック対象**：オンラインか
- **フィルタータイプ**：False

#### 接続時間が500ミリ秒を超えた場合にアラートを発する

- **チェック対象**：レスポンスタイム（ミリ秒）
- **フィルタータイプ**：より大きい
- **値**：500

#### 接続が遅い場合にパフォーマンス低下としてマークする

- **チェック対象**：レスポンスタイム（ミリ秒）
- **フィルタータイプ**：より大きい
- **値**：200
