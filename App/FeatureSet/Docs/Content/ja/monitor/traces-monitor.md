# トレースモニター

トレースモニタリングを使用すると、アプリケーションの分散トレースを監視し、スパンのパターン、件数、ステータスに基づいてアラートをトリガーできます。OneUptimeはテレメトリーサービスからのトレースデータを時間ウィンドウで評価します。

## 概要

トレースモニターは特定のフィルターに一致するスパンを検索・カウントします。これにより、以下のことが可能になります。

- サービス内のエラースパンの急増に関するアラート
- 特定の操作やエンドポイントの監視
- スパンの量とパターンの追跡
- スパンのステータス、名前、カスタム属性によるフィルタリング
- トレースデータからパフォーマンスと信頼性の問題を検出

## トレースモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **トレース** を選択します
4. 監視するテレメトリーサービスを選択します
5. 必要に応じてスパンフィルターと条件を設定します

## 設定オプション

### テレメトリーサービス

トレースを監視する1つ以上のサービスを選択します。サービスはOpenTelemetryを介してOneUptimeにトレースを送信している必要があります。

### スパンフィルター

| フィルター       | 説明                                                       | 必須   |
| ---------------- | ---------------------------------------------------------- | ------ |
| スパンステータス | スパンステータスコードでフィルタリング（OK、ERROR、UNSET） | いいえ |
| スパン名         | 特定のスパン名のテキスト検索（操作やエンドポイント名など） | いいえ |
| 属性             | カスタムスパン属性でフィルタリングするキーと値のペア       | いいえ |
| 時間ウィンドウ   | スパンを検索する過去の期間（秒単位、デフォルト：60）       | いいえ |

### スパンステータスコード

- **OK** — 操作が正常に完了した
- **ERROR** — 操作でエラーが発生した
- **UNSET** — ステータスが明示的に設定されていない

## 監視条件

### 利用可能なチェックタイプ

| チェックタイプ | 説明                                               |
| -------------- | -------------------------------------------------- |
| スパン件数     | 時間ウィンドウ内でフィルターに一致するスパンの件数 |

### フィルタータイプ

- **より大きい** — スパン件数がしきい値を超える
- **より小さい** — スパン件数がしきい値を下回る
- **以上** — スパン件数がしきい値以上
- **以下** — スパン件数がしきい値以下
- **等しい** — スパン件数が完全一致
- **等しくない** — スパン件数が一致しない

### 条件の例

#### 60秒間で50件以上のエラースパンが発生した場合にアラートを発する

- **スパンステータス**：ERROR
- **時間ウィンドウ**：60秒
- **チェック対象**：スパン件数
- **フィルタータイプ**：より大きい
- **値**：50

#### 特定のエンドポイントでエラーが発生した場合にアラートを発する

- **スパン名**：`POST /api/checkout`
- **スパンステータス**：ERROR
- **時間ウィンドウ**：120秒
- **チェック対象**：スパン件数
- **フィルタータイプ**：より大きい
- **値**：0

## セットアップ要件

トレースモニタリングには、OpenTelemetryを介してOneUptimeに分散トレースを送信するアプリケーションが必要です。セットアップ手順については、[OpenTelemetry](/docs/telemetry/open-telemetry)のドキュメントを参照してください。
