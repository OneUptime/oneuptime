# SNMPモニター

SNMP（Simple Network Management Protocol）モニタリングを使用すると、SNMP OID（オブジェクト識別子）を照会することで、スイッチ、ルーター、ファイアウォール、その他のネットワーク機器などのネットワークデバイスを監視できます。

## 概要

SNMPモニターはOIDを使用してネットワークデバイスから特定の管理情報を照会します。これにより、以下のことが可能になります。

- デバイスの可用性と正常性の監視
- インターフェース統計（トラフィック、エラー、ステータス）の追跡
- システムメトリクス（CPU、メモリ、稼働時間）の監視
- カスタムベンダー固有のOIDの確認
- OID値に基づくアラートの設定

## SNMPモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **SNMP** を選択します
4. 以下に説明するSNMP設定を構成します

## 設定オプション

### 基本設定

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| SNMPバージョン | プロトコルバージョン：v1、v2c、またはv3 | はい |
| ホスト名/IP | SNMPデバイスのホスト名またはIPアドレス | はい |
| ポート | SNMPポート（デフォルト：161） | はい |

### 認証

#### SNMP v1/v2c

SNMP v1とv2cの場合、コミュニティ文字列のみ必要です。

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| コミュニティ文字列 | SNMPコミュニティ文字列（例：「public」） | はい |

#### SNMP v3

SNMPv3は認証と暗号化によって強化されたセキュリティを提供します。

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| セキュリティレベル | noAuthNoPriv、authNoPriv、またはauthPriv | はい |
| ユーザー名 | SNMPv3ユーザー名 | はい |
| 認証プロトコル | MD5、SHA、SHA256、またはSHA512 | authNoPrivまたはauthPrivの場合 |
| 認証キー | 認証パスワード | authNoPrivまたはauthPrivの場合 |
| プライバシープロトコル | DES、AES、またはAES256 | authPrivの場合 |
| プライバシーキー | プライバシー/暗号化パスワード | authPrivの場合 |

### 監視するOID

デバイスから照会するOIDを追加します。各OIDに対して、以下を指定できます。

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| OID | 数値OID（例：1.3.6.1.2.1.1.1.0） | はい |
| 名前 | OIDのフレンドリーネーム（例：sysDescr） | いいえ |
| 説明 | このOIDが何を表すかの説明 | いいえ |

### 一般的なOIDテンプレート

OneUptimeは一般的に監視されるOIDのテンプレートを提供しています。

#### システムMIB

| OID | 名前 | 説明 |
|-----|------|-------------|
| 1.3.6.1.2.1.1.1.0 | sysDescr | システムの説明 |
| 1.3.6.1.2.1.1.3.0 | sysUpTime | システムの稼働時間（ティック単位） |
| 1.3.6.1.2.1.1.5.0 | sysName | システム名 |
| 1.3.6.1.2.1.1.6.0 | sysLocation | システムの場所 |
| 1.3.6.1.2.1.1.4.0 | sysContact | システムの連絡先 |

#### インターフェースMIB

| OID | 名前 | 説明 |
|-----|------|-------------|
| 1.3.6.1.2.1.2.1.0 | ifNumber | ネットワークインターフェース数 |
| 1.3.6.1.2.1.2.2.1.8.X | ifOperStatus | インターフェースの動作ステータス（X = インターフェースインデックス） |
| 1.3.6.1.2.1.2.2.1.10.X | ifInOctets | 入力バイト数（X = インターフェースインデックス） |
| 1.3.6.1.2.1.2.2.1.16.X | ifOutOctets | 出力バイト数（X = インターフェースインデックス） |

#### ホストリソースMIB

| OID | 名前 | 説明 |
|-----|------|-------------|
| 1.3.6.1.2.1.25.1.1.0 | hrSystemUptime | ホストシステムの稼働時間 |
| 1.3.6.1.2.1.25.1.5.0 | hrSystemNumUsers | ユーザー数 |
| 1.3.6.1.2.1.25.1.6.0 | hrSystemProcesses | 実行中のプロセス数 |
| 1.3.6.1.2.1.25.3.3.1.2.X | hrProcessorLoad | CPU負荷（X = プロセッサインデックス） |

### 詳細設定

| フィールド | 説明 | デフォルト |
|-------|-------------|---------|
| タイムアウト | レスポンスを待つ時間（ミリ秒） | 5000 |
| リトライ回数 | 失敗時の再試行回数 | 3 |

## 監視条件

SNMPレスポンスをチェックしてアラートやインシデントをトリガーする条件を設定できます。

### 利用可能なチェックタイプ

| チェックタイプ | 説明 |
|------------|-------------|
| SNMPデバイスはオンラインか | デバイスがSNMPクエリに応答するか |
| SNMPレスポンスタイム | クエリのレスポンスタイム（ミリ秒） |
| SNMP OID値 | 特定のOIDが返した値 |
| SNMP OIDが存在するか | OIDが値を返すか（nullでないか） |

### 条件の例

#### デバイスがオンラインか確認する
- **チェック対象**：SNMPデバイスはオンラインか
- **フィルタータイプ**：True

#### レスポンスタイムがしきい値を超えた場合にアラートを発する
- **チェック対象**：SNMPレスポンスタイム（ミリ秒）
- **フィルタータイプ**：より大きい
- **値**：1000

#### インターフェースステータスの確認
- **チェック対象**：SNMP OID値
- **OID**：1.3.6.1.2.1.2.2.1.8.1
- **フィルタータイプ**：等しい
- **値**：1（1 = アップ、2 = ダウン）

#### CPU負荷のしきい値確認
- **チェック対象**：SNMP OID値
- **OID**：1.3.6.1.2.1.25.3.3.1.2.1
- **フィルタータイプ**：より大きい
- **値**：80

## モニターシークレットの使用

セキュリティのため、コミュニティ文字列やSNMPv3の認証情報などの機密情報をシークレットとして保存できます。

### シークレットの追加

1. **プロジェクト設定** -> **モニターシークレット** -> **モニターシークレットの作成** に移動します
2. シークレット（コミュニティ文字列やSNMPv3パスワードなど）を追加します
3. このシークレットにアクセスできるSNMPモニターを選択します

### SNMP設定でシークレットを使用する

機密フィールドで `{{monitorSecrets.SECRET_NAME}}` 構文を使用します。

- **コミュニティ文字列**：`{{monitorSecrets.SnmpCommunity}}`
- **SNMPv3認証キー**：`{{monitorSecrets.SnmpAuthKey}}`
- **SNMPv3プライバシーキー**：`{{monitorSecrets.SnmpPrivKey}}`

## アラート用テンプレート変数

インシデントやアラートのテンプレートを設定する際、以下の変数を使用できます。

| 変数 | 説明 |
|----------|-------------|
| `{{isOnline}}` | デバイスがオンラインか（true/false） |
| `{{responseTimeInMs}}` | クエリのレスポンスタイム（ミリ秒） |
| `{{failureCause}}` | クエリが失敗した場合のエラーメッセージ |
| `{{oidResponses}}` | OIDレスポンスオブジェクトの配列 |
| `{{OID_NAME}}` | 名前で特定のOID値にアクセス（例：`{{sysUpTime}}`） |

## トラブルシューティング

### 一般的な問題

#### デバイスが応答しない場合
- デバイスのIP/ホスト名が正しいか確認します
- デバイスでSNMPが有効になっているか確認します
- ファイアウォールルールがUDPポート161を許可しているか確認します
- コミュニティ文字列が正しいか確認します

#### 認証エラー（v3）
- ユーザー名、認証プロトコル、認証キーを確認します
- セキュリティレベルがデバイスの設定と一致しているか確認します
- authPrivレベルのプライバシープロトコルとキーが正しいか確認します

#### OIDが見つからない場合
- OIDがデバイスでサポートされているか確認します
- OIDに特定のMIBのロードが必要かどうか確認します
- snmpget/snmpwalkツールを使ってOIDを直接照会してみます

### SNMP接続のテスト

監視をセットアップする前に、コマンドラインツールを使ってSNMP接続をテストできます。

```bash
# SNMP v2c
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# SNMP v3（authPriv）
snmpget -v3 -u username -l authPriv -a SHA -A authpassword -x AES -X privpassword 192.168.1.1 1.3.6.1.2.1.1.1.0
```

## ベストプラクティス

1. **可能な限りSNMPv3を使用する** — 認証と暗号化によってより良いセキュリティを提供します
2. **認証情報をシークレットとして保存する** — コミュニティ文字列やパスワードをハードコードしないでください
3. **必要なOIDのみを監視する** — ネットワークのオーバーヘッドを削減するために、必要なものだけを照会します
4. **適切なタイムアウトを設定する** — ネットワークデバイスによってレスポンスタイムが異なる場合があります
5. **わかりやすいOID名を使用する** — アラートメッセージを理解しやすくします
6. **デプロイ前にテストする** — モニターを作成する前にSNMP接続を確認します
