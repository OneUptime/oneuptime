# IPモニター

IPモニタリングを使用すると、任意のIPv4またはIPv6アドレスの可用性と応答性を監視できます。OneUptimeは定期的にターゲットIPアドレスへの接続をテストし、その状態を報告します。

## 概要

IPモニターは特定のIPアドレスが到達可能で応答しているかを確認します。これにより、以下のことが可能になります。

- IPv4およびIPv6アドレスの可用性の監視
- レスポンスタイムとレイテンシーの追跡
- ネットワーク接続の問題の検出
- インフラストラクチャのエンドポイントへの到達可能性の確認

## IPモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **IP** を選択します
4. 監視するIPアドレスを入力します
5. 必要に応じて監視条件を設定します

## 設定オプション

### IPアドレス

監視するIPv4またはIPv6アドレスを入力します（例：`192.168.1.1` または `2001:db8::1`）。有効なIPアドレス形式である必要があります。

## 監視条件

以下の条件に基づいて、IPアドレスがオンライン、パフォーマンス低下、またはオフラインと判定されるタイミングを設定できます。

### 利用可能なチェックタイプ

| チェックタイプ             | 説明                           |
| -------------------------- | ------------------------------ |
| オンラインか               | IPアドレスに到達可能か         |
| レスポンスタイム（ミリ秒） | レスポンスタイム（ミリ秒）     |
| リクエストタイムアウトか   | リクエストがタイムアウトしたか |

### フィルタータイプ

**オンラインか** および **リクエストタイムアウトか** の場合：

- **True** — 条件が真
- **False** — 条件が偽

**レスポンスタイム** の場合：

- **より大きい** — レスポンスタイムがしきい値を超える
- **より小さい** — レスポンスタイムがしきい値を下回る
- **以上** — レスポンスタイムがしきい値以上
- **以下** — レスポンスタイムがしきい値以下
- **等しい** — レスポンスタイムが完全一致
- **等しくない** — レスポンスタイムが一致しない
- **時間経過で評価** — 時間ウィンドウで集計（平均、合計、最大、最小、すべての値、いずれかの値）を使って評価

### 条件の例

#### IPアドレスに到達できない場合にオフラインとしてマークする

- **チェック対象**：オンラインか
- **フィルタータイプ**：False

#### レイテンシーが100ミリ秒を超えた場合にアラートを発する

- **チェック対象**：レスポンスタイム（ミリ秒）
- **フィルタータイプ**：より大きい
- **値**：100
