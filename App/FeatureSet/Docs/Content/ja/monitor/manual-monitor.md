# 手動モニター

手動モニタリングを使用すると、ステータスを手動またはAPIで管理するモニターを作成できます。OneUptimeは自動チェックを実行しません。モニターのステータスは直接制御します。

## 概要

手動モニターは自分で更新するプレースホルダーです。以下のような場合に役立ちます。

- OneUptime APIを介してステータスを更新する外部監視ツールとの統合
- 自動的に監視できないサービスやシステムの追跡
- 自動ヘルスチェックなしのコンポーネントのインシデント管理
- 手動でステータスを追跡するサードパーティの依存関係の表示

## 手動モニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **手動** を選択します
4. モニターの名前と説明を入力します

## 仕組み

手動モニターには、監視間隔、プローブ、または自動条件評価がありません。モニターのステータスは変更するまでそのまま維持されます。

### ステータスの更新

手動モニターのステータスを更新するには2つの方法があります。

- **ダッシュボード** — OneUptime ダッシュボードから直接モニターのステータスを変更する
- **API** — OneUptime APIを使用してプログラムでモニターのステータスを更新する

### インシデントとアラート

他のモニタータイプと同様に、手動モニターに対してインシデントとアラートを作成できます。これにより、以下のことが可能になります。

- 外部で監視されているサービスのダウンタイムを追跡する
- 問題が報告されたときに手動でインシデントを作成する
- ステータスページで手動モニターを使用してユーザーにステータスを伝える

## 手動モニターの使いどころ

| ユースケース                       | 説明                                                                        |
| ---------------------------------- | --------------------------------------------------------------------------- |
| サードパーティサービス             | 依存しているが直接監視できない外部サービスのステータスを追跡する            |
| 物理インフラストラクチャ           | ネットワーク監視のないハードウェアや物理システムを表現する                  |
| ビジネスプロセス                   | サービスステータスに影響する非技術的なプロセスを追跡する                    |
| APIドリブンのステータス            | 外部ツールがOneUptime APIを介してモニターのステータスを更新できるようにする |
| ステータスページのプレースホルダー | OneUptime外部で管理されているコンポーネントをステータスページに表示する     |
