# SSL証明書モニター

SSL証明書モニタリングを使用すると、ウェブサイトやサービス上のSSL/TLS証明書の有効性と有効期限を監視できます。OneUptimeは定期的に証明書を確認し、有効期限が近づいたり問題が検出されたりした場合にアラートを発します。

## 概要

SSL証明書モニターはHTTPSエンドポイントに接続してSSL/TLS証明書を検査します。これにより、以下のことが可能になります。

- 証明書の有効期限日の監視
- 期限切れまたは間もなく期限切れになる証明書の検出
- 自己署名証明書の特定
- 証明書の有効性の確認
- 期限切れ証明書によるサービス障害の防止

## SSL証明書モニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **SSL証明書** を選択します
4. 確認するHTTPSエンドポイントのURLを入力します
5. 必要に応じて監視条件を設定します

## 設定オプション

### URL

監視するSSL証明書のHTTPSエンドポイントの完全なURLを入力します（例：`https://example.com` または `https://example.com:8443`）。

## 監視条件

以下の条件に基づいて、証明書のステータスがオンライン、パフォーマンス低下、またはオフラインと判定されるタイミングを設定できます。

### 利用可能なチェックタイプ

| チェックタイプ | 説明 |
|------------|-------------|
| オンラインか | サーバーに到達可能か |
| 有効な証明書か | 証明書が有効か（期限切れでなく、自己署名でない） |
| 自己署名証明書か | 証明書が自己署名であるか |
| 期限切れ証明書か | 証明書の有効期限が切れているか |
| 無効な証明書か | 証明書が無効であるか |
| 有効期限まで何時間か | 証明書が有効期限を迎えるまでの時間（時間単位） |
| 有効期限まで何日か | 証明書が有効期限を迎えるまでの日数 |
| リクエストタイムアウトか | 接続がタイムアウトしたか |

### フィルタータイプ

**オンラインか**、**有効な証明書か**、**自己署名証明書か**、**期限切れ証明書か**、**無効な証明書か**、**リクエストタイムアウトか** の場合：

- **True** — 条件が真
- **False** — 条件が偽

**有効期限まで何時間か** および **有効期限まで何日か** の場合：

- **より大きい** — 有効期限まで指定値以上の時間/日数がある
- **より小さい** — 有効期限まで指定値未満の時間/日数しかない
- **以上** — 有効期限まで指定値以上の時間/日数がある
- **以下** — 有効期限まで指定値以下の時間/日数しかない
- **等しい** — 有効期限まで指定値と完全一致
- **等しくない** — 有効期限まで指定値と一致しない

### 条件の例

#### 証明書が30日以内に有効期限を迎える場合にパフォーマンス低下としてマークする

- **チェック対象**：有効期限まで何日か
- **フィルタータイプ**：より小さい
- **値**：30

#### 証明書が期限切れの場合にオフラインとしてマークする

- **チェック対象**：期限切れ証明書か
- **フィルタータイプ**：True

#### 証明書が自己署名の場合にアラートを発する

- **チェック対象**：自己署名証明書か
- **フィルタータイプ**：True

#### 証明書が無効の場合にオフラインとしてマークする

- **チェック対象**：無効な証明書か
- **フィルタータイプ**：True

## ベストプラクティス

1. **複数のしきい値を設定する** — 有効期限の30日前にパフォーマンス低下ステータスを、7日前にオフラインステータスを設定して更新する時間を確保します
2. **すべてのエンドポイントを監視する** — 複数のドメインやサブドメインがある場合は、それぞれにモニターを作成します
3. **非標準ポートを含める** — 非標準ポートでHTTPSを実行しているサービスを忘れずに監視します
4. **更新後に監視する** — 証明書を更新した後、モニターが有効であることを確認します
