# DNSSECモニター

DNSSECモニタリングを使用すると、ゾーンのDNSレスポンスの暗号学的整合性を検証できます。OneUptimeは定期的に完全なDNSSEC検証を実行し、DNSKEYレコード、親ゾーンでのDS委任、RRSIG署名の有効性、リゾルバー間のAD flagの合意、および権威ネームサーバー間の一貫性を確認します。

## 概要

DNSSECモニターは、ルートゾーンからお使いのドメインまでの信頼チェーン全体を検証します。これにより、以下のことが可能になります。

- リゾルバーがユーザーにSERVFAILを返し始める前に、DNSSECチェーンの破損を検出
- ゾーン署名鍵の有効期限切れ前に警告を受け取る
- DSレコードが親ゾーンに正しく公開されていることを確認
- 権威ネームサーバー間の不一致（プライマリ/セカンダリの同期ずれ）を検出
- 検証リゾルバーがゾーンに対して実際にAD flagを設定していることを確認

## DNSSECモニターの作成

1. OneUptime ダッシュボードで **モニター** を開きます
2. **モニターの作成** をクリックします
3. モニタータイプとして **DNSSEC** を選択します
4. 検証するゾーン（ドメイン）を入力します
5. 必要に応じてリゾルバーと監視条件を設定します

## 設定オプション

### 基本設定

| フィールド | 説明 | 必須 |
|-------|-------------|----------|
| ゾーン（ドメイン名） | DNSSECで検証するゾーン（例：`example.com`） | はい |
| リゾルバー | 照会する検証リゾルバーのカンマ区切りリスト（例：`1.1.1.1, 8.8.8.8, 9.9.9.9`） | はい |
| ネームサーバーの一貫性チェック | 各権威ネームサーバーに直接問い合わせ、同じSOAシリアルを返すことを確認 | いいえ |

### 詳細設定

| フィールド | 説明 | デフォルト |
|-------|-------------|---------|
| 署名有効期限警告（日数） | RRSIG有効期限フィルターのデフォルトしきい値 | 7 |
| タイムアウト（ミリ秒） | 各DNSクエリの待機時間 | 10000 |
| リトライ回数 | 失敗時の再試行回数 | 3 |

## 監視条件

以下の条件に基づいて、ゾーンがオンライン、パフォーマンス低下、またはオフラインと判定されるタイミングを設定できます。

### 利用可能なチェックタイプ

| チェックタイプ | 説明 |
|------------|-------------|
| DNSSECチェーンは有効か | 検証チェーン全体（ルート → TLD → ゾーン）が正しく解決される |
| DNSSEC DNSKEYレコードが存在するか | ゾーンが少なくとも1つのDNSKEYレコードを公開している |
| 親でDNSSEC DSレコードが存在するか | 親ゾーンがゾーンのKSKに一致するDSレコードを公開している |
| DNSSEC署名の残り有効日数 | 最も早く期限切れになるRRSIG署名までの日数 |
| DNSSECリゾルバー合意（AD Flag） | 照会したすべてのリゾルバーがAD（Authenticated Data）flagを返す |
| DNSSECネームサーバーは一貫しているか | すべての権威ネームサーバーが同じSOAシリアルを返す |
| DNSSECは有効か | すべての検証チェックを通した総合的な合否 |

### フィルタータイプ

**DNSSECチェーンは有効か**、**DNSSEC DNSKEYレコードが存在するか**、**親でDNSSEC DSレコードが存在するか**、**DNSSECリゾルバー合意（AD Flag）**、**DNSSECネームサーバーは一貫しているか**、**DNSSECは有効か** の場合：

- **True** — 条件が真
- **False** — 条件が偽

**DNSSEC署名の残り有効日数** の場合：

- **より大きい**、**より小さい**、**以上**、**以下**、**等しい**、**等しくない**

### 条件の例

#### DNSSECチェーンが壊れている場合にアラートを発する

- **チェック対象**：DNSSECチェーンは有効か
- **フィルタータイプ**：False

#### 署名の有効期限切れ前に警告する

- **チェック対象**：DNSSEC署名の残り有効日数
- **フィルタータイプ**：より小さい
- **値**：7

#### 親でのDSの欠落（委任の破損）を検出する

- **チェック対象**：親でDNSSEC DSレコードが存在するか
- **フィルタータイプ**：False

#### リゾルバー間の不一致を検出する

- **チェック対象**：DNSSECリゾルバー合意（AD Flag）
- **フィルタータイプ**：False

#### ネームサーバー間のデータ不一致を検出する

- **チェック対象**：DNSSECネームサーバーは一貫しているか
- **フィルタータイプ**：False

## ベストプラクティス

1. **複数のパブリックリゾルバーを使用する** — `1.1.1.1`、`8.8.8.8`、`9.9.9.9` をデフォルトで使用し、単一リゾルバーの障害で誤検知が発生しないようにする
2. **有効期限の十分前に警告する** — 署名有効期限の7日前にパフォーマンス低下アラート、2日前にオフラインアラートを設定する。鍵のロールオーバーは静かに失敗することがあります
3. **署名済みのすべてのゾーンを監視する** — Apexドメイン、署名済みサブドメイン、別の運用者に委任されたすべてのゾーンを含める
4. **ネームサーバーの一貫性チェックを有効にする** — DNSSEC検証だけでは見逃すプライマリ/セカンダリの同期問題を検出します（ネットワークが任意のIPへのアウトバウンドDNSをブロックしている場合を除く）
