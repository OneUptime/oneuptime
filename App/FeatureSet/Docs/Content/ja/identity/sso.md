# SSO（シングルサインオン）

OneUptime は、エンタープライズ認証のための SAML 2.0 ベースのシングルサインオン（SSO）をサポートしています。SSO により、チームメンバーが組織のアイデンティティプロバイダー（IdP）を使用して OneUptime にログインでき、集中アクセス管理とセキュリティの強化が実現します。

## 概要

SSO 統合は以下のメリットを提供します。

- **集中認証**: ユーザーが既存の会社の認証情報でログインできます
- **セキュリティの強化**: IdP の多要素認証とセキュリティポリシーを活用します
- **簡素化されたユーザー管理**: 既存の ID 管理システムからアクセスを管理します
- **パスワード疲労の軽減**: ユーザーが別の OneUptime パスワードを覚える必要がありません

## SSO のセットアップ

1. **プロジェクト設定に移動する**
   - OneUptime プロジェクトに移動します
   - **プロジェクト設定** > **認証** > **SSO** に移動します

2. **SSO 設定を作成する**
   - **SSO を作成** をクリックします
   - SSO 設定の **名前** を入力します（例: "Keycloak SAML" または "Okta SAML"）
   - アイデンティティプロバイダーの **サインオン URL** を入力します
   - アイデンティティプロバイダーの **発行者**（エンティティ ID）を入力します
   - アイデンティティプロバイダーの **公開証明書** を貼り付けます
   - **署名アルゴリズム** を選択します（例: `RSA-SHA-256`）
   - **ダイジェストアルゴリズム** を選択します（例: `SHA256`）

3. **OneUptime SSO メタデータを取得する**
   - 保存後、**SSO 設定を表示** ボタンをクリックします
   - **識別子（エンティティ ID）** をコピーします - IdP の設定に必要です
   - **返信 URL（アサーションコンシューマーサービス URL）** をコピーします - IdP の設定に必要です

## Keycloak SAML 設定

Keycloak は、人気のあるオープンソースの ID およびアクセス管理ソリューションです。OneUptime の SAML アイデンティティプロバイダーとして Keycloak を設定する手順を以下に示します。

### 前提条件

- 設定済みレルムを持つ稼働中の Keycloak インスタンス
- Keycloak と OneUptime の両方への管理者アクセス
- SSO サポートを持つ OneUptime アカウント

### ステップ 1: OneUptime SSO を設定する

1. OneUptime ダッシュボードにログインします
2. **プロジェクト設定** > **認証** > **SSO** に移動します
3. **SSO を作成** をクリックして以下を入力します。
   - **名前**: わかりやすい名前（例: `my-project-oneuptime`）
   - **サインオン URL**: `https://<your-keycloak-domain>/auth/realms/<your-realm>/protocol/saml`
   - **発行者**: `https://<your-keycloak-domain>/auth/realms/<your-realm>`
   - **証明書**: 以下の[ステップ 2](#step-2-get-the-keycloak-certificate)を参照
   - **署名アルゴリズム**: `RSA-SHA-256`
   - **ダイジェストアルゴリズム**: `SHA256`
4. 設定を保存します

### ステップ 2: Keycloak 証明書を取得する

1. Keycloak でクライアント設定に移動します
2. **エクスポート** をクリックします（または Keycloak のバージョンによっては **キー** タブに移動）
3. エクスポートされた JSON ファイルで、名前に `certificate` が含まれるキーを見つけます
4. 証明書の値をコピーし、以下の形式で OneUptime に貼り付けます。

```
-----BEGIN CERTIFICATE-----
MIICnzCCAYcCBgFyPZ8QFzANBgkqhkiG.......
-----END CERTIFICATE-----
```

### ステップ 3: Keycloak クライアントを設定する

1. Keycloak でレルムの **クライアント** に移動します
2. 新しいクライアントを作成するか、既存のものを編集します
3. **クライアントプロトコル** を `saml` に設定します
4. **クライアント ID** を OneUptime の **SSO 設定を表示** の **識別子（エンティティ ID）** の値に設定します
5. **有効なリダイレクト URI** を OneUptime の URL に設定します
6. **ルート URL** を OneUptime のベース URL に設定します
7. OneUptime の **返信 URL（アサーションコンシューマーサービス URL）** を **アサーションコンシューマーサービス POST バインディング URL** フィールドに貼り付けます

### ステップ 4: Keycloak クライアント設定を構成する

1. **署名キーの設定** を無効にします（[キー] タブで）
2. **Name ID 形式** を `email` に設定します
3. **Name ID 形式を強制する** オプションが有効になっていることを確認し、Keycloak が常にメールアドレスを Name ID として送信するようにします

### ステップ 5: 設定を確認する

1. Keycloak と OneUptime の両方ですべての設定を保存します
2. SSO を使用して OneUptime へのログインを試みます
3. Keycloak のログインページにリダイレクトされ、認証成功後に OneUptime に戻ることを確認します

### Keycloak のトラブルシューティング

- **署名エラーでログインが失敗する**: `BEGIN CERTIFICATE` と `END CERTIFICATE` の行を含む証明書が正しくコピーされていることを確認します
- **Name ID エラー**: Keycloak で **Name ID 形式** が `email` に設定されていることを確認します
- **リダイレクトループ**: **有効なリダイレクト URI** と **アサーションコンシューマーサービス POST バインディング URL** が正しく設定されていることを確認します
- **証明書が見つからない**: 正しいレルムの正しいクライアントからエクスポートしていることを確認します

## Microsoft Entra ID（旧 Azure AD / Active Directory）SAML 設定

Microsoft Entra ID は、Microsoft のクラウドベースの ID およびアクセス管理サービスです。OneUptime の SAML アイデンティティプロバイダーとして Entra ID を設定する手順を以下に示します。

### 前提条件

- Microsoft Entra ID テナント（SAML SSO を持つエンタープライズアプリケーションをサポートする任意のティア）
- Microsoft Entra ID と OneUptime の両方への管理者アクセス
- SSO サポートを持つ OneUptime アカウント

### ステップ 1: OneUptime SSO を設定する

1. OneUptime ダッシュボードにログインします
2. **プロジェクト設定** > **認証** > **SSO** に移動します
3. **SSO を作成** をクリックして以下を入力します。
   - **名前**: わかりやすい名前（例: `Azure AD SAML`）
   - **サインオン URL**: [ステップ 3](#step-3-copy-entra-id-saml-metadata-to-oneuptime) で Entra ID から取得します
   - **発行者**: [ステップ 3](#step-3-copy-entra-id-saml-metadata-to-oneuptime) で Entra ID から取得します
   - **証明書**: [ステップ 3](#step-3-copy-entra-id-saml-metadata-to-oneuptime) で Entra ID から取得します
   - **署名アルゴリズム**: `RSA-SHA-256`
   - **ダイジェストアルゴリズム**: `SHA256`
4. **SSO 設定を表示** をクリックして **識別子（エンティティ ID）** と **返信 URL（アサーションコンシューマーサービス URL）** をコピーします - Entra ID の設定に必要です

### ステップ 2: Microsoft Entra ID でエンタープライズアプリケーションを作成する

1. [Microsoft Entra 管理センター](https://entra.microsoft.com) にサインインします
2. **ID** > **アプリケーション** > **エンタープライズアプリケーション** に移動します
3. **+ 新しいアプリケーション** をクリックします
4. **+ 独自のアプリケーションを作成する** をクリックします
5. 名前を入力します（例: "OneUptime"）
6. **ギャラリーに見つからない他のアプリケーションを統合する（非ギャラリー）** を選択します
7. **作成** をクリックします

### ステップ 3: Entra ID で SAML SSO を設定する

1. 新しいエンタープライズアプリケーションで **シングルサインオン** に移動します
2. シングルサインオンの方法として **SAML** を選択します
3. **基本的な SAML 構成** で **編集** をクリックして設定します。
   - **識別子（エンティティ ID）**: OneUptime の **SSO 設定を表示** の **識別子（エンティティ ID）** を貼り付けます
   - **返信 URL（アサーションコンシューマーサービス URL）**: OneUptime の **SSO 設定を表示** の **返信 URL** を貼り付けます
4. **保存** をクリックします
5. **SAML 証明書** セクションで:
   - **証明書（Base64）** をダウンロードします
   - テキストエディタでダウンロードした証明書ファイルを開き、内容をコピーします
6. **OneUptime のセットアップ** セクションで以下をコピーします。
   - **ログイン URL** - OneUptime の **サインオン URL** に貼り付けます
   - **Azure AD 識別子** - OneUptime の **発行者** に貼り付けます
7. OneUptime に戻り、証明書と URL を貼り付けて保存します

### ステップ 4: ユーザー属性とクレームを設定する

1. SAML 設定ページで **属性とクレーム** の **編集** をクリックします
2. 以下のクレームが設定されていることを確認します。

| クレーム名 | 値 |
|-----------|-------|
| `Unique User Identifier (Name ID)` | `user.userprincipalname` または `user.mail` |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress` | `user.mail` |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/givenname` | `user.givenname` |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/surname` | `user.surname` |

3. **名前識別子の形式** を `メールアドレス` に設定します
4. **保存** をクリックします

### ステップ 5: ユーザーとグループを割り当てる

1. エンタープライズアプリケーションで **ユーザーとグループ** に移動します
2. **+ ユーザー/グループの追加** をクリックします
3. SSO アクセスを付与するユーザーやグループを選択します
4. **割り当て** をクリックします

### ステップ 6: 設定を確認する

1. Entra ID と OneUptime の両方ですべての設定を保存します
2. SSO を使用して OneUptime へのログインを試みます
3. Microsoft のログインページにリダイレクトされ、認証成功後に OneUptime に戻ることを確認します

### Microsoft Entra ID のトラブルシューティング

- **AADSTS700016 エラー**: Entra ID の識別子（エンティティ ID）が OneUptime と一致しない - 両方の値が同一であることを確認します
- **証明書エラー**: **Base64** 証明書（生のバイナリ形式ではなく）をダウンロードし、`BEGIN CERTIFICATE` / `END CERTIFICATE` の行が含まれていることを確認します
- **ユーザーが割り当てられていない**: SSO 経由でログインする前に、ユーザーがエンタープライズアプリケーションに明示的に割り当てられている必要があります
- **Name ID の不一致**: Name ID クレームが OneUptime のユーザーのメールアドレスに一致するメールアドレスに設定されていることを確認します

## Okta SAML 設定

Okta は、堅牢な SAML SSO 機能を提供する広く使用されている ID プラットフォームです。OneUptime の SAML アイデンティティプロバイダーとして Okta を設定する手順を以下に示します。

### 前提条件

- 管理者アクセスを持つ Okta 組織
- SSO サポートを持つ OneUptime アカウント

### ステップ 1: OneUptime SSO を設定する

1. OneUptime ダッシュボードにログインします
2. **プロジェクト設定** > **認証** > **SSO** に移動します
3. **SSO を作成** をクリックして以下を入力します。
   - **名前**: わかりやすい名前（例: `Okta SAML`）
   - **サインオン URL**: [ステップ 3](#step-3-copy-okta-saml-metadata-to-oneuptime) で Okta から取得します
   - **発行者**: [ステップ 3](#step-3-copy-okta-saml-metadata-to-oneuptime) で Okta から取得します
   - **証明書**: [ステップ 3](#step-3-copy-okta-saml-metadata-to-oneuptime) で Okta から取得します
   - **署名アルゴリズム**: `RSA-SHA-256`
   - **ダイジェストアルゴリズム**: `SHA256`
4. **SSO 設定を表示** をクリックして **識別子（エンティティ ID）** と **返信 URL（アサーションコンシューマーサービス URL）** をコピーします - Okta の設定に必要です

### ステップ 2: Okta で SAML アプリケーションを作成する

1. Okta 管理コンソールにサインインします
2. **アプリケーション** > **アプリケーション** に移動します
3. **アプリ統合を作成** をクリックします
4. **SAML 2.0** を選択して **次へ** をクリックします
5. **アプリ名** に "OneUptime" を入力して **次へ** をクリックします
6. **SAML 設定** セクションで以下を設定します。
   - **シングルサインオン URL**: OneUptime の **SSO 設定を表示** の **返信 URL（アサーションコンシューマーサービス URL）** を貼り付けます
   - **オーディエンス URI（SP エンティティ ID）**: OneUptime の **SSO 設定を表示** の **識別子（エンティティ ID）** を貼り付けます
   - **Name ID 形式**: `EmailAddress` を選択します
   - **アプリケーションのユーザー名**: `Email` を選択します
7. **次へ** をクリックし、**社内アプリを追加している Okta のお客様です** を選択して **完了** をクリックします

### ステップ 3: Okta SAML メタデータを OneUptime にコピーする

1. Okta アプリケーションで **サインオン** タブに移動します
2. **SAML 署名証明書** セクションで、アクティブな証明書を見つけて **アクション** > **IdP メタデータを表示** をクリックします
3. メタデータ XML から、または **サインオン** タブの詳細から:
   - **サインオン URL**（**アイデンティティプロバイダーのシングルサインオン URL** とも呼ばれます）をコピーし、OneUptime の **サインオン URL** に貼り付けます
   - **発行者**（**アイデンティティプロバイダーの発行者** とも呼ばれます）をコピーし、OneUptime の **発行者** に貼り付けます
4. 署名証明書をダウンロードします。
   - **SAML 署名証明書** セクションで、アクティブな証明書の **アクション** > **証明書をダウンロード** をクリックします
   - テキストエディタでダウンロードした `.cert` ファイルを開き、内容をコピーします
   - OneUptime に証明書を貼り付けます（`BEGIN CERTIFICATE` と `END CERTIFICATE` の行を含む）
5. OneUptime の SSO 設定を保存します

### ステップ 4: 属性ステートメントを設定する（任意）

1. Okta アプリケーションで **全般** タブに移動します
2. **SAML 設定** セクションで **編集** をクリックし、**次へ** をクリックして SAML 設定に進みます
3. **属性ステートメント** セクションで以下を追加します。

| 名前 | 値 |
|------|-------|
| `email` | `user.email` |
| `firstName` | `user.firstName` |
| `lastName` | `user.lastName` |

4. **次へ** をクリックし、**完了** をクリックします

### ステップ 5: ユーザーとグループを割り当てる

1. Okta アプリケーションで **割り当て** タブに移動します
2. **割り当て** > **ユーザーに割り当て** または **グループに割り当て** をクリックします
3. SSO アクセスを付与するユーザーまたはグループを選択します
4. 各選択で **割り当て** をクリックし、**完了** をクリックします

### ステップ 6: 設定を確認する

1. Okta と OneUptime の両方ですべての設定を保存します
2. SSO を使用して OneUptime へのログインを試みます
3. Okta のログインページにリダイレクトされ、認証成功後に OneUptime に戻ることを確認します

### Okta のトラブルシューティング

- **404 または無効な SSO URL**: Okta の **シングルサインオン URL** が OneUptime の **返信 URL** と完全に一致していることを確認します
- **オーディエンスの不一致**: Okta の **オーディエンス URI** が OneUptime の **識別子（エンティティ ID）** と完全に一致していることを確認します
- **証明書エラー**: 非アクティブな証明書ではなく、**アクティブな** 署名証明書の証明書をダウンロードしていることを確認します
- **ユーザーが割り当てられていない**: SSO 経由でログインする前に、ユーザーが Okta アプリケーションに割り当てられている必要があります
- **Name ID エラー**: **Name ID 形式** が `EmailAddress` に設定されており、**アプリケーションのユーザー名** が `Email` に設定されていることを確認します

## その他のアイデンティティプロバイダー

OneUptime の SSO 実装は SAML 2.0 プロトコルを使用しており、準拠した任意のアイデンティティプロバイダーで動作します。一般的な設定手順は以下の通りです。

1. OneUptime で SSO 設定を作成し、**SSO 設定を表示** ボタンから **識別子（エンティティ ID）** と **返信 URL（アサーションコンシューマーサービス URL）** をメモします
2. アイデンティティプロバイダーで以下を使用して SAML アプリケーションを作成します。
   - **アサーションコンシューマーサービス URL / 返信 URL**: OneUptime の SSO 設定から
   - **エンティティ ID / オーディエンス URI**: OneUptime の SSO 設定から
   - **Name ID 形式**: メールアドレス
3. アイデンティティプロバイダーから以下を OneUptime にコピーします。
   - **サインオン URL**（SSO エンドポイント）
   - **発行者**（IdP のエンティティ ID）
   - **公開証明書**（X.509 署名証明書）
4. **署名アルゴリズム** を `RSA-SHA-256` に、**ダイジェストアルゴリズム** を `SHA256` に設定します

## SSO とロールに関する注意事項

現在、OneUptime はアイデンティティプロバイダーからの SAML ロールのマッピングをサポートしていません。ロールベースのアクセスは、OneUptime の **プロジェクト設定** > **SSO** 設定内で個別に設定する必要があります。ここで SSO ユーザーのデフォルトロールを割り当てることができます。
