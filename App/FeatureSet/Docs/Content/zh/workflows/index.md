# 工作流概览

工作流让你无需写代码就能自动化 OneUptime 里的各种任务。把几个模块拖到画布上、连起来,就有了一个会在事件发生时运行的自动化——比如打开一个事件、定时触发,或者其他工具向 OneUptime 推送数据。

可以把工作流看作项目的后台帮手:它们响应事件、与其他工具对话,在你专注于工作时悄悄保持各处同步。

## 工作流能做什么

- **把 OneUptime 接入其他工具**——把事件发送到 Slack、在 Jira 中创建工单、向你技术栈中的 webhook 发起调用。
- **响应 OneUptime 中发生的事情**——当一个严重事件被创建时,通知值班团队并自动开工单。
- **按计划运行任务**——每五分钟、每晚、每周一早上。
- **从外部接收数据**——通过专属 URL 让其他系统把数据推送到 OneUptime。
- **复用常用自动化**——构建一次,从其他任何工作流里调用。

## 工作流的工作原理

每个工作流都有三个部分:

1. **一个触发器**——决定工作流何时启动。可以是手动按钮、定时计划、入站 webhook,或者 OneUptime 内部的事件(比如新建事件)。
2. **一个或多个组件**——工作流要做的事。发送一条消息、发起一次 HTTP 调用、运行一次快速检查、按条件分支。
3. **它们之间的连接**——你从一个模块画线到下一个,决定执行顺序。

所有这些都在画布上以可视化方式构建。大多数工作流不需要编码,不过当你确实需要时,也可以加入一段 JavaScript。

## 关键术语

| 术语 | 含义 |
| --- | --- |
| **工作流** | 整个自动化——一个名称、一张画布,以及一个开关来启用或停用它。 |
| **触发器** | 第一个模块。决定工作流何时运行。每个工作流恰好有一个触发器。 |
| **组件** | 动作模块——发送消息、发起请求、检查条件。 |
| **运行** | 工作流的一次执行。保存了时间戳和每个模块的输出。 |
| **全局变量** | 一次保存便可在任意工作流中复用的值(比如 API 密钥)。 |

## 在 OneUptime 中哪里找工作流

在左侧导航中打开 **工作流**。在那里:

- **工作流**——你的工作流列表。新建一个或打开已有的。
- **构建器标签**——你设计工作流的画布。
- **日志标签**——这个工作流的每一次运行,带有详情。
- **设置标签**——名称、描述、所有者、标签、启用/停用。
- **全局变量**——所有工作流之间共享的值。
- **运行与日志**——项目中所有工作流的执行历史。

## 构建你的第一个工作流

1. **创建**——给工作流起一个名称和简短描述。
2. **选择触发器**——手动、定时、webhook,或来自 OneUptime 的事件。
3. **添加组件**——把动作拖到画布上并连接起来。
4. **测试**——点击 **手动运行** 并在日志中查看结果。
5. **打开它**——准备好后在设置中切换 **已启用** 开关。

## 一个简单的例子

假设你想在创建严重事件时在 Slack 中发布消息:

1. 创建一个名为 "严重事件推送到 Slack" 的工作流。
2. 选择 **事件 → 创建时** 触发器。
3. 添加一个 **条件** 模块。设置它检查事件标题是否包含 "Sev 1"。
4. 从 **是** 分支添加一个 **Slack** 模块。选择频道并写好消息。
5. 启用工作流。

下次有人打开标题包含 "Sev 1" 的事件时,Slack 就会亮起来。

## 工作流如何与 OneUptime 的其他部分配合

- **监控** 发现问题。**事件** 记录它。**工作流** 响应它。
- **Runbook** 是给人用的分步指南。工作流是无人值守的自动化。当需要人来做决定时使用 Runbook;当步骤完全自动时使用工作流。
- **工作区连接**(Slack、Teams)是工作流发送消息的地方。
