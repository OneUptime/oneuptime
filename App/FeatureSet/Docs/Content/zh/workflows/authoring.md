# 创建工作流

在 **工作流 → 创建工作流** 下新建一个工作流，给它起个名字和可选的描述，然后打开 **构建器** 标签开始往画布上拖节点。

## 画布

构建器是一个可缩放、可平移的图。你从组件面板添加节点，用边连接它们，并在侧面板里配置每个节点的参数。头部的保存指示器告诉你最近一次编辑是否已保存。

工作流始终从一个 **触发器** 节点开始。触发器没有输入端口——执行从这里开始。下游的一切都是 **组件**。

## 节点的构成

每个节点都有：

| 字段 | 用途 |
| --- | --- |
| **标题** | 画布上显示的标签。默认是组件名；为了让复杂工作流更易读，可以覆盖它。 |
| **参数** | 组件完成工作所需的配置——URL、Slack 频道、JavaScript 片段等。必填参数会用星号标注。 |
| **输入端口** | 节点左侧的接口，接收进入的边。组件有一个名为 `in` 的输入端口；触发器没有。 |
| **输出端口** | 节点右侧的接口，发出的边从这里开始。组件定义诸如 `success`、`error`、`yes`、`no` 的端口。 |
| **返回值** | 节点产出的数据——也就是其输出端口的载荷。下游节点通过 `{{NodeId.fieldName}}` 引用。 |

## 连接节点

从一个输出端口拖到下游节点的输入端口来创建一条边。从 `success` 出来的边只在上游节点成功时运行那条分支；从 `error` 出来的边只在它失败时运行。如果你不连接某个端口，那条分支就直接结束。

可以扇出：一个输出端口可以喂多个下游节点，从那一点开始它们会并行执行。

## 配置参数

点击一个节点来打开它的侧面板。每个参数都有类型化编辑器：

- **Text / URL / Email / Number / Password** — 单行输入。
- **JSON** — 带语法高亮和校验指示的 JSON 编辑器。
- **JavaScript** — 用于 **Custom Code** 组件片段的代码编辑器。
- **Markdown / HTML** — 邮件和消息组件的富文本正文。
- **CronTab** — 调度表达式（用于 Schedule 触发器）。
- **Boolean** — 开关。
- **Select / Query** — 用于取固定值集合或模型查询的下拉框。

任何文本字段都支持变量插值——规则见 [工作流变量](/docs/workflows/variables)。

## 一个最简的入门工作流

最快感受画布的方式：

1. 丢一个 **Manual** 触发器。
2. 丢一个 **Log** 组件（在 **Utils** 下）。把触发器的输出端口连到 Log 组件的输入端口。
3. 在 Log 组件的参数里输入 `Hello from {{Manual.JSON.name}}`。
4. 保存并启用工作流。
5. 点击 **手动运行**，把 `{ "name": "Ada" }` 作为输入粘贴进去，提交。
6. 打开 **日志** 标签。最近一次运行显示 Log 节点捕获的输出：`Hello from Ada`。

这套往返流程——拖、连、配、跑、看——就是创作每个工作流的节奏。

## 保存、启用和在生产环境测试

工作流以 JSON 图的形式存在 `Workflow.graph` 列上。构建器会随编辑保存；头部的保存指示器告诉你最新更改是否已到达服务器。没有单独的"发布"步骤。

但是：工作流只在 **isEnabled** 为开时才会触发它的触发器。新建的工作流出厂是禁用的。把这个开关当作你的"准备好上线"开关——构建好之后，点击 **手动运行** 用一个示例负载干跑一遍，看看 **日志**，然后才打开启用开关。

如果你需要暂停一个工作流但不删除它（例如在无关的事件期间），到 **设置** 里把 **isEnabled** 关掉。已经在进行中的运行会继续；不会再启动新的。

## 重排序与整理

- 拖动节点重新摆位。位置会保存在图里，下一个打开画布的人看到的是同样的布局。
- 在边上右键删除它；在节点上右键有删除和复制选项。
- 对于较宽的工作流，按从左到右的方向布局，让执行方向与你的阅读方向一致。
