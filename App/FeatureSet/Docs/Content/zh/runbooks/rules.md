# Runbook 规则

Runbook 规则会在 **事件**、**告警** 或 **计划维护事件** 创建时自动附加 Runbook。规则从每种实体的设置菜单管理：

- 事件 → 设置 → **Runbook 规则**
- 告警 → 设置 → **Runbook 规则**
- 计划维护 → 设置 → **Runbook 规则**

三处页面编辑的是同一份底层规则模型——它们只是被过滤成只显示对应实体类型的规则。

## 规则的结构

| 字段 | 用途 |
| --- | --- |
| **名称** | 简短、可读的标签。出现在审计日志中。 |
| **描述** | 给同事看的可选上下文。 |
| **启用** | 在不删除规则的情况下暂停的开关。 |
| **标题模式** | 与实体标题匹配的大小写不敏感正则。空 = 任意标题匹配。 |
| **描述模式** | 与实体描述匹配的大小写不敏感正则。空 = 任意描述匹配。 |
| **要启动的 Runbook** | 规则触发时启动的一个或多个 Runbook。 |

## 匹配语义

规则只有在**所有指定条件都通过**时才匹配。空条件被跳过：

- 没有设置任何模式的规则会在该类型的每个事件上运行（全局"始终运行"规则）。
- 只有标题模式的规则会在标题匹配正则的事件上触发。
- 多条规则可以匹配同一事件——每条匹配都会触发，且它们所引用的 Runbook 集合都会运行（每个 Runbook 各自一次执行）。

## 示例：数据库事件的 DB failover

```
名称：       数据库事件启动 DB failover
触发器：     事件
标题模式：   (?:^|\b)(db|database|postgres|mysql|mongo)
Runbook：    [DB failover playbook, 通知 DBA 团队]
```

只要事件标题里出现 "db"、"database"、"postgres" 等，就会创建两次 Runbook 执行。

## 示例：始终运行的卫生规则

```
名称：           每个事件的预飞检查
触发器：         事件
标题模式：       （空）
描述模式：       （空）
Runbook：        [捕获事件前状态]
```

在每个事件上触发——非常适合捕获系统状态快照、页面指标等。

## 规则触发时会发生什么

1. Runbook 被加载。
2. 它的步骤作为 **快照** 复制到新的 Runbook 执行上。
3. 执行被加入 Runbook worker 队列。
4. 执行与源实体关联——出现在事件、告警或维护页面以及 Runbook 的执行列表中。

可以在 **Runbooks → 执行** 中按状态、Runbook 或日期筛选，看到所有由规则触发的运行。

## 已禁用的 Runbook

如果规则引用的 Runbook 的 `isEnabled = false`，规则仍然匹配，但执行会被跳过。重新启用 Runbook 即可恢复。

## 测试一条规则

在生产中依赖某条规则之前，创建一个标题与模式匹配的测试事件（或告警），确认期望的 Runbook 会触发。规则在创建瞬间被评估——之后修改事件标题不会再次触发规则。
