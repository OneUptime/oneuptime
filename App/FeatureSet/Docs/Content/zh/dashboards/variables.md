# 仪表板变量与过滤器

变量把单个仪表板变成模板。定义一个 `service` 变量，同一张图就能为 `checkout`、`payments`、`search` 分别重新渲染——在顶部下拉里选一个，而不是搭三个几乎一样的仪表板。

本页讲四种变量类型、它们的值如何注入到小部件查询中，以及紧邻它们的全局时间范围和刷新控件。

## 变量类型

在 **仪表板 → 设置 → Variables** 下添加变量。每个变量有一个名称（在小部件查询中以 `{{name}}` 引用）、一个可选标签和一个类型。

### Custom List

静态下拉。你提供一个逗号分隔的值列表；查看者从中选择。

什么时候用：选项集合较小、固定、只对你团队有意义时。`environment` 取值 `prod, staging, dev`。`region` 取值 `us-east-1, eu-west-1, ap-south-1`。

### Query

下拉的选项在渲染时由一个 ClickHouse 查询计算得到。

什么时候用：选项是动态的且存在于你的遥测中。"过去 24 小时登录过的每个客户 ID"，通过 `SELECT DISTINCT customer_id FROM ...`。查询针对项目数据运行；即使是你自己的数据，也把结果当作不可信的输入对待。

### Text Input

一个自由文本字段。查看者输入什么就注入什么。

什么时候用：你希望仪表板像一个搜索工具。一个"按 IP 过滤"或"按请求 ID 过滤"的仪表板。

### Telemetry Attribute

选项是某个 OpenTelemetry 属性 key 在仪表板时间范围内、跨项目遥测中的不同取值。

配置 **attribute key**（例如 `k8s.cluster.name`、`service.name`、`host.name`）。小部件从日志/指标/追踪中拉出 distinct 值，作为下拉选项提供。

什么时候用：选项恰好就是你已经给遥测打过标签的实体。集群名、服务名、区域、客户 ID、部署环境——任何你已经作为 OpenTelemetry 资源属性或 span 属性发出的字段。

这是面向服务的仪表板里最常用的变量类型，因为它会自动更新：当你发布了一个带 `service.name = inventory` 标签的新服务，那个值就会出现在下拉里，不需要任何人去编辑仪表板。

## 多选

每个变量都可以配置为 **multi-select**。打开后，查看者可选一个或多个值；仪表板按 `value IN (...)` 而不是 `value = ...` 过滤。

什么时候用多选：你想在不离开仪表板的情况下一起看"checkout + payments"。当图表的数学在所选值上不成立时（例如对均值再取均值），避免使用。

## 默认值

每个变量都可设可选默认值。在查看者改下拉之前，仪表板用默认值渲染。对于公开仪表板，默认值就是访客落地时看到的内容。

## 插值的工作方式

任何接收字符串过滤的小部件查询位置——指标查询的 `WHERE` 子句、列表小部件的过滤、日志流的属性匹配——你都可以引用 `{{variable_name}}`。

例如，一个 Chart 的指标查询可能是：

```
SELECT avg(latency_ms) FROM spans WHERE service.name = '{{service}}'
```

当 `service` 被设为 `checkout` 时，查询以 `service.name = 'checkout'` 运行。当查看者切到 `payments`，查询以 `service.name = 'payments'` 重新运行。

特别地，对于 **Telemetry Attribute** 变量，OneUptime 知道属性 key，并会把过滤注入到每个提到同一属性的小部件——变量改变时你不必逐个手动编辑小部件查询。这就是让服务模板化仪表板开箱即用的"魔法"。

## 时间范围

仪表板头部有一个全局 **时间范围** 选择器。每个指标小部件都针对这个窗口查询。可选：

- **预设** — 过去 1 小时、24 小时、7 天、30 天、90 天（取决于你的留存）。
- **自定义范围** — 选起止时间戳。

时间范围是仪表板 URL 的一部分——分享 URL 就分享了窗口。这在事件期间很方便：把时间范围钉到 "10:00–10:30 UTC today"，把链接发到事件频道。

## 刷新间隔

时间范围旁边，选择小部件重新查询的频率：

- **关闭** — 小部件在加载时查询一次。
- **5s / 10s / 30s / 1m / 5m / 15m** — 自动刷新。

自动刷新便于挂墙屏幕和当前事件视图。对临时排查，关掉它好让视图在你滚动时保持稳定。

## 综合运用

一个服务模板化的仪表板通常包含：

1. 一个 `service` 变量，类型为 **Telemetry Attribute**，绑定到 `service.name`。默认：你最常关注的服务。多选：关（让图表始终一次只显示一个服务）。
2. 一个 `environment` 变量，类型为 **Custom List**。默认：`prod`。
3. 一个 `cluster` 变量，类型为 **Telemetry Attribute**，绑定到 `k8s.cluster.name`。多选：开（让你能跨集群汇总）。
4. 仪表板的小部件在它们的过滤里引用这些变量。

结果：一个仪表板、覆盖整个机群、顶部几个下拉。
