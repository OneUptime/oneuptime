# 常见问题与故障排查

OneUptime 移动端和桌面端应用（PWA）的常见问题及解决方案。

## 通用常见问题

### 什么是渐进式 Web 应用（PWA）？

渐进式 Web 应用是一种使用现代 Web 技术提供类原生应用体验的 Web 应用程序。PWA 可以直接从浏览器安装，无需应用商店，支持离线工作、推送通知，并可与您设备的操作系统集成。

### OneUptime 为什么不使用传统应用商店？

OneUptime 使用 PWA 技术，因为它具有以下优势：
- **即时更新**：无需等待应用商店审核或手动更新
- **跨平台**：单一代码库适用于所有设备
- **无下载大小限制**：完整功能，不受大小限制
- **直接分发**：直接从您的 OneUptime 实例安装
- **始终最新**：用户始终拥有最新版本
- **安全性**：与 Web 应用程序相同的安全优势


### OneUptime PWA 占用多少存储空间？

- **初始安装**：10-20MB
- **缓存增长**：正常使用后 50-100MB
- **最大缓存**：浏览器通常限制为 200MB
- **自动清理**：浏览器自动管理存储

### OneUptime PWA 是否支持推送通知？

是的，OneUptime PWA 支持丰富的推送通知：
- **事件告警**：实时事件通知
- **状态更新**：监控器状态变化告警
- **自定义触发器**：配置通知规则
- **富内容**：图片、操作和详细信息
- **角标更新**：应用图标上显示未读数量

## 安装常见问题

### 为什么看不到"安装"按钮？

常见原因及解决方案：
1. **浏览器兼容性**：使用 Chrome、Edge 或 Safari
2. **需要 HTTPS**：确保 OneUptime 实例使用 HTTPS
3. **PWA 要求**：服务器必须满足 PWA 清单要求
4. **缓存问题**：清除浏览器缓存并重新加载
5. **已安装**：应用可能已经安装
6. **等待时间**：某些浏览器需要在页面上停留 30 秒以上

### 我可以在多台设备上安装吗？

可以！您可以在以下设备上安装 OneUptime PWA：
- 每个用户不限设备数量
- 同一设备上的多个浏览器
- 不同操作系统
- 共享/家庭设备（使用不同账号）

### 如何更新已安装的应用？

OneUptime PWA 自动更新：
- **自动更新**：在线访问时应用自动更新
- **后台更新**：更新在后台下载
- **立即可用**：新功能即时可用
- **无需用户操作**：与应用商店应用不同，无需手动更新

### 安装过程中可以自定义应用名称吗？

是的，安装过程中您可以：
- 更改应用名称（默认：「OneUptime」）
- 添加您的组织名称
- 使用自定义命名规范
- 修改图标标签（取决于平台）

### 如何卸载 OneUptime PWA？

卸载方式因平台而异：

**Android：**
- 长按应用图标 → 卸载
- 设置 → 应用 → OneUptime → 卸载

**iOS：**
- 长按应用图标 → 移除应用 → 删除应用

**Windows：**
- 设置 → 应用 → OneUptime → 卸载
- 右键单击开始菜单项 → 卸载

**macOS：**
- 从应用程序文件夹拖到废纸篓
- 右键单击 Dock 图标 → 移除

**Linux：**
- 从应用启动器中移除
- 删除 .desktop 文件


## 通知常见问题

### 为什么我没有收到通知？

常见通知问题及解决方法：

**检查权限：**
```
1. 启用浏览器通知权限
2. 操作系统通知权限
3. 已配置 OneUptime 通知设置
4. 已禁用勿扰模式
```

**特定平台：**
- **Android**：检查电池优化设置
- **iOS**：在设置应用中验证通知设置
- **Windows**：检查专注助手设置
- **macOS**：验证通知中心权限
- **Linux**：检查通知守护进程状态

### 我可以自定义通知声音吗？

通知自定义选项：
- **系统声音**：使用操作系统通知声音设置
- **浏览器设置**：在浏览器通知偏好中配置
- **OneUptime 设置**：在控制台中设置通知偏好
- **优先级**：为不同严重程度配置不同的声音

### 如何临时禁用通知？

临时禁用通知：
- **勿扰模式**：启用系统 DND 模式
- **浏览器设置**：临时禁用网站通知
- **OneUptime 控制台**：在设置中暂停通知
- **专注模式**：使用操作系统专注/集中模式

## 安全常见问题

### OneUptime PWA 安全吗？

安全功能和注意事项：
- **HTTPS 加密**：所有数据安全传输
- **同源策略**：适用浏览器安全限制
- **沙盒环境**：在浏览器安全沙盒中运行
- **定期更新**：安全补丁自动应用
- **无根访问**：与原生应用相比，系统访问受限


*注意：敏感数据已加密，并遵循浏览器安全标准。*

### 我可以在企业网络上使用 OneUptime PWA 吗？

企业网络注意事项：
- **防火墙规则**：确保 HTTPS（端口 443）访问
- **代理配置**：配置浏览器代理设置
- **证书信任**：如需要，安装企业证书
- **VPN 访问**：使用 VPN 进行远程访问
- **安全策略**：遵守 IT 安全要求

## 故障排查

### 安装问题

**问题**：安装按钮不出现
```
解决方案：
1. 在 OneUptime 页面等待 30 秒以上
2. 刷新页面后再等待
3. 清除浏览器缓存和 Cookie
4. 尝试不同浏览器（推荐 Chrome/Edge）
5. 验证 HTTPS 连接（检查锁形图标）
6. 检查是否已安装
```

**问题**：安装失败或崩溃
```
解决方案：
1. 确保有足够存储空间（100MB 以上）
2. 关闭其他浏览器标签页和应用程序
3. 将浏览器更新到最新版本
4. 临时禁用浏览器扩展
5. 在私密/隐身模式下尝试安装
6. 重启浏览器后重试
```

**问题**：应用已安装但不出现
```
解决方案：
1. 检查所有应用启动器位置
2. 在设备搜索中搜索「OneUptime」
3. 在浏览器的应用管理部分查找
4. 等待 1-2 分钟让系统刷新
5. 重启设备后再检查
```

**问题**：应用频繁崩溃
```
解决方案：
1. 将浏览器更新到最新版本
2. 清除 OneUptime 的所有浏览器数据
3. 禁用浏览器扩展
4. 检查可用存储空间
5. 重启操作系统
6. 重新安装 OneUptime PWA
```

**问题**：推送通知不工作
```
解决方案：
1. 检查浏览器中的通知权限
2. 验证系统通知设置
3. 先测试简单通知
4. 清除通知数据并重新授予权限
5. 检查勿扰/专注模式设置
6. 验证 OneUptime 通知配置
```

**问题**：应用未同步最新数据
```
解决方案：
1. 下拉刷新（移动端）
2. 按 Ctrl+F5（Windows/Linux）或 Cmd+R（Mac）
3. 关闭并重新打开应用
4. 清除应用缓存并重新加载
5. 检查网络连接
```

### 特定平台问题

**Android 问题：**
```
问题：应用未出现在应用抽屉中
解决方案：检查"最近添加"的应用部分，在应用抽屉中搜索

问题：通知延迟
解决方案：为浏览器应用禁用电池优化

问题：启动时应用崩溃
解决方案：清除 Chrome 应用数据，重启设备
```

**iOS 问题：**
```
问题：无法添加到主屏幕
解决方案：使用 Safari 浏览器，确保 iOS 11.3 或更高版本

问题：应用图标缺失
解决方案：检查所有主屏幕页面和应用程序库

问题：Face ID 不工作
解决方案：在设置中为 Safari 启用 Face ID
```

**Windows 问题：**
```
问题：应用未出现在开始菜单中
解决方案：搜索应用名称，检查已安装应用列表

问题：通知不显示
解决方案：检查 Windows 通知设置，为浏览器启用

问题：窗口大小问题
解决方案：手动调整大小，应用会记住尺寸
```

**macOS 问题：**
```
问题：无法通过 Safari 安装
解决方案：更新到 macOS Sonoma 或更高版本，使用 文件 → 添加到 Dock

问题：应用不在应用程序文件夹中
解决方案：检查 Launchpad，使用 Spotlight 搜索

问题：通知不工作
解决方案：检查 系统偏好设置 → 通知
```

**Linux 问题：**
```
问题：缺少 PWA 安装选项
解决方案：使用 Chrome/Chromium，确保桌面环境支持

问题：图标未出现在启动器中
解决方案：更新桌面数据库，检查 .desktop 文件

问题：音频通知不工作
解决方案：检查 PulseAudio，验证浏览器音频权限
```

### 错误消息

**"此站点无法安装"**
```
原因：
- OneUptime 实例不满足 PWA 要求
- Web 应用清单缺失或无效
- HTTPS 未正确配置
- 浏览器不支持 PWA 安装

解决方案：
- 联系管理员验证 PWA 设置
- 尝试不同浏览器
- 检查浏览器控制台以获取详细错误
```
