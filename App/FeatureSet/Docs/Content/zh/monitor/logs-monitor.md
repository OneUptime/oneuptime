# 日志监控器

日志监控允许您监控应用程序日志，并根据日志模式、数量和严重程度触发告警。OneUptime 评估来自遥测服务的日志，并根据您配置的标准进行检查。

## 概述

日志监控器在时间窗口内搜索并统计符合特定过滤器的日志。这使您能够：

- 对错误日志峰值发出告警
- 监控特定的日志模式或消息
- 按严重程度跟踪日志量
- 按服务、属性和内容过滤日志
- 从日志模式中检测应用程序问题

## 创建日志监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **日志** 作为监控器类型
4. 选择要监控的遥测服务
5. 根据需要配置日志过滤器和标准

## 配置选项

### 遥测服务

选择一个或多个服务来监控其日志。服务必须通过 OpenTelemetry 向 OneUptime 发送日志。

### 日志过滤器

| 过滤器 | 描述 | 是否必填 |
|--------|------|---------|
| 严重程度 | 按日志严重程度过滤（ERROR、WARN、INFO、DEBUG 等） | 否 |
| 正文 | 在日志消息正文中进行文本搜索 | 否 |
| 属性 | 用于过滤自定义日志属性的键值对 | 否 |
| 时间窗口 | 搜索日志的时间范围（秒，默认：60） | 否 |

### 严重程度

按一个或多个严重程度过滤日志：

- **FATAL** / **EMERGENCY** / **CRITICAL**
- **ERROR**
- **WARN** / **WARNING**
- **INFO** / **INFORMATIONAL**
- **DEBUG**
- **TRACE**
- **UNSPECIFIED**

## 监控标准

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| 日志数量 | 时间窗口内符合过滤条件的日志数量 |

### 过滤类型

- **大于** — 日志数量超过阈值
- **小于** — 日志数量低于阈值
- **大于或等于** — 日志数量等于或超过阈值
- **小于或等于** — 日志数量等于或低于阈值
- **等于** — 日志数量完全匹配
- **不等于** — 日志数量不匹配

### 示例标准

#### 当 60 秒内超过 100 条错误日志时发出告警

- **严重程度**：ERROR
- **时间窗口**：60 秒
- **检查项**：日志数量
- **过滤类型**：大于
- **值**：100

#### 当出现任何致命日志时发出告警

- **严重程度**：FATAL
- **时间窗口**：60 秒
- **检查项**：日志数量
- **过滤类型**：大于
- **值**：0

#### 监控包含特定错误消息的日志

- **正文**：`database connection timeout`
- **时间窗口**：300 秒
- **检查项**：日志数量
- **过滤类型**：大于
- **值**：5

## 设置要求

日志监控需要您的应用程序通过 OpenTelemetry 向 OneUptime 发送日志。设置说明请参见 [OpenTelemetry](/docs/telemetry/open-telemetry) 文档。
