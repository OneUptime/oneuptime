# DNS 监控器

DNS 监控允许您监控域名的 DNS 解析健康状况和正确性。OneUptime 定期查询 DNS 记录并根据您配置的标准验证响应。

## 概述

DNS 监控器查询 DNS 服务器以获取特定记录类型并评估结果。这使您能够：

- 监控 DNS 服务可用性
- 验证 DNS 记录返回正确的值
- 跟踪 DNS 解析响应时间
- 验证 DNSSEC 配置
- 检测 DNS 传播问题或劫持

## 创建 DNS 监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **DNS** 作为监控器类型
4. 输入要查询的域名和记录类型
5. 根据需要配置监控标准

## 配置选项

### 基本设置

| 字段 | 描述 | 是否必填 |
|------|------|---------|
| 域名 | 要查询的域（例如 `example.com`） | 是 |
| 记录类型 | 要查询的 DNS 记录类型 | 是 |
| DNS 服务器 | 要使用的自定义 DNS 服务器（例如 `8.8.8.8`）。留空使用系统默认值 | 否 |

### 支持的记录类型

| 记录类型 | 描述 |
|---------|------|
| A | IPv4 地址记录 |
| AAAA | IPv6 地址记录 |
| CNAME | 规范名称（别名）记录 |
| MX | 邮件交换记录 |
| NS | 名称服务器记录 |
| TXT | 文本记录（SPF、DKIM 等） |
| SOA | 授权起始记录 |
| PTR | 指针记录（反向 DNS） |
| SRV | 服务定位器记录 |
| CAA | 证书授权机构授权记录 |

### 高级设置

| 字段 | 描述 | 默认值 |
|------|------|--------|
| 端口 | DNS 端口号 | 53 |
| 超时（毫秒） | 等待响应的时间 | 5000 |
| 重试次数 | 失败时的重试次数 | 3 |

## 监控标准

您可以配置标准来判断 DNS 何时处于在线、降级或离线状态，基于以下条件：

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| DNS 是否在线 | DNS 服务器是否响应查询 |
| DNS 响应时间（毫秒） | 查询响应时间（毫秒） |
| DNS 记录是否存在 | 查询的 DNS 记录是否存在 |
| DNS 记录值 | DNS 记录返回的值 |
| DNSSEC 是否有效 | DNSSEC 验证是否通过 |

### 过滤类型

对于 **DNS 是否在线**、**DNS 记录是否存在** 和 **DNSSEC 是否有效**：

- **True** — 条件为真
- **False** — 条件为假

对于 **DNS 响应时间**：

- **大于**、**小于**、**大于或等于**、**小于或等于**、**等于**、**不等于**

对于 **DNS 记录值**：

- **包含** — 记录值包含指定文本
- **不包含** — 记录值不包含指定文本
- **以...开头** — 记录值以指定文本开头
- **以...结尾** — 记录值以指定文本结尾
- **等于** — 记录值完全匹配
- **不等于** — 记录值不匹配

### 示例标准

#### 检查 DNS 是否正常解析

- **检查项**：DNS 是否在线
- **过滤类型**：True

#### 验证 A 记录指向正确的 IP

- **检查项**：DNS 记录值
- **过滤类型**：等于
- **值**：`93.184.216.34`

#### 当 DNS 响应缓慢时告警

- **检查项**：DNS 响应时间（毫秒）
- **过滤类型**：大于
- **值**：500

#### 验证 DNSSEC 是否有效

- **检查项**：DNSSEC 是否有效
- **过滤类型**：True
