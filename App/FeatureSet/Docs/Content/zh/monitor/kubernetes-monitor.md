# Kubernetes 监控器

Kubernetes 监控允许您监控 Kubernetes 集群的健康状况和性能，包括节点、Pod、工作负载和控制平面组件。OneUptime 从您的集群收集指标并根据您配置的标准进行评估。

## 概述

Kubernetes 监控器使用来自集群的指标，为您的基础设施提供深度可见性。这使您能够：

- 监控集群、命名空间、工作负载、节点和 Pod 的健康状况
- 跟踪资源的 CPU、内存、磁盘和网络使用情况
- 检测 Pod 崩溃、重启和调度失败
- 监控 Deployment 副本可用性
- 就控制平面问题（etcd、API 服务器、调度器）发出告警
- 跟踪资源请求和限制

## 创建 Kubernetes 监控器

1. 在 OneUptime 控制台中转到 **监控器**
2. 点击 **创建监控器**
3. 选择 **Kubernetes** 作为监控器类型
4. 选择要监控的集群和资源范围
5. 配置资源过滤器和指标查询
6. 根据需要配置监控标准

## 配置选项

### 集群

选择要监控的 Kubernetes 集群。集群必须通过 OpenTelemetry 与 OneUptime 集成。

### 资源范围

选择监控资源的级别：

| 范围 | 描述 |
|------|------|
| 集群 | 监控整个集群 |
| 命名空间 | 监控特定命名空间内的资源 |
| 工作负载 | 监控特定的 Deployment、StatefulSet、DaemonSet、Job 或 CronJob |
| 节点 | 监控特定的集群节点 |
| Pod | 监控特定的 Pod |

### 资源过滤器

使用可选过滤器缩小范围：

| 过滤器 | 描述 | 适用范围 |
|--------|------|---------|
| 命名空间 | Kubernetes 命名空间 | 命名空间、工作负载、Pod |
| 工作负载类型 | deployment、statefulset、daemonset、job、cronjob | 工作负载 |
| 工作负载名称 | 工作负载的名称 | 工作负载 |
| 节点名称 | 节点的名称 | 节点 |
| Pod 名称 | Pod 的名称 | Pod |

### 指标查询

配置一个或多个指标查询进行评估。每个查询指定：

- **指标名称** — 要查询的 Kubernetes 指标
- **聚合** — 如何聚合指标值
- **过滤器** — 基于属性的额外过滤

您还可以创建使用数学表达式组合多个指标查询的**公式**。

### 滚动时间窗口

选择指标评估的时间窗口：

- 过去 1 分钟
- 过去 5 分钟
- 过去 10 分钟
- 过去 15 分钟
- 过去 30 分钟
- 过去 60 分钟

## 常用 Kubernetes 指标

### Pod 指标

| 指标 | 描述 |
|------|------|
| Pod CPU 使用率 | Pod 的 CPU 消耗 |
| Pod 内存使用量 | Pod 的内存消耗 |
| Pod 文件系统使用量 | Pod 的磁盘使用量 |
| Pod 网络接收/发送 | 网络流量 |
| Pod 阶段 | 当前 Pod 阶段（Running、Pending、Failed 等） |

### 节点指标

| 指标 | 描述 |
|------|------|
| 节点 CPU 使用率 | 每个节点的 CPU 利用率 |
| 节点内存使用量 | 每个节点的内存利用率 |
| 节点文件系统使用量 | 每个节点的磁盘使用量 |
| 节点磁盘 I/O | 读写操作 |
| 节点就绪状态 | 节点是否就绪 |

### 容器指标

| 指标 | 描述 |
|------|------|
| 容器重启次数 | 容器重启次数 |
| 容器 CPU/内存限制 | 资源限制 |
| 容器 CPU/内存请求 | 资源请求 |
| 容器就绪状态 | 容器是否就绪 |

### 工作负载指标

| 指标 | 描述 |
|------|------|
| Deployment 可用/不可用副本数 | 副本数量 |
| DaemonSet 调度失败节点数 | 调度问题 |
| StatefulSet 就绪副本数 | 就绪副本数 |
| Job 活跃/失败/成功 Pod 数 | Job 状态 |

## 监控标准

### 可用检查类型

| 检查类型 | 描述 |
|---------|------|
| 指标值 | 配置的指标查询或公式的值 |

### 聚合类型

| 聚合 | 描述 |
|------|------|
| 平均值 | 时间窗口内的平均值 |
| 求和 | 所有值的总和 |
| 最大值 | 时间窗口内的最高值 |
| 最小值 | 时间窗口内的最低值 |
| 所有值 | 所有值必须满足标准 |
| 任意值 | 至少一个值必须满足标准 |

### 过滤类型

- **大于**、**小于**、**大于或等于**、**小于或等于**、**等于**、**不等于**

## 预置告警模板

OneUptime 为常见的 Kubernetes 监控场景提供模板：

| 模板 | 描述 | 阈值 |
|------|------|------|
| CrashLoopBackOff 检测 | 容器重启次数 | > 5 次重启 |
| Pod 卡在 Pending 状态 | Pending 阶段的 Pod | > 0 个 Pod |
| 节点未就绪 | 节点就绪状态 | = 0（未就绪） |
| 高节点 CPU | 节点 CPU 利用率 | > 90% |
| 高节点内存 | 节点内存利用率 | > 85% |
| Deployment 副本不匹配 | 不可用副本数 | > 0 个副本 |
| Job 失败 | Job 中失败的 Pod | > 0 次失败 |
| etcd 无领导者 | etcd 集群领导者缺失 | = 0（无领导者） |
| API 服务器限速 | 丢弃的 API 请求 | > 0 个请求 |
| 调度器积压 | 调度器中的待处理 Pod | > 0 个 Pod |
| 高节点磁盘使用率 | 节点文件系统使用率 | > 90% |
| DaemonSet 不可用 | 调度失败的节点 | > 0 个节点 |

## 设置要求

要使用 Kubernetes 监控，您需要在集群中安装 OneUptime Kubernetes Agent。该 Agent 通过 OTLP 将集群指标、事件和 Pod 日志发送到 OneUptime。

请参见[安装 Kubernetes Agent](/docs/monitor/kubernetes-agent) 指南——它涵盖了一条命令的 Helm 安装以及用于为您的集群（standard、GKE Autopilot、EKS Fargate）选择正确配置的 `preset` 选项。
