# Push-уведомления

Нативные push-уведомления (iOS/Android) работают на основе **Expo Push** и **не требуют серверной настройки** для самостоятельно размещённых экземпляров.

## Принцип работы

Мобильное приложение OneUptime регистрирует токен Expo Push в бэкенде. Когда бэкенду необходимо отправить уведомление, он выполняет POST-запрос к публичному Expo Push API, который маршрутизирует сообщение в Apple APNs или Google FCM от имени приложения.

Веб-push-уведомления по-прежнему используют ключи VAPID и протокол Web Push.

## Настройка для самостоятельного хостинга

Настройка push-уведомлений не требуется. Бинарный файл мобильного приложения автоматически обрабатывает регистрацию на платформе через инфраструктуру push-уведомлений Expo.

## Устранение неполадок

### Push-уведомления не поступают

- Убедитесь, что мобильное приложение собрано с помощью EAS Build (Expo Go не поддерживает push-уведомления)
- Проверьте наличие устройства в таблице `UserPush` вашей базы данных
- Проверьте журналы сервера OneUptime на наличие ошибок Expo Push API
- Убедитесь, что на устройстве есть активное интернет-соединение и включены разрешения уведомлений

### Ошибки «DeviceNotRegistered» в журналах

Токен Expo Push больше недействителен. Как правило, это означает, что приложение было удалено или пользователь отозвал разрешения на уведомления. Токен будет автоматически очищен.
