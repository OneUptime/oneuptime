# Интеграция входящей электронной почты SendGrid

**Монитор входящей электронной почты** OneUptime позволяет создавать и устранять оповещения на основе писем, отправляемых на уникальные адреса электронной почты, привязанные к конкретному монитору. Это удобно для интеграции с устаревшими системами, инструментами оповещения или любыми сервисами, поддерживающими отправку email.

В данном руководстве описана настройка SendGrid Inbound Parse для пересылки входящих писем на ваш самостоятельно размещённый экземпляр OneUptime.

## Предварительные требования

- Учётная запись SendGrid (подходит бесплатный тариф)
- Домен, которым вы управляете, с доступом к настройкам DNS
- Ваш экземпляр OneUptime должен быть публично доступен (для отправки вебхуков SendGrid)

## Принцип работы

1. Вы создаёте **монитор входящей электронной почты** в OneUptime
2. OneUptime генерирует уникальный адрес для этого монитора (например, `monitor-abc123@inbound.yourdomain.com`)
3. При получении письма на этот адрес SendGrid принимает его и пересылает в OneUptime через вебхук
4. OneUptime оценивает письмо по заданным критериям для создания или устранения оповещений

## Инструкции по настройке

### Шаг 1: Выбор домена для входящей электронной почты

Вам понадобится поддомен, выделенный для приёма входящих писем. Рекомендуется использовать поддомен вида:

- `inbound.yourdomain.com`
- `email.yourdomain.com`
- `monitor.yourdomain.com`

Этот поддомен будет использоваться исключительно для email-адресов мониторов OneUptime.

### Шаг 2: Настройка DNS MX-записи

Добавьте MX-запись в конфигурацию DNS для маршрутизации писем вашего входящего поддомена на SendGrid.

| Тип | Хост/Имя | Приоритет | Значение |
|-----|----------|-----------|---------|
| MX | inbound | 10 | mx.sendgrid.net |

**Пример:** Если ваш домен `example.com` и используется `inbound.example.com`:

```
inbound.example.com.  IN  MX  10  mx.sendgrid.net.
```

**Примечание:** Изменения DNS могут занять до 48 часов для распространения, но обычно завершаются за несколько часов.

### Шаг 3: Верификация домена в SendGrid (необязательно, но рекомендуется)

Для улучшения доставляемости и предотвращения попадания писем в спам:

1. Войдите в [панель управления SendGrid](https://app.sendgrid.com)
2. Перейдите в **Настройки** > **Аутентификация отправителя**
3. Нажмите **Аутентифицировать ваш домен**
4. Следуйте инструкциям для добавления необходимых DNS-записей (CNAME для DKIM)

### Шаг 4: Настройка SendGrid Inbound Parse

1. Войдите в [панель управления SendGrid](https://app.sendgrid.com)
2. Перейдите в **Настройки** > **Inbound Parse**
3. Нажмите **Add Host & URL**
4. Настройте следующее:

| Поле | Значение |
|------|---------|
| **Receiving Domain** | Ваш входящий поддомен (например, `inbound.yourdomain.com`) |
| **Destination URL** | `https://your-oneuptime-domain.com/incoming-email/sendgrid/YOUR_SECRET` |
| **Check incoming emails for spam** | Необязательно — включите при желании |
| **Send raw, full MIME message** | Оставьте отключённым (не требуется) |
| **POST the raw, full MIME message** | Оставьте отключённым (не требуется) |

5. Нажмите **Add**

### Шаг 5: Настройка переменных среды OneUptime

#### Docker Compose

Добавьте следующие переменные среды в файл `config.env`:

```bash
# Конфигурация входящей электронной почты
INBOUND_EMAIL_PROVIDER=SendGrid
INBOUND_EMAIL_DOMAIN=inbound.yourdomain.com
# INBOUND_EMAIL_WEBHOOK_SECRET=your-optional-secret  # Необязательно: для дополнительной безопасности
```

#### Kubernetes с Helm

Добавьте следующие значения в файл `values.yaml`:

```yaml
inboundEmail:
  provider: "SendGrid"
  domain: "inbound.yourdomain.com"
  # webhookSecret: "your-optional-secret"  # Необязательно
```

**Важно:** Перезапустите сервер OneUptime после добавления переменных среды.

### Шаг 6: Создание монитора входящей электронной почты

1. Войдите на панель управления OneUptime
2. Перейдите в **Мониторы** > **Создать монитор**
3. Выберите тип **Входящая электронная почта**
4. Настройте монитор:
   - **Имя:** задайте описательное название
   - **Описание:** опишите назначение монитора
5. Настройте **Критерии создания оповещения** (при каких условиях создавать оповещение):
   - Например: тема письма содержит «ALERT» или «CRITICAL»
6. Настройте **Критерии устранения оповещения** (при каких условиях устранять оповещение):
   - Например: тема письма содержит «RESOLVED» или «OK»
7. Нажмите **Создать**

После создания будет отображён уникальный адрес для данного монитора (например, `monitor-abc123def456@inbound.yourdomain.com`).

### Шаг 7: Тестирование интеграции

1. Скопируйте адрес монитора с панели управления OneUptime
2. Отправьте тестовое письмо на этот адрес с темой, соответствующей критериям оповещения
3. Проверьте панель управления OneUptime:
   - Письмо получено (видно в сводке монитора)
   - Оповещение создано (если критерии совпали)

## Справочник переменных среды

| Переменная | Описание | Обязательно | По умолчанию |
|------------|----------|-------------|--------------|
| `INBOUND_EMAIL_PROVIDER` | Используемый поставщик входящей электронной почты | Да | — |
| `INBOUND_EMAIL_DOMAIN` | Поддомен, настроенный для входящих писем | Да | — |
| `INBOUND_EMAIL_WEBHOOK_SECRET` | Секрет для валидации вебхук-запросов. При задании добавьте его к URL вебхука: `/incoming-email/sendgrid/YOUR_SECRET` | Нет | — |

## Поддерживаемые критерии для электронной почты

При настройке монитора входящей электронной почты можно создавать критерии на основе:

| Поле | Описание | Доступные фильтры |
|------|----------|-------------------|
| **Тема письма** | Строка темы письма | Содержит, Не содержит, Равно, Не равно, Начинается с, Заканчивается на, Пустое, Не пустое |
| **Отправитель письма** | Адрес электронной почты отправителя | Содержит, Не содержит, Равно, Не равно, Начинается с, Заканчивается на, Пустое, Не пустое |
| **Тело письма** | Текстовое тело письма | Содержит, Не содержит, Равно, Не равно, Начинается с, Заканчивается на, Пустое, Не пустое |
| **Получатель письма** | Адрес электронной почты получателя | Содержит, Не содержит, Равно, Не равно, Начинается с, Заканчивается на, Пустое, Не пустое |
| **Письмо получено** | Время с момента получения последнего письма | Получено в течение минут, Не получено в течение минут |

## Примеры использования

### Оповещения от устаревших систем

Многие устаревшие системы могут только отправлять email-оповещения. Создайте монитор входящей электронной почты для:
- Создания оповещений OneUptime при получении писем `[CRITICAL]` от устаревшей системы
- Устранения оповещений при получении писем `[RESOLVED]`

### Интеграция со сторонними сервисами

Интеграция с сервисами, отправляющими email-уведомления:
- Инструменты мониторинга без API-интеграций
- Уведомления облачных поставщиков
- Инструменты сканирования безопасности

### Мониторинг пульса через электронную почту

Используйте критерий «Письмо получено» для контроля периодических писем:
- Создание оповещения при отсутствии писем в течение 60 минут
- Полезно для мониторинга пакетных заданий или запланированных задач, отправляющих письма о завершении

## Устранение неполадок

### Письма не поступают

1. **Проверьте распространение DNS:**
   ```bash
   dig MX inbound.yourdomain.com
   ```
   Должно вернуть `mx.sendgrid.net`

2. **Проверьте настройки SendGrid Inbound Parse:**
   - Войдите в панель управления SendGrid
   - Перейдите в Настройки > Inbound Parse
   - Убедитесь в правильности домена и URL вебхука

3. **Проверьте журналы OneUptime:**
   - Ищите вебхук-запросы в журналах сервиса ProbeIngest
   - Проверьте наличие сообщений об ошибках

### Вебхуки не срабатывают

1. **Убедитесь в публичной доступности OneUptime:**
   - URL вебхука должен быть доступен из интернета
   - Проверьте: `curl -X POST https://your-oneuptime-domain.com/incoming-email/sendgrid`

2. **Проверьте правила брандмауэра:**
   - Разрешите входящий HTTPS-трафик с IP-диапазонов SendGrid

3. **Проверьте SSL-сертификат:**
   - SendGrid требует действительный SSL-сертификат
   - Самоподписанные сертификаты могут вызвать проблемы

### Монитор не создаёт оповещения

1. **Проверьте конфигурацию критериев:**
   - Убедитесь, что критерии создания оповещений соответствуют содержимому письма
   - Сначала тестируйте с точными совпадениями строк, затем переходите к сопоставлению по шаблону

2. **Проверьте статус монитора:**
   - Убедитесь, что монитор не отключён
   - Убедитесь, что тип монитора — «Входящая электронная почта»

3. **Просмотрите сводку монитора:**
   - Проверьте, получено ли письмо и обработано ли оно
   - Просмотрите журналы оценки для проверки соответствия критериев

### Журналы доставки вебхуков SendGrid

Для проверки успешной отправки вебхуков SendGrid:

1. К сожалению, SendGrid не предоставляет подробных журналов для Inbound Parse
2. Проверьте журналы вашего сервера OneUptime на наличие входящих вебхук-запросов
3. Временно используйте инструмент вроде [RequestBin](https://requestbin.com) для тестирования доставки вебхуков

## Рекомендации по безопасности

1. **Используйте HTTPS:** всегда используйте HTTPS для конечной точки вебхука
2. **Секрет вебхука:** настройте `INBOUND_EMAIL_WEBHOOK_SECRET` и включите его в URL вебхука (например, `/incoming-email/sendgrid/your-secret`) для дополнительной валидации
3. **Верификация домена:** верифицируйте ваш домен в SendGrid для улучшения безопасности email
4. **Ограничивайте доступ:** создавайте мониторы только для доверенных источников электронной почты
5. **Мониторинг журналов:** регулярно просматривайте журналы входящих писем на наличие подозрительной активности
