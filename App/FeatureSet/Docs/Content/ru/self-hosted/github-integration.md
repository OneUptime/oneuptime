# Интеграция с GitHub

Для интеграции GitHub с вашим самостоятельно размещённым экземпляром OneUptime необходимо создать GitHub App и настроить необходимые переменные среды. Это позволяет OneUptime подключаться к вашим репозиториям GitHub для управления кодовой базой.

## Предварительные требования

- Учётная запись GitHub с правами администратора организации (для репозиториев организации) или доступом к личной учётной записи
- Доступ к конфигурации вашего сервера OneUptime

## Инструкции по настройке

### Шаг 1: Создание GitHub App

1. Перейдите на GitHub и откройте настройки вашей организации или личной учётной записи:
   - **Для организаций:** перейдите на `https://github.com/organizations/YOUR_ORG/settings/apps`
   - **Для личной учётной записи:** перейдите на `https://github.com/settings/apps`

2. Нажмите **«New GitHub App»**

3. Заполните форму регистрации:
   - **GitHub App name:** OneUptime (или любое уникальное имя) — **Запомните это имя, оно понадобится для переменной среды `GITHUB_APP_NAME`**
   - **Homepage URL:** `https://your-oneuptime-domain.com`
   - **Callback URL:** `https://your-oneuptime-domain.com/api/github/auth/callback`
   - **Setup URL:** `https://your-oneuptime-domain.com/api/github/auth/callback` — **Важно: на этот URL GitHub перенаправляет пользователей после установки приложения. Он должен быть задан для работы перенаправления.**
   - **Redirect on update:** установите этот флажок для перенаправления пользователей после обновления установки приложения
   - **Webhook URL:** `https://your-oneuptime-domain.com/api/github/webhook`
   - **Webhook secret:** сгенерируйте случайную строку (сохраните на потом)

### Шаг 2: Настройка разрешений приложения

В разделе «Permissions & events» настройте следующие разрешения:

**Разрешения для репозиториев:**

| Разрешение | Уровень доступа | Назначение |
|------------|-----------------|------------|
| Contents | Чтение и запись | Чтение файлов репозитория, отправка веток (требуется для AI-агента) |
| Pull requests | Чтение и запись | Создание и управление запросами на слияние |
| Issues | Чтение и запись | Чтение задач и добавление комментариев |
| Commit statuses | Чтение | Проверка статуса сборки/CI |
| Actions | Чтение | Чтение запусков и журналов рабочих процессов GitHub Actions |
| Metadata | Чтение | Базовые метаданные репозитория (обязательно) |

**Разрешения для организаций (при использовании с организациями):**

| Разрешение | Уровень доступа | Назначение |
|------------|-----------------|------------|
| Members | Чтение | Список участников организации |

**Разрешения для учётной записи:**

| Разрешение | Уровень доступа | Назначение |
|------------|-----------------|------------|
| Email addresses | Чтение | Чтение email пользователя для уведомлений |

### Шаг 3: Подписка на события вебхука

События для получения OneUptime обновлений в реальном времени — подпишитесь на следующие события вебхука:

- **Pull request** — получение уведомлений при открытии, закрытии или слиянии PR
- **Push** — получение уведомлений при отправке кода
- **Workflow run** — получение обновлений статуса CI/CD

### Шаг 4: Установка доступа для установки

В разделе «Where can this GitHub App be installed?» выберите:
- **Only on this account** — для приватного/внутреннего использования
- **Any account** — если другие должны иметь возможность устанавливать ваше приложение

### Шаг 5: Создание GitHub App

1. Нажмите **«Create GitHub App»**
2. Вы будете перенаправлены на страницу настроек приложения
3. Запишите следующие значения:
   - **App ID** — находится в верхней части страницы настроек приложения
   - **Client ID** — находится в разделе «About»

### Шаг 6: Генерация секрета клиента

1. В настройках GitHub App прокрутите до раздела «Client secrets»
2. Нажмите **«Generate a new client secret»**
3. Немедленно скопируйте секрет — просмотреть его повторно не получится

### Шаг 7: Генерация закрытого ключа

1. Прокрутите до раздела «Private keys»
2. Нажмите **«Generate a private key»**
3. Файл `.pem` будет загружен автоматически
4. Храните этот файл в безопасном месте — он используется для аутентификации как GitHub App

### Шаг 8: Настройка переменных среды OneUptime

#### Docker Compose

При использовании Docker Compose добавьте следующие переменные среды в файл `config.env`:

```bash
# Конфигурация GitHub App
GITHUB_APP_ID=YOUR_APP_ID
GITHUB_APP_NAME=YOUR_APP_NAME  # Точное имя вашего GitHub App (например, «OneUptime»)
GITHUB_APP_CLIENT_ID=YOUR_CLIENT_ID
GITHUB_APP_CLIENT_SECRET=YOUR_CLIENT_SECRET
GITHUB_APP_PRIVATE_KEY="<BASE64_ENCODED_PRIVATE_KEY_CONTENT>"
GITHUB_APP_WEBHOOK_SECRET=YOUR_WEBHOOK_SECRET
```

**Примечание:** Если ваша среда не поддерживает многострочные строки, закодируйте закрытый ключ в base64 и вставьте без переносов строк.

#### Kubernetes с Helm

При использовании Kubernetes с Helm добавьте следующие значения в файл `values.yaml`:

```yaml
gitHubApp:
  id: "YOUR_APP_ID"
  name: "YOUR_APP_NAME"  # Точное имя вашего GitHub App
  clientId: "YOUR_CLIENT_ID"
  clientSecret: "YOUR_CLIENT_SECRET"
  privateKey: "<BASE64_ENCODED_PRIVATE_KEY_CONTENT>"
  webhookSecret: "YOUR_WEBHOOK_SECRET"
```

**Важно:** Перезапустите сервер OneUptime после добавления переменных среды, чтобы изменения вступили в силу.

### Шаг 9: Установка GitHub App

1. Перейдите на публичную страницу вашего GitHub App: `https://github.com/apps/YOUR_APP_NAME`
2. Нажмите **«Install»** или **«Configure»**
3. Выберите организацию или учётную запись для установки приложения
4. Выберите, к каким репозиториям будет иметь доступ приложение:
   - **All repositories** — доступ ко всем текущим и будущим репозиториям
   - **Only select repositories** — выбор конкретных репозиториев
5. Нажмите **«Install»**

### Шаг 10: Подключение репозиториев в OneUptime

1. Войдите на панель управления OneUptime
2. Перейдите в **Ещё** > **Репозитории кода**
3. Нажмите **«Создать репозиторий»** или воспользуйтесь потоком установки GitHub App
4. Если перенаправлены с GitHub, идентификатор установки будет захвачен автоматически
5. Выберите репозитории для подключения из списка
6. Нажмите **«Подключить»** для привязки репозитория к вашему проекту OneUptime

## Справочник переменных среды

| Переменная | Описание | Обязательно |
|------------|----------|-------------|
| `GITHUB_APP_ID` | App ID из настроек GitHub App | Да |
| `GITHUB_APP_NAME` | Точное имя GitHub App (используется для URL установки) | Да |
| `GITHUB_APP_CLIENT_ID` | Client ID из настроек GitHub App | Да |
| `GITHUB_APP_CLIENT_SECRET` | Сгенерированный секрет клиента | Да |
| `GITHUB_APP_PRIVATE_KEY` | Содержимое файла закрытого ключа (.pem) | Да |
| `GITHUB_APP_WEBHOOK_SECRET` | Секрет вебхука для верификации полезных нагрузок | Нет (рекомендуется) |

## Устранение неполадок

### Распространённые проблемы

**После установки GitHub App нет перенаправления обратно в OneUptime:**
- Убедитесь, что **Setup URL** настроен в настройках GitHub App: `https://your-oneuptime-domain.com/api/github/auth/callback`
- Перейдите в настройки GitHub App > раздел «Post installation» и проверьте правильность Setup URL
- Параметр «Redirect on update» также должен быть включён
- Примечание: Setup URL отличается от Callback URL — оба должны указывать на конечную точку `/api/github/auth/callback`

**Ошибка «GitHub App is not configured»:**
- Убедитесь, что переменная среды `GITHUB_APP_CLIENT_ID` задана
- Перезапустите сервер OneUptime после установки переменных среды

**Ошибка «Invalid webhook signature»:**
- Убедитесь, что `GITHUB_APP_WEBHOOK_SECRET` совпадает с секретом, настроенным в GitHub
- Убедитесь в правильности URL вебхука и его доступности из интернета

**Ошибка «Failed to get installation access token»:**
- Убедитесь в правильном форматировании `GITHUB_APP_PRIVATE_KEY`
- Проверьте, что закрытый ключ содержит маркеры BEGIN/END
- Убедитесь в правильности App ID

**Репозитории не отображаются после установки:**
- Убедитесь, что GitHub App имеет доступ к нужным репозиториям
- Проверьте разрешения установки в GitHub (Настройки > Applications > Installed GitHub Apps)

**События вебхука не поступают:**
- Убедитесь, что URL вебхука доступен из публичного интернета
- Проверьте журналы доставки вебхука в настройках GitHub App
- Убедитесь в правильности настройки секрета вебхука

### Проверка доставки вебхуков

1. Перейдите в настройки GitHub App
2. Нажмите «Advanced» в боковой панели
3. Просмотрите «Recent Deliveries» для проверки попыток вебхука и ответов

## Рекомендации по безопасности

1. **Регулярно обновляйте секреты** — периодически генерируйте новые секреты клиента и закрытые ключи
2. **Используйте секреты вебхука** — всегда настраивайте секрет вебхука для верификации подлинности полезных нагрузок
3. **Ограничивайте доступ к репозиториям** — предоставляйте доступ только к репозиториям, которые необходимо подключить
4. **Отслеживайте доставку вебхуков** — регулярно проверяйте наличие неудачных доставок или подозрительной активности
5. **Надёжно храните закрытые ключи** — никогда не фиксируйте закрытые ключи в системе контроля версий
