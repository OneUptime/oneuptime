# Интеграция с Microsoft Teams

Для интеграции Microsoft Teams с вашим самостоятельно размещённым экземпляром OneUptime необходимо настроить регистрацию приложения Azure и задать необходимые переменные среды.

## Предварительные требования

- Учётная запись Azure — создайте на [https://azure.com](https://azure.com)
- Доступ к конфигурации вашего сервера OneUptime

## Инструкции по настройке

### Шаг 1: Создание регистрации приложения Azure

1. Перейдите на [портал Azure](https://portal.azure.com)
2. Откройте «Регистрации приложений» и нажмите «Новая регистрация»
3. Заполните форму регистрации:
   - **Имя:** oneuptime
   - **Поддерживаемые типы учётных записей:** Учётные записи в любом каталоге организации (Any Microsoft Entra ID tenant - Multitenant)
   - **URI перенаправления:** Web - `https://your-oneuptime-domain.com/api/microsoft-teams/auth`
   - Также добавьте: `https://your-oneuptime-domain.com/api/microsoft-teams/admin-consent/callback`
4. Нажмите «Зарегистрировать»
5. Запишите «Идентификатор приложения (клиента)» — он понадобится позже

### Шаг 2: Настройка разрешений приложения

1. В регистрации приложения перейдите в «Разрешения API»
2. Нажмите «Добавить разрешение» и выберите «Microsoft Graph»

**Добавьте делегированные разрешения** (при действии от имени вошедшего пользователя):

- **User.Read** — необходимо для получения профиля аутентифицированного пользователя (отображаемое имя, email) в процессе OAuth
- **Team.ReadBasic.All** — необходимо для просмотра команд, в которых состоит пользователь, при выборе команды для подключения
- **Channel.ReadBasic.All** — необходимо для чтения информации о каналах и их списков для доставки уведомлений
- **ChannelMessage.Send** — необходимо для отправки оповещений и уведомлений об инцидентах в каналы Teams

**Добавьте разрешения для приложения** (при действии от имени самого приложения без вошедшего пользователя):

- **Team.ReadBasic.All** — необходимо для просмотра всех команд в организации после предоставления согласия администратора
- **Channel.ReadBasic.All** — необходимо для проверки существования канала и получения его сведений
- **ChannelMessage.Send** — необходимо для программной отправки сообщений в каналы

**Примечание:** Bot Framework обрабатывает доставку сообщений с использованием разрешений Resource-Specific Consent (RSC), определённых в манифесте приложения Teams:

- **ChannelMessage.Send.Group** — позволяет боту отправлять сообщения в каналы команды
- **ChannelMessage.Read.Group** — позволяет боту читать сообщения в каналах для интерактивных команд
- **Channel.Create.Group** — позволяет боту создавать каналы при необходимости

3. Нажмите «Предоставить согласие администратора» для вашей организации

### Шаг 3: Создание секрета клиента

1. Перейдите в «Сертификаты и секреты» в регистрации приложения
2. Нажмите «Новый секрет клиента»
3. Добавьте описание и установите срок действия (рекомендуется 24 месяца)
4. Нажмите «Добавить» и немедленно скопируйте значение секрета — просмотреть его повторно не получится

**Важно:** Скопируйте значение секрета, а не его идентификатор — значение обычно длиннее и содержит больше символов.

### Шаг 4: Создание службы Bot

1. На портале Azure откройте «Azure Bot» и нажмите «Создать»
2. Заполните форму создания бота:

   - **Bot handle:** oneuptime-bot
   - **Подписка:** ваша подписка Azure
   - **Группа ресурсов:** создайте новую или используйте существующую
   - **Расположение:** выберите ближайшее к пользователям
   - **Ценовая категория:** F0 (Бесплатно) достаточно для тестирования
   - Используйте идентификатор приложения (клиента) и идентификатор клиента из регистрации приложения, созданной ранее

3. Нажмите «Проверить + создать», затем «Создать»

4. После развёртывания откройте ресурс бота и перейдите в «Конфигурация»
5. Установите «Конечная точка для обмена сообщениями» на `https://your-oneuptime-domain.com/api/microsoft-bot/messages`
6. Сохраните конфигурацию

### Шаг 5: Добавление канала Microsoft Teams к боту

1. В ресурсе Azure Bot перейдите в «Каналы»
2. Найдите и выберите «Microsoft Teams», нажмите «Открыть» или «Добавить»
3. Просмотрите параметры (включите для Teams, оставьте параметры обмена сообщениями по умолчанию, если нет особых требований)
4. Нажмите «Сохранить» (и «Готово»/«Опубликовать» при запросе) для включения канала Teams

### Шаг 6: Настройка переменных среды OneUptime

#### Docker Compose

При использовании Docker Compose добавьте следующие переменные среды в вашу конфигурацию:

```bash
MICROSOFT_TEAMS_APP_CLIENT_ID=YOUR_TEAMS_APP_CLIENT_ID
MICROSOFT_TEAMS_APP_CLIENT_SECRET=YOUR_TEAMS_APP_CLIENT_SECRET
MICROSOFT_TEAMS_APP_TENANT_ID=YOUR_MICROSOFT_TENANT_ID
```

#### Kubernetes с Helm

При использовании Kubernetes с Helm добавьте следующие значения в файл `values.yaml`:

```yaml
microsoftTeamsApp:
  clientId: YOUR_TEAMS_APP_CLIENT_ID
  clientSecret: YOUR_TEAMS_APP_CLIENT_SECRET
   tenantId: YOUR_MICROSOFT_TENANT_ID
```

**Важно:** Перезапустите сервер OneUptime после добавления переменных среды, чтобы изменения вступили в силу.

### Шаг 7: Загрузка манифеста приложения Teams

1. Перейдите в **Настройки** проекта > **Интеграции** > **Microsoft Teams**
2. Загрузите манифест приложения Teams оттуда
3. Откройте Microsoft Teams и нажмите «Приложения» на боковой панели
4. Внизу нажмите «Управление приложениями»
5. Нажмите «Загрузить пользовательское приложение»
6. Выберите «Загрузить для меня или моих команд»
7. Загрузите ранее скачанный zip-файл манифеста

## Устранение неполадок

При возникновении проблем:

- Убедитесь, что приложению предоставлены необходимые разрешения
- Проверьте точное совпадение URI перенаправления (замените `your-oneuptime-domain.com` на ваш реальный домен)
- Убедитесь в правильности переменных среды
- Убедитесь, что конечная точка обмена сообщениями бота доступна из интернета
- Убедитесь в правильности настройки бота с каналом Teams
- Убедитесь, что манифест приложения Teams был успешно загружен
