# Руководство по установке для Android

Установите нативное Android-приложение **OneUptime On-Call** из Google Play Store или установите APK напрямую на устройствах без Google Play.

## Требования

- Телефон или планшет Android с **Android 8.0 (Oreo) или более поздней версией**
- Активная учётная запись OneUptime (или URL вашего self-hosted экземпляра OneUptime)
- Подключение к интернету для входа в систему и получения push-уведомлений

## Вариант 1: Установка из Google Play (рекомендуется)

1. Откройте **Google Play Store** на вашем устройстве.
2. Введите в поиске **«OneUptime On-Call»** или откройте эту ссылку на вашем устройстве:
   [https://play.google.com/store/apps/details?id=com.oneuptime.oncall](https://play.google.com/store/apps/details?id=com.oneuptime.oncall)
3. Нажмите **Установить**.
4. После установки нажмите **Открыть** или запустите **OneUptime On-Call** из меню приложений.

## Вариант 2: Прямая установка APK

Для устройств без Google Play (например, GrapheneOS, /e/OS или устройств Huawei) установите официальный APK из GitHub Releases:

1. На вашем Android-устройстве откройте эту ссылку:
   [https://github.com/OneUptime/oneuptime/releases/latest/download/oneuptime-on-call-android-app.apk](https://github.com/OneUptime/oneuptime/releases/latest/download/oneuptime-on-call-android-app.apk)
2. Когда появится запрос, разрешите вашему браузеру устанавливать неизвестные приложения:
   **Настройки → Приложения → \[Ваш браузер\] → Установка неизвестных приложений → Разрешить из этого источника**.
3. Откройте загруженный APK и нажмите **Установить**.
4. Запустите **OneUptime On-Call** из меню приложений.

APK собирается и подписывается OneUptime из того же исходного кода, что и версия в Play Store. Обновления приложения не происходят автоматически при установке через APK — загружайте последний APK по ссылке выше, когда выходит новая версия.

## Первый запуск и вход в систему

1. **URL сервера**
   - Если вы используете OneUptime Cloud, оставьте значение по умолчанию `https://oneuptime.com`.
   - Если вы используете self-hosted установку, введите URL вашего экземпляра OneUptime (например, `https://oneuptime.example.com`).
   - Приложение проверит доступность сервера перед продолжением.
2. **Вход**
   - Введите электронную почту и пароль вашей учётной записи OneUptime.
   - При желании включите **биометрическую разблокировку** (отпечаток пальца) для более быстрой разблокировки при последующих запусках.
3. **Разрешение уведомлений**
   - Когда появится запрос, нажмите **Разрешить**, чтобы приложение могло доставлять дежурные оповещения, уведомления об инцидентах и подтверждения.

## Push-уведомления

Push-уведомления доставляются через Firebase Cloud Messaging (FCM) посредством Expo Push. Чтобы оповещения надёжно доходили до вас во время дежурства:

1. Откройте **Настройки → Приложения → OneUptime On-Call → Уведомления** и убедитесь, что все категории включены.
2. Откройте **Настройки → Приложения → OneUptime On-Call → Батарея** и выберите **Без ограничений** (или отключите оптимизацию батареи), чтобы ОС не задерживала фоновые push-уведомления.
3. Разрешите приложению работать в фоновом режиме и отключите для него любые ограничения «Экономии трафика».
4. Если вы используете устройства Samsung, также отключите **Настройки → Обслуживание устройства → Батарея → Ограничения фоновой работы** для OneUptime On-Call.
5. Добавьте OneUptime On-Call в списки исключений **режима «Не беспокоить»**, чтобы оповещения по-прежнему звонили во время вашей смены дежурства.

## Обновления

**Google Play:**

- Обновления устанавливаются автоматически. Чтобы запустить обновление вручную, откройте **Play Store → Профиль → Управление приложениями и устройством → Доступные обновления → OneUptime On-Call → Обновить**.

**Установка APK вручную:**

- Повторно загрузите последний APK по ссылке GitHub Releases выше и установите поверх существующего приложения — ваши данные, URL сервера и вход в систему сохранятся.

## Удаление

1. **Удерживайте** значок **OneUptime On-Call**, затем нажмите **Удалить**.
2. Или откройте **Настройки → Приложения → OneUptime On-Call → Удалить**.
3. Подтвердите удаление приложения.

Ваша учётная запись OneUptime и графики дежурств хранятся на стороне сервера и не удаляются при удалении приложения.

## Устранение неполадок

**«Ошибка сети» при входе:**

- Убедитесь, что **URL сервера** указан правильно и доступен с вашего устройства.
- Если вы находитесь в корпоративной сети или используете VPN, убедитесь, что экземпляр OneUptime доступен.
- Убедитесь, что сервер обслуживается по HTTPS с действительным сертификатом.

**Не приходят push-уведомления:**

- Убедитесь, что уведомления включены в **Настройки → Приложения → OneUptime On-Call → Уведомления**.
- Отключите оптимизацию батареи для OneUptime On-Call (см. раздел «Push-уведомления» выше).
- Убедитесь, что режим «Не беспокоить» отключён или OneUptime On-Call находится в списке исключений.
- Выйдите из системы и войдите снова, чтобы обновить push-токен, зарегистрированный на сервере.
- Пользователи self-hosted установок: убедитесь, что push-уведомления настроены в вашем экземпляре OneUptime (см. руководство [Push Notifications](/docs/self-hosted/push-notifications) для self-hosted установок).

**Биометрическая разблокировка не работает:**

- Зарегистрируйте отпечаток пальца в **Настройки → Безопасность → Отпечаток пальца**.
- Снова включите биометрическую разблокировку на экране **Настройки** внутри приложения OneUptime On-Call.

**Установка APK заблокирована:**

- Вы должны предоставить браузеру разрешение на установку неизвестных приложений (см. вариант 2 выше).
- Некоторые операторы связи или корпоративные профили устройств полностью блокируют установку APK; в этом случае используйте версию из Google Play.

**Приложение аварийно завершается при запуске:**

- Обновите приложение до последней версии из Google Play или установите последний APK.
- Перезагрузите устройство.
- Если проблема сохраняется, удалите и переустановите приложение, затем войдите снова.
