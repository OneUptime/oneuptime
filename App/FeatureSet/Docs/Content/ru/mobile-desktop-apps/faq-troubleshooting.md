# FAQ и устранение неполадок

Часто задаваемые вопросы и решения для мобильных и настольных приложений OneUptime.

## Как OneUptime распространяет свои приложения?

- **Мобильные (iOS и Android):** OneUptime поставляет нативное приложение под названием **OneUptime On-Call**. Оно опубликовано в [Apple App Store](https://apps.apple.com/us/app/oneuptime-on-call/id6759615391) и [Google Play](https://play.google.com/store/apps/details?id=com.oneuptime.oncall). Также доступна подписанная [загрузка APK](https://github.com/OneUptime/oneuptime/releases/latest/download/oneuptime-on-call-android-app.apk) для Android-устройств без Google Play.
- **Настольные (Windows, macOS, Linux):** Веб-панель управления OneUptime является Progressive Web App (PWA). Вы можете установить её как настольное приложение непосредственно из браузера на базе Chromium или Safari — учётная запись магазина не требуется.

## FAQ по мобильному приложению

### Какие устройства поддерживаются?

- **iOS:** iPhone или iPad с iOS 15.0 или более поздней версией.
- **Android:** Телефоны и планшеты с Android 8.0 (Oreo) или более поздней версией.

### Бесплатное ли приложение?

Да. Приложение OneUptime On-Call бесплатно для установки. Вы входите в систему с помощью существующей учётной записи OneUptime.

### Могу ли я использовать приложение с self-hosted экземпляром OneUptime?

Да. При первом запуске приложение запрашивает **URL сервера**. Введите URL вашего self-hosted экземпляра (например, `https://oneuptime.example.com`). Приложение проверит доступность сервера, прежде чем позволить вам войти в систему.

Для получения push-уведомлений на self-hosted экземплярах следуйте руководству [Push Notifications](/docs/self-hosted/push-notifications).

### Как доставляются обновления?

- **iOS:** Через App Store. Включите автоматические обновления в **Настройки → App Store** или обновляйте вручную из вашего профиля App Store.
- **Android (Google Play):** Автоматические обновления включены по умолчанию.
- **Android (установка APK вручную):** Загрузите и установите последний APK по ссылке GitHub Releases выше.

### Почему я не получаю push-уведомления?

Мобильные push-уведомления используют APNs (iOS) и FCM (Android) посредством Expo Push. Проверьте следующее:

1. Уведомления включены на уровне ОС для **OneUptime On-Call**.
2. Оптимизация батареи отключена и фоновая активность разрешена (Android).
3. Режимы «Не беспокоить» или Фокусирования отключены, либо приложение находится в списке исключений.
4. Вы вошли в систему — push-токен регистрируется на сервере только после входа.
5. **Только для self-hosted:** Push-уведомления настроены в вашем экземпляре OneUptime. См. руководство [Push Notifications](/docs/self-hosted/push-notifications).

### Безопасны ли данные на моём телефоне?

- Весь API-трафик использует HTTPS.
- Токены доступа и обновления хранятся в защищённом хранилище ключей устройства (Keychain на iOS, Keystore на Android).
- Вы можете требовать разблокировку по Face ID / Touch ID / отпечатку пальца на экране **Настройки** внутри приложения.

### Могу ли я установить приложение на нескольких устройствах?

Да. Войдите в систему с одной и той же учётной записью OneUptime на любом количестве устройств. Каждое устройство получает свои собственные push-уведомления.

### Как удалить приложение?

- **iOS:** Удерживайте значок → **Удалить приложение** → **Удалить приложение**.
- **Android:** Удерживайте значок → **Удалить** или **Настройки → Приложения → OneUptime On-Call → Удалить**.

Ваша учётная запись OneUptime и данные хранятся на сервере и не удаляются при удалении приложения.

## FAQ по настольному приложению (PWA)

### Что такое Progressive Web App (PWA)?

Progressive Web App — это веб-приложение, которое можно установить как нативное настольное приложение. После установки оно запускается в собственном окне, имеет свой значок в панели запуска и может доставлять настольные уведомления — без обращения к Windows Store, Mac App Store или другим каналам распространения.

### Почему настольное приложение использует технологию PWA?

- **Мгновенные обновления** — приложение остаётся синхронизированным с вашим экземпляром OneUptime сразу после развёртывания.
- **Учётная запись магазина не требуется** — устанавливайте напрямую из любого современного браузера.
- **Единая кодовая база** — одна и та же панель управления работает в Windows, macOS и Linux.

### Почему не отображается кнопка «Установить»?

1. Используйте браузер на базе Chromium (Chrome, Edge, Brave, Arc) или Safari (macOS Sonoma+).
2. Убедитесь, что ваш экземпляр OneUptime обслуживается по HTTPS с действительным сертификатом.
3. Очистите кэш браузера и перезагрузите страницу.
4. Приложение может быть уже установлено — проверьте раздел «Приложения» / меню «Пуск».

### Как обновить настольное приложение?

PWA обновляется автоматически каждый раз, когда вы открываете его при подключении к интернету. Чтобы принудительно обновить, обновите окно с помощью **Ctrl+R** (Windows/Linux) или **Cmd+R** (macOS).

### Как удалить настольное PWA?

- **Windows:** **Настройки → Приложения → OneUptime → Удалить** или щёлкните правой кнопкой мыши на записи в меню «Пуск».
- **macOS:** Перетащите приложение из **Программ** в Корзину или щёлкните правой кнопкой мыши на значке в Dock и выберите **Удалить**.
- **Linux:** Используйте опцию удаления в панели запуска приложений или удалите соответствующий файл `.desktop`.

## Устранение неполадок

### Проблемы с мобильным приложением

**Не удаётся войти в приложение / «Ошибка сети»:**
- Убедитесь, что **URL сервера** указан правильно и доступен с вашего телефона.
- Убедитесь, что ваш телефон подключён к интернету.
- Для self-hosted экземпляров за VPN убедитесь, что VPN активен.

**Push-уведомления задерживаются или отсутствуют (Android):**
- Отключите оптимизацию батареи: **Настройки → Приложения → OneUptime On-Call → Батарея → Без ограничений**.
- Отключите Экономию трафика для приложения.
- На устройствах Samsung отключите **Обслуживание устройства → Батарея → Ограничения фоновой работы** для OneUptime On-Call.

**Push-уведомления задерживаются или отсутствуют (iOS):**
- Не закрывайте приложение принудительно — iOS может приостанавливать фоновую доставку.
- Отключите Режим энергосбережения, пока вы находитесь на дежурстве.
- Добавьте OneUptime On-Call в список разрешений любого активного режима Фокусирования.

**Face ID / Touch ID / отпечаток пальца не работает:**
- Убедитесь, что биометрические данные зарегистрированы в настройках ОС.
- Снова включите биометрическую разблокировку на экране **Настройки** внутри приложения OneUptime On-Call.

### Проблемы с настольным приложением (PWA)

**Отсутствует кнопка «Установить»:**
- Используйте поддерживаемый браузер (на базе Chromium или Safari на macOS Sonoma+).
- Убедитесь, что экземпляр OneUptime обслуживается по HTTPS.
- Дождитесь полной загрузки страницы, затем проверьте адресную строку на наличие значка установки.

**Не появляются настольные уведомления:**
- Разрешите уведомления, когда браузер запросит их.
- Проверьте настройки уведомлений ОС (Focus Assist в Windows, Уведомления в macOS, демон уведомлений в Linux).
- Для self-hosted экземпляров убедитесь, что настройка [Push Notifications](/docs/self-hosted/push-notifications) завершена.

**Приложение не отображает последние данные:**
- Обновите с помощью **Ctrl+R** / **Cmd+R**.
- Закройте и снова откройте окно.
- Проверьте ваше сетевое подключение.
