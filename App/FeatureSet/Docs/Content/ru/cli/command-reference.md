# Справочник команд

Полный справочник по всем командам CLI OneUptime.

## Команды аутентификации

### `oneuptime login`

Аутентификация в экземпляре OneUptime.

```bash
oneuptime login <api-key> <instance-url> [--context-name <name>]
```

| Параметр         | Тип      | Обязательный | Описание                                  |
| ---------------- | -------- | ------------ | ----------------------------------------- |
| `<api-key>`      | аргумент | Да           | Ключ API для аутентификации               |
| `<instance-url>` | аргумент | Да           | URL экземпляра OneUptime                  |
| `--context-name` | опция    | Нет          | Имя контекста (по умолчанию: `"default"`) |

---

### `oneuptime context list`

Список всех сохранённых контекстов.

```bash
oneuptime context list
```

---

### `oneuptime context use`

Переключение на именованный контекст.

```bash
oneuptime context use <name>
```

| Параметр | Тип      | Обязательный | Описание                    |
| -------- | -------- | ------------ | --------------------------- |
| `<name>` | аргумент | Да           | Имя активируемого контекста |

---

### `oneuptime context current`

Отображение активного контекста со скрытым ключом API.

```bash
oneuptime context current
```

---

### `oneuptime context delete`

Удаление сохранённого контекста.

```bash
oneuptime context delete <name>
```

| Параметр | Тип      | Обязательный | Описание                 |
| -------- | -------- | ------------ | ------------------------ |
| `<name>` | аргумент | Да           | Имя удаляемого контекста |

---

## Команды ресурсов

Все команды ресурсов следуют одному шаблону. Замените `<resource>` на любое поддерживаемое имя ресурса (например, `incident`, `monitor`, `alert`, `status-page`).

### `oneuptime <resource> list`

Список ресурсов с фильтрацией и пагинацией.

```bash
oneuptime <resource> list [options]
```

| Опция            | Тип    | По умолчанию | Описание                            |
| ---------------- | ------ | ------------ | ----------------------------------- |
| `--query <json>` | строка | Нет          | Критерии фильтрации в формате JSON  |
| `--limit <n>`    | число  | `10`         | Максимальное количество результатов |
| `--skip <n>`     | число  | `0`          | Количество пропускаемых результатов |
| `--sort <json>`  | строка | Нет          | Порядок сортировки в формате JSON   |
| `-o, --output`   | строка | `table`      | Формат вывода                       |

---

### `oneuptime <resource> get`

Получение одного ресурса по ID.

```bash
oneuptime <resource> get <id> [-o <format>]
```

| Параметр       | Тип      | Обязательный | Описание          |
| -------------- | -------- | ------------ | ----------------- |
| `<id>`         | аргумент | Да           | ID ресурса (UUID) |
| `-o, --output` | опция    | Нет          | Формат вывода     |

---

### `oneuptime <resource> create`

Создание нового ресурса.

```bash
oneuptime <resource> create [--data <json> | --file <path>] [-o <format>]
```

| Опция           | Тип    | Обязательный                  | Описание                      |
| --------------- | ------ | ----------------------------- | ----------------------------- |
| `--data <json>` | строка | Одно из `--data` или `--file` | Данные ресурса в формате JSON |
| `--file <path>` | строка | Одно из `--data` или `--file` | Путь к файлу JSON             |
| `-o, --output`  | строка | Нет                           | Формат вывода                 |

---

### `oneuptime <resource> update`

Обновление существующего ресурса.

```bash
oneuptime <resource> update <id> --data <json> [-o <format>]
```

| Параметр        | Тип      | Обязательный | Описание                           |
| --------------- | -------- | ------------ | ---------------------------------- |
| `<id>`          | аргумент | Да           | ID ресурса                         |
| `--data <json>` | опция    | Да           | Поля для обновления в формате JSON |
| `-o, --output`  | опция    | Нет          | Формат вывода                      |

---

### `oneuptime <resource> delete`

Удаление ресурса.

```bash
oneuptime <resource> delete <id> [--force]
```

| Параметр  | Тип      | Обязательный | Описание                        |
| --------- | -------- | ------------ | ------------------------------- |
| `<id>`    | аргумент | Да           | ID ресурса                      |
| `--force` | опция    | Нет          | Пропустить запрос подтверждения |

---

### `oneuptime <resource> count`

Подсчёт ресурсов, соответствующих фильтру.

```bash
oneuptime <resource> count [--query <json>]
```

| Опция            | Тип    | По умолчанию | Описание                           |
| ---------------- | ------ | ------------ | ---------------------------------- |
| `--query <json>` | строка | Нет          | Критерии фильтрации в формате JSON |

---

## Служебные команды

### `oneuptime version`

Отображение версии CLI.

```bash
oneuptime version
```

---

### `oneuptime whoami`

Отображение текущих данных аутентификации.

```bash
oneuptime whoami
```

Отображает URL экземпляра и скрытый ключ API. Если активен сохранённый контекст, также отображается имя контекста.

---

### `oneuptime resources`

Список всех доступных типов ресурсов.

```bash
oneuptime resources [--type <type>]
```

| Опция           | Тип    | По умолчанию | Описание                                  |
| --------------- | ------ | ------------ | ----------------------------------------- |
| `--type <type>` | строка | Нет          | Фильтр по типу `database` или `analytics` |

---

## Глобальные опции

Эти флаги доступны для всех команд:

| Опция                   | Описание                               |
| ----------------------- | -------------------------------------- |
| `--api-key <key>`       | Переопределить ключ API                |
| `--url <url>`           | Переопределить URL экземпляра          |
| `--context <name>`      | Использовать конкретный контекст       |
| `-o, --output <format>` | Формат вывода: `json`, `table`, `wide` |
| `--no-color`            | Отключить цветной вывод                |
| `--help`                | Показать справку                       |
| `--version`             | Показать версию                        |

## Маршруты API

Для справки: CLI сопоставляет команды со следующими конечными точками API:

| Команда  | Метод  | Конечная точка                  |
| -------- | ------ | ------------------------------- |
| `list`   | POST   | `/api/<resource>/get-list`      |
| `get`    | POST   | `/api/<resource>/<id>/get-item` |
| `create` | POST   | `/api/<resource>`               |
| `update` | PUT    | `/api/<resource>/<id>/`         |
| `delete` | DELETE | `/api/<resource>/<id>/`         |
| `count`  | POST   | `/api/<resource>/count`         |

Все запросы включают заголовок `APIKey` для аутентификации.
