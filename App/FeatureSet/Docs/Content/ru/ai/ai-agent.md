# AI-агенты

AI-агенты в OneUptime автоматически исправляют ошибки, проблемы производительности и запросы к базе данных в вашем коде. Работая на основе данных наблюдаемости OpenTelemetry, AI-агенты создают pull request'ы с исправлениями — не просто алерты.

## Что могут делать AI-агенты?

AI-агенты анализируют данные наблюдаемости (трейсы, логи и метрики), чтобы обнаруживать и автоматически исправлять проблемы в вашей кодовой базе:

- **Автоматическое исправление ошибок**: Когда AI-агент замечает исключения в трейсах или логах, он автоматически исправляет проблему и создаёт pull request.
- **Исправление проблем производительности**: Анализирует трейсы с наибольшим временем выполнения и создаёт pull request'ы с оптимизациями производительности.
- **Оптимизация запросов к базе данных**: Выявляет медленные или неэффективные запросы и оптимизирует их с помощью правильной индексации и переписывания запросов.
- **Исправление проблем фронтенда**: Автоматически устраняет специфические проблемы производительности фронтенда, ошибки рендеринга и JavaScript.
- **Автоматическое добавление телеметрии**: Добавляет трейсинг, метрики и логи в вашу кодовую базу одним нажатием. Ручная инструментация не требуется.
- **Интеграция с GitHub и GitLab**: Бесшовно интегрируется с вашими существующими репозиториями. PR создаются непосредственно в вашем рабочем процессе.
- **Интеграция с CI/CD**: Интегрируется с вашими существующими CI/CD конвейерами. Исправления тестируются и проверяются до создания PR.
- **Поддержка Terraform**: Автоматически исправляет проблемы инфраструктуры. Поддерживает Terraform и OpenTofu для инфраструктуры как кода.
- **Интеграция с трекерами задач**: Подключается к Jira, Linear и другим трекерам задач. Автоматически связывает исправления с соответствующими задачами.

## Как это работает

1. **Сбор данных**: OpenTelemetry собирает трейсы, логи и метрики из вашего приложения
2. **Обнаружение проблем**: AI выявляет ошибки, узкие места производительности и медленные запросы
3. **Создание исправления**: AI анализирует вашу кодовую базу и автоматически создаёт исправление
4. **Создание PR**: Pull request с исправлением и подробным отчётом готов для проверки

## Гибкость в выборе LLM-провайдера

OneUptime работает с любым LLM-провайдером. Вы можете использовать:

- Модели **OpenAI GPT**
- Модели **Anthropic Claude**
- **Meta Llama** (через Ollama или других провайдеров)
- **Самостоятельно размещённые** модели

Разместите AI-модель самостоятельно и держите свой код полностью закрытым.

## Конфиденциальность

Независимо от вашего тарифного плана, OneUptime никогда не просматривает, не хранит и не обучает модели на вашем коде:

- **Нет доступа к коду**: Ваш код остаётся в вашей инфраструктуре
- **Нет хранения данных**: Политика нулевого хранения данных
- **Нет обучения**: Ваш код никогда не используется для обучения AI

## Глобальные AI-агенты и самостоятельно размещённые AI-агенты

### Глобальные AI-агенты

Если вы используете **OneUptime SaaS** (облачную версию), глобальные AI-агенты предоставляются OneUptime, уже настроены и готовы к использованию. Эти агенты управляются OneUptime и не требуют дополнительной настройки.

Глобальные AI-агенты автоматически доступны для всех проектов, если не отключены в настройках проекта.

### Самостоятельно размещённые AI-агенты

Для организаций, которым необходимо запускать AI-агентов в собственной инфраструктуре (например, по соображениям безопасности, соответствия требованиям или доступа к сети), OneUptime поддерживает самостоятельно размещённые AI-агенты.

Самостоятельно размещённые AI-агенты:

- Работают в вашей частной сети
- Могут получать доступ к внутренним ресурсам и системам
- Предоставляют вам полный контроль над средой агента
- Могут быть настроены под ваши конкретные нужды

## Настройка самостоятельно размещённого AI-агента

### Шаг 1: Создание AI-агента в OneUptime

1. Войдите в панель управления OneUptime
2. Перейдите в **Project Settings** > **AI Agents**
3. Нажмите **Create AI Agent**, чтобы добавить нового агента
4. Заполните обязательные поля:
   - **Name**: Понятное имя для вашего AI-агента
   - **Description** (необязательно): Описание назначения агента
5. После создания вы получите `AI_AGENT_ID` и `AI_AGENT_KEY`

**Важно**: Сохраните ваш `AI_AGENT_KEY` в надёжном месте. Он будет показан только один раз и не может быть получен позже.

### Шаг 2: Развёртывание AI-агента

#### Docker

Для запуска AI-агента убедитесь, что у вас установлен Docker. Запустите агента с помощью:

```bash
docker run --name oneuptime-ai-agent --network host \
  -e AI_AGENT_KEY=<ai-agent-key> \
  -e AI_AGENT_ID=<ai-agent-id> \
  -e ONEUPTIME_URL=https://oneuptime.com \
  -d oneuptime/ai-agent:release
```

Если вы самостоятельно размещаете OneUptime, замените `ONEUPTIME_URL` на URL вашего пользовательского экземпляра.

#### Docker Compose

Вы также можете запустить AI-агента с помощью docker-compose. Создайте файл `docker-compose.yml`:

```yaml
version: "3"

services:
  oneuptime-ai-agent:
    image: oneuptime/ai-agent:release
    container_name: oneuptime-ai-agent
    environment:
      - AI_AGENT_KEY=<ai-agent-key>
      - AI_AGENT_ID=<ai-agent-id>
      - ONEUPTIME_URL=https://oneuptime.com
    network_mode: host
    restart: always
```

Затем запустите:

```bash
docker compose up -d
```

#### Kubernetes

Создайте файл `oneuptime-ai-agent.yaml`:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: oneuptime-ai-agent
spec:
  selector:
    matchLabels:
      app: oneuptime-ai-agent
  template:
    metadata:
      labels:
        app: oneuptime-ai-agent
    spec:
      containers:
        - name: oneuptime-ai-agent
          image: oneuptime/ai-agent:release
          env:
            - name: AI_AGENT_KEY
              value: "<ai-agent-key>"
            - name: AI_AGENT_ID
              value: "<ai-agent-id>"
            - name: ONEUPTIME_URL
              value: "https://oneuptime.com"
```

Примените конфигурацию:

```bash
kubectl apply -f oneuptime-ai-agent.yaml
```

### Переменные окружения

AI-агент поддерживает следующие переменные окружения:

#### Обязательные переменные

| Переменная      | Описание                                                              |
| --------------- | --------------------------------------------------------------------- |
| `AI_AGENT_KEY`  | Ключ AI-агента из вашей панели управления OneUptime                   |
| `AI_AGENT_ID`   | Идентификатор AI-агента из вашей панели управления OneUptime          |
| `ONEUPTIME_URL` | URL вашего экземпляра OneUptime (по умолчанию: https://oneuptime.com) |

## Проверка AI-агента

После развёртывания AI-агента:

1. Перейдите в **Project Settings** > **AI Agents** в панели управления OneUptime
2. Ваш агент должен отображаться как **Connected** в течение нескольких минут
3. Если статус показывает **Disconnected**, проверьте логи контейнера на наличие ошибок

Для просмотра логов контейнера:

```bash
# Docker
docker logs oneuptime-ai-agent

# Kubernetes
kubectl logs deployment/oneuptime-ai-agent
```

## Устранение неполадок

### Агент не подключается

1. **Проверьте учётные данные**: Убедитесь, что `AI_AGENT_KEY` и `AI_AGENT_ID` верны
2. **Проверьте сеть**: Убедитесь, что агент может связаться с вашим экземпляром OneUptime
3. **Просмотрите логи**: Проверьте логи контейнера на наличие сообщений об ошибках
4. **Правила брандмауэра**: Убедитесь, что исходящий HTTPS (порт 443) разрешён

### Агент постоянно отключается

1. **Проверьте ограничения ресурсов**: Убедитесь, что контейнер имеет достаточно памяти и CPU
2. **Стабильность сети**: Проверьте стабильность сетевого соединения
3. **Просмотрите логи**: Ищите ошибки тайм-аута или соединения в логах
