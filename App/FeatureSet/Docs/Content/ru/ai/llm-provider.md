# LLM-провайдеры

OneUptime поддерживает интеграцию с различными провайдерами больших языковых моделей (LLM) для обеспечения функций на основе AI во всей платформе. Это руководство поможет вам настроить собственного LLM-провайдера.

## Что могут делать LLM-провайдеры?

LLM-провайдеры в OneUptime помогают автоматизировать и улучшить рабочий процесс управления инцидентами:

- **Заметки по инцидентам**: Автоматическая генерация подробных заметок и обновлений об инцидентах
- **Заметки по алертам**: Создание содержательных описаний алертов и контекста
- **Заметки по плановому обслуживанию**: Автоматическая генерация заметок о событиях технического обслуживания
- **Постмортемы инцидентов**: Автоматическое составление исчерпывающих отчётов по результатам инцидентов
- **Улучшения кода**: Если вы подключите репозиторий кода к OneUptime, мы будем использовать ваш LLM-провайдер для анализа данных телеметрии (логи, трейсы, метрики, исключения) и предложения улучшений кода

## Пользователи OneUptime SaaS

Если вы используете **OneUptime SaaS** (облачную версию), вы можете использовать **глобальный LLM-провайдер** по умолчанию без дополнительной настройки. Глобальный LLM-провайдер предварительно настроен и готов к использованию для всех AI-функций.

Если вы предпочитаете использовать собственные ключи API или конкретного провайдера, вы всё равно можете настроить пользовательского LLM-провайдера, следуя инструкциям ниже.

## Поддерживаемые провайдеры

OneUptime в настоящее время поддерживает следующих LLM-провайдеров:

| Провайдер     | Описание                                                                                                  | Требуется ключ API | Требуется базовый URL           |
| ------------- | --------------------------------------------------------------------------------------------------------- | ------------------ | ------------------------------- |
| **OpenAI**    | GPT-4, GPT-4o, GPT-3.5 Turbo и другие модели OpenAI                                                       | Да                 | Нет (используется по умолчанию) |
| **Anthropic** | Claude 3 Opus, Claude 3 Sonnet, Claude 3 Haiku и другие модели Claude                                     | Да                 | Нет (используется по умолчанию) |
| **Ollama**    | Самостоятельно размещённые модели с открытым исходным кодом, такие как Llama 2, Mistral, CodeLlama и т.д. | Нет                | Да                              |

## Настройка LLM-провайдера

### Шаг 1: Перейдите в настройки LLM-провайдеров

1. Войдите в панель управления OneUptime
2. Перейдите в **Project Settings** > **AI** > **LLM Providers**
3. Нажмите **Create LLM Provider**, чтобы добавить нового провайдера

### Шаг 2: Настройте провайдера

Заполните следующие поля:

- **Name**: Понятное имя для этой конфигурации LLM (например, "Production OpenAI", "Local Ollama")
- **Description** (необязательно): Описание для идентификации назначения этого провайдера
- **LLM Type**: Выберите тип провайдера (OpenAI, Anthropic или Ollama)
- **API Key**: Ваш ключ API (обязателен для OpenAI и Anthropic)
- **Model Name**: Конкретная используемая модель (например, `gpt-4o`, `claude-3-opus-20240229`, `llama2`)
- **Base URL** (необязательно): Пользовательский URL конечной точки API (обязателен для Ollama, необязателен для других)

## Настройка для конкретных провайдеров

### OpenAI

1. Получите ключ API на [OpenAI Platform](https://platform.openai.com/api-keys)
2. Выберите **OpenAI** в качестве типа LLM
3. Введите ваш ключ API
4. Выберите имя модели:
   - `gpt-4o` — наиболее мощная модель, лучшая для сложных задач
   - `gpt-4o-mini` — более быстрая и экономичная
   - `gpt-4-turbo` — хороший баланс возможностей и скорости
   - `gpt-3.5-turbo` — быстрая и экономичная

**Пример конфигурации:**

```
Name: Production OpenAI
LLM Type: OpenAI
API Key: sk-xxxxxxxxxxxxxxxxxxxx
Model Name: gpt-4o
```

### Anthropic

1. Получите ключ API в [Anthropic Console](https://console.anthropic.com/)
2. Выберите **Anthropic** в качестве типа LLM
3. Введите ваш ключ API
4. Выберите имя модели:
   - `claude-3-opus-20240229` — наиболее мощная модель
   - `claude-3-sonnet-20240229` — хороший баланс интеллекта и скорости
   - `claude-3-haiku-20240307` — самая быстрая и компактная
   - `claude-3-5-sonnet-20241022` — последняя модель Sonnet

**Пример конфигурации:**

```
Name: Production Anthropic
LLM Type: Anthropic
API Key: sk-ant-xxxxxxxxxxxxxxxxxxxx
Model Name: claude-3-5-sonnet-20241022
```

### Ollama (самостоятельное размещение)

Ollama позволяет запускать модели с открытым исходным кодом локально или в собственной инфраструктуре.

1. Установите Ollama с [ollama.ai](https://ollama.ai)
2. Загрузите нужную модель: `ollama pull llama2`
3. Убедитесь, что Ollama запущена и доступна
4. Выберите **Ollama** в качестве типа LLM
5. Введите базовый URL (например, `http://localhost:11434`)
6. Введите имя загруженной модели

**Пример конфигурации:**

```
Name: Local Ollama
LLM Type: Ollama
Base URL: http://localhost:11434
Model Name: llama2
```

**Популярные модели Ollama:**

- `llama2` — модель Llama 2 от Meta
- `llama3` — модель Llama 3 от Meta
- `mistral` — модель от Mistral AI
- `codellama` — специализированная на коде модель Llama
- `mixtral` — модель смеси экспертов от Mistral

## Использование пользовательских базовых URL

Для корпоративных развёртываний или при использовании прокси-сервисов вы можете указать пользовательский базовый URL:

- **Azure OpenAI**: Используйте URL вашей конечной точки Azure
- **API, совместимые с OpenAI**: Любой API, следующий спецификации API OpenAI
- **Частные экземпляры Ollama**: URL вашего внутреннего сервера Ollama

## Рекомендации

1. **Используйте описательные имена**: Называйте провайдеров понятно (например, "Production GPT-4", "Development Ollama")
2. **Защищайте ключи API**: Ключи API шифруются в состоянии покоя, но избегайте их передачи другим
3. **Тестируйте конфигурацию**: После настройки проверьте работу провайдера с AI-функциями
4. **Следите за использованием**: Отслеживайте использование API для управления расходами

## Устранение неполадок

### Проблемы с подключением

- **OpenAI/Anthropic**: Убедитесь, что ваш ключ API действителен и имеет достаточный баланс
- **Ollama**: Убедитесь, что сервер Ollama запущен и базовый URL указан правильно
- **Брандмауэр**: Убедитесь, что ваша сеть разрешает исходящие соединения с API провайдера

### Модель не найдена

- Убедитесь, что имя модели написано правильно
- Для Ollama убедитесь, что вы загрузили модель с помощью `ollama pull <model-name>`
- Проверьте, доступна ли модель в вашем регионе (некоторые модели имеют региональные ограничения)
