# Бесплатное развёртывание OneUptime с помощью Docker Compose

Если вы предпочитаете размещать OneUptime на собственном сервере, вы можете использовать Docker Compose для развёртывания одиночного экземпляра OneUptime на Debian, Ubuntu или RHEL. Эта опция даёт вам больший контроль и возможности настройки вашего экземпляра, но также требует более высоких технических навыков и ресурсов для развёртывания и обслуживания.

#### Выбор системных требований

В зависимости от вашего использования и бюджета вы можете выбрать различные системные требования для вашего сервера. Для оптимальной производительности мы рекомендуем использовать OneUptime с:

- **Рекомендуемые системные требования**
  - 16 ГБ ОЗУ
  - 8 ядер
  - 400 ГБ диска
  - Ubuntu 22.04
  - Docker и Docker Compose установлены
- **Требования для домашней лаборатории / минимальные требования**
  - Если вы хотите запустить OneUptime для личного или экспериментального использования в домашней среде (некоторые наши пользователи устанавливают его даже на Raspberry Pi), вы можете использовать следующие минимальные требования:
    - 8 ГБ ОЗУ
    - 4 ядра
    - 20 ГБ диска
    - Docker и Docker Compose установлены

#### Предварительные требования для развёртывания на одном сервере

Обучающее видео по установке: [https://youtu.be/j1SWmMW2oL4](https://youtu.be/j1SWmMW2oL4)

Перед началом процесса развёртывания убедитесь, что у вас есть:

- Сервер под управлением Debian, Ubuntu или производного от RHEL
- Docker и Docker Compose, установленные на вашем сервере

Для установки OneUptime:

```
# Клонируйте репозиторий только ветку release и перейдите в неё.
git clone --depth 1 --single-branch --branch release https://github.com/OneUptime/oneuptime.git
cd oneuptime

# Скопируйте config.example.env в config.env
cp config.example.env config.env

# ВАЖНО: Отредактируйте файл config.env. Убедитесь, что у вас случайные секреты.

npm start
```

Если вы не хотите использовать npm или он не установлен, выполните вместо этого:

```
# Прочитайте переменные окружения из файла config.env и запустите docker compose up.
(export $(grep -v '^#' config.env | xargs) && docker compose up --remove-orphans -d)

# Используйте sudo, если у вас возникают проблемы с правами при привязке портов.
sudo bash -c "(export $(grep -v '^#' config.env | xargs) && docker compose up --remove-orphans -d)"
```

### Доступ к OneUptime

OneUptime должен запуститься по адресу: http://localhost. Вам необходимо зарегистрировать новую учётную запись для вашего экземпляра, чтобы начать его использование.

### Настройка SSL/TLS-сертификатов

OneUptime **не** поддерживает самостоятельную настройку SSL/TLS-сертификатов. Вам необходимо настроить SSL/TLS-сертификаты самостоятельно.

Если вам нужно использовать SSL/TLS-сертификаты, выполните следующие шаги:

1. Используйте обратный прокси-сервер, например Nginx или Caddy.
2. Используйте Let's Encrypt для выдачи сертификатов.
3. Направьте обратный прокси на сервер OneUptime.
4. Обновите следующие настройки:
   - Установите переменную окружения `HTTP_PROTOCOL` на `https`.
   - Измените переменную окружения `HOST` на доменное имя сервера, на котором размещён обратный прокси.

## Контрольный список готовности к production

В идеале не развёртывайте OneUptime в production с docker-compose. Мы настоятельно рекомендуем использовать Kubernetes. Для OneUptime доступен helm-чарт [здесь](https://artifacthub.io/packages/helm/oneuptime/oneuptime).

Если вы всё же хотите развернуть OneUptime в production с docker-compose, пожалуйста, учтите следующее:

- **SSL/TLS**: Настройте SSL/TLS-сертификаты. OneUptime не поддерживает самостоятельную настройку SSL/TLS-сертификатов. Вам необходимо настроить их самостоятельно. Смотрите выше.
- **Секреты**: Убедитесь, что в файле `config.env` установлены случайные секреты. В файле есть некоторые секреты по умолчанию. Пожалуйста, замените их случайными длинными строками.
- **Резервные копии**: Регулярно создавайте резервные копии ваших баз данных (Clickhouse, Postgres). Redis используется как кэш, является stateless и может быть безопасно проигнорирован.
- **Обновления**: Пожалуйста, регулярно обновляйте OneUptime. Мы выпускаем обновления каждый день. Мы рекомендуем обновлять программное обеспечение не реже одного раза в неделю в production.

### Обновление OneUptime

Для обновления:

```
git checkout release # Убедитесь, что вы находитесь на ветке release.
git pull
npm run update
```

### Что следует учесть

- В нашей конфигурации Docker мы используем локальный драйвер логирования. OneUptime, особенно в контейнерах probe и ingest, генерирует значительное количество логов. Чтобы предотвратить заполнение хранилища, крайне важно ограничить хранилище логов в Docker. Подробные инструкции по этому вопросу см. в официальной документации Docker [здесь](https://docs.docker.com/config/containers/logging/local/).

### Удаление OneUptime

Для удаления OneUptime выполните следующую команду:

```
npm run down
```

Это остановит и удалит все контейнеры, сети и тома, созданные OneUptime. Файл `config.env` и клонированный репозиторий удалены не будут.
