# Монитор портов

Мониторинг портов позволяет отслеживать доступность конкретных портов TCP или UDP на хосте. OneUptime периодически пытается подключиться к указанному порту и проверяет, открыт ли он и принимает ли соединения.

## Обзор

Мониторы портов проверяют, принимает ли определённый сетевой порт подключения. Это позволяет:

- Мониторинг доступности сервисов на конкретных портах
- Отслеживание времени отклика порта
- Проверка работоспособности таких сервисов, как базы данных, почтовые серверы и серверы приложений
- Обнаружение сбоев сервисов до того, как они затронут пользователей

## Создание монитора портов

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Порт**
4. Введите имя хоста или IP-адрес и номер порта
5. При необходимости настройте критерии мониторинга

## Параметры конфигурации

### Имя хоста или IP-адрес

Введите имя хоста или IP-адрес целевого хоста (например, `example.com` или `192.168.1.1`).

### Порт

Введите номер порта для мониторинга (1–65535). Распространённые примеры:

| Порт  | Сервис     |
| ----- | ---------- |
| 22    | SSH        |
| 25    | SMTP       |
| 80    | HTTP       |
| 443   | HTTPS      |
| 3306  | MySQL      |
| 5432  | PostgreSQL |
| 6379  | Redis      |
| 27017 | MongoDB    |

## Критерии мониторинга

Можно настроить критерии, определяющие, считается ли порт доступным, деградированным или недоступным, на основе:

### Доступные типы проверок

| Тип проверки       | Описание                                   |
| ------------------ | ------------------------------------------ |
| Доступен           | Открыт ли порт и принимает ли соединения   |
| Время отклика (мс) | Время установки соединения в миллисекундах |
| Тайм-аут запроса   | Истёк ли тайм-аут попытки подключения      |

### Типы фильтров

Для **Доступен** и **Тайм-аут запроса**:

- **True** — условие истинно
- **False** — условие ложно

Для **Времени отклика**:

- **Больше чем** — время отклика превышает порог
- **Меньше чем** — время отклика ниже порога
- **Больше или равно** — время отклика на уровне порога или выше
- **Меньше или равно** — время отклика на уровне порога или ниже
- **Равно** — время отклика точно совпадает
- **Не равно** — время отклика не совпадает
- **Оценка за период** — оценка с использованием агрегации (Среднее, Сумма, Максимум, Минимум, Все значения, Любое значение) за временное окно

### Примеры критериев

#### Статус «Недоступен» при закрытом порте

- **Проверять**: Доступен
- **Тип фильтра**: False

#### Оповещение при времени подключения свыше 500 мс

- **Проверять**: Время отклика (мс)
- **Тип фильтра**: Больше чем
- **Значение**: 500

#### Статус «Деградирован» при медленном подключении

- **Проверять**: Время отклика (мс)
- **Тип фильтра**: Больше чем
- **Значение**: 200
