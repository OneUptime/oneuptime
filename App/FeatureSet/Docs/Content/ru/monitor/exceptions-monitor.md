# Монитор исключений

Мониторинг исключений позволяет отслеживать исключения и ошибки приложений и формировать оповещения при превышении настроенных порогов. OneUptime оценивает данные об исключениях из ваших телеметрических сервисов за определённое временное окно.

## Обзор

Мониторы исключений подсчитывают и фильтруют исключения, соответствующие заданным критериям. Это позволяет:

- Оповещение при всплесках исключений в приложениях
- Мониторинг исключений определённых типов
- Поиск исключений по тексту сообщения об ошибке
- Отдельное отслеживание устранённых и активных исключений
- Обнаружение проблем со стабильностью приложения по шаблонам ошибок

## Создание монитора исключений

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Исключения**
4. Выберите телеметрические сервисы для мониторинга
5. При необходимости настройте фильтры и критерии исключений

## Параметры конфигурации

### Телеметрические сервисы

Выберите один или несколько сервисов для мониторинга исключений. Сервисы должны отправлять данные об исключениях в OneUptime через OpenTelemetry.

### Фильтры исключений

| Фильтр | Описание | Обязательно |
|--------|----------|-------------|
| Типы исключений | Фильтрация по именам типов исключений (например, `NullPointerException`, `TypeError`) | Нет |
| Сообщение | Текстовый поиск в сообщениях исключений | Нет |
| Включать устранённые | Включать исключения, помеченные как устранённые (по умолчанию: false) | Нет |
| Включать архивированные | Включать архивированные исключения (по умолчанию: false) | Нет |
| Временное окно | Глубина поиска исключений (в секундах, по умолчанию: 60) | Нет |

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки | Описание |
|--------------|----------|
| Количество исключений | Число исключений, соответствующих фильтрам, за временное окно |

### Типы фильтров

- **Больше чем** — количество исключений превышает порог
- **Меньше чем** — количество исключений ниже порога
- **Больше или равно** — количество исключений на уровне порога или выше
- **Меньше или равно** — количество исключений на уровне порога или ниже
- **Равно** — количество исключений точно совпадает
- **Не равно** — количество исключений не совпадает

### Примеры критериев

#### Оповещение при более чем 10 исключениях за 60 секунд

- **Временное окно**: 60 секунд
- **Проверять**: Количество исключений
- **Тип фильтра**: Больше чем
- **Значение**: 10

#### Оповещение при любом NullPointerException

- **Типы исключений**: `NullPointerException`
- **Временное окно**: 60 секунд
- **Проверять**: Количество исключений
- **Тип фильтра**: Больше чем
- **Значение**: 0

#### Мониторинг исключений с определённым сообщением

- **Сообщение**: `out of memory`
- **Временное окно**: 300 секунд
- **Проверять**: Количество исключений
- **Тип фильтра**: Больше чем
- **Значение**: 0

## Требования для настройки

Мониторинг исключений требует, чтобы ваши приложения отправляли данные об исключениях в OneUptime через OpenTelemetry. Инструкции по настройке см. в документации [OpenTelemetry](/docs/telemetry/open-telemetry).
