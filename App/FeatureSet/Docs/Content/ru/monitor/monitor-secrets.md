# Секреты монитора

Для безопасного хранения конфиденциальных данных, используемых в проверках мониторинга, можно применять секреты. Секреты шифруются и хранятся в защищённом виде.

### Добавление секрета

Чтобы добавить секрет, перейдите в OneUptime Dashboard -> Настройки проекта -> Секреты монитора -> Создать секрет монитора.

![Создание секрета](/docs/static/images/CreateMonitorSecret.png)

Можно выбрать, какие мониторы имеют доступ к секрету. В данном случае мы добавили секрет `ApiKey` и выбрали мониторы, которые получат к нему доступ.

**Обратите внимание**: секреты шифруются и хранятся безопасно. Если вы потеряете секрет, потребуется создать новый. После сохранения просмотреть или обновить секрет невозможно.

### Использование секрета

Секреты можно использовать в следующих типах мониторинга:

- API (в заголовках запроса, теле запроса и URL)
- Сайт, IP, Порт, Ping, SSL-сертификат (в URL)
- Синтетический монитор, Монитор пользовательского кода (в коде)
- SNMP-монитор (в строке сообщества, ключе аутентификации и ключе шифрования SNMPv3)

![Использование секрета](/docs/static/images/UsingMonitorSecret.png)

Для использования секрета добавьте `{{monitorSecrets.SECRET_NAME}}` в поле, где требуется использовать секрет. Например, в данном случае мы добавили `{{monitorSecrets.ApiKey}}` в поле «Заголовки запроса».

Секреты внедряются в зонд до выполнения скриптов синтетического монитора или монитора пользовательского кода, поэтому ссылки вида `{{monitorSecrets.ApiKey}}` разрешаются в расшифрованное значение внутри выполняемого скрипта.

### Права доступа к секретам монитора

Можно выбрать, какие мониторы имеют доступ к секрету. Права доступа можно обновлять в любое время. Таким образом, если нужно предоставить доступ к секрету новому монитору, это можно сделать, обновив права доступа.
