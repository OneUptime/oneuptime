# Ручной монитор

Ручной мониторинг позволяет создавать мониторы, статус которых управляется вручную или через API. OneUptime не выполняет никаких автоматических проверок — вы сами управляете статусом монитора.

## Обзор

Ручные мониторы — это плейсхолдеры, которые вы обновляете самостоятельно. Они полезны для:

- Интеграции с внешними инструментами мониторинга, которые обновляют статус через API OneUptime
- Отслеживания сервисов или систем, которые не поддаются автоматическому мониторингу
- Управления инцидентами для компонентов без автоматических проверок работоспособности
- Представления зависимостей от сторонних систем, статус которых вы отслеживаете вручную

## Создание ручного монитора

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Ручной**
4. Введите имя и описание монитора

## Принцип работы

Ручные мониторы не имеют интервалов мониторинга, зондов и автоматической оценки критериев. Статус монитора остаётся таким, каким вы его установили, до тех пор пока вы его не измените.

### Обновление статуса

Статус ручного монитора можно обновить двумя способами:

- **Панель управления** — изменение статуса монитора непосредственно из панели управления OneUptime
- **API** — программное обновление статуса монитора через API OneUptime

### Инциденты и оповещения

Инциденты и оповещения для ручных мониторов можно создавать так же, как и для мониторов любого другого типа. Это позволяет:

- Отслеживать простои для сервисов, контролируемых внешними системами
- Создавать инциденты вручную при обнаружении проблем
- Использовать ручные мониторы на страницах статуса для информирования пользователей о состоянии

## Когда использовать ручные мониторы

| Вариант использования         | Описание                                                                                         |
| ----------------------------- | ------------------------------------------------------------------------------------------------ |
| Сторонние сервисы             | Отслеживание статуса внешних сервисов, от которых вы зависите, но не можете отслеживать напрямую |
| Физическая инфраструктура     | Представление аппаратного обеспечения или физических систем без сетевого мониторинга             |
| Бизнес-процессы               | Отслеживание нетехнических процессов, влияющих на статус сервиса                                 |
| Управление статусом через API | Позволяет внешним инструментам обновлять статус монитора через API OneUptime                     |
| Плейсхолдеры страниц статуса  | Отображение компонентов на странице статуса, управляемых вне OneUptime                           |
