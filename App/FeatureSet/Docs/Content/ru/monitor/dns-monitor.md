# Монитор DNS

Мониторинг DNS позволяет отслеживать работоспособность и корректность разрешения имён для ваших доменов. OneUptime периодически делает DNS-запросы и проверяет полученные ответы на соответствие заданным критериям.

## Обзор

Мониторы DNS выполняют запросы к DNS-серверам для получения записей определённых типов и оценивают результаты. Это позволяет:

- Мониторинг доступности DNS-сервиса
- Проверка корректности возвращаемых DNS-записей
- Отслеживание времени отклика DNS-разрешения
- Валидация конфигурации DNSSEC
- Обнаружение проблем распространения DNS или перехвата запросов

## Создание монитора DNS

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **DNS**
4. Введите имя домена и тип запрашиваемой записи
5. При необходимости настройте критерии мониторинга

## Параметры конфигурации

### Основные настройки

| Поле       | Описание                                                                                                             | Обязательно |
| ---------- | -------------------------------------------------------------------------------------------------------------------- | ----------- |
| Имя домена | Домен для запроса (например, `example.com`)                                                                          | Да          |
| Тип записи | Тип запрашиваемой DNS-записи                                                                                         | Да          |
| DNS-сервер | Пользовательский DNS-сервер (например, `8.8.8.8`). Оставьте пустым для использования системного сервера по умолчанию | Нет         |

### Поддерживаемые типы записей

| Тип записи | Описание                                |
| ---------- | --------------------------------------- |
| A          | Записи адресов IPv4                     |
| AAAA       | Записи адресов IPv6                     |
| CNAME      | Записи канонических имён (псевдонимов)  |
| MX         | Записи почтового обмена                 |
| NS         | Записи серверов имён                    |
| TXT        | Текстовые записи (SPF, DKIM и др.)      |
| SOA        | Записи «Начало зоны»                    |
| PTR        | Записи обратного преобразования DNS     |
| SRV        | Записи обнаружения сервисов             |
| CAA        | Записи авторизации центров сертификации |

### Дополнительные настройки

| Поле              | Описание                              | По умолчанию |
| ----------------- | ------------------------------------- | ------------ |
| Порт              | Номер порта DNS                       | 53           |
| Тайм-аут (мс)     | Время ожидания ответа                 | 5000         |
| Повторные попытки | Количество повторных попыток при сбое | 3            |

## Критерии мониторинга

Можно настроить критерии, определяющие, считается ли ваш DNS доступным, деградированным или недоступным, на основе:

### Доступные типы проверок

| Тип проверки           | Описание                                     |
| ---------------------- | -------------------------------------------- |
| DNS доступен           | Отвечает ли DNS-сервер на запросы            |
| Время отклика DNS (мс) | Время отклика запроса в миллисекундах        |
| DNS-запись существует  | Существуют ли DNS-записи для данного запроса |
| Значение DNS-записи    | Значение, возвращённое DNS-записью           |
| DNSSEC действителен    | Прошла ли валидация DNSSEC                   |

### Типы фильтров

Для **DNS доступен**, **DNS-запись существует** и **DNSSEC действителен**:

- **True** — условие истинно
- **False** — условие ложно

Для **Времени отклика DNS**:

- **Больше чем**, **Меньше чем**, **Больше или равно**, **Меньше или равно**, **Равно**, **Не равно**

Для **Значения DNS-записи**:

- **Содержит** — значение записи содержит указанный текст
- **Не содержит** — значение записи не содержит указанный текст
- **Начинается с** — значение записи начинается с указанного текста
- **Заканчивается на** — значение записи заканчивается на указанный текст
- **Равно** — значение записи точно совпадает
- **Не равно** — значение записи не совпадает

### Примеры критериев

#### Проверка разрешения DNS

- **Проверять**: DNS доступен
- **Тип фильтра**: True

#### Проверка, что A-запись указывает на правильный IP

- **Проверять**: Значение DNS-записи
- **Тип фильтра**: Равно
- **Значение**: `93.184.216.34`

#### Оповещение при медленном отклике DNS

- **Проверять**: Время отклика DNS (мс)
- **Тип фильтра**: Больше чем
- **Значение**: 500

#### Проверка действительности DNSSEC

- **Проверять**: DNSSEC действителен
- **Тип фильтра**: True
