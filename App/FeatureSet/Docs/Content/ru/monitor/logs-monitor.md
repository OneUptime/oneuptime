# Монитор журналов

Мониторинг журналов позволяет отслеживать журналы приложений и формировать оповещения на основе шаблонов, количества записей и уровней серьёзности. OneUptime оценивает журналы из ваших телеметрических сервисов и проверяет их на соответствие заданным критериям.

## Обзор

Мониторы журналов выполняют поиск и подсчёт записей журналов, соответствующих заданным фильтрам, за определённое временное окно. Это позволяет:

- Оповещение при всплесках журналов ошибок
- Мониторинг определённых шаблонов и сообщений в журналах
- Отслеживание объёма журналов по уровню серьёзности
- Фильтрация журналов по сервису, атрибутам и содержимому
- Обнаружение проблем с приложением по шаблонам журналов

## Создание монитора журналов

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Журналы**
4. Выберите телеметрические сервисы для мониторинга
5. При необходимости настройте фильтры и критерии для журналов

## Параметры конфигурации

### Телеметрические сервисы

Выберите один или несколько сервисов для мониторинга журналов. Сервисы должны отправлять журналы в OneUptime через OpenTelemetry.

### Фильтры журналов

| Фильтр | Описание | Обязательно |
|--------|----------|-------------|
| Уровни серьёзности | Фильтрация по уровню серьёзности (ERROR, WARN, INFO, DEBUG и др.) | Нет |
| Тело | Текстовый поиск в теле сообщения журнала | Нет |
| Атрибуты | Пары ключ-значение для фильтрации по пользовательским атрибутам журнала | Нет |
| Временное окно | Глубина поиска журналов (в секундах, по умолчанию: 60) | Нет |

### Уровни серьёзности

Фильтрация журналов по одному или нескольким уровням серьёзности:

- **FATAL** / **EMERGENCY** / **CRITICAL**
- **ERROR**
- **WARN** / **WARNING**
- **INFO** / **INFORMATIONAL**
- **DEBUG**
- **TRACE**
- **UNSPECIFIED**

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки | Описание |
|--------------|----------|
| Количество записей журнала | Число записей, соответствующих фильтрам, за временное окно |

### Типы фильтров

- **Больше чем** — количество записей превышает порог
- **Меньше чем** — количество записей ниже порога
- **Больше или равно** — количество записей на уровне порога или выше
- **Меньше или равно** — количество записей на уровне порога или ниже
- **Равно** — количество записей точно совпадает
- **Не равно** — количество записей не совпадает

### Примеры критериев

#### Оповещение при более чем 100 записях ошибок за 60 секунд

- **Уровни серьёзности**: ERROR
- **Временное окно**: 60 секунд
- **Проверять**: Количество записей журнала
- **Тип фильтра**: Больше чем
- **Значение**: 100

#### Оповещение при любых критических записях

- **Уровни серьёзности**: FATAL
- **Временное окно**: 60 секунд
- **Проверять**: Количество записей журнала
- **Тип фильтра**: Больше чем
- **Значение**: 0

#### Мониторинг журналов с определённым сообщением об ошибке

- **Тело**: `database connection timeout`
- **Временное окно**: 300 секунд
- **Проверять**: Количество записей журнала
- **Тип фильтра**: Больше чем
- **Значение**: 5

## Требования для настройки

Мониторинг журналов требует, чтобы ваши приложения отправляли журналы в OneUptime через OpenTelemetry. Инструкции по настройке см. в документации [OpenTelemetry](/docs/telemetry/open-telemetry).
