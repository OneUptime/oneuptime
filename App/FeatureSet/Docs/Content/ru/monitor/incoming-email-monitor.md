# Монитор входящей электронной почты

Монитор входящей электронной почты позволяет создавать и устранять оповещения на основе писем, отправляемых на уникальные адреса электронной почты, привязанные к конкретному монитору. Это удобно для интеграции с устаревшими системами, сторонними инструментами оповещения или любыми сервисами, поддерживающими отправку email-уведомлений.

## Принцип работы

1. При создании монитора входящей электронной почты OneUptime генерирует для него уникальный адрес электронной почты
2. Любое письмо, отправленное на этот адрес, принимается и оценивается по заданным критериям
3. В зависимости от критериев OneUptime создаёт новые оповещения или устраняет существующие

Это мощный способ интеграции систем email-оповещения с рабочим процессом управления инцидентами OneUptime.

## Создание монитора входящей электронной почты

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Входящая электронная почта**
4. Настройте параметры монитора:
   - **Имя:** описательное название монитора
   - **Описание:** назначение данного монитора
5. Настройте **Критерии создания оповещения** (условия, при которых создаются оповещения)
6. Настройте **Критерии устранения оповещения** (условия, при которых оповещения устраняются)
7. Нажмите **Создать**

После создания на странице сведений о мониторе будет отображён уникальный адрес электронной почты.

## Формат адреса электронной почты

Каждый монитор входящей электронной почты получает уникальный адрес в формате:

```
monitor-{secret-key}@{inbound-domain}
```

Например: `monitor-abc123def456@inbound.yourdomain.com`

Скопируйте этот адрес со страницы сведений о мониторе и настройте внешние системы для отправки писем на него.

## Доступные поля для критериев

Критерии можно создавать на основе следующих полей письма:

| Поле | Описание |
|------|----------|
| **Тема письма** | Строка темы входящего письма |
| **Отправитель письма** | Адрес электронной почты отправителя |
| **Тело письма** | Текстовое содержимое тела письма |
| **Получатель письма** | Адрес электронной почты получателя |
| **Письмо получено** | Критерии на основе времени получения писем |

## Доступные типы фильтров

### Строковые фильтры (Тема, Отправитель, Тело, Получатель)

| Фильтр | Описание | Пример |
|--------|----------|--------|
| **Содержит** | Поле содержит указанный текст | Тема содержит «КРИТИЧНО» |
| **Не содержит** | Поле не содержит указанный текст | Тема не содержит «ТЕСТ» |
| **Равно** | Поле точно совпадает с указанным текстом | Отправитель равно «alerts@service.com» |
| **Не равно** | Поле не совпадает с указанным текстом | Тема не равна «ОК» |
| **Начинается с** | Поле начинается с указанного текста | Тема начинается с «[ОПОВЕЩЕНИЕ]» |
| **Заканчивается на** | Поле заканчивается на указанный текст | Тема заканчивается на «- Production» |
| **Пустое** | Поле пустое или содержит пробелы | Тело пустое |
| **Не пустое** | Поле имеет содержимое | Тема не пустая |

### Фильтры на основе времени (Письмо получено)

| Фильтр | Описание | Пример |
|--------|----------|--------|
| **Получено в течение минут** | Письмо получено в течение X минут | Письмо получено в течение 30 минут |
| **Не получено в течение минут** | Письмо не получено в течение X минут | Письмо не получено в течение 60 минут |

## Примеры конфигурации

### Пример 1: Создание оповещения при критических письмах

**Критерии создания оповещения:**
- Тема письма **Содержит** «КРИТИЧНО»
- ИЛИ Тема письма **Содержит** «ОПОВЕЩЕНИЕ»
- ИЛИ Тема письма **Содержит** «ОШИБКА»

**Критерии устранения оповещения:**
- Тема письма **Содержит** «УСТРАНЕНО»
- ИЛИ Тема письма **Содержит** «ОК»
- ИЛИ Тема письма **Содержит** «ВОССТАНОВЛЕНО»

### Пример 2: Мониторинг конкретного отправителя

**Критерии создания оповещения:**
- Отправитель письма **Равно** «monitoring@legacy-system.com»
- И Тема письма **Содержит** «Сбой»

**Критерии устранения оповещения:**
- Отправитель письма **Равно** «monitoring@legacy-system.com»
- И Тема письма **Содержит** «Успешно»

### Пример 3: Монитор пульса (нет письма = оповещение)

**Критерии создания оповещения:**
- Письмо получено **Не получено в течение минут** со значением `60`

Оповещение создаётся, если письмо не получено в течение 60 минут — полезно для мониторинга запланированных заданий или пакетных процессов, которые должны отправлять письма о завершении работы.

**Критерии устранения оповещения:**
- Письмо получено **Получено в течение минут** со значением `5`

Оповещение устраняется при получении письма.

## Варианты использования

### Интеграция с устаревшими системами

Многие устаревшие системы поддерживают только email-оповещения. Используйте монитор входящей электронной почты для:
- Преобразования email-оповещений в инциденты OneUptime
- Автоматического устранения инцидентов при получении писем о восстановлении
- Централизации оповещений из нескольких устаревших систем

### Мониторинг сторонних сервисов

Интеграция с сервисами, отправляющими email-уведомления:
- Оповещения облачных поставщиков (AWS, GCP, Azure)
- Инструменты сканирования безопасности
- Уведомления о завершении резервного копирования
- Предупреждения об истечении SSL-сертификатов

### Мониторинг запланированных заданий

Мониторинг пакетных заданий и запланированных задач:
- Создание оповещений при незавершении писем о выполнении задания в срок
- Отслеживание сбоев заданий через email-уведомления об ошибках
- Мониторинг завершения конвейеров обработки данных

### Агрегация оповещений от нескольких поставщиков

Объединение оповещений из нескольких инструментов мониторинга:
- Получение оповещений от Nagios, Zabbix и других инструментов по электронной почте
- Унификация управления инцидентами в OneUptime
- Единый источник истины для всех оповещений

## Переменные шаблона

При настройке шаблонов инцидентов можно использовать следующие переменные из входящих писем:

| Переменная | Описание |
|------------|----------|
| `{{emailSubject}}` | Тема полученного письма |
| `{{emailFrom}}` | Адрес электронной почты отправителя |
| `{{emailTo}}` | Адрес электронной почты получателя |
| `{{emailBody}}` | Текстовое тело письма |
| `{{emailReceivedAt}}` | Время получения письма |

## Сводный вид монитора

В сводке монитора отображается:
- **Последнее письмо получено в:** время получения последнего письма
- **От кого:** отправитель последнего письма
- **Тема:** строка темы последнего письма
- **Заголовки письма:** полные заголовки последнего письма (раскрываются)
- **Тело письма:** содержимое последнего письма (раскрывается)

## Настройка для самостоятельного хостинга

При самостоятельном хостинге OneUptime необходимо настроить поставщика входящей электронной почты. В настоящее время поддерживается:

- **SendGrid Inbound Parse** — инструкции по настройке см. в разделе [Интеграция входящей электронной почты SendGrid](/docs/self-hosted/sendgrid-inbound-email)

## Важные замечания

- **Безопасность адреса электронной почты:** адрес монитора содержит секретный ключ. Обращайтесь с ним как с паролем и не публикуйте в открытом доступе.
- **Размер письма:** очень большие письма (с крупными вложениями) могут быть усечены или отклонены поставщиком электронной почты.
- **Время обработки:** письма обрабатываются асинхронно. Между отправкой письма и созданием оповещения может быть небольшая задержка в несколько секунд.
- **Нечувствительность к регистру:** все строковые сравнения (Содержит, Равно и др.) нечувствительны к регистру.
- **Текстовый формат:** критерии для тела письма используют текстовую версию письма. HTML-форматирование удаляется.

## Устранение неполадок

### Письма не поступают

1. Проверьте правильность адреса электронной почты (нет ли опечаток)
2. Проверьте, не блокирует ли письма спам-фильтр
3. Убедитесь в правильности настройки поставщика входящей электронной почты
4. Проверьте журналы OneUptime на наличие сообщений об ошибках

### Оповещения не создаются

1. Убедитесь, что критерии соответствуют содержимому письма
2. Проверьте, что монитор не отключён
3. Просмотрите журналы оценки в сведениях о мониторе
4. Проверьте с точными совпадениями строк, прежде чем использовать сопоставление по шаблону

### Оповещения не устраняются

1. Убедитесь, что критерии устранения соответствуют письму о восстановлении
2. Убедитесь в наличии активного оповещения для устранения
3. Проверьте, отправляется ли письмо об устранении на тот же адрес монитора
