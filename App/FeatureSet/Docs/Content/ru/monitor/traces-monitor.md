# Монитор трассировок

Мониторинг трассировок позволяет отслеживать распределённые трассировки приложений и формировать оповещения на основе шаблонов, количества и статусов спанов. OneUptime оценивает данные трассировок из ваших телеметрических сервисов за определённое временное окно.

## Обзор

Мониторы трассировок выполняют поиск и подсчёт спанов, соответствующих заданным фильтрам. Это позволяет:

- Оповещение при всплесках ошибок спанов в сервисах
- Мониторинг конкретных операций и конечных точек
- Отслеживание объёма и шаблонов спанов
- Фильтрация по статусу спана, имени и пользовательским атрибутам
- Обнаружение проблем с производительностью и надёжностью по данным трассировок

## Создание монитора трассировок

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Трассировки**
4. Выберите телеметрические сервисы для мониторинга
5. При необходимости настройте фильтры и критерии для спанов

## Параметры конфигурации

### Телеметрические сервисы

Выберите один или несколько сервисов для мониторинга трассировок. Сервисы должны отправлять трассировки в OneUptime через OpenTelemetry.

### Фильтры спанов

| Фильтр | Описание | Обязательно |
|--------|----------|-------------|
| Статусы спанов | Фильтрация по коду статуса спана (OK, ERROR, UNSET) | Нет |
| Имя спана | Текстовый поиск по именам спанов (например, имена операций или конечных точек) | Нет |
| Атрибуты | Пары ключ-значение для фильтрации по пользовательским атрибутам спанов | Нет |
| Временное окно | Глубина поиска спанов (в секундах, по умолчанию: 60) | Нет |

### Коды статуса спанов

- **OK** — операция завершена успешно
- **ERROR** — операция завершилась с ошибкой
- **UNSET** — статус не задан явно

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки | Описание |
|--------------|----------|
| Количество спанов | Число спанов, соответствующих фильтрам, за временное окно |

### Типы фильтров

- **Больше чем** — количество спанов превышает порог
- **Меньше чем** — количество спанов ниже порога
- **Больше или равно** — количество спанов на уровне порога или выше
- **Меньше или равно** — количество спанов на уровне порога или ниже
- **Равно** — количество спанов точно совпадает
- **Не равно** — количество спанов не совпадает

### Примеры критериев

#### Оповещение при более чем 50 спанах ошибок за 60 секунд

- **Статусы спанов**: ERROR
- **Временное окно**: 60 секунд
- **Проверять**: Количество спанов
- **Тип фильтра**: Больше чем
- **Значение**: 50

#### Оповещение при ошибках в конкретной конечной точке

- **Имя спана**: `POST /api/checkout`
- **Статусы спанов**: ERROR
- **Временное окно**: 120 секунд
- **Проверять**: Количество спанов
- **Тип фильтра**: Больше чем
- **Значение**: 0

## Требования для настройки

Мониторинг трассировок требует, чтобы ваши приложения отправляли распределённые трассировки в OneUptime через OpenTelemetry. Инструкции по настройке см. в документации [OpenTelemetry](/docs/telemetry/open-telemetry).
