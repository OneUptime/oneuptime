# Монитор внешних страниц статуса

Мониторинг внешних страниц статуса позволяет отслеживать статус-страницы сторонних поставщиков и получать оповещения при сбоях или деградации зависимых сервисов. OneUptime периодически проверяет внешние страницы статуса (AWS, GCP, Azure, GitHub и другие) и оценивает их состояние.

## Обзор

Мониторы внешних страниц статуса проверяют работоспособность зависимых сервисов, запрашивая их публичные страницы статуса. Это позволяет:

- Мониторинг доступности сторонних сервисов, от которых зависит ваше приложение
- Получение оповещений при сбоях у поставщиков вышестоящего уровня
- Отслеживание статусов отдельных компонентов (например, «AWS EC2 us-east-1»)
- Обнаружение деградации производительности до того, как она затронет пользователей
- Корреляция собственных инцидентов с проблемами у поставщиков вышестоящего уровня

## Поддерживаемые поставщики

OneUptime поддерживает мониторинг страниц статуса через следующие методы:

| Тип поставщика           | Описание                                                   |
| ------------------------ | ---------------------------------------------------------- |
| **Авто** (по умолчанию)  | Автоматически определяет формат страницы статуса           |
| **Atlassian Statuspage** | Страницы статуса на основе Atlassian Statuspage (JSON API) |
| **RSS**                  | Страницы статуса, предоставляющие RSS-канал                |
| **Atom**                 | Страницы статуса, предоставляющие Atom-канал               |

### Автоматическое определение

При выборе варианта **Авто** OneUptime пытается автоматически определить формат страницы статуса:

1. Сначала пробует JSON API Atlassian Statuspage (`/api/v2/status.json` и `/api/v2/components.json`)
2. При неудаче пытается разобрать страницу как RSS- или Atom-канал
3. В качестве последнего варианта выполняет базовую проверку доступности HTTP

## Создание монитора внешней страницы статуса

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Внешняя страница статуса**
4. Введите URL страницы статуса для мониторинга
5. При желании выберите конкретный тип поставщика (или оставьте «Авто»)
6. При желании введите имя компонента для ограничения мониторинга конкретным компонентом
7. При необходимости настройте критерии мониторинга

## Параметры конфигурации

### URL страницы статуса

Введите URL внешней страницы статуса для мониторинга. Для сайтов на платформе Atlassian Statuspage это обычно корневой URL (например, `https://status.example.com`). Для RSS/Atom-каналов введите URL канала напрямую.

### Тип поставщика

Выберите тип поставщика страницы статуса. Используйте **Авто** (по умолчанию), чтобы позволить OneUptime определить формат автоматически, или укажите конкретный тип поставщика, если он вам известен.

### Фильтр по имени компонента

Если страница статуса отображает несколько компонентов, можно указать имя компонента для мониторинга только этого конкретного компонента. Например, для мониторинга только AWS EC2 в us-east-1 введите `EC2 us-east-1` (точное имя компонента, как указано на странице статуса).

Если имя компонента не указано, мониторинг выполняется для общего статуса страницы.

### Дополнительные параметры

#### Тайм-аут

Максимальное время (в миллисекундах) ожидания ответа от страницы статуса. По умолчанию 10000 мс (10 секунд).

#### Повторные попытки

Количество повторных попыток при неудачном запросе. По умолчанию 3 попытки.

## Критерии мониторинга

Можно настроить критерии, определяющие, считается ли внешний сервис доступным, деградированным или недоступным, на основе:

- **Доступен** — доступна ли страница статуса и возвращает ли данные о состоянии
- **Общий статус** — общий индикатор статуса страницы (например, «operational», «major_outage»)
- **Статус компонента** — статус конкретного компонента (при использовании фильтра по имени компонента)
- **Активные инциденты** — количество активных инцидентов, отображаемых на странице статуса
- **Время отклика** — время, необходимое для получения данных страницы статуса

## Шаблонизация инцидентов и оповещений

При создании инцидентов или оповещений на основе мониторов внешних страниц статуса можно использовать следующие переменные шаблона:

| Переменная                | Описание                                  |
| ------------------------- | ----------------------------------------- |
| `{{isOnline}}`            | Доступна ли страница статуса (true/false) |
| `{{responseTimeInMs}}`    | Время отклика в миллисекундах             |
| `{{failureCause}}`        | Причина сбоя, если применимо              |
| `{{overallStatus}}`       | Значение общего индикатора статуса        |
| `{{activeIncidentCount}}` | Количество активных инцидентов            |
| `{{componentStatuses}}`   | Массив JSON со статусами компонентов      |

## Рекомендации по использованию

- **Используйте тип поставщика «Авто»**, если формат неизвестен — автоматическое определение хорошо работает для большинства страниц статуса
- **Отслеживайте конкретные компоненты**, если вы зависите только от отдельных сервисов (например, конкретного региона AWS)
- **Настройте корреляцию инцидентов** — когда мониторы обнаруживают проблемы, а страница статуса поставщика также показывает неполадки, это помогает быстрее определить первопричину
- **Комбинируйте с другими мониторами** — используйте мониторы внешних страниц статуса совместно с мониторами API и сайтов для комплексного контроля
