# Монитор метрик

Мониторинг метрик позволяет отслеживать пользовательские метрики приложений и инфраструктуры, собираемые через OpenTelemetry. OneUptime оценивает значения метрик за определённое временное окно и формирует оповещения на основе заданных критериев.

## Обзор

Мониторы метрик запрашивают и оценивают числовые метрики из ваших телеметрических сервисов. Это позволяет:

- Мониторинг пользовательских метрик приложений (частота запросов, глубина очередей, частота ошибок и др.)
- Отслеживание метрик инфраструктуры (CPU, память, диск, сеть)
- Создание сложных метрических запросов с фильтрами и агрегацией
- Объединение нескольких метрик с помощью математических формул
- Настройка оповещений на основе пороговых значений метрик

## Создание монитора метрик

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Метрики**
4. Настройте метрические запросы и необязательные формулы
5. Выберите стратегию агрегации
6. При необходимости настройте критерии мониторинга

## Параметры конфигурации

### Метрические запросы

Определите один или несколько метрических запросов. Каждый запрос включает:

| Поле            | Описание                                                             | Обязательно |
| --------------- | -------------------------------------------------------------------- | ----------- |
| Имя метрики     | Название метрики для запроса                                         | Да          |
| Тип агрегации   | Способ агрегации необработанных значений (sum, avg, min, max, count) | Да          |
| Атрибуты        | Фильтры ключ-значение для сужения данных метрики                     | Нет         |
| Агрегировать по | Измерения для группировки метрики                                    | Нет         |

Каждому запросу присваивается псевдоним (например, `a`, `b`, `c`) для использования в формулах.

### Формулы

Комбинируйте несколько метрических запросов с помощью математических выражений. Например:

- `a / b * 100` — вычисление процентного соотношения из двух запросов
- `a + b` — сумма двух метрик
- `a - b` — разность метрик

### Скользящее временное окно

Выберите временное окно для оценки метрик:

- Последняя 1 минута
- Последние 5 минут
- Последние 10 минут
- Последние 15 минут
- Последние 30 минут
- Последние 60 минут

### Стратегия агрегации

Выберите способ агрегации значений метрик для оценки:

| Стратегия             | Описание                                              |
| --------------------- | ----------------------------------------------------- |
| Среднее               | Среднее значение за временное окно                    |
| Сумма                 | Сумма всех значений                                   |
| Максимальное значение | Наибольшее значение за временное окно                 |
| Минимальное значение  | Наименьшее значение за временное окно                 |
| Все значения          | Все значения должны соответствовать критерию          |
| Любое значение        | Хотя бы одно значение должно соответствовать критерию |

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки     | Описание                                                              |
| ---------------- | --------------------------------------------------------------------- |
| Значение метрики | Агрегированное значение настроенного метрического запроса или формулы |

### Типы фильтров

- **Больше чем** — значение метрики превышает порог
- **Меньше чем** — значение метрики ниже порога
- **Больше или равно** — значение метрики на уровне порога или выше
- **Меньше или равно** — значение метрики на уровне порога или ниже
- **Равно** — значение метрики точно совпадает
- **Не равно** — значение метрики не совпадает

### Примеры критериев

#### Оповещение при частоте ошибок свыше 5%

- **Запрос a**: `http_requests_total` с фильтром `status=5xx`
- **Запрос b**: `http_requests_total`
- **Формула**: `a / b * 100`
- **Проверять**: Значение метрики
- **Тип фильтра**: Больше чем
- **Значение**: 5

#### Оповещение при высокой глубине очереди запросов

- **Запрос**: `request_queue_size`, агрегация: Максимальное значение
- **Проверять**: Значение метрики
- **Тип фильтра**: Больше чем
- **Значение**: 1000

## Требования для настройки

Мониторинг метрик требует, чтобы ваши приложения или инфраструктура отправляли метрики в OneUptime через OpenTelemetry. Инструкции по настройке см. в документации [OpenTelemetry](/docs/telemetry/open-telemetry).
