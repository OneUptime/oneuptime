# Критерии мониторинга: выражения JavaScript

Для создания пользовательских критериев мониторинга можно использовать выражения JavaScript. Выражение вычисляется в контексте отслеживаемого объекта и должно возвращать логическое значение. Если выражение возвращает `true`, критерий мониторинга выполнен. Если `false` — не выполнен.

Выражения JavaScript в качестве критериев мониторинга доступны для следующих типов мониторов: API, Сайт и Входящий запрос.

### Мониторы сайтов и API

В контексте отслеживаемого объекта доступны следующие переменные:

| Переменная | Описание | Тип |
| --- | --- | --- |
| `responseBody` | Объект тела ответа. Если тело ответа в формате HTML/XML — строка; если JSON — объект JSON. | `string` или `JSON` |
| `responseHeaders` | Объект заголовков ответа. | `Dictionary<string>` |
| `responseStatusCode` | Код состояния ответа. | `number` |
| `responseTimeInMs` | Время ответа в миллисекундах. | `number` |

#### Пример

В следующем примере показано, как использовать выражение JavaScript для мониторинга сайта на наличие определённой строки в теле ответа:

```javascript

/**
 *  
 * Если тело ответа в формате JSON, responseBody будет объектом JSON
 * {
 *    "item": "hello"
 * }
 * 
 *  **/

"{{responseBody.item}}" === "hello"

// или можно использовать заголовки ответа

"{{responseHeaders.contentType}} === "application/json"


// можно также использовать регулярные выражения

"{{responseBody.item}}".match(/hello/)

// можно использовать код состояния ответа

{{responseStatusCode}} === 200

// можно объединять несколько выражений с помощью логических операторов

"{{responseBody.item}}" === "hello" && {{responseStatusCode}} === 200

// для массивов используйте следующий синтаксис

/**
 *  
 * Если тело ответа: 
 * {
 *    "item": [{
 *          "name": "hello"
 *      }]
 * }
 * 
 *  **/

"{{responseBody.items[0].name}}" === "hello"
```

### Мониторы входящих запросов

В контексте отслеживаемого объекта доступны следующие переменные:

| Переменная | Описание | Тип |
| --- | --- | --- |
| `requestBody` | Объект тела запроса. | `string` или `JSON` |
| `requestHeaders` | Объект заголовков запроса. | `Dictionary<string>` |


#### Пример

В следующем примере показано, как использовать выражение JavaScript для мониторинга входящего запроса на наличие определённой строки в теле запроса:

```javascript
"{{requestBody.item}}" === "hello"

// или можно использовать заголовки запроса

"{{requestHeaders.contentType}}" === "text/html"

// можно также использовать регулярные выражения

"{{requestBody.item}}".match(/hello/)

// можно объединять несколько выражений с помощью логических операторов

"{{requestBody.item}}" === "hello" && "{{requestHeaders.contentType}}" === "text/html"

// для массивов используйте следующий синтаксис

"{{requestBody.items[0].name}}" === "hello"
```

### Важные замечания

* Скрипты имеют тайм-аут 1 секунда; при превышении этого времени возвращается `false`.
* `{{var}}` заменяется значением переменной, поэтому при сравнении строк необходимо заключать их в кавычки (например, `"{{responseBody.item}}" === "hello"`), а при сравнении чисел кавычки не нужны (например, `{{responseStatusCode}} === 200`).
