# SNMP-монитор

Мониторинг SNMP (Simple Network Management Protocol) позволяет отслеживать работу сетевых устройств — коммутаторов, маршрутизаторов, межсетевых экранов и другой сетевой инфраструктуры — путём запроса OID (Object Identifiers) по SNMP.

## Обзор

SNMP-мониторы запрашивают сетевые устройства на предмет конкретных управляющих данных с использованием OID. Это позволяет:

- Мониторинг доступности и работоспособности устройств
- Отслеживание статистики интерфейсов (трафик, ошибки, статус)
- Мониторинг системных метрик (CPU, память, время работы)
- Проверка специфических OID от производителей
- Настройка оповещений на основе значений OID

## Создание SNMP-монитора

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **SNMP**
4. Настройте параметры SNMP, описанные ниже

## Параметры конфигурации

### Основные настройки

| Поле         | Описание                               | Обязательно |
| ------------ | -------------------------------------- | ----------- |
| Версия SNMP  | Версия протокола: v1, v2c или v3       | Да          |
| Имя хоста/IP | Имя хоста или IP-адрес SNMP-устройства | Да          |
| Порт         | Порт SNMP (по умолчанию: 161)          | Да          |

### Аутентификация

#### SNMP v1/v2c

Для SNMP v1 и v2c требуется только строка сообщества:

| Поле              | Описание                                    | Обязательно |
| ----------------- | ------------------------------------------- | ----------- |
| Строка сообщества | Строка сообщества SNMP (например, «public») | Да          |

#### SNMP v3

SNMPv3 обеспечивает повышенную безопасность с аутентификацией и шифрованием:

| Поле                    | Описание                              | Обязательно                 |
| ----------------------- | ------------------------------------- | --------------------------- |
| Уровень безопасности    | noAuthNoPriv, authNoPriv или authPriv | Да                          |
| Имя пользователя        | Имя пользователя SNMPv3               | Да                          |
| Протокол аутентификации | MD5, SHA, SHA256 или SHA512           | При authNoPriv или authPriv |
| Ключ аутентификации     | Пароль аутентификации                 | При authNoPriv или authPriv |
| Протокол шифрования     | DES, AES или AES256                   | При authPriv                |
| Ключ шифрования         | Пароль шифрования                     | При authPriv                |

### OID для мониторинга

Добавьте OID, которые нужно запрашивать у устройства. Для каждого OID можно указать:

| Поле     | Описание                                   | Обязательно |
| -------- | ------------------------------------------ | ----------- |
| OID      | Числовой OID (например, 1.3.6.1.2.1.1.1.0) | Да          |
| Имя      | Дружественное имя OID (например, sysDescr) | Нет         |
| Описание | Описание того, что представляет данный OID | Нет         |

### Готовые шаблоны OID

OneUptime предоставляет шаблоны для часто используемых OID:

#### MIB системы

| 1.3.6.1.2.1.1.3.0 | sysUpTime   | Время работы системы (в тиках) |
| 1.3.6.1.2.1.1.5.0 | sysName     | Имя системы                    |
| 1.3.6.1.2.1.1.6.0 | sysLocation | Расположение системы           |
| 1.3.6.1.2.1.1.4.0 | sysContact  | Контактные данные системы      |

### Дополнительные настройки

| Поле              | Описание                              | По умолчанию |
| ----------------- | ------------------------------------- | ------------ |
| Тайм-аут          | Время ожидания ответа (мс)            | 5000         |
| Повторные попытки | Количество повторных попыток при сбое | 3            |

## Критерии мониторинга

Можно настроить критерии для проверки ответов SNMP и формирования оповещений или инцидентов.

### Доступные типы проверок

| Тип проверки             | Описание                                         |
| ------------------------ | ------------------------------------------------ |
| SNMP-устройство доступно | Проверка отклика устройства на SNMP-запросы      |
| Время отклика SNMP       | Проверка времени отклика запроса в миллисекундах |
| Значение SNMP OID        | Проверка значения, возвращённого конкретным OID  |
| Существование SNMP OID   | Проверка наличия значения OID (не null)          |

### Примеры критериев

#### Проверка доступности устройства

- **Проверять**: SNMP-устройство доступно
- **Тип фильтра**: True

#### Оповещение при превышении порога времени отклика

- **Проверять**: Время отклика SNMP (мс)
- **Тип фильтра**: Больше чем
- **Значение**: 1000

#### Проверка статуса интерфейса

- **Проверять**: Значение SNMP OID
- **OID**: 1.3.6.1.2.1.2.2.1.8.1
- **Тип фильтра**: Равно
- **Значение**: 1 (1 = активен, 2 = неактивен)

#### Проверка порога нагрузки CPU

- **Проверять**: Значение SNMP OID
- **OID**: 1.3.6.1.2.1.25.3.3.1.2.1
- **Тип фильтра**: Больше чем
- **Значение**: 80

## Использование секретов монитора

Для безопасного хранения конфиденциальных данных, таких как строки сообщества и учётные данные SNMPv3, можно использовать секреты.

### Добавление секрета

1. Перейдите в **Настройки проекта** -> **Секреты монитора** -> **Создать секрет монитора**
2. Добавьте секрет (например, строку сообщества или пароль SNMPv3)
3. Выберите SNMP-мониторы, которые должны иметь доступ к данному секрету

### Использование секретов в конфигурации SNMP

Используйте синтаксис `{{monitorSecrets.SECRET_NAME}}` в любом конфиденциальном поле:

- **Строка сообщества**: `{{monitorSecrets.SnmpCommunity}}`
- **Ключ аутентификации SNMPv3**: `{{monitorSecrets.SnmpAuthKey}}`
- **Ключ шифрования SNMPv3**: `{{monitorSecrets.SnmpPrivKey}}`

## Переменные шаблона для оповещений

При создании шаблонов инцидентов и оповещений можно использовать следующие переменные:

| Переменная             | Описание                                                      |
| ---------------------- | ------------------------------------------------------------- |
| `{{isOnline}}`         | Доступно ли устройство (true/false)                           |
| `{{responseTimeInMs}}` | Время отклика запроса в миллисекундах                         |
| `{{failureCause}}`     | Сообщение об ошибке при неудачном запросе                     |
| `{{oidResponses}}`     | Массив объектов ответа OID                                    |
| `{{OID_NAME}}`         | Значение конкретного OID по имени (например, `{{sysUpTime}}`) |

## Устранение неполадок

### Распространённые проблемы

#### Устройство не отвечает

- Убедитесь в правильности IP-адреса/имени хоста устройства
- Убедитесь, что SNMP включён на устройстве
- Проверьте правила брандмауэра, разрешающие UDP-порт 161
- Убедитесь в правильности строки сообщества

#### Ошибки аутентификации (v3)

- Проверьте имя пользователя, протокол аутентификации и ключ аутентификации
- Убедитесь, что уровень безопасности соответствует конфигурации устройства
- Убедитесь в правильности протокола и ключа шифрования для уровня authPriv

#### OID не найден

- Убедитесь, что OID поддерживается вашим устройством
- Проверьте, требует ли OID загрузки определённого MIB
- Попробуйте запросить OID напрямую с помощью инструментов snmpget/snmpwalk

### Тестирование подключения SNMP

Перед настройкой мониторинга можно проверить подключение SNMP с помощью инструментов командной строки:

```bash
# SNMP v2c
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0

# SNMP v3 (authPriv)
snmpget -v3 -u username -l authPriv -a SHA -A authpassword -x AES -X privpassword 192.168.1.1 1.3.6.1.2.1.1.1.0
```

## Рекомендации по использованию

1. **По возможности используйте SNMPv3** — он обеспечивает аутентификацию и шифрование для повышенной безопасности
2. **Храните учётные данные в секретах** — никогда не указывайте строки сообщества и пароли в открытом виде
3. **Отслеживайте только необходимые OID** — запрашивайте только нужные данные для снижения сетевой нагрузки
4. **Устанавливайте подходящие тайм-ауты** — сетевые устройства могут иметь разное время отклика
5. **Используйте описательные имена OID** — это упрощает понимание сообщений оповещений
6. **Тестируйте перед развёртыванием** — проверяйте подключение SNMP перед созданием мониторов
