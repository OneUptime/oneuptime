# Монитор профилей

Мониторинг профилей позволяет отслеживать данные непрерывного профилирования приложений и формировать оповещения на основе количества и шаблонов профилей. OneUptime оценивает данные профилирования из ваших телеметрических сервисов за определённое временное окно.

## Обзор

Мониторы профилей подсчитывают и фильтруют данные профилирования, соответствующие заданным критериям. Это позволяет:

- Мониторинг данных непрерывного профилирования приложений
- Фильтрация профилей по типу (CPU, память, горутины и др.)
- Отслеживание объёма и шаблонов профилей
- Оповещение при аномалиях профилирования
- Фильтрация по пользовательским атрибутам профилей

## Создание монитора профилей

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Профили**
4. Выберите телеметрические сервисы для мониторинга
5. При необходимости настройте фильтры и критерии для профилей

## Параметры конфигурации

### Телеметрические сервисы

Выберите один или несколько сервисов для мониторинга профилей. Сервисы должны отправлять данные непрерывного профилирования в OneUptime через OpenTelemetry.

### Фильтры профилей

| Фильтр         | Описание                                                                 | Обязательно |
| -------------- | ------------------------------------------------------------------------ | ----------- |
| Типы профилей  | Фильтрация по именам типов профилей (например, CPU, memory, goroutines)  | Нет         |
| Атрибуты       | Пары ключ-значение для фильтрации по пользовательским атрибутам профилей | Нет         |
| Временное окно | Глубина поиска профилей (в секундах, по умолчанию: 60)                   | Нет         |

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки        | Описание                                                    |
| ------------------- | ----------------------------------------------------------- |
| Количество профилей | Число профилей, соответствующих фильтрам, за временное окно |

### Типы фильтров

- **Больше чем** — количество профилей превышает порог
- **Меньше чем** — количество профилей ниже порога
- **Больше или равно** — количество профилей на уровне порога или выше
- **Меньше или равно** — количество профилей на уровне порога или ниже
- **Равно** — количество профилей точно совпадает
- **Не равно** — количество профилей не совпадает

### Примеры критериев

#### Оповещение при отсутствии профилей за 5 минут

- **Временное окно**: 300 секунд
- **Проверять**: Количество профилей
- **Тип фильтра**: Равно
- **Значение**: 0

## Требования для настройки

Мониторинг профилей требует, чтобы ваши приложения отправляли данные непрерывного профилирования в OneUptime через OpenTelemetry. Инструкции по настройке см. в документации [OpenTelemetry](/docs/telemetry/open-telemetry).
