# Монитор сервера / виртуальной машины

Мониторинг серверов и виртуальных машин позволяет отслеживать работоспособность и производительность серверов, виртуальных машин и другой инфраструктуры путём установки лёгкого агента, передающего системные метрики в OneUptime.

## Обзор

Мониторы серверов используют агент инфраструктуры, установленный на ваших серверах, для сбора и передачи системных метрик. Это позволяет:

- Мониторинг доступности и работоспособности серверов
- Отслеживание использования CPU, памяти и диска
- Мониторинг запущенных процессов
- Настройка оповещений на основе порогов загрузки ресурсов
- Обнаружение проблем с инфраструктурой до того, как они затронут сервисы

## Создание монитора сервера

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Сервер / Виртуальная машина**
4. Для данного монитора будет сгенерирован **Секретный ключ** — он потребуется для настройки агента
5. Следуйте инструкциям по установке агента на вашем сервере

## Установка агента инфраструктуры

Агент инфраструктуры OneUptime — это лёгкий демон на Go, который собирает системные метрики и каждые 30 секунд отправляет их в OneUptime. Поддерживается Linux, macOS и Windows.

### Linux / macOS

```bash
# Установите агент
curl -sSL https://oneuptime.com/docs/static/scripts/infrastructure-agent/install.sh | sudo bash

# Настройте агент
sudo oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com

# Запустите агент
sudo oneuptime-infrastructure-agent start
```

Замените `YOUR_SECRET_KEY` секретным ключом, отображаемым в настройках монитора, а `https://oneuptime.com` — URL вашего экземпляра OneUptime при самостоятельном хостинге.

### Windows

1. Загрузите последнюю версию агента с [GitHub Releases](https://github.com/OneUptime/oneuptime/releases/latest)
   - `oneuptime-infrastructure-agent_windows_amd64.zip` для систем x64
   - `oneuptime-infrastructure-agent_windows_arm64.zip` для систем ARM64
2. Распакуйте zip-архив
3. Откройте командную строку от имени администратора и выполните:

```bash
# Настройте агент
oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com

# Запустите агент
oneuptime-infrastructure-agent start
```

### Поддержка прокси

Если сервер подключается к интернету через прокси, можно настроить агент для работы с ним:

```bash
sudo oneuptime-infrastructure-agent configure --secret-key=YOUR_SECRET_KEY --oneuptime-url=https://oneuptime.com --proxy-url=http://proxy.example.com:8080
```

## Команды агента

Агент инфраструктуры поддерживает следующие команды:

| Команда     | Описание                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| `configure` | Настройка агента с секретным ключом и URL OneUptime                                                     |
| `start`     | Запуск службы агента                                                                                    |
| `stop`      | Остановка службы агента                                                                                 |
| `restart`   | Перезапуск службы агента                                                                                |
| `status`    | Отображение текущего статуса службы                                                                     |
| `logs`      | Просмотр журналов агента (используйте `-n` для указания количества строк, `-f` для непрерывного вывода) |
| `uninstall` | Удаление службы агента                                                                                  |

## Собираемые метрики

Агент собирает следующие метрики с вашего сервера:

### CPU

- **Процент использования CPU** — общая загрузка CPU в процентах
- **Ядра CPU** — количество ядер процессора

### Память

- **Общая память** — общий доступный объём памяти
- **Используемая память** — объём памяти в текущем использовании
- **Свободная память** — доступная свободная память
- **Процент использования памяти** — загрузка памяти в процентах

### Диск

Для каждого смонтированного диска/раздела:

- **Общий объём диска** — общая ёмкость диска
- **Используемое дисковое пространство** — занятое пространство
- **Свободное дисковое пространство** — доступное свободное пространство
- **Процент использования диска** — загрузка диска в процентах
- **Путь диска** — точка монтирования диска

### Процессы

- **Имя процесса** — название запущенного процесса
- **Идентификатор процесса (PID)** — идентификатор процесса
- **Команда процесса** — полная команда для запуска процесса

## Критерии мониторинга

Можно настроить критерии, определяющие, считается ли сервер доступным, деградированным или недоступным.

### Доступные типы проверок

| Тип проверки                 | Описание                                              |
| ---------------------------- | ----------------------------------------------------- |
| Доступен                     | Отправляет ли агент сервера данные (на основе пульса) |
| Процент использования CPU    | Текущий процент загрузки CPU                          |
| Процент использования памяти | Текущий процент загрузки памяти                       |
| Процент использования диска  | Текущий процент загрузки диска (для конкретного пути) |
| Имя процесса сервера         | Проверка наличия процесса с конкретным именем         |
| Команда процесса сервера     | Проверка наличия процесса с конкретной командой       |
| PID процесса сервера         | Проверка наличия процесса с конкретным PID            |

### Типы фильтров

Для числовых метрик (CPU, память, диск):

- **Больше чем** — значение превышает порог
- **Меньше чем** — значение ниже порога
- **Больше или равно** — значение на уровне порога или выше
- **Меньше или равно** — значение на уровне порога или ниже
- **Оценка за период** — оценка с использованием агрегации (Среднее, Сумма, Максимум, Минимум, Все значения, Любое значение) за временное окно

Для проверок процессов:

- **Выполняется** — процесс в настоящее время запущен
- **Не выполняется** — процесс не запущен

### Примеры критериев

#### Статус «Недоступен» при прекращении передачи данных агентом

- **Проверять**: Доступен
- **Тип фильтра**: False

#### Оповещение при загрузке CPU свыше 90%

- **Проверять**: Процент использования CPU
- **Тип фильтра**: Больше чем
- **Значение**: 90

#### Оповещение при заполненности диска более чем на 85%

- **Проверять**: Процент использования диска
- **Путь диска**: `/`
- **Тип фильтра**: Больше чем
- **Значение**: 85

#### Оповещение при загрузке памяти свыше 80%

- **Проверять**: Процент использования памяти
- **Тип фильтра**: Больше чем
- **Значение**: 80

#### Оповещение при остановке критического процесса

- **Проверять**: Имя процесса сервера
- **Тип фильтра**: Не выполняется
- **Значение**: `nginx`

## Устранение неполадок

### Агент не передаёт данные

- Убедитесь, что агент запущен: `sudo oneuptime-infrastructure-agent status`
- Проверьте журналы агента: `sudo oneuptime-infrastructure-agent logs -n 50`
- Убедитесь в правильности секретного ключа
- Убедитесь, что сервер может получить доступ к URL вашего экземпляра OneUptime
- Проверьте правила брандмауэра, разрешающие исходящие HTTPS-соединения

### Высокое потребление ресурсов агентом

Агент разработан для минимальной нагрузки. При обнаружении высокого потребления ресурсов:

- Перезапустите агент: `sudo oneuptime-infrastructure-agent restart`
- Проверьте журналы агента на наличие ошибок

### Проблемы с прокси

- Убедитесь в правильности URL и порта прокси
- Убедитесь, что прокси разрешает соединения с вашим экземпляром OneUptime
- Повторно настройте: `sudo oneuptime-infrastructure-agent configure --proxy-url=http://proxy:port --secret-key=YOUR_KEY --oneuptime-url=YOUR_URL`

## Рекомендации по использованию

1. **Устанавливайте значимые пороги** — настройте критерии деградации и недоступности, соответствующие нормальным рабочим диапазонам вашего сервера
2. **Отслеживайте критически важные процессы** — используйте мониторинг процессов для обеспечения постоянной работы таких сервисов, как веб-серверы и базы данных
3. **Проактивно контролируйте использование диска** — проблемы с дисковым пространством могут вызвать сбои приложений; устанавливайте оповещения задолго до заполнения дисков
4. **Используйте «Оценку за период»** — для метрик с кратковременными пиками, таких как CPU, применяйте агрегацию за период времени для избежания ложных срабатываний
5. **Поддерживайте агент в актуальном состоянии** — периодически обновляйте агент инфраструктуры для получения последних улучшений и исправлений
