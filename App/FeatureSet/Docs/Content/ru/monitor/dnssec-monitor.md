# Монитор DNSSEC

Мониторинг DNSSEC позволяет проверять криптографическую целостность DNS-ответов для ваших зон. OneUptime периодически выполняет полную проверку DNSSEC — проверяет записи DNSKEY, делегирование DS в родительской зоне, действительность подписей RRSIG, согласие резолверов по AD flag, а также консистентность между авторитативными серверами имён.

## Обзор

Мониторы DNSSEC проверяют всю цепочку доверия от корневой зоны до вашего домена. Это позволяет:

- Обнаружить нарушенные цепочки DNSSEC до того, как резолверы начнут возвращать SERVFAIL вашим пользователям
- Получать предупреждения до истечения срока действия ключей подписи зоны
- Убедиться, что ваши записи DS корректно опубликованы в родительской зоне
- Отслеживать расхождения между авторитативными серверами имён (рассинхронизация primary/secondary)
- Подтверждать, что проверяющие резолверы действительно устанавливают AD flag для вашей зоны

## Создание монитора DNSSEC

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **DNSSEC**
4. Введите зону (домен) для проверки
5. При необходимости настройте резолверы и критерии мониторинга

## Параметры конфигурации

### Основные настройки

| Поле                                   | Описание                                                                                                     | Обязательно |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ----------- |
| Зона (имя домена)                      | Зона для проверки через DNSSEC (например, `example.com`)                                                     | Да          |
| Резолверы                              | Список проверяющих резолверов через запятую (например, `1.1.1.1, 8.8.8.8, 9.9.9.9`)                          | Да          |
| Проверка консистентности серверов имён | Запрашивать каждый авторитативный сервер имён напрямую и убедиться, что они возвращают одинаковый SOA serial | Нет         |

### Дополнительные настройки

| Поле                                      | Описание                                                    | По умолчанию |
| ----------------------------------------- | ----------------------------------------------------------- | ------------ |
| Предупреждение об истечении подписи (дни) | Пороговое значение по умолчанию для фильтра истечения RRSIG | 7            |
| Тайм-аут (мс)                             | Время ожидания для каждого DNS-запроса                      | 10000        |
| Повторные попытки                         | Количество повторных попыток при сбое                       | 3            |

## Критерии мониторинга

Можно настроить критерии, определяющие, считается ли ваша зона доступной, деградированной или недоступной, на основе:

### Доступные типы проверок

| Тип проверки                           | Описание                                                           |
| -------------------------------------- | ------------------------------------------------------------------ |
| Цепочка DNSSEC действительна           | Вся цепочка проверки (корень → TLD → зона) разрешается корректно   |
| Запись DNSSEC DNSKEY существует        | Зона публикует хотя бы одну запись DNSKEY                          |
| Запись DNSSEC DS существует у родителя | Родительская зона публикует запись DS, соответствующую KSK зоны    |
| Дней до истечения подписи DNSSEC       | Количество дней до истечения ближайшей подписи RRSIG               |
| Согласие резолверов DNSSEC (AD Flag)   | Каждый опрошенный резолвер возвращает AD (Authenticated Data) flag |
| Серверы имён DNSSEC консистентны       | Все авторитативные серверы имён возвращают одинаковый SOA serial   |
| DNSSEC действителен                    | Совокупный успех/неудача по всем проверкам валидации               |

### Типы фильтров

Для **Цепочка DNSSEC действительна**, **Запись DNSSEC DNSKEY существует**, **Запись DNSSEC DS существует у родителя**, **Согласие резолверов DNSSEC (AD Flag)**, **Серверы имён DNSSEC консистентны** и **DNSSEC действителен**:

- **True** — условие истинно
- **False** — условие ложно

Для **Дней до истечения подписи DNSSEC**:

- **Больше чем**, **Меньше чем**, **Больше или равно**, **Меньше или равно**, **Равно**, **Не равно**

### Примеры критериев

#### Оповещение при нарушении цепочки DNSSEC

- **Проверять**: Цепочка DNSSEC действительна
- **Тип фильтра**: False

#### Предупреждение до истечения подписей

- **Проверять**: Дней до истечения подписи DNSSEC
- **Тип фильтра**: Меньше чем
- **Значение**: 7

#### Обнаружение отсутствия DS у родителя (нарушение делегирования)

- **Проверять**: Запись DNSSEC DS существует у родителя
- **Тип фильтра**: False

#### Обнаружение разногласий между резолверами

- **Проверять**: Согласие резолверов DNSSEC (AD Flag)
- **Тип фильтра**: False

#### Обнаружение несогласованности серверов имён

- **Проверять**: Серверы имён DNSSEC консистентны
- **Тип фильтра**: False

## Лучшие практики

1. **Используйте несколько публичных резолверов** — по умолчанию используйте `1.1.1.1`, `8.8.8.8` и `9.9.9.9`, чтобы сбой одного резолвера не приводил к ложным срабатываниям
2. **Предупреждайте задолго до истечения срока** — настройте оповещения о деградации за 7 дней и оповещения об отключении за 2 дня до истечения срока действия подписи; смена ключей может незаметно завершиться неудачей
3. **Мониторьте каждую подписанную зону** — включайте apex-домены, подписанные поддомены и любую зону, делегированную другому оператору
4. **Включите проверки консистентности серверов имён** — это обнаруживает проблемы синхронизации primary/secondary, которые сама по себе валидация DNSSEC пропустит, при условии что ваша сеть не блокирует исходящий DNS на произвольные IP-адреса
