# Монитор Kubernetes

Мониторинг Kubernetes позволяет отслеживать работоспособность и производительность ваших кластеров Kubernetes, включая узлы, Pod, рабочие нагрузки и компоненты плоскости управления. OneUptime собирает метрики из вашего кластера и оценивает их по заданным критериям.

## Обзор

Мониторы Kubernetes используют метрики из вашего кластера, чтобы обеспечить глубокую видимость вашей инфраструктуры. Это позволяет:

- Отслеживать работоспособность кластера, пространства имён, рабочей нагрузки, узла и Pod
- Отслеживать использование CPU, памяти, диска и сети по ресурсам
- Обнаруживать сбои Pod, перезапуски и ошибки планирования
- Отслеживать доступность реплик Deployment
- Получать оповещения о проблемах плоскости управления (etcd, API server, scheduler)
- Отслеживать запросы и лимиты ресурсов

## Создание монитора Kubernetes

1. Перейдите в раздел **Monitors** на дашборде OneUptime
2. Нажмите **Create Monitor**
3. Выберите **Kubernetes** в качестве типа монитора
4. Выберите кластер и область ресурсов для мониторинга
5. Настройте фильтры ресурсов и запросы метрик
6. При необходимости настройте критерии мониторинга

## Опции конфигурации

### Кластер

Выберите кластер Kubernetes для мониторинга. Кластеры должны быть интегрированы с OneUptime через OpenTelemetry.

### Область ресурсов

Выберите уровень, на котором мониторить ресурсы:

| Область   | Описание                                                                     |
| --------- | ---------------------------------------------------------------------------- |
| Cluster   | Мониторинг всего кластера                                                    |
| Namespace | Мониторинг ресурсов внутри определённого пространства имён                   |
| Workload  | Мониторинг определённого Deployment, StatefulSet, DaemonSet, Job или CronJob |
| Node      | Мониторинг определённого узла кластера                                       |
| Pod       | Мониторинг определённого Pod                                                 |

### Фильтры ресурсов

Сузьте область с помощью опциональных фильтров:

| Фильтр        | Описание                                         | Применимые области       |
| ------------- | ------------------------------------------------ | ------------------------ |
| Namespace     | Пространство имён Kubernetes                     | Namespace, Workload, Pod |
| Workload Type | deployment, statefulset, daemonset, job, cronjob | Workload                 |
| Workload Name | Имя рабочей нагрузки                             | Workload                 |
| Node Name     | Имя узла                                         | Node                     |
| Pod Name      | Имя Pod                                          | Pod                      |

### Запросы метрик

Настройте один или несколько запросов метрик для оценки. Каждый запрос задаёт:

- **Имя метрики** — метрика Kubernetes для запроса
- **Агрегация** — как агрегировать значения метрики
- **Фильтры** — дополнительная фильтрация на основе атрибутов

Вы также можете создавать **формулы**, которые объединяют несколько запросов метрик с помощью математических выражений.

### Скользящее временное окно

Выберите временное окно для оценки метрик:

- За последнюю 1 минуту
- За последние 5 минут
- За последние 10 минут
- За последние 15 минут
- За последние 30 минут
- За последние 60 минут

| Node Disk I/O         | Операции чтения/записи       |

### Метрики контейнеров

| Метрика                       | Описание                            |

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки | Описание                                          |
| ------------ | ------------------------------------------------- |
| Metric Value | Значение настроенного запроса метрики или формулы |

### Типы агрегации

| Агрегация     | Описание                                      |
| ------------- | --------------------------------------------- |
| Average       | Среднее значение за временное окно            |
| Sum           | Сумма всех значений                           |
| Maximum Value | Наибольшее значение во временном окне         |
| Minimum Value | Наименьшее значение во временном окне         |
| All Values    | Все значения должны соответствовать критериям |
| Any Value     | Хотя бы одно значение должно соответствовать  |

### Типы фильтров

- **Greater Than**, **Less Than**, **Greater Than or Equal To**, **Less Than or Equal To**, **Equal To**, **Not Equal To**

## Готовые шаблоны оповещений

OneUptime предоставляет шаблоны для распространённых сценариев мониторинга Kubernetes:

| Шаблон                      | Описание                            | Порог            |
| --------------------------- | ----------------------------------- | ---------------- |
| CrashLoopBackOff Detection  | Счётчик перезапусков контейнера     | > 5 перезапусков |
| Pod Stuck in Pending        | Pod в фазе Pending                  | > 0 Pod          |
| Node Not Ready              | Состояние готовности узла           | = 0 (не готов)   |
| High Node CPU               | Использование CPU узла              | > 90%            |
| High Node Memory            | Использование памяти узла           | > 85%            |
| Deployment Replica Mismatch | Недоступные реплики                 | > 0 реплик       |
| Job Failures                | Неуспешные Pod в Job                | > 0 ошибок       |
| etcd No Leader              | Отсутствие лидера в кластере etcd   | = 0 (нет лидера) |
| API Server Throttling       | Отклонённые запросы API             | > 0 запросов     |
| Scheduler Backlog           | Ожидающие Pod в scheduler           | > 0 Pod          |
| High Node Disk Usage        | Использование файловой системы узла | > 90%            |
| DaemonSet Unavailable       | Misscheduled узлы                   | > 0 узлов        |

## Требования к настройке

Чтобы использовать мониторинг Kubernetes, вам необходимо установить агент OneUptime Kubernetes в вашем кластере. Агент отправляет метрики кластера, события, логи Pod и — по умолчанию — **трассировки приложений и HTTP RED-метрики, перехваченные через eBPF** в OneUptime по OTLP. Никаких изменений в коде или SDK для каждого приложения для просмотра трафика уровня сервисов не требуется.

См. руководство [Установка агента Kubernetes](/docs/monitor/kubernetes-agent) — оно охватывает установку Helm одной командой, опцию `preset` для выбора правильной конфигурации для вашего кластера (standard, GKE Autopilot, EKS Fargate) и переключатели `ebpf.features.*` для отдельных семейств сигналов (HTTP RED-метрики, service graph, network flows, TCP-статистика).
