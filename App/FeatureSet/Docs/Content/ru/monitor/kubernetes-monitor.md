# Монитор Kubernetes

Мониторинг Kubernetes позволяет отслеживать работоспособность и производительность кластеров Kubernetes, включая узлы, поды, рабочие нагрузки и компоненты плоскости управления. OneUptime собирает метрики из вашего кластера и оценивает их на соответствие заданным критериям.

## Обзор

Мониторы Kubernetes используют метрики кластера для обеспечения глубокой видимости инфраструктуры. Это позволяет:

- Мониторинг работоспособности кластера, пространств имён, рабочих нагрузок, узлов и подов
- Отслеживание использования CPU, памяти, диска и сети по ресурсам
- Обнаружение сбоев, перезапусков и ошибок планирования подов
- Мониторинг доступности реплик Deployment
- Оповещения о проблемах с плоскостью управления (etcd, сервер API, планировщик)
- Отслеживание запросов и лимитов ресурсов

## Создание монитора Kubernetes

1. Перейдите в раздел **Мониторы** на панели управления OneUptime
2. Нажмите **Создать монитор**
3. Выберите тип монитора **Kubernetes**
4. Выберите кластер и область ресурсов для мониторинга
5. Настройте фильтры ресурсов и метрические запросы
6. При необходимости настройте критерии мониторинга

## Параметры конфигурации

### Кластер

Выберите кластер Kubernetes для мониторинга. Кластеры должны быть интегрированы с OneUptime через OpenTelemetry.

### Область ресурсов

Выберите уровень для мониторинга ресурсов:

| Область | Описание |
|---------|----------|
| Кластер | Мониторинг всего кластера |
| Пространство имён | Мониторинг ресурсов в конкретном пространстве имён |
| Рабочая нагрузка | Мониторинг конкретного Deployment, StatefulSet, DaemonSet, Job или CronJob |
| Узел | Мониторинг конкретного узла кластера |
| Под | Мониторинг конкретного пода |

### Фильтры ресурсов

Сужение области мониторинга с помощью дополнительных фильтров:

| Фильтр | Описание | Применимые области |
|--------|----------|---------------------|
| Пространство имён | Пространство имён Kubernetes | Пространство имён, Рабочая нагрузка, Под |
| Тип рабочей нагрузки | deployment, statefulset, daemonset, job, cronjob | Рабочая нагрузка |
| Имя рабочей нагрузки | Имя рабочей нагрузки | Рабочая нагрузка |
| Имя узла | Имя узла | Узел |
| Имя пода | Имя пода | Под |

### Метрические запросы

Настройте один или несколько метрических запросов. Каждый запрос задаёт:

- **Имя метрики** — метрика Kubernetes для запроса
- **Агрегация** — способ агрегации значений метрик
- **Фильтры** — дополнительная фильтрация по атрибутам

Также можно создавать **формулы**, объединяющие несколько метрических запросов с помощью математических выражений.

### Скользящее временное окно

Выберите временное окно для оценки метрик:

- Последняя 1 минута
- Последние 5 минут
- Последние 10 минут
- Последние 15 минут
- Последние 30 минут
- Последние 60 минут

## Распространённые метрики Kubernetes

### Метрики подов

| Метрика | Описание |
|---------|----------|
| Использование CPU подом | Потребление CPU подами |
| Использование памяти подом | Потребление памяти подами |
| Использование файловой системы подом | Использование диска подами |
| Входящий/исходящий сетевой трафик пода | Сетевой трафик |
| Фаза пода | Текущая фаза пода (Running, Pending, Failed и др.) |

### Метрики узлов

| Метрика | Описание |
|---------|----------|
| Использование CPU узлом | Загрузка CPU по узлам |
| Использование памяти узлом | Загрузка памяти по узлам |
| Использование файловой системы узлом | Использование диска по узлам |
| Дисковый ввод-вывод узла | Операции чтения/записи |
| Состояние готовности узла | Готов ли узел к работе |

### Метрики контейнеров

| Метрика | Описание |
|---------|----------|
| Перезапуски контейнера | Количество перезапусков контейнера |
| Лимиты CPU/памяти контейнера | Лимиты ресурсов |
| Запросы CPU/памяти контейнера | Запросы ресурсов |
| Статус готовности контейнера | Готов ли контейнер |

### Метрики рабочих нагрузок

| Метрика | Описание |
|---------|----------|
| Доступные/недоступные реплики Deployment | Количество реплик |
| Неправильно запланированные узлы DaemonSet | Проблемы с планированием |
| Готовые реплики StatefulSet | Количество готовых реплик |
| Активные/сбойные/завершённые поды Job | Статус задания |

## Критерии мониторинга

### Доступные типы проверок

| Тип проверки | Описание |
|--------------|----------|
| Значение метрики | Значение настроенного метрического запроса или формулы |

### Типы агрегации

| Агрегация | Описание |
|-----------|----------|
| Среднее | Среднее значение за временное окно |
| Сумма | Сумма всех значений |
| Максимальное значение | Наибольшее значение за временное окно |
| Минимальное значение | Наименьшее значение за временное окно |
| Все значения | Все значения должны соответствовать критерию |
| Любое значение | Хотя бы одно значение должно соответствовать критерию |

### Типы фильтров

- **Больше чем**, **Меньше чем**, **Больше или равно**, **Меньше или равно**, **Равно**, **Не равно**

## Готовые шаблоны оповещений

OneUptime предоставляет шаблоны для типовых сценариев мониторинга Kubernetes:

| Шаблон | Описание | Порог |
|--------|----------|-------|
| Обнаружение CrashLoopBackOff | Количество перезапусков контейнера | > 5 перезапусков |
| Под застрял в состоянии Pending | Поды в фазе Pending | > 0 подов |
| Узел не готов | Состояние готовности узла | = 0 (не готов) |
| Высокая нагрузка на CPU узла | Загрузка CPU узла | > 90% |
| Высокое использование памяти узла | Загрузка памяти узла | > 85% |
| Несоответствие реплик Deployment | Недоступные реплики | > 0 реплик |
| Сбои Job | Сбойные поды в задании | > 0 сбоев |
| etcd без лидера | Отсутствие лидера в кластере etcd | = 0 (нет лидера) |
| Троттлинг сервера API | Отброшенные API-запросы | > 0 запросов |
| Очередь планировщика | Поды в очереди планировщика | > 0 подов |
| Высокое использование диска узла | Использование файловой системы узла | > 90% |
| DaemonSet недоступен | Неправильно запланированные узлы | > 0 узлов |

## Требования для настройки

Для использования мониторинга Kubernetes необходимо установить агент OneUptime Kubernetes в ваш кластер. Агент передаёт метрики кластера, события и журналы подов в OneUptime по OTLP.

См. руководство [Установка агента Kubernetes](/docs/monitor/kubernetes-agent) — в нём описана однострочная установка через Helm и параметр `preset` для выбора подходящей конфигурации для вашего кластера (standard, GKE Autopilot, EKS Fargate).
