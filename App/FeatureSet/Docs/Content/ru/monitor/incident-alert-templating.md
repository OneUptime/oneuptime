# Динамическая шаблонизация инцидентов и оповещений

Для динамического заполнения заголовков, описаний и заметок по устранению инцидентов и оповещений, создаваемых автоматически из критериев мониторов, можно использовать тот же синтаксис плейсхолдеров `{{variable}}`, что и в выражениях JavaScript.

## Поддерживаемые типы мониторов и переменные

Следующие типы мониторов поддерживают динамическую шаблонизацию с соответствующими переменными:

- **Мониторы сайтов и API**: данные ответов, заголовки, коды состояния, время выполнения
- **Мониторы входящих запросов**: данные запросов, заголовки, методы, время выполнения
- **Ping-мониторы**: статус подключения, время отклика, причины сбоев
- **Мониторы портов**: доступность портов, время отклика, статус тайм-аута
- **IP-мониторы**: доступность IP-адресов, время ping, информация о сбоях
- **Мониторы SSL-сертификатов**: сведения о сертификате, статус валидации, информация об истечении срока
- **Мониторы серверов/виртуальных машин**: системные метрики (CPU, память, диск), процессы, имя хоста
- **Синтетические мониторы**: результаты выполнения скрипта, скриншоты, сведения о браузере
- **Мониторы пользовательского кода JavaScript**: результаты выполнения, время, сообщения об ошибках
- **SNMP-мониторы**: статус устройства, время отклика, значения OID

> **Примечание**: Мониторы журналов, трассировок и метрик в настоящее время не поддерживают шаблонизацию инцидентов/оповещений, так как используют другие механизмы срабатывания.

## Поддерживаемые типы мониторов и переменные

| --- | --- | --- |

| `responseHeaders` | Объект заголовков ответа (ключи приведены к нижнему регистру). | `Dictionary<string>` |
| `responseStatusCode` | HTTP-код состояния ответа. | `number` |
| `responseTimeInMs` | Время ответа в миллисекундах. | `number` |
| `isOnline` | Считается ли монитор доступным. | `boolean` |

### Синтетические мониторы

Синтетические мониторы запускают один и тот же скрипт в нескольких браузерах (Chromium, Firefox, Webkit) и размерах экрана (мобильный, планшет, десктоп), создавая один результат для каждой конфигурации. Каждый запуск доступен через массив `syntheticResponses` — обращайтесь к конкретному запуску по индексу (`{{syntheticResponses[0].browserType}}`) или перебирайте их с помощью `{{#each syntheticResponses}}`.

| Переменная | Описание | Тип |
| --- | --- | --- |
| `failureCause` | Причина сбоя, если синтетическая проверка не удалась. | `string` |
| `syntheticResponses` | Массив с одной записью для каждой комбинации браузера и размера экрана. | `Array<Object>` |
| `syntheticResponses[].executionTimeInMs` | Время выполнения в миллисекундах для данного запуска. | `number` |
| `syntheticResponses[].result` | Результат данного запуска. | `string`, `number`, `boolean` или `JSON` |
| `syntheticResponses[].scriptError` | Ошибка, возникшая при данном запуске. | `string` |
| `syntheticResponses[].logMessages` | Сообщения журнала, созданные при данном запуске. | `Array<string>` |
| `syntheticResponses[].screenshots` | Скриншоты, сделанные при данном запуске. | `Object` |
| `syntheticResponses[].browserType` | Браузер, использованный для данного запуска. | `string` |
| `syntheticResponses[].screenSizeType` | Размер экрана, использованный для данного запуска. | `string` |

### Мониторы пользовательского кода JavaScript

| Переменная | Описание | Тип |
| --- | --- | --- |
| `executionTimeInMs` | Время выполнения пользовательского кода в миллисекундах. | `number` |
| `result` | Результат, возвращённый пользовательским кодом. | `string`, `number`, `boolean` или `JSON` |
| `scriptError` | Ошибка, возникшая при выполнении кода. | `string` |
| `logMessages` | Массив сообщений журнала, созданных при выполнении. | `Array<string>` |

### SNMP-мониторы

| Переменная | Описание | Тип |
| --- | --- | --- |
| `isOnline` | Доступно ли SNMP-устройство и отвечает ли оно. | `boolean` |
| `responseTimeInMs` | Время отклика SNMP-запроса в миллисекундах. | `number` |
| `failureCause` | Причина сбоя, если SNMP-запрос не удался. | `string` |
| `isTimeout` | Истёк ли тайм-аут SNMP-запроса. | `boolean` |
| `oidResponses` | Массив объектов ответа OID с полями oid, name, value и type. | `Array<Object>` |
| `oidResponses[].oid` | Запрошенный OID. | `string` |
| `oidResponses[].name` | Дружественное имя OID (если указано). | `string` |
| `oidResponses[].value` | Значение, возвращённое OID. | `string` или `number` |
| `oidResponses[].type` | Тип данных SNMP значения. | `string` |
| `{{OID_NAME}}` | Прямой доступ к значению OID по имени (например, `{{sysUpTime}}`). | `string` или `number` |


## Базовое использование

В форме инцидента/оповещения внутри экземпляра критерия монитора можно написать:

```
API вернул {{responseStatusCode}} за {{responseTimeInMs}}мс
```

Если код состояния ответа монитора равен `502`, а время — `842`, сохранённый заголовок принимает вид:

```
API вернул 502 за 842мс
```

Доступ к вложенному JSON работает так же, как в выражениях JavaScript:

```
ID проблемы: {{responseBody.error.id}}
Сообщение: {{responseBody.error.message}}
```

Поддерживается индексирование массивов:

```
Первый пользователь: {{responseBody.users[0].name}}
```

Если путь не существует, он разрешается в пустую строку по умолчанию.

## Расширенное использование

### Доступ к элементам массива
```
Использование первого диска: {{diskMetrics[0].usagePercent}}%
Последний процесс: {{processes[-1].name}}
```

### Доступ к вложенным объектам
```
Сообщение об ошибке: {{responseBody.error.details.message}}
Местоположение сервера: {{sslCertificate.locality}} {{sslCertificate.country}}
```

### Перебор массивов с помощью `{{#each}}`

Для перебора массивов используйте синтаксис блока `{{#each path}}...{{/each}}`. Это удобно, когда данные содержат список элементов, которые необходимо включить в описание инцидента или оповещения.

**Синтаксис:**
```
{{#each arrayPath}}
  ...тело с {{property}} из каждого элемента...
{{/each}}
```

Внутри тела цикла:
- `{{propertyName}}` разрешается относительно текущего элемента массива
- `{{nested.property}}` — доступ через точку к текущему элементу
- `{{@index}}` разрешается в 0-индексированный номер текущей итерации
- `{{this}}` разрешается в текущее значение элемента (полезно для массивов строк/чисел)
- Переменные, не найденные в текущем элементе, ищутся в родительской карте хранилища

**Пример — входящий запрос с массивом оповещений (например, вебхуки Grafana):**

Если тело входящего запроса выглядит так:
```json
{
  "status": "firing",
  "alerts": [
    { "status": "firing", "labels": { "label": "Coralpay" } },
    { "status": "firing", "labels": { "label": "capitecpay" } },
    { "status": "resolved", "labels": { "label": "capricorn" } }
  ]
}
```

Можно написать шаблон:
```
Метки оповещений:
{{#each requestBody.alerts}}
- {{labels.label}} ({{status}})
{{/each}}
```

Который даст:
```
Метки оповещений:
- Coralpay (firing)
- capitecpay (firing)
- capricorn (resolved)
```

**Пример — метрики дисков сервера:**
```
Использование дисков:
{{#each diskMetrics}}
- {{diskPath}}: {{usagePercent}}% использовано
{{/each}}
```

**Пример — использование `{{@index}}`:**
```
Процессы:
{{#each processes}}
{{@index}}. {{name}} (PID: {{pid}})
{{/each}}
```

**Пример — примитивный массив с `{{this}}`:**
```
Сообщения журнала:
{{#each logMessages}}
- {{this}}
{{/each}}
```

**Пример — вложенные циклы:**

Блоки `{{#each}}` можно вкладывать для многоуровневых массивов:
```
{{#each requestBody.groups}}
Группа: {{name}}
{{#each members}}
  - {{id}}: {{role}}
{{/each}}
{{/each}}
```

> **Примечание**: Если путь не разрешается в массив, весь блок `{{#each}}...{{/each}}` удаляется из вывода. Пустые массивы не создают вывода для блока.


## Примеры

### Заголовок инцидента монитора сайта/API
```
Высокая задержка: {{responseTimeInMs}}мс (> порога)
```

### Описание инцидента монитора сайта/API
```
### Ошибка API
Статус: **{{responseStatusCode}}**  
Задержка: **{{responseTimeInMs}}мс**  
Фрагмент тела: `{{responseBody.error.message}}`
```

### Заголовок оповещения о входящем запросе
```
Неверный входящий запрос: method={{requestMethod}} auth={{requestHeaders.authorization}}
```

### Заголовок оповещения SSL-сертификата
```
Истекает SSL-сертификат: {{commonName}} истекает {{expiresAt}}
```

### Описание оповещения монитора сервера
```
### Оповещение сервера: {{hostname}}
Использование CPU: **{{cpuUsagePercent}}%**  
Использование памяти: **{{memoryUsagePercent}}%**  
Использование первого диска: **{{diskMetrics[0].usagePercent}}%**  
Последняя проверка: {{requestReceivedAt}}
```

### Заголовок оповещения ping-монитора
```
Ping не удался для цели: {{failureCause}} ({{responseTimeInMs}}мс)
```

### Описание оповещения монитора порта
```
Проблема с подключением к порту
Статус целевого порта: {{isOnline}}
Время отклика: {{responseTimeInMs}}мс
Причина сбоя: {{failureCause}}
```

### Оповещение синтетического монитора

Доступ к конкретному запуску браузера/размера экрана по индексу:
```
Первый запуск: {{syntheticResponses[0].browserType}} / {{syntheticResponses[0].screenSizeType}}
Результат: {{syntheticResponses[0].result}} за {{syntheticResponses[0].executionTimeInMs}}мс
```

Перебор всех комбинаций браузера/размера экрана с помощью `{{#each}}`:
```
### Результаты синтетического монитора
{{#each syntheticResponses}}
- **{{browserType}} / {{screenSizeType}}**: {{result}} за {{executionTimeInMs}}мс
  - Ошибка скрипта: {{scriptError}}
  - Первый журнал: {{logMessages[0]}}
{{/each}}
```

### Оповещение монитора пользовательского кода
```
Выполнение пользовательского кода: {{executionTimeInMs}}мс
Вывод журнала: {{logMessages[0]}}
```

### Заголовок оповещения SNMP-монитора
```
SNMP-устройство недоступно: {{failureCause}} ({{responseTimeInMs}}мс)
```

### Описание оповещения SNMP-монитора
```
### Оповещение SNMP-устройства
Статус: **{{isOnline}}**
Время отклика: **{{responseTimeInMs}}мс**
Время работы системы: {{sysUpTime}}
Имя системы: {{sysName}}
Значение первого OID: {{oidResponses[0].value}}
```

### Входящий запрос с циклом по массиву (вебхук Grafana)

Заголовок:
```
[{{requestBody.status}}] {{requestBody.receiver}}
```

Описание:
```
### Оповещения от {{requestBody.receiver}}

{{#each requestBody.alerts}}
**Оповещение {{@index}}**: {{labels.alertname}}
- Метка: {{labels.label}}
- Статус: {{status}}
- Значения: {{valueString}}
- Источник: {{generatorURL}}
{{/each}}
```

### Монитор сервера с циклом по дискам

Описание:
```
### Оповещение сервера: {{hostname}}
Использование CPU: **{{cpuUsagePercent}}%**
Использование памяти: **{{memoryUsagePercent}}%**

**Использование дисков:**
{{#each diskMetrics}}
- {{diskPath}}: {{usagePercent}}% использовано ({{freePercent}}% свободно)
{{/each}}

**Запущенные процессы:**
{{#each processes}}
- [{{pid}}] {{name}}: {{command}}
{{/each}}
```

### SNMP-монитор с циклом по OID

Описание:
```
### Статус SNMP-устройства
Доступен: {{isOnline}}
Отклик: {{responseTimeInMs}}мс

**Значения OID:**
{{#each oidResponses}}
- {{name}} ({{oid}}): {{value}}
{{/each}}
```
