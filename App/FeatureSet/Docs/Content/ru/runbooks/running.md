# Выполнение runbook-а

Есть три способа создать исполнение runbook-а:

1. **Автоматически по правилу** — см. [Правила runbook-ов](/docs/runbooks/rules).
2. **Вручную со страницы runbook-а** — нажмите **Запустить сейчас** на обзоре runbook-а. Не привязано ни к инциденту, ни к оповещению, ни к плановой работе.
3. **Вручную из ленты сущности** — нажмите **Запустить runbook** на инциденте, оповещении или плановой работе. Исполнение привязано к этой сущности.

## Вид исполнения

Откройте любое исполнение, чтобы увидеть чек-лист. Каждый шаг показывает:

- **Статус** — Ожидает, Выполняется, Ждёт вас, Готово, Пропущено, Ошибка.
- **Заголовок и описание** — скопированы из runbook-а в момент исполнения.
- **Вывод** (сворачивается) — stdout, возвращаемые значения, HTTP-ответы.
- **Сообщение об ошибке**, если шаг упал.
- Для ручных шагов в `WaitingForUser`: кнопки **Отметить готовым** и **Пропустить**.

Пока исполнение не в финальном статусе, страница обновляется каждые 3 секунды, поэтому автоматические шаги вы видите почти в реальном времени.

## Чередование ручных и автоматических шагов

Классический поток:

1. **Скрипт-шаг**: захват состояния системы, запись в S3.
2. **Ручной шаг**: «Сообщите клиентам через баннер на странице статуса.» Ответственный отмечает.
3. **HTTP-шаг**: вызов DBA через PagerDuty.
4. **Ручной шаг**: «Подтвердите, что secondary DB стала primary.» Ответственный отмечает.
5. **Скрипт-шаг**: отправить «всё в порядке» в Slack.

Шаги 2 и 4 ставят исполнение на паузу до отметки. Шаги 1, 3, 5 идут автоматически. Весь прогон — это одно исполнение, одна шкала времени, один источник правды.

## Отмена прогона

Нажмите **Отменить исполнение** на странице. Текущий шаг (если есть) завершается; последующие шаги не стартуют. Статус становится `Cancelled`.

## Хранение вывода

Вывод на шаг ограничен **50 КБ**, чтобы убежавшие скрипты не раздували базу. Если нужны большие артефакты, пишите их из скрипта в S3 или в логгер и сохраняйте URL в возвращаемом значении.

## Повторный запуск runbook-а

Исполнение — это разовая, неизменяемая запись. Чтобы повторить, ещё раз нажмите **Запустить сейчас** — это создаст новое исполнение со свежим снимком текущих шагов runbook-а. Исходное исполнение остаётся нетронутым для аудиторского следа.

## Поиск прошлых исполнений

У каждого runbook-а есть вкладка **Исполнения** со списком всех прогонов, с фильтрами по статусу, диапазону дат и сущности-источнику. На инциденте, оповещении или плановой работе вкладка **Runbooks** показывает прогоны, привязанные к этой сущности.
