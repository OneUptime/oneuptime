# Виджеты

Виджет — это одна плитка на дашборде. Эта страница перечисляет все виджеты, которые можно добавить, что они показывают и в каком случае к ним обращаться.

О том, как перетаскивать виджеты по холсту, читайте в разделе [Создание дашборда](/docs/dashboards/authoring).

## Графики и числа

### Chart

Линейный, столбчатый или площадной график одной или нескольких метрических серий за временной диапазон дашборда.

**Настройки**:

- Один или несколько запросов к метрикам.
- Опциональная формула, объединяющая два запроса (например, `errors / total * 100`, чтобы получить процент ошибок).
- Опция «показывать как rate» для накопительных счётчиков, которые растут без сброса.
- Параметры отображения: с накоплением или наложением, единица оси Y, положение легенды, тип графика.

Используйте, когда: важны тренды. Задержки во времени, число ошибок, глубина очереди — всё, где форма линии рассказывает историю.

### Value

Одно большое число с опциональными цветными порогами.

**Настройки**:

- Запрос к метрике, возвращающий одно число (последнее значение, среднее или максимум за временной диапазон).
- Опциональный порог **warning** (жёлтый, когда выше).
- Опциональный порог **critical** (красный, когда выше).
- Формат числа и единица измерения.

Используйте, когда: на вопрос отвечает одно число. Текущая частота ошибок, P95 задержек прямо сейчас, число открытых инцидентов.

### Gauge

Круговой индикатор с минимумом, максимумом, полосой предупреждения и критической полосой.

**Настройки**: запрос к метрике и четыре границы.

Используйте, когда: значение укладывается в известный диапазон. Процент CPU (0–100%), загрузка диска, ёмкость очереди.

### Table

Таблица результатов метрики, по одной строке на группу.

**Настройки**: запрос к метрике (обычно сгруппированный по метке вроде host или service), отображаемые столбцы и лимит строк.

Используйте, когда: нужна разбивка, а не тренд. Топ-10 самых шумных хостов, число ошибок по сервисам, запросы по эндпоинтам.

## Text

Статический блок Markdown.

**Настройки**: тело в формате Markdown. Заголовки, списки, ссылки, выделение и блоки кода — всё рендерится.

Используйте, когда: нужен заголовок секции, абзац с контекстом, список ссылок на runbook-и или временный баннер во время инцидента.

## Логи и трейсы

### Log Stream

Живой поток строк лога, подходящих под фильтр.

**Настройки**: фильтры лога (service, severity, атрибуты) и отображаемые столбцы.

Используйте, когда: хочется видеть, что приложение говорит прямо сейчас, не покидая дашборд.

### Trace List

Список свежих трейсов, подходящих под фильтр, с длительностью, статусом и сервисом.

**Настройки**: фильтры трейсов (service, status, атрибуты).

Используйте, когда: нужен список свежей активности, а не график. Типичный приём — график задержек сверху и список медленных трейсов под ним.

## Живые списки

### Incident List

Живой список инцидентов, подходящих под фильтр.

**Настройки**: фильтры по состоянию, уровню критичности, меткам, монитору или команде.

Используйте, когда: дашборд отвечает на вопрос «что сломано прямо сейчас?»

### Alert List

Живой список оповещений, подходящих под фильтр.

**Настройки**: фильтры по состоянию, уровню критичности, меткам.

Используйте, когда: командный дашборд отслеживает оповещения по своим сервисам.

### Monitor List

Живой список мониторов и их текущего состояния.

**Настройки**: фильтры по типу монитора, меткам или текущему состоянию.

Используйте, когда: нужен обзор флота — «все ли сайты живы?»

## Списки ресурсов Kubernetes

Для проектов с установленным [Kubernetes Agent](/docs/monitor/kubernetes-agent). У каждого виджета есть опциональные фильтры по кластеру, пространству имён и меткам.

- **Kubernetes Pod List** — поды с их фазой, перезапусками и нодой.
- **Kubernetes Node List** — ноды с их условиями и ёмкостью.
- **Kubernetes Namespace List** — пространства имён и количество рабочих нагрузок.
- **Kubernetes Deployment List** — deployment-ы с желаемым и готовым числом реплик.
- **Kubernetes StatefulSet List** — stateful set-ы с готовыми репликами.
- **Kubernetes DaemonSet List** — daemon set-ы с желаемыми и готовыми.
- **Kubernetes Job List** — задания и их статус завершения.
- **Kubernetes CronJob List** — cron-задания с расписанием и последним запуском.

Используйте, когда: хочется одного дашборда, объединяющего состояние Kubernetes с телеметрией от этих рабочих нагрузок.

## Списки ресурсов Docker

Для проектов с настроенным мониторингом Docker.

- **Docker Host List** — хосты, на которых работает Docker, с количеством контейнеров.
- **Docker Container List** — контейнеры со статусом, образом, хостом, временем работы.
- **Docker Image List** — образы и их размеры.
- **Docker Network List** — сети Docker и подключённые контейнеры.
- **Docker Volume List** — тома Docker и их использование.

## Инфраструктура

### Host List

Хосты, отслеживаемые серверным монитором OneUptime, со статусом, CPU, памятью и временем работы.

**Настройки**: фильтры по меткам или текущему состоянию.

## Какой виджет выбрать?

Несколько быстрых правил:

- **Тренд во времени?** Chart.
- **Одно число, которое важно прямо сейчас?** Value (или Gauge, если есть чёткие min/max).
- **Разбивка по множеству объектов?** Table.
- **Что происходит в системе прямо сейчас?** Log Stream, Trace List, Incident List.
- **Состояние конкретной группы ресурсов?** Подходящий виджет-список.
- **Заголовок, абзац или ссылка?** Text.

Большинство дашбордов сочетают несколько типов — график сверху, рядом одно-два значения, текстовый разделитель, ниже один-два списка.
