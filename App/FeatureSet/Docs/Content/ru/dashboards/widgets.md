# Виджеты дашборда

Виджет — это одна плитка на дашборде. У каждого виджета есть тип (chart, value, list, …), позиция, размер и конфигурация. Эта страница — каталог: что показывает каждый виджет, что принимает на вход, когда к нему обращаться.

Механика холста описана в [Создание дашборда](/docs/dashboards/authoring).

## Виджеты временных рядов

### Chart

Линейный / столбчатый / area-график одной или нескольких серий метрик по временному диапазону дашборда.

**Настройка**:

- Один или несколько запросов метрик (`metricQueryConfig` для одной серии, `metricQueryConfigs` для нескольких).
- Опциональная **formula**, комбинирующая несколько запросов (например, `errors / total * 100`).
- Опциональный **transformAsRate** для кумулятивных счётчиков OpenTelemetry (например, `system.disk.io`) — виджет вычисляет `(value - previousValue) / Δt` для каждого бакета.
- Отображение: сложенные vs. наложенные серии, единица оси Y, легенда вкл/выкл, тип графика.

К нему обращаются, когда: важны тренды. Задержка запросов, число ошибок во времени, глубина очереди — всё, где форма кривой что-то рассказывает.

### Value

Одно крупное число с опциональными порогами и опциональным sparkline.

**Настройка**:

- Запрос метрики (одно значение — обычно `last`, `avg` или `max` по временному диапазону).
- Опциональный **warning threshold** (жёлтый выше).
- Опциональный **critical threshold** (красный выше).
- Отображение: формат числа, суффикс единицы.

К нему обращаются, когда: одно число отвечает на вопрос. Текущая частота ошибок, P95-задержка прямо сейчас, число открытых инцидентов.

### Gauge

Круговой индикатор с min, max, полосой warning и полосой critical.

**Настройка**: запрос метрики и четыре границы (min, max, warning, critical).

К нему обращаются, когда: значение лежит в известном диапазоне. Утилизация CPU (0–100%), заполненность диска, наполненность очереди.

### Table

Табличный показ результатов запроса метрики, по одной строке на группу.

**Настройка**: запрос метрики (обычно с группировкой по метке вроде `host.name` или `service.name`), колонки для показа и ограничение по строкам.

К нему обращаются, когда: нужна разбивка, а не тренд. Топ-10 самых шумных хостов, число ошибок на сервис, частота запросов на endpoint.

## Виджет-аннотация

### Text

Статический блок Markdown.

**Настройка**: тело Markdown. Заголовки, списки, ссылки, выделение, inline-код, ограждённые блоки кода — всё рисуется.

К нему обращаются, когда: нужен заголовок раздела, абзац контекста («этот дашборд покрывает сервис checkout»), список ссылок на runbook-и или связанные дашборды или временный баннер во время инцидента.

## Логи и трейсы

### LogStream

Live-хвост строк логов, соответствующих фильтру.

**Настройка**: фильтры логов (сервис, severity, совпадения по атрибутам), колонки для показа.

К нему обращаются, когда: вы хотите видеть, что приложение говорит *прямо сейчас* на дашборде, не уходя на страницу с логами.

### TraceList

Список недавних трейсов, соответствующих фильтру, с длительностью, статусом и именем сервиса.

**Настройка**: фильтры трейсов (сервис, статус, совпадения по атрибутам).

К нему обращаются, когда: нужен пагинированный показ недавней активности, а не график. Типичная пара: Chart с задержкой сверху, TraceList медленных трейсов снизу.

## Операционные списки

### IncidentList

Live-список инцидентов, соответствующих фильтру.

**Настройка**: фильтры по состоянию, severity, меткам, монитору или назначенной команде.

К нему обращаются, когда: дашборд должен отвечать на «что сейчас сломано?»

### AlertList

Live-список оповещений, соответствующих фильтру.

**Настройка**: фильтры по состоянию, severity, меткам.

К нему обращаются, когда: нужны дашборды для процессов на основе оповещений (например, дашборды команды разработки, наблюдающие за оповещениями своего сервиса).

### MonitorList

Live-список мониторов, соответствующих фильтру, с текущим статусом каждого монитора.

**Настройка**: фильтры по типу монитора, меткам или текущему состоянию.

К нему обращаются, когда: нужен взгляд «все ли сайты в строю?» на уровне всего парка или per-team-список наблюдаемых endpoint-ов.

## Списки ресурсов Kubernetes

Для проектов с установленным [Kubernetes Agent](/docs/monitor/kubernetes-agent) доступны следующие live-виджеты ресурсов. Каждый принимает опциональные фильтры по `cluster`, `namespace` и меткам.

- **KubernetesPodList** — поды с фазой, числом перезапусков и назначением на ноду.
- **KubernetesNodeList** — ноды с условиями, ёмкостью и аллокациями.
- **KubernetesNamespaceList** — namespace-ы и счётчики их workload-ов.
- **KubernetesDeploymentList** — deployment-ы с желаемым vs. готовым числом реплик.
- **KubernetesStatefulSetList** — stateful-set-ы с готовыми репликами.
- **KubernetesDaemonSetList** — daemon-set-ы с желаемым vs. готовым.
- **KubernetesJobList** — jobs со статусом завершения.
- **KubernetesCronJobList** — cron jobs с расписанием и последним запуском.

К ним обращаются, когда: нужен один дашборд, смешивающий состояние ресурсов Kubernetes с телеметрией этих workload-ов.

## Списки ресурсов Docker

Для проектов с установленным монитором Docker:

- **DockerHostList** — хосты, на которых работает Docker, с количеством контейнеров.
- **DockerContainerList** — контейнеры с состоянием, образом, хостом, uptime.
- **DockerImageList** — образы и их размеры.
- **DockerNetworkList** — сети Docker и счётчики подключённых контейнеров.
- **DockerVolumeList** — тома Docker и их использование.

## Инфраструктура

### HostList

Хосты, наблюдаемые серверным монитором OneUptime, — с текущим статусом, CPU, памятью и uptime.

**Настройка**: фильтры по меткам или текущему состоянию здоровья.

## Выбор правильного виджета

Несколько быстрых эвристик:

- **Тренд во времени?** Chart.
- **Одно число, важное прямо сейчас?** Value (или Gauge, если у него есть естественный диапазон).
- **Разбивка по многим элементам?** Table.
- **Что сейчас происходит в системе?** LogStream, TraceList, IncidentList.
- **Состояние конкретного парка ресурсов?** Подходящий list-виджет.
- **Заголовок, абзац или ссылка?** Text.

Большинство дашбордов используют смесь — Chart сверху, одно-два Value рядом, текстовый разделитель, ниже один-два списка.
