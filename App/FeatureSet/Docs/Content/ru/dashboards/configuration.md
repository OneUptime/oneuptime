# Конфигурация и разрешения дашборда

На этой странице собраны настройки и ручки контроля доступа, о которых стоит знать, когда у вас есть дашборд, который вы реально хотите оставить.

## Владение

**Владельцы** дашборда — это пользователи и команды, которым предоставлены явные разрешения на него (отдельно от роли в масштабе проекта).

В **Dashboard → Owners**:

- Добавьте **владельца-пользователя**, чтобы дать конкретному человеку дополнительный доступ к этому дашборду.
- Добавьте **владельца-команду**, чтобы дать то же самое каждому участнику команды.

Используйте владение, когда роль чтения уровня проекта слишком широкая — например, дашборд с чувствительными данными по клиентам, который должен видеть только команда customer-success.

## Метки

Метки — это many-to-many теги для организации дашбордов. Применяйте их в **Dashboard → Overview**.

Распространённые паттерны меток:

- **По команде**: `team:platform`, `team:checkout`, `team:growth`.
- **По окружению**: `env:prod`, `env:staging`.
- **По назначению**: `purpose:oncall`, `purpose:exec`, `purpose:investigation`.

Список **Dashboards** позволяет фильтровать по метке, что является самым быстрым способом найти дашборд в проекте, накопившем десятки.

## Права

Дашборды — first-class-ресурсы в ролевом контроле доступа OneUptime. Релевантные права:

| Право | Разрешает |
| --- | --- |
| `CreateDashboard` | Создавать новые дашборды в проекте. |
| `ReadDashboard` | Смотреть дашборды (в приватном режиме). |
| `EditDashboard` | Изменять виджеты, переменные, настройки дашборда. |
| `DeleteDashboard` | Удалять дашборд. |

Для вспомогательных сущностей есть соответствующие права: у владельцев дашборда (user / team) и собственных доменов свои пары create / read / edit / delete, так что можно дать «управлять владельцами», не давая «редактировать сам дашборд».

Назначайте их на проектные роли в **Project Settings → Teams & Roles**.

## Контроль доступа в публичном режиме

Доступ в публичном режиме (см. [Совместный доступ и публичные дашборды](/docs/dashboards/sharing)) управляется тремя слоями в таком порядке:

1. Переключатель **Public Dashboard** — если выключен, публичный URL возвращает 404.
2. **Master Password** — если задан, посетители должны ввести его до рендеринга дашборда.
3. **IP Whitelist** (план Scale) — если задан, запросы с неперечисленных IP получают 403.

У дашборда может быть любая комбинация. Самая защитная конфигурация — «Public on, password set, IP allowlist active» — полезна для партнёрских порталов, где нужны все три.

## Retention

Сами дашборды не истекают. Данные, которые они отображают, следуют retention телеметрии проекта — метрики, логи и трейсы доступны для запросов, пока их хранит ваш план. Виджет, направленный на «последние 90 дней» на плане с retention 30 дней, нарисует то, что ещё лежит в хранилище.

## Клонирование дашборда

Чтобы продублировать существующий дашборд, откройте его и используйте действие **Duplicate** из списка дашбордов. Копия включает каждый виджет, переменную и настройку, кроме конфигурации публичного режима (которая всегда стартует в off — вы решаете, включать ли её на копии).

Это правильный паттерн, когда нужно форкнуть шаблон («наш oncall-дашборд») в сервис-специфичную версию.

## Удаление дашборда

В **Dashboard → Delete**. Это необратимо — конфигурация холста и любые привязки собственного домена удаляются. Телеметрические данные не затрагиваются (они живут в хранилищах метрик / логов / трейсов, а не на дашборде).

Если дашборд опубликован публично с собственным доменом, публичный URL перестаёт резолвиться в момент удаления. Сначала снимите домен, если вам нужно его переподключить.

## Миграция и резервное копирование

Для self-hosted-инсталляций: полная конфигурация дашборда (виджеты, переменные, настройки) живёт в таблице `Dashboard` в Postgres. Достаточно регулярного бэкапа БД — отдельного формата экспорта дашбордов нет.

Для OneUptime Cloud: регулярные бэкапы выполняются за вас. Если нужна локальная копия конфигурации дашборда, используйте [OneUptime API](/docs/api-reference/api-reference) для чтения записи `Dashboard`.
