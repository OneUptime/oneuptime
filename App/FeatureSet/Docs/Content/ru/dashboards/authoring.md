# Создание дашборда

Создайте дашборд в **Dashboards → Create Dashboard**, дайте ему имя и откройте. Холст откроется в режиме **Edit**, готовый к виджетам.

## Холст

Дашборд — это сетка. По умолчанию холст имеет **12 единиц дашборда в ширину** и **60 единиц в высоту** — высоту можно увеличить, добавляя ряды ниже. Каждая единица — это квадрат, масштабирующийся под viewport: на десктопе он шире, чем на телефоне, но каждый виджет сохраняет свои пропорции.

Виджеты занимают прямоугольник единиц. Вы решаете и позицию (верхний левый угол, отсчитываемый в единицах от верхнего левого угла холста), и размер (ширина и высота в единицах). Минимальные размеры гарантируют, что крошечный виджет остаётся читаемым.

## Edit vs. View

Переключатель в заголовке страницы переключает режимы:

- **Edit** — открыта палитра виджетов, виджеты можно перетаскивать и изменять им размер, у каждого виджета есть шестерёнка настроек. Используйте при сборке.
- **View** — дашборд рисуется только для чтения, ровно так, как его видит человек с правом только-чтения (или публичный посетитель). Используйте, чтобы проверить результат перед публикацией.

В обоих режимах показан один и тот же дашборд — отдельного шага «publish» нет. Сохранение правки немедленно действует для каждого зрителя.

## Добавление виджета

1. Откройте палитру виджетов (кнопка **+** в режиме Edit).
2. Выберите тип виджета. См. [Виджеты](/docs/dashboards/widgets) для каталога.
3. Виджет приземляется на холсте в следующей свободной позиции с размером по умолчанию.
4. Кликните по шестерёнке виджета, чтобы открыть панель настроек.
5. Настройте источник данных (запрос метрики, фильтр списка, тело текста и т. д.) и любые опции отображения (пороги, единицы, оси, колонки).
6. Перетащите виджет, чтобы расположить. Тяните за угол, чтобы изменить размер.

Повторяйте. Сетка привязывает виджеты к границам целых единиц.

## Конфигурация источников данных

Большинство виджетов читают из одного из трёх мест:

- **Метрики** — запрос метрики на базе ClickHouse. Виджет собирает `metricQueryConfig` (одна серия) или `metricQueryConfigs` (несколько серий, наложенных или сложенных). Опциональный `transformAsRate` преобразует кумулятивный счётчик OpenTelemetry в скорость изменения. Опциональный `formula` позволяет комбинировать два запроса (например, число ошибок / общее число).
- **Live-списки ресурсов** — инциденты, оповещения, мониторы, ресурсы Kubernetes, ресурсы Docker, хосты. Каждый виджет-список принимает фильтр (например, метки, статус, namespace) и отображает соответствующие строки в реальном времени.
- **Статический контент** — виджет **Text** принимает тело Markdown. Используйте его для заголовков, разделителей, ссылок на runbook-и и аннотаций «что это за дашборд?».

Для виджетов-метрик конфигурация зеркалит конструктор запросов, который вы видите в других местах OneUptime, — выберите метрику, выберите агрегацию, добавьте фильтры `WHERE`, выберите группировку по времени. Запрос выполняется по телеметрическим данным проекта.

## Пороги и форматирование

Виджеты, отображающие одно число (**Value**, **Gauge**), принимают опциональные пороги:

- **Warning threshold** — рисовать значение жёлтым, когда оно пересекает порог.
- **Critical threshold** — рисовать значение красным, когда оно пересекает порог.

В графиках можно задать единицу оси Y, позицию легенды и нужно ли складывать серии. В таблицах можно выбрать, какие колонки показывать и ограничение по строкам.

## Временной диапазон и обновление

В заголовке дашборда есть два глобальных контрола, влияющих на каждый виджет-метрику:

- **Time range** — выберите пресет (последний час, 24 часа, 7 дней, 30 дней) или произвольный диапазон. Каждый виджет-метрика делает запрос по этому окну.
- **Refresh interval** — Off, 5с, 10с, 30с, 1м, 5м, 15м. Перезапускает запрос каждого виджета с выбранной частотой. Виджеты-списки, нативно поддерживающие websocket-ы, обновляются по push независимо от выбранного интервала.

Для виджетов, игнорирующих глобальный временной диапазон (например, текстовый блок), контрол ничего не делает.

## Сохранение

Холст автосохраняется по мере редактирования. Маленький индикатор в заголовке подсказывает, когда последнее изменение сохранено. Шага «publish» нет — каждое сохранение сразу видно в реальном времени. Если делаете рискованное изменение, сначала продублируйте дашборд.

## Паттерны, которые хорошо работают

- **Одна тема на дашборд.** Сопротивляйтесь искушению поставить «всё, что мы мониторим» на одну страницу. Три дашборда с метками `oncall-checkout`, `oncall-payments`, `oncall-search` стареют лучше, чем один мега-дашборд.
- **Заякорьте верх страницы самым важным виджетом.** Люди читают сверху — пусть первое, что они видят, — это ответ на «здорова ли эта система?».
- **Используйте виджеты Text для маркировки разделов.** Короткий заголовок каждые несколько рядов («Latency» / «Errors» / «Capacity») делает дашборд сканируемым с другого конца комнаты.
- **Используйте переменные вместо дублирования.** Если вы дважды строите один и тот же дашборд для двух сервисов, вам нужна переменная `service`. См. [Переменные и фильтры](/docs/dashboards/variables).
