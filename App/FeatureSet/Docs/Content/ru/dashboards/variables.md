# Переменные и фильтры

Переменная превращает один дашборд в шаблон. Добавьте переменную `service` в свой дашборд — и те же графики перерисуются для `checkout`, `payments` или `search`: пользователи выбирают значение в выпадающем списке наверху, а вы не строите три почти одинаковых дашборда.

## Типы переменных

Добавьте переменные в разделе **Dashboard → Settings → Variables**. У каждой переменной есть имя (используется как `{{name}}` в виджетах), опциональная метка и тип.

### Custom List

Статический выпадающий список. Варианты вы вводите вручную.

Используйте, когда: вариантов немного и они фиксированы. `environment` со значениями `prod, staging, dev`. `region` со значениями `us-east-1, eu-west-1, ap-south-1`.

### Query

Варианты приходят из запроса к вашим данным.

Используйте, когда: набор вариантов меняется со временем, и вы хотите, чтобы выпадающий список поспевал. «Каждый ID клиента, увиденный за прошедшие 24 часа». Запрос выполняется по данным вашего проекта, а его результаты становятся выпадающим списком.

### Text Input

Свободное текстовое поле. Используется то, что введёт зритель.

Используйте, когда: дашборд должен работать как инструмент поиска. Фильтрация по IP-адресу, ID запроса или любому другому свободному значению.

### Telemetry Attribute

Варианты — это различные значения какого-то атрибута в вашей телеметрии за временной диапазон дашборда.

Настройте **ключ атрибута** (например, `service.name`, `host.name`, `k8s.cluster.name`). Выпадающий список заполняется всеми уникальными значениями, встреченными в ваших логах, метриках и трейсах.

Используйте, когда: варианты соответствуют тегам, которые вы уже отправляете с телеметрией. Это самый распространённый тип, потому что он обновляется автоматически — когда вы выкатите новый сервис с тегом `service.name = inventory`, это имя появится в списке без правок в дашборде.

## Множественный выбор

Каждая переменная может разрешать множественный выбор. Когда он включён, зритель может выбрать одно или несколько значений; дашборд фильтрует по любому из них.

Используйте множественный выбор, когда: хочется сравнить «checkout и payments вместе», не покидая дашборд. Избегайте его, когда математика не работает между выбранными значениями (например, усреднение средних).

## Значения по умолчанию

У каждой переменной может быть значение по умолчанию. Дашборд отрисовывается с этим значением, пока зритель его не изменит. Для публичных дашбордов значение по умолчанию — это то, что посетители увидят первым.

## Как использовать переменную в виджете

Везде, где виджет принимает фильтр — в `WHERE` запроса к метрике, в фильтре списка, в условии для атрибута в Log Stream — можно использовать `{{variable_name}}`.

Например, график, отфильтрованный по сервису:

```
service.name = '{{service}}'
```

Когда выпадающий список выставлен в `checkout`, график фильтруется по сервису checkout. Когда зритель переключается на `payments`, график перерисовывается для payments.

Для переменных типа **Telemetry Attribute** OneUptime знает, на какой атрибут отображается переменная, и применяет фильтр ко всем виджетам, использующим тот же атрибут — править каждый виджет вручную не нужно.

## Временной диапазон

В заголовке дашборда есть глобальный временной диапазон. Каждый виджет метрик запрашивает данные за это окно. Варианты:

- **Пресеты** — прошедший час, 24 часа, 7 дней, 30 дней, 90 дней (зависит от срока хранения ваших данных).
- **Произвольный** — выберите начало и конец.

Временной диапазон является частью URL дашборда — поделившись URL, вы делитесь и окном. Полезно во время инцидента: зафиксируйте диапазон на «10:00–10:30 UTC сегодня» и вставьте ссылку в канал инцидента.

## Интервал обновления

Рядом с временным диапазоном выберите, как часто виджеты обновляются:

- **Off** — виджеты выполняют запрос один раз при загрузке страницы.
- **5s / 10s / 30s / 1m / 5m / 15m** — автообновление.

Автообновление хорошо для экрана на стене или живого вида инцидента. Отключайте его на время расследования, чтобы вид не дёргался у вас перед глазами.

## Собираем всё вместе

Дашборд-шаблон по сервисам обычно содержит:

1. Переменную `service` типа **Telemetry Attribute** для `service.name`. По умолчанию — самый отслеживаемый вами сервис. Множественный выбор выключен (чтобы графики всегда показывали один сервис).
2. Переменную `environment` типа **Custom List**. По умолчанию — `prod`.
3. Переменную `cluster` типа **Telemetry Attribute** для `k8s.cluster.name`. Множественный выбор включён (чтобы можно было сравнивать кластеры).
4. Виджеты, которые ссылаются на эти переменные в своих фильтрах.

В результате: один дашборд, все сервисы покрыты, три выпадающих списка наверху.
