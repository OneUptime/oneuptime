# Обзор дашбордов

Дашборды превращают данные, которые OneUptime уже собирает — метрики, логи, трейсы, инциденты, мониторы, ресурсы Kubernetes, хосты, — в одну страницу, на которую можно взглянуть и понять, что происходит.

Поставьте график задержек запросов рядом со списком открытых инцидентов, рядом с индикатором загрузки CPU, рядом с абзацем пояснений. Сохраните. Поделитесь ссылкой.

## Для чего хороши дашборды

- **Страница «всё ли в порядке?»** — для дежурного, командного стендапа или ТВ-панели на стене.
- **Поиск связей** — всплеск CPU в то же время, что и рост задержек и открытый инцидент, гораздо проще заметить на одной странице, чем перепрыгивая между тремя вкладками.
- **Расследование** — когда вы отлаживаете проблему, дашборд, собранный на лету, бьёт по эффективности десять отдельных запросов подряд.
- **Внешний обмен** — страница с метриками для клиентов, дашборд для партнёров, публичный дашборд для open-source проекта.

## Что можно разместить на дашборде

- **Графики** трендов во времени — задержки, ошибки, пропускная способность.
- **Плитки и индикаторы** одного значения — текущий процент ошибок, CPU, число открытых инцидентов.
- **Таблицы** для разбивок — топ-10 самых шумных хостов, число ошибок по сервисам.
- **Текстовые блоки** для заголовков, контекста и ссылок на runbook-и.
- **Живые списки** инцидентов, оповещений, мониторов, логов, трейсов, ресурсов Kubernetes, ресурсов Docker и хостов.

См. [Виджеты](/docs/dashboards/widgets) — полный список и описание каждого.

## Ключевые понятия

| Термин                 | Смысл                                                                                                         |
| ---------------------- | ------------------------------------------------------------------------------------------------------------- |
| **Дашборд**            | Вся страница целиком — имя, сетка виджетов, элементы управления временным диапазоном и список переменных.     |
| **Виджет**             | Одна плитка на странице — график, число, список, абзац.                                                       |
| **Переменная**         | Выпадающий список наверху, который фильтрует все виджеты сразу (кластер, сервис, клиент, окружение).          |
| **Временной диапазон** | Окно времени, которое используют все графики и числа. Задаётся один раз вверху страницы.                      |
| **Обновление**         | Как часто виджеты повторно запрашивают данные. Отключено, каждые несколько секунд или каждые несколько минут. |
| **Режим**              | **Edit** (перетаскиваете виджеты) или **View** (только чтение, так дашборд видят посетители).                 |

## Где найти дашборды

Откройте **Dashboards** в левой панели навигации.

| Страница                 | Что вы там делаете                                                      |
| ------------------------ | ----------------------------------------------------------------------- |
| **Dashboards**           | Список ваших дашбордов. Создайте новый, ищите или фильтруйте по меткам. |
| **Dashboard → View**     | Холст. Переключайтесь между **Edit** и **View** в заголовке.            |
| **Dashboard → Overview** | Описание, владельцы и метки.                                            |
| **Dashboard → Settings** | Публичный доступ, пароль, белый список IP, кастомный домен, брендинг.   |
| **Dashboard → Owners**   | Пользователи и команды с явным доступом.                                |
| **Dashboard → Delete**   | Удаление дашборда.                                                      |

## Сборка дашборда

1. **Создайте** — выберите имя. Холст откроется пустым.
2. **Добавьте виджеты** — выберите тип виджета, настройте его данные, перетащите в нужное место.
3. **(Опционально) добавьте переменные** — например, выпадающий список `service`, чтобы один и тот же дашборд работал для каждого сервиса.
4. **Задайте временной диапазон** — значения по умолчанию подойдут; настроите позже.
5. **(Опционально) поделитесь публично** — переключите тумблер в Settings, добавьте пароль или белый список IP при необходимости.
6. **(Опционально) кастомный домен** — разместите дашборд на `status.your-domain.com`.

## Быстрый пример

Цель: дашборд для дежурного по сервису checkout — с задержками, частотой ошибок, открытыми инцидентами и живым потоком логов.

1. Создайте дашборд с именем «Checkout on-call».
2. Добавьте переменную `service`. Значение по умолчанию — `checkout`.
3. Добавьте виджет **Chart** с задержкой P95, отфильтрованной по переменной `service`.
4. Рядом добавьте виджет **Value** для частоты ошибок с предупреждением на 1% и критическим порогом на 5%.
5. Ниже добавьте виджет **Incident List** для инцидентов с меткой `checkout`.
6. Ещё ниже — виджет **Log Stream**, показывающий логи того же сервиса.
7. Сохраните. Переключите выпадающий список на `payments` — тот же дашборд теперь показывает сервис payments.

## Как дашборды вписываются в остальной OneUptime

- **Мониторы и телеметрия** — источники данных. Каждая метрика, лог и трейс, которые вы собираете, доступны для запроса в виджетах.
- **Инциденты и оповещения** появляются в виджетах **Incident List** и **Alert List**. Для них дашборды работают только на чтение — создавать и изменять их нужно в других местах.
- **Страницы статуса** — это публичная коммуникация («работает ли система?»). Дашборды — это подробное наблюдение за поведением системы. Эти инструменты дополняют друг друга, а не заменяют.
- **Рабочие процессы** — это то, как OneUptime выполняет действия. Дашборды — это то, как вы видите, что происходит.
