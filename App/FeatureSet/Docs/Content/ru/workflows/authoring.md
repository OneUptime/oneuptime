# Создание рабочего процесса

Создайте рабочий процесс в разделе **Workflows → Create Workflow**, дайте ему имя и опциональное описание, затем откройте вкладку **Builder**, чтобы начать бросать узлы на холст.

## Холст

Builder — это граф, который можно масштабировать и панорамировать. Вы добавляете узлы из палитры компонентов, соединяете их рёбрами и конфигурируете аргументы каждого узла в боковой панели. Индикатор сохранения в заголовке подсказывает, было ли последнее изменение сохранено.

Рабочий процесс всегда начинается ровно с одного узла-**триггера**. У триггеров нет входного порта — это место, где начинается выполнение. Всё, что идёт дальше, — это **компоненты**.

## Анатомия узла

У каждого узла есть:

| Поле | Назначение |
| --- | --- |
| **Title** | Подпись, видимая на холсте. По умолчанию — название компонента; переопределите, чтобы сложные рабочие процессы было легче читать. |
| **Arguments** | Конфигурация, нужная компоненту для работы — URL, канал Slack, JavaScript-сниппет и т. д. Обязательные аргументы помечены звёздочкой. |
| **Input ports** | Разъёмы слева на узле, куда приходят входящие рёбра. У компонентов один входной порт под названием `in`; у триггеров их нет. |
| **Output ports** | Разъёмы справа, откуда выходят рёбра. Компоненты определяют такие порты, как `success`, `error`, `yes`, `no`. |
| **Return values** | Данные, которые узел производит — payload-ы его выходных портов. На них ссылаются последующие узлы как `{{NodeId.fieldName}}`. |

## Соединение узлов

Перетащите от выходного порта к входному порту узла ниже по потоку, чтобы создать ребро. Ребро из `success` запускает эту ветку только тогда, когда вышестоящий узел отработал успешно; ребро из `error` запускается только при сбое. Если вы не подключите порт, эта ветка просто заканчивается.

Можно делать «веер»: один выходной порт может питать несколько узлов ниже по потоку, и все они выполняются параллельно от этой точки.

## Конфигурация аргументов

Кликните по узлу, чтобы открыть его боковую панель. Каждый аргумент имеет типизированный редактор:

- **Text / URL / Email / Number / Password** — однострочный ввод.
- **JSON** — JSON-редактор с подсветкой синтаксиса и индикатором валидации.
- **JavaScript** — редактор кода для сниппетов, используемых компонентом **Custom Code**.
- **Markdown / HTML** — тела с форматированным текстом для компонентов email и сообщений.
- **CronTab** — выражение расписания (используется триггером Schedule).
- **Boolean** — переключатель.
- **Select / Query** — выпадающие списки для полей с фиксированным набором значений или с запросом в стиле модели.

Любое текстовое поле принимает интерполяцию переменных — см. [Переменные](/docs/workflows/variables) для правил.

## Минимальный первый рабочий процесс

Самый быстрый способ почувствовать холст:

1. Бросьте триггер **Manual**.
2. Бросьте компонент **Log** (в разделе **Utils**). Соедините выходной порт триггера со входом компонента Log.
3. В аргументе компонента Log введите `Hello from {{Manual.JSON.name}}`.
4. Сохраните и включите рабочий процесс.
5. Нажмите **Run Manually**, вставьте `{ "name": "Ada" }` как ввод и отправьте.
6. Откройте вкладку **Logs**. В последнем запуске виден захваченный вывод узла Log: `Hello from Ada`.

Этот круг — перетащить, соединить, настроить, запустить, проверить — это ритм создания любого рабочего процесса.

## Сохранение, включение и тестирование в production

Рабочие процессы хранятся как граф JSON в колонке `Workflow.graph`. Builder сохраняет по мере редактирования; индикатор сохранения в заголовке показывает, когда последнее изменение попало на сервер. Отдельного шага «publish» нет.

Но: рабочий процесс срабатывает по триггеру только когда **isEnabled** включён. Новые рабочие процессы поставляются выключенными. Считайте этот флаг своим переключателем «готов к prod» — собирайте, нажмите **Run Manually** для прогона на тестовом payload, посмотрите **Logs** — и только потом включайте Enable.

Если нужно приостановить рабочий процесс, не удаляя его (например, во время постороннего инцидента), переключите **isEnabled** в положение «off» в **Settings**. Уже выполняющиеся запуски продолжатся; новые не начнутся.

## Перестановка и реорганизация

- Перетащите узел, чтобы переместить его. Позиция сохраняется в графе, так что следующий, кто откроет холст, увидит ту же раскладку.
- Кликните правой кнопкой по ребру, чтобы удалить его; правой кнопкой по узлу — для опций удаления и дублирования.
- Для широких рабочих процессов располагайте их слева направо, чтобы направление выполнения совпадало с направлением чтения.
