# 手動監測器

手動監測讓您可以建立狀態完全由人工或透過 API 管理的監測器。OneUptime 不會執行任何自動化檢查，您可以直接控制監測器的狀態。

## 概觀

手動監測器是由您自行更新的佔位項目。這在以下情境中很有用：

- 與透過 OneUptime API 更新狀態的外部監測工具整合
- 追蹤無法自動監測的服務或系統
- 為沒有自動化健康檢查的元件管理事件
- 表示由您手動追蹤其狀態的第三方相依項目

## 建立手動監測器

1. 前往 OneUptime 儀表板中的 **Monitors**
2. 點選 **Create Monitor**
3. 選擇 **Manual** 作為監測器類型
4. 輸入監測器的名稱與描述

## 運作方式

手動監測器沒有監測間隔、探針或自動化條件評估。監測器的狀態會維持在您所設定的值，直到您變更為止。

### 更新狀態

您可以透過兩種方式更新手動監測器的狀態：

- **儀表板** — 直接從 OneUptime 儀表板變更監測器狀態
- **API** — 使用 OneUptime API 以程式方式更新監測器狀態

### 事件與警示

您可以針對手動監測器建立事件與警示，就如同任何其他監測器類型一樣。這讓您可以：

- 追蹤由外部監測的服務的停機時間
- 在回報問題時手動建立事件
- 在狀態頁面上使用手動監測器，向使用者傳達狀態

## 何時使用手動監測器

| 使用案例         | 描述                                            |
| ---------------- | ----------------------------------------------- |
| 第三方服務       | 追蹤您所依賴但無法直接監測的外部服務狀態        |
| 實體基礎設施     | 表示沒有網路監測的硬體或實體系統                |
| 業務流程         | 追蹤會影響服務狀態的非技術性流程                |
| API 驅動的狀態   | 讓外部工具透過 OneUptime API 更新監測器狀態     |
| 狀態頁面佔位項目 | 在您的狀態頁面上顯示由 OneUptime 之外管理的元件 |
