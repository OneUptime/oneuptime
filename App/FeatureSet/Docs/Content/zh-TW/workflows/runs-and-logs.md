# 執行與記錄

每當工作流程執行時，OneUptime 都會儲存一筆關於發生了什麼的紀錄 — 何時執行、是否成功，以及每個區塊做了什麼。這筆紀錄稱為**執行（run）**。執行讓你能夠確認工作流程是否正常運作、為失敗的工作流程進行除錯，並回顧過去的活動。

## 在哪裡找到它們

| 頁面                        | 你會看到什麼                                                 |
| --------------------------- | ------------------------------------------------------------ |
| **Workflows → Runs & Logs** | 專案中每個工作流程的每次執行。可依工作流程、狀態與時間篩選。 |
| **Workflow → Logs 分頁**    | 僅限這一個工作流程的執行。                                   |
| **單一執行**                | 一次執行，包含每個區塊的輸出。                               |

## 執行狀態

| 狀態          | 代表的意義                                                                                     |
| ------------- | ---------------------------------------------------------------------------------------------- |
| **Scheduled** | 觸發器已觸發，執行即將開始。通常只需要不到一秒的時間。                                         |
| **Running**   | 工作流程正在進行中。長時間執行的區塊會讓執行保持在此狀態。                                     |
| **Success**   | 每個執行的區塊都已完成且沒有錯誤。（刻意走 **error** 分支仍算成功 — 工作流程本身並沒有失敗。） |
| **Error**     | 某個區塊失敗了，而且沒有連接 **error** 路徑來處理它。執行在那裡停止。                          |
| **Timeout**   | 執行的時間超過了允許的範圍。請參閱 [Configuration & Safety](/docs/workflows/configuration)。   |

## 解讀一次執行

點擊任何一次執行即可開啟詳細資訊。你會看到：

- **標頭** — 觸發器、開始與結束時間、總時長，以及狀態。
- **區塊清單** — 每個執行的區塊，依序排列。每個區塊都會顯示它收到的值、它的輸出，以及它走了哪條路徑。
- **錯誤** — 如果某個區塊失敗了，會顯示錯誤訊息以及（可用時）更多細節。

顯示的值正是該區塊所看到的內容 — 在所有變數都被填入之後。這是最有用的除錯檢視：如果一則 Slack 訊息顯示的是字面文字 `{{Incident.title}}` 而不是實際的標題，你就知道該變數沒有被解析。

## 常見除錯

### 「我的工作流程沒有執行。」

1. 確認該工作流程在 Settings 中已**啟用**。新的工作流程預設為停用。
2. 對於 OneUptime 事件觸發器：確認該事件確實發生了。開啟該紀錄並檢查其歷史記錄。
3. 對於 webhook 觸發器：確認另一個系統正在傳送到正確的 URL。大多數工具在傳送 webhook 時都會記錄 — 到那裡檢查。
4. 對於排程觸發器：確認 cron 運算式符合你預期的時間。

如果觸發器已觸發但沒有出現任何執行，請在 **Project Settings → Billing** 下檢查你的執行配額。

### 「後面的某個區塊從未執行。」

一個沒有執行的區塊通常是接線問題。開啟畫布並檢查：

- 前一個區塊的輸出是否已連接到這個區塊的輸入？
- 前一個區塊是否走了與你預期不同的輸出（例如，走了 **error** 而不是 **success**，或走了 **No** 而不是 **Yes**）？執行詳細資訊會顯示走了哪條路徑。

### 「某個變數傳過來是空的。」

開啟該執行並查看失敗區塊的值。

- 如果你看到字面的 `{{BlockName.field}}` 文字，表示該參照沒有被解析 — 很可能是區塊名稱或欄位名稱有拼字錯誤。
- 如果你看到的是空字串，表示前一個區塊有執行，但沒有產生那個欄位。

### 「我手動執行時可以運作，但從觸發器執行卻不行。」

使用 **Run Manually** 並搭配一個與真實觸發器所傳送內容相似的 JSON 酬載。然後將手動執行中的值與真實執行並排比較。差異通常出在單一的欄位名稱或型別。

## 重新執行工作流程

沒有「重試這次執行」的按鈕。我們不會自動重新執行舊的執行，因為其副作用（Slack 訊息、API 呼叫、工單）重複執行可能並不安全。若要重做這項工作，請修正工作流程，並讓下一次真實的觸發器觸發它。

對於手動工作流程，只要點擊 **Run Manually** 並使用相同的酬載即可。

## 執行紀錄會保留多久？

執行紀錄會為專案無限期保留。如果某個工作流程執行得非常頻繁，並使你的歷史記錄變得雜亂（例如每分鐘觸發一次的除錯工作流程），請停用或刪除它，以停止增加雜訊。
