# 工作流程概觀

工作流程讓你無需撰寫程式碼即可自動化 OneUptime 中的任務。在畫布上拖放幾個區塊、把它們連接起來，你就擁有了一套會在某些事件發生時自動執行的自動化流程 — 像是某個事件（incident）被開啟、某個排程觸發，或是另一個工具把資料送進 OneUptime。

可以把工作流程想成是專案的背景小幫手：它們會對事件做出反應、與其他工具溝通，並在你專注於自己的工作時，悄悄地讓各項資料保持同步。

## 你可以用工作流程做什麼

- **將 OneUptime 連接到你的其他工具** — 把事件（incident）送到 Slack、建立 Jira 工單、向你技術堆疊中的某個 webhook 發送請求。
- **對 OneUptime 中發生的事情做出反應** — 當一個嚴重事件（incident）被建立時，自動通知待命團隊並開立工單。
- **依排程執行任務** — 每五分鐘、每天晚上、每週一早上。
- **接收來自外部的資料** — 讓其他系統透過一個唯一的 URL 將資料推送進 OneUptime。
- **重複使用常見的自動化** — 建立一次，就能從任何其他工作流程呼叫它。

## 工作流程如何運作

每個工作流程都有三個部分：

1. **一個觸發器（trigger）** — 啟動工作流程的東西。它可以是一個手動按鈕、一個排程、一個傳入的 webhook，或是 OneUptime 中的某個事件（例如新的 incident）。
2. **一個或多個元件（component）** — 工作流程所執行的動作。發送訊息、發出 HTTP 呼叫、執行快速檢查、依條件分支。
3. **它們之間的連線** — 你從一個區塊拉線到下一個區塊，以決定執行的順序。

你會在畫布上以視覺化的方式建立這一切。大多數工作流程都不需要撰寫程式碼，不過當你有需要時，也可以放入一段 JavaScript 程式碼。

## 重要術語

| 術語 | 代表的意思 |
| --- | --- |
| **工作流程（Workflow）** | 整個自動化 — 一個名稱、一個畫布，以及一個用來開啟或關閉它的開關。 |
| **觸發器（Trigger）** | 第一個區塊。它決定工作流程何時執行。每個工作流程都剛好有一個觸發器。 |
| **元件（Component）** | 一個動作區塊 — 發送訊息、發出請求、檢查條件。 |
| **執行（Run）** | 工作流程的一次執行。會連同時間戳記以及每個區塊的輸出一起儲存。 |
| **全域變數（Global variable）** | 一個你儲存一次後就能在任何工作流程中重複使用的值（例如 API 金鑰）。 |

## 在 OneUptime 中何處可以找到工作流程

在左側導覽中開啟 **Workflows**。從那裡：

- **Workflows** — 你的工作流程清單。建立一個新的，或開啟一個現有的。
- **Builder 分頁** — 你設計工作流程的畫布。
- **Logs 分頁** — 這個工作流程的每一次執行，包含詳細資訊。
- **Settings 分頁** — 名稱、描述、擁有者、標籤、啟用／停用。
- **Global Variables** — 在你所有工作流程之間共用的值。
- **Runs & Logs** — 你專案中每個工作流程的執行歷史。

## 建立你的第一個工作流程

1. **建立** — 為你的工作流程取一個名稱與一段簡短的描述。
2. **挑選一個觸發器** — 手動、排程、webhook，或是來自 OneUptime 的某個事件。
3. **加入元件** — 把動作拖到畫布上並將它們連接起來。
4. **測試** — 點擊 **Run Manually**，並在記錄（logs）中觀察會發生什麼事。
5. **將它開啟** — 當你準備好時，在 Settings 中切換 **Enabled** 開關。

## 一個快速範例

假設你想在每次建立嚴重事件（incident）時於 Slack 中發文：

1. 建立一個名為「Critical incidents to Slack」的工作流程。
2. 挑選 **Incident → On Create** 觸發器。
3. 加入一個 **Conditions** 區塊。將它設定為檢查事件標題是否包含「Sev 1」。
4. 從 **Yes** 分支，加入一個 **Slack** 區塊。挑選頻道並撰寫訊息。
5. 將工作流程開啟。

下次有人開啟一個標題中含有「Sev 1」的事件時，Slack 就會亮起來。

## 工作流程如何與 OneUptime 的其餘部分搭配

- **監控器（Monitors）** 發現問題。**事件（Incidents）** 記錄它。**工作流程（Workflows）** 對它做出反應。
- **操作手冊（Runbooks）** 是給人看的逐步指南。工作流程則是無人值守的自動化。當需要由人來做決策時，請使用操作手冊；當步驟是自動化的時候，請使用工作流程。
- **工作區連線（Workspace connections）**（Slack、Teams）是工作流程發送訊息的目的地。
