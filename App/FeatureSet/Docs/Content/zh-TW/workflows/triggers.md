# 觸發器

觸發器是工作流程中的第一個區塊——它決定工作流程何時執行。每個工作流程都剛好有一個觸發器。你可以從四種類型中選擇。

## 手動

在工作流程頁面上點擊 **Run Manually** 即可隨需執行工作流程。你可以貼上一段 JSON 酬載，讓工作流程的其餘部分讀取。

適用於：你想用按鈕觸發的一鍵自動化，例如「輪替這把金鑰」或「發送一則測試警報」。

**輸出**：你貼入的 JSON，若未貼入則為空物件。

## 排程

使用 cron 運算式，依照重複的排程執行工作流程。

適用於：每晚清理、每小時同步、每週報表。

**設定**：一個 cron 運算式。以下是幾個常見的：

- `0 * * * *` — 每小時的整點執行。
- `*/5 * * * *` — 每 5 分鐘執行一次。
- `0 9 * * 1` — 每週一上午 9:00 執行。

如果系統短暫無法使用，當它恢復後就會立即接手執行該次任務——對於短暫的中斷，你不需要擔心錯過的觸發時點。

## Webhook

OneUptime 會建立一個唯一的 URL。任何存取該 URL 的請求都會啟動工作流程。請求的標頭、查詢參數與主體都會被傳入。

適用於：從另一個工具接收資料進入 OneUptime——CI/CD 回呼、來自其他監控的警報、你 CRM 中的註冊事件。

**輸出**：

- **Request Headers** — 來自傳入請求的所有標頭。
- **Request Query Params** — 已解析的查詢字串。
- **Request Body** — 已解析的主體（若不是 JSON 則為原始文字）。

該 URL 同時接受 `GET` 與 `POST`。呼叫端會收到一個快速的確認回應——工作流程本身則在背景執行。

請將該 URL 當成密碼來看待。任何擁有它的人都能啟動你的工作流程。

## OneUptime 事件觸發器

OneUptime 中幾乎每一樣東西——監控器、事故、警報、排程維護、狀態頁面、待命政策、團隊——都可以觸發工作流程。每一種都提供三個事件：

- **On Create** — 當新增一個時觸發。
- **On Update** — 當某個被變更時觸發。
- **On Delete** — 當某個被刪除時觸發。

這就是你建立「當 OneUptime 中發生 X 時，就執行 Y」的方式，而不需要在迴圈中持續檢查事物。

完整的紀錄會被傳遞給下一個區塊。例如，**Incident → On Create** 觸發器會傳遞新的事故，因此下一個區塊可以讀取它的標題、描述、嚴重程度，以及任何其他欄位。

### 團隊最常使用的事件

- **Incident** — 在事故被開啟、更新（已確認、已解決）或刪除時做出反應。
- **Alert** — 對警報的相同三種反應。
- **Monitor** — 在監控器被新增、編輯或移除時做出反應。
- **Scheduled Maintenance** — 在排定維護時段時自動公告該維護時段。
- **Status Page Subscriber** — 歡迎訂閱狀態頁面的人。
- **On-Call Duty Policy** — 將排程變更同步到另一個輪值系統。

依名稱搜尋觸發器面板，即可找到你想要的那個。

## 我該使用哪個觸發器？

| 如果你想要…                       | 選擇               |
| --------------------------------- | ------------------ |
| 點擊按鈕來執行工作流程            | **手動**           |
| 依重複的排程執行                  | **排程**           |
| 讓另一個系統推送資料進來          | **Webhook**        |
| 對 OneUptime 內部的某件事做出反應 | **OneUptime 事件** |

一個工作流程只能有一個觸發器。如果你需要兩種方式來啟動同一個自動化，請在一個工作流程中建立共用邏輯，並使用 **Execute Workflow** 元件從兩個輕量的「包裝」工作流程中呼叫它。
