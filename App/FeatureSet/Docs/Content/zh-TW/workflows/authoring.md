# 撰寫工作流程

若要建立工作流程，請開啟 **Workflows → Create Workflow**，為它命名，然後點進 **Builder** 分頁。你會看到一個空白畫布，你將在這裡建構自動化流程。

## 畫布

Builder 是一個拖放式畫布。你從旁邊的元件面板加入區塊，用線條將它們連接在一起，並點按每個區塊來設定它的功能。變更會自動儲存——儲存完成後，你會在頂端看到一個指示器。

每個工作流程都以開頭的一個 **trigger** 為起點。其餘的一切都是會執行某項作業的 **component**。

## 區塊上有什麼

| 欄位         | 它的作用                                                                                                                 |
| ------------ | ------------------------------------------------------------------------------------------------------------------------ |
| **Title**    | 顯示在畫布上的名稱。重新命名它可以讓複雜的工作流程更易於閱讀。                                                           |
| **Settings** | 該區塊完成其工作所需要的內容——一個 URL、一個 Slack 頻道、一段訊息內文等。必填欄位會以星號標示。                          |
| **Input**    | 左側的圓點，來自先前區塊的線條會連入此處。                                                                               |
| **Outputs**  | 右側的圓點，線條會由此連出到下一個區塊。許多區塊有各自獨立的 **success** 與 **error** 輸出，讓你可以同時處理這兩種情況。 |

## 連接區塊

從某個區塊的輸出圓點拖曳到下一個區塊的輸入圓點。你所繪製的線條決定了接下來執行什麼。

- 如果你從 **success** 連接，下一個區塊只會在先前的區塊成功時才執行。
- 如果你從 **error** 連接，下一個區塊只會在先前的區塊失敗時才執行。
- 如果你沒有連接某個輸出，該路徑就會在此停止。

你可以將一個輸出連接到多個區塊。從該點開始，它們會同時執行。

## 設定區塊

點按一個區塊以在旁邊開啟它的設定。每個設定都有適合的輸入類型——文字欄位、下拉選單、程式碼編輯器、開關等等。

大多數文字欄位都接受變數——資料就是這樣從一個區塊流向下一個區塊的。語法請參閱 [Variables](/docs/workflows/variables)。

## 你的第一個工作流程

最快熟悉這個畫布的方式：

1. 將一個 **Manual** trigger 拖曳到畫布上。
2. 將一個 **Log** component（在 **Utils** 之下）拖曳到它旁邊。把 trigger 連接到 Log component。
3. 在 Log 區塊的訊息欄位中，輸入 `Hello from {{Manual.JSON.name}}`。
4. 儲存並開啟工作流程。
5. 點按 **Run Manually**，貼上 `{ "name": "Ada" }` 作為輸入，然後送出。
6. 開啟 **Logs** 分頁。最近一次的執行會顯示 `Hello from Ada`。

這個循環——拖曳、連接、設定、執行、檢查日誌——就是你建構每個工作流程的方式。

## 儲存並開啟

畫布會在你作業時自動儲存。沒有獨立的「publish」步驟。

但工作流程只有在 Settings 中的 **Enabled** 開啟時才會真正執行。新的工作流程一開始是停用的。把那個開關當作你的安全網——先建構它，用 **Run Manually** 測試，檢查日誌，然後再開啟它。

若要在不刪除的情況下暫停工作流程，將 **Enabled** 關閉。已在進行中的執行會完成；不會有新的執行開始。

## 整理

- 拖曳區塊以移動它們。版面配置會被儲存，下一個人看到的會是相同的排列。
- 在線條上按右鍵以刪除它。在區塊上按右鍵以刪除或複製它。
- 對於較寬的工作流程，將它們由左至右排列，讓它們以執行的方向被閱讀。
