# 儀表板總覽

儀表板將 OneUptime 已經在收集的資料 — 指標、日誌、追蹤、事件、監測器、Kubernetes 資源、主機 — 轉化為一個頁面，讓人只要看一眼就能了解目前的狀況。

把請求延遲的圖表放在開放中事件清單旁邊，旁邊再放一個 CPU 量表，再旁邊放一段說明文字。儲存它。分享連結。

## 儀表板適合用來做什麼

- **一個「一切都還好嗎?」的頁面** — 給待命人員、團隊站立會議,或牆上掛的電視看。
- **發現關聯** — 在同一個頁面上,比在三個分頁之間切換,更容易看出 CPU 尖峰與延遲上升和開放中事件在同一時間發生。
- **調查問題** — 當你在除錯時,一個臨時搭建的儀表板,勝過一次跑十個查詢。
- **對外分享** — 面向客戶的效能頁面、合作夥伴的狀態頁面,或開源專案的公開儀表板。

## 你可以在儀表板上放什麼

- **圖表** 用於呈現一段時間的趨勢 — 延遲、錯誤、吞吐量。
- **單一數值方塊與量表** — 目前的錯誤率、CPU、開放中事件。
- **表格** 用於拆解明細 — 最吵的前 10 台主機、每個服務的錯誤計數。
- **文字區塊** 用於標題、說明背景,以及連往執行手冊的連結。
- **即時清單**,包含事件、警示、監測器、日誌、追蹤、Kubernetes 資源、Docker 資源,以及主機。

完整清單以及每個元件顯示的內容,請參閱 [Widgets](/docs/dashboards/widgets)。

## 重要名詞

| 名詞 | 代表的意義 |
| --- | --- |
| **儀表板（Dashboard）** | 整個頁面 — 一個名稱、一個由元件組成的格狀版面、時間範圍控制項,以及一份變數清單。 |
| **元件（Widget）** | 頁面上的一個方塊 — 一張圖表、一個數字、一份清單、一段文字。 |
| **變數（Variable）** | 頁面頂端的下拉選單,可一次篩選每一個元件（叢集、服務、客戶、環境）。 |
| **時間範圍（Time range）** | 每張圖表和每個數字所使用的時間窗口。在頁面頂端設定一次即可。 |
| **重新整理（Refresh）** | 元件多久重新查詢一次資料。關閉、每隔幾秒,或每隔幾分鐘。 |
| **模式（Mode）** | 不是 **編輯（Edit）**（拖曳元件位置）就是 **檢視（View）**（唯讀,訪客看到的樣子）。 |

## 在哪裡找到儀表板

在左側導覽列開啟 **Dashboards**。

| 頁面 | 你在那裡做什麼 |
| --- | --- |
| **Dashboards** | 你的儀表板清單。建立新的、搜尋,或依標籤篩選。 |
| **Dashboard → View** | 畫布。在標頭中於 **編輯（Edit）** 與 **檢視（View）** 之間切換。 |
| **Dashboard → Overview** | 描述、擁有者,以及標籤。 |
| **Dashboard → Settings** | 公開分享、密碼、IP 允許清單、自訂網域、品牌設定。 |
| **Dashboard → Owners** | 具有明確存取權的使用者與團隊。 |
| **Dashboard → Delete** | 移除這個儀表板。 |

## 建立儀表板

1. **建立** — 取一個名稱。畫布會以空白開啟。
2. **新增元件** — 選擇元件類型,設定它的資料,把它拖曳到你想要的位置。
3. **（選用）新增變數** — 例如,一個 `service` 下拉選單,讓同一個儀表板適用於每一個服務。
4. **設定時間範圍** — 預設值就很合用;之後再微調。
5. **（選用）公開分享** — 在 Settings 中切換開關,必要時加上密碼或 IP 允許清單。
6. **（選用）自訂網域** — 將儀表板架設在 `status.your-domain.com`。

## 一個快速範例

目標:為結帳服務做一個待命頁面,包含延遲、錯誤率、開放中事件,以及即時日誌串流。

1. 建立一個名為「Checkout on-call」的儀表板。
2. 新增一個 `service` 變數。將它的預設值設為 `checkout`。
3. 新增一個 **Chart** 元件,顯示 P95 延遲,並以 `service` 變數篩選。
4. 在它旁邊,新增一個 **Value** 元件顯示錯誤率,警告門檻設在 1%,嚴重門檻設在 5%。
5. 在下方,新增一個 **Incident List** 元件,顯示標記為 `checkout` 的事件。
6. 在那下方,新增一個 **Log Stream** 元件,顯示來自同一個服務的日誌。
7. 儲存。把下拉選單切換成 `payments` — 同一個儀表板現在就會顯示 payments 服務。

## 儀表板如何與 OneUptime 的其他部分搭配

- **監測器與遙測** 是資料的來源。你收集的每一個指標、日誌和追蹤,都可以在元件上查詢。
- **事件與警示** 會出現在 **Incident List** 和 **Alert List** 元件中。儀表板對這些資料是唯讀的 — 請在別處建立與更新它們。
- **狀態頁面** 是面向客戶的溝通管道（「系統正常運作嗎?」）。儀表板則用來詳細查看系統的運作行為。兩者相輔相成,而不是互相取代。
- **工作流程** 是 OneUptime 採取行動的方式。儀表板則是你解讀正在發生什麼事的方式。
