# 設定與權限

本頁說明當你有一個想要長期保留的儀表板時，值得了解的設定與存取控制。

## 擁有者

儀表板的**擁有者**是你已明確授予存取權的使用者與團隊（在他們專案層級角色之外另行授予）。

在 **Dashboard → Owners** 下：

- 新增**使用者擁有者**，讓某個人對此儀表板擁有額外的存取權。
- 新增**團隊擁有者**，將相同的存取權授予團隊中的每位成員。

當專案層級的讀取角色過於寬泛時，請使用擁有者機制——例如，一個包含客戶層級細節、僅應對客戶成功團隊可見的儀表板。

## 標籤

標籤是用來組織儀表板的標記。請在 **Dashboard → Overview** 下套用標籤。

常見模式：

- **依團隊**：`team:platform`、`team:checkout`、`team:growth`。
- **依環境**：`env:prod`、`env:staging`。
- **依用途**：`purpose:oncall`、`purpose:exec`、`purpose:investigation`。

**Dashboards** 清單可讓你依標籤篩選，這是在累積了大量儀表板的專案中，找到特定儀表板最快的方式。

## 權限

儀表板搭配你專案的角色型存取控制運作。相關權限如下：

| 權限                 | 允許的操作                   |
| -------------------- | ---------------------------- |
| **Create Dashboard** | 建立新的儀表板。             |
| **Read Dashboard**   | 檢視儀表板（在私人模式下）。 |
| **Edit Dashboard**   | 變更小工具、變數與設定。     |
| **Delete Dashboard** | 刪除儀表板。                 |

針對儀表板擁有者與自訂網域也有相對應的權限，因此你可以授予「管理擁有者」而不必授予「編輯儀表板」。

請在 **Project Settings → Teams & Roles** 下的專案角色上指派這些權限。

## 公開儀表板的存取權

當你將儀表板設為公開時（請參閱 [Sharing & Public Dashboards](/docs/dashboards/sharing)），有三項設定控制誰可以檢視它：

1. **Public Dashboard** 開關——若關閉，公開網址會回傳 404。
2. **Master Password**——若已設定，訪客需先輸入密碼，儀表板才會顯示。
3. **IP Whitelist**（Scale 方案）——若已設定，來自其他 IP 的請求會被拒絕。

你可以將上述任何項目組合使用。鎖定程度最高的組合是「公開開啟、已設定密碼、IP 允許清單啟用」——適用於你希望三層防護兼具的合作夥伴入口網站。

## 資料保留

儀表板本身不會過期。它們所顯示的資料會遵循你專案的保留設定——指標、記錄與追蹤可查詢的時間長度，取決於你的方案保留多久。一個指向「過去 90 天」的小工具，若方案只保留 30 天，便只會顯示仍儲存著的資料。

## 複製儀表板

若要複製現有的儀表板，請開啟儀表板清單並選擇 **Duplicate**。副本會包含每個小工具、變數與設定，但公開分享除外——該項目一律從關閉開始，讓你決定是否要重新開啟。

當你想將某個範本（例如「我們的待命儀表板」）分支成特定服務的副本時，這正是合適的做法。

## 刪除儀表板

在 **Dashboard → Delete** 下操作。此動作無法復原——儀表板的版面配置以及任何附加於其上的自訂網域都會被移除。你的遙測資料不受影響。

如果儀表板在自訂網域上公開，刪除後該網址會立即停止解析。若你想保留該網址的運作，請先將網域移至其他儀表板。

## 備份

如果你自行託管 OneUptime，定期的資料庫備份就已足夠——儀表板的設定會與專案的其餘部分一同儲存。

在 OneUptime Cloud 上，備份由我們為你處理。如果你想要自己的副本，可以透過 [OneUptime API](/docs/api-reference/api-reference) 讀取儀表板。
