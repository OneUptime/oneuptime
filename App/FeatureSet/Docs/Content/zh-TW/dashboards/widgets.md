# 小工具

小工具是儀表板上的一個圖塊。本頁列出您可以新增的每一種小工具、它顯示的內容，以及何時該使用它。

關於如何在畫布上拖曳小工具，請參閱[編寫儀表板](/docs/dashboards/authoring)。

## 圖表與數字

### 圖表

在儀表板的時間範圍內，將一個或多個指標序列繪製成折線圖、長條圖或區域圖。

**設定**：

- 一個或多個指標查詢。
- 一個可選的公式，用來組合兩個查詢（例如，使用 `errors / total * 100` 來取得錯誤率）。
- 針對持續增長而不重置的累積計數器，提供「顯示為速率」選項。
- 顯示選項：堆疊或疊加、Y 軸單位、圖例位置、圖表類型。

何時使用：當趨勢很重要時。隨時間變化的延遲、錯誤計數、佇列深度，任何由折線形狀來說明狀況的情境。

### 數值

一個大型的單一數字，並可選擇加上彩色的閾值。

**設定**：

- 一個會回傳單一數字的指標查詢（時間範圍內的最後一個值、平均值或最大值）。
- 一個可選的**警告**閾值（超過時顯示黃色）。
- 一個可選的**嚴重**閾值（超過時顯示紅色）。
- 數字格式與單位。

何時使用：當一個數字就能回答問題時。目前的錯誤率、此刻的 P95 延遲、未解決事件的數量。

### 量規

一個圓形量規，具有最小值、最大值、警告區段與嚴重區段。

**設定**：一個指標查詢與四個邊界值。

何時使用：當數值落在已知範圍內時。CPU 百分比（0–100%）、磁碟使用量、佇列容量。

### 表格

一個顯示指標結果的表格，每個群組一列。

**設定**：一個指標查詢（通常依主機或服務之類的標籤分組）、要顯示的欄位，以及列數上限。

何時使用：當您想要的是分項拆解而非趨勢時。前 10 個最吵雜的主機、每個服務的錯誤計數、每個端點的請求數。

## 文字

一個靜態的 Markdown 區塊。

**設定**：Markdown 內文。標題、清單、連結、強調與程式碼區塊皆可呈現。

何時使用：當您想要一個區段標題、一段背景說明、一份指向應變手冊的連結清單，或在事件期間放置臨時橫幅時。

## 記錄檔與追蹤

### 記錄檔串流

即時追蹤符合篩選條件的記錄檔行。

**設定**：記錄檔篩選條件（服務、嚴重性、屬性）與要顯示的欄位。

何時使用：當您想要不離開儀表板，就能看到應用程式此刻正在說什麼時。

### 追蹤清單

一份符合篩選條件的近期追蹤清單，包含持續時間、狀態與服務。

**設定**：追蹤篩選條件（服務、狀態、屬性）。

何時使用：當您想要的是近期活動清單而非圖表時。一種常見模式是頂部放一個延遲圖表，下方放一份緩慢追蹤的清單。

## 即時清單

### 事件清單

一份符合篩選條件的即時事件清單。

**設定**：依狀態、嚴重性、標籤、監控器或團隊篩選。

何時使用：當儀表板要回答「現在哪裡出問題了？」時。

### 警示清單

一份符合篩選條件的即時警示清單。

**設定**：依狀態、嚴重性、標籤篩選。

何時使用：當團隊儀表板要追蹤其服務上的警示時。

### 監控器清單

一份監控器及其目前狀態的即時清單。

**設定**：依監控器類型、標籤或目前狀態篩選。

何時使用：當您想要機群檢視時——「所有網站都正常嗎？」

## Kubernetes 資源清單

適用於已安裝 [Kubernetes Agent](/docs/monitor/kubernetes-agent) 的專案。每一種都接受針對叢集、命名空間與標籤的可選篩選條件。

- **Kubernetes Pod List**——顯示 Pod 及其階段、重啟次數與節點。
- **Kubernetes Node List**——顯示節點及其狀況與容量。
- **Kubernetes Namespace List**——顯示命名空間與工作負載計數。
- **Kubernetes Deployment List**——顯示部署及其期望與就緒副本數的對比。
- **Kubernetes StatefulSet List**——顯示有狀態集及其就緒副本數。
- **Kubernetes DaemonSet List**——顯示常駐集及其期望與就緒數的對比。
- **Kubernetes Job List**——顯示工作及其完成狀態。
- **Kubernetes CronJob List**——顯示排程工作及其排程與上次執行時間。

何時使用：當您想要用單一儀表板，將 Kubernetes 狀態與這些工作負載的遙測資料混合呈現時。

## Docker 資源清單

適用於已設定 Docker 監控的專案。

- **Docker Host List**——顯示執行 Docker 的主機及其容器計數。
- **Docker Container List**——顯示容器及其狀態、映像檔、主機、執行時間。
- **Docker Image List**——顯示映像檔及其大小。
- **Docker Network List**——顯示 Docker 網路與已連線的容器。
- **Docker Volume List**——顯示 Docker 磁碟區及其使用量。

## 基礎設施

### 主機清單

由 OneUptime 伺服器監控器所監控的主機，包含狀態、CPU、記憶體與執行時間。

**設定**：依標籤或目前狀態篩選。

## 我該使用哪一種小工具？

幾條快速規則：

- **隨時間的趨勢？** 圖表。
- **此刻很重要的一個數字？** 數值（若有明確的最小值/最大值則用量規）。
- **跨許多事物的分項拆解？** 表格。
- **系統現在正在發生什麼？** 記錄檔串流、追蹤清單、事件清單。
- **某一組特定資源的狀態？** 對應的清單小工具。
- **一個標題、一段文字或一個連結？** 文字。

大多數儀表板都會混搭幾種——頂部放一個圖表、旁邊放一兩個數值、一個文字分隔線，下方放一兩份清單。
