# 撰寫儀表板

若要建立儀表板，請開啟 **Dashboards → Create Dashboard**，為其命名，然後開啟它。畫布會以**編輯**模式開啟，準備好讓你開始新增小工具。

## 畫布

儀表板是一個格線。小工具會自動對齊定位——你決定每個小工具的位置以及大小。隨著你新增更多列，可以將頁面往下延伸。每個小工具在較大或較小的螢幕上都會維持其比例。

## 編輯與檢視

標頭中的切換按鈕可在兩種模式之間切換：

- **編輯**——小工具面板會開啟，你可以拖曳小工具、調整其大小，並點選任何小工具來變更其設定。
- **檢視**——儀表板為唯讀，呈現的內容與訪客及其他團隊成員所看到的完全相同。在分享之前，使用這個模式來檢查結果。

兩種模式下都是同一個儀表板。沒有獨立的「發布」步驟——每次編輯在儲存的那一刻就會即時生效。

## 新增小工具

1. 點選 **+** 按鈕來開啟小工具面板。
2. 挑選小工具類型。型錄請參閱 [Widgets](/docs/dashboards/widgets)。
3. 小工具會出現在畫布上。
4. 點選小工具上的齒輪圖示來開啟其設定。
5. 選擇資料來源（指標、清單篩選條件、一段文字等）以及任何顯示選項。
6. 拖曳小工具來移動它。拖曳邊角來調整大小。

## 資料的來源

大多數小工具會從以下三個位置之一讀取資料：

- **指標（Metrics）**——挑選一個指標和一種彙總方式（平均值、最大值、計數、百分位數）。新增篩選條件。選擇如何將結果分組。這與你在 OneUptime 其他地方看到的查詢建構器相同。
- **即時清單（Live lists）**——事件、警示、監控器、Kubernetes pod、Docker 容器、主機。每個清單小工具會接受一個篩選條件，並顯示符合條件的項目，且會即時更新。
- **靜態內容（Static content）**——**文字**小工具會接受一段 Markdown。可用於標題、背景說明、指向操作手冊的連結，或事件期間的臨時筆記。

## 閾值與格式化

單一數值小工具（**數值**、**儀表**）讓你可以設定：

- **警告閾值**——當數值越過該閾值時，顏色會變為黃色。
- **嚴重閾值**——當數值越過該閾值時，顏色會變為紅色。

圖表讓你可以設定 Y 軸單位、選擇圖例的位置，以及挑選各系列要彼此堆疊還是重疊。表格讓你可以挑選要顯示的欄位以及列數。

## 時間範圍與重新整理

在儀表板的頂部，有兩個控制項會影響每個指標小工具：

- **時間範圍**——一個預設值（過去一小時、24 小時、7 天、30 天）或自訂範圍。每個圖表和數字都使用這個時間範圍。
- **重新整理**——小工具重新查詢的頻率。關閉、5 秒、10 秒、30 秒、1 分鐘、5 分鐘、15 分鐘。即時清單會自行更新，與此設定無關。

不使用時間範圍的小工具（例如文字小工具）會忽略這兩個控制項。

## 儲存

畫布會在你作業時自行儲存。標頭中有一個小型指示器會告訴你最新的變更何時已儲存。如果你要進行大幅變更，請先複製儀表板，這樣你就會有一份安全的副本。

## 讓儀表板歷久彌新的訣竅

- **每個儀表板只放一個主題。** 別把「我們監控的所有東西」都放在同一頁。幾個焦點明確的儀表板勝過一個龐大的頁面。
- **將最重要的小工具放在頂部。** 人們會由上往下瀏覽——讓他們看到的第一樣東西就是「這個系統是否健康？」的答案。
- **用文字小工具為各區段加上標籤。** 每隔幾列放一個簡短的標題（「延遲」、「錯誤」、「容量」），能讓這一頁從遠處也能一目了然。
- **使用變數而非重複建立。** 如果你正打算為第二個服務建立相同的儀表板，請改用一個帶有 `service` 變數的儀表板。請參閱 [Variables & Filters](/docs/dashboards/variables)。
