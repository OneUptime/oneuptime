# 分享與公開儀表板

預設情況下，儀表板僅供您專案私下使用 — 只有登入的團隊成員才能查看。但 OneUptime 也讓您可以公開分享儀表板、用密碼保護它、限制特定 IP 存取，以及將它託管於您自己的網域。本頁涵蓋這四種方式。

## 私有儀表板（預設）

儀表板僅供您專案已登入的成員存取。其 URL 看起來像 `https://oneuptime.com/dashboards/<id>/view`，且需要登入。

在專案內，由擁有者與標籤控制誰可以看到什麼 — 請參閱[組態與權限](/docs/dashboards/configuration)。

## 公開儀表板

在 **Dashboard → Settings** 下，將 **Public Dashboard** 開啟。儀表板現在會有第二個不需要登入的 URL。將它分享給供應商、合作夥伴、客戶，或貼到公開的 README 中。

公開儀表板：

- 一律以 **View** 模式開啟。公開訪客無法編輯，也看不到小工具面板。
- 包含您已新增的變數。訪客從與您團隊相同的下拉選單中挑選。
- 使用您在 Settings 中設定的**品牌識別** — 頁面標題、描述、標誌、favicon。

啟用公開儀表板就像發佈一個網頁。上面的每個小工具都會變成全世界可讀。在切換開關之前，先看看畫布上有什麼。

## 主密碼

要為公開儀表板設定密碼：

1. 開啟 **Public Dashboard**。
2. 開啟 **Master Password**。
3. 設定密碼。

訪客在儀表板出現之前會看到密碼提示。密碼以雜湊形式儲存 — 我們永遠看不到實際的密碼。

在以下情況使用主密碼：

- 您想要與合作夥伴或客戶分享，但不希望 URL 在外洩時仍可使用。
- 儀表板屬於「半公開」 — 開放程度足以讓您不想將每位檢視者都邀請為團隊成員，但又沒有開放到可以放在公開網際網路上。

若需要更強的把關（每位檢視者各自的帳號、誰看了什麼的稽核軌跡），請改為將儀表板保持私有，並將檢視者邀請為唯讀團隊成員。

## IP 允許清單

在 **Scale** 方案中，您可以將公開儀表板限制於一份 IP 位址或範圍的清單。請在 **Dashboard → Settings → IP Whitelist** 下進行設定。

在以下情況使用此功能：

- 儀表板應該只能從您的辦公室或 VPN 存取。
- 供應商入口網站應該只能從他們已知的 IP 存取。
- 您想要在主密碼之上再加一層額外保護。

來自任何其他 IP 的請求都會被拒絕。

## 自訂網域

開箱即用時，公開儀表板會在 `oneuptime.com` 上提供。要將它託管於您自己的子網域，例如 `dashboard.acme.com`：

1. 在您的 DNS 上新增一筆 CNAME 記錄，將子網域指向 OneUptime 的目標。
2. 在 **Dashboard → Settings → Custom Domains** 下，新增該網域。
3. 驗證它。OneUptime 會替您檢查 DNS 記錄。
4. 一旦驗證完成，儀表板便可在您的自訂網域和原始 URL 兩者上存取。

自訂網域適用於：

- 以您自己品牌呈現的面向客戶儀表板。
- 共同品牌的合作夥伴儀表板。
- 擁有自己 URL 的公開健康狀態頁面。

如果您要將相同內容提供給多個受眾，您可以為單一儀表板附加多個自訂網域。

## 品牌識別

在 **Dashboard → Settings** 下，您可以設定：

- **Page title** — 顯示在瀏覽器分頁和頁面頂端的內容。
- **Page description** — 搜尋引擎和社群預覽所使用的描述。
- **Logo** — 上傳 PNG 或 SVG 以顯示在頁首。
- **Favicon** — 瀏覽器分頁中的小圖示。

品牌識別僅在公開檢視儀表板時套用。內部檢視者一律看到 OneUptime 的品牌識別。

## 嵌入

您可以使用 iframe 將公開儀表板嵌入您自己的網站：

```html
<iframe src="https://dashboard.acme.com/view"
        width="100%" height="800"
        frameborder="0"></iframe>
```

如果儀表板設有主密碼，訪客將會在 iframe 內看到密碼提示。

## 可分享的 URL

儀表板 URL 將目前的變數選擇和時間範圍以查詢參數的形式包含在內。調整下拉選單、複製 URL、貼到聊天中 — 開啟連結的人會看到完全相同檢視的儀表板。

這是讓隊友看到「事件開始當下的儀表板」最快的方式。釘選時間範圍、複製、貼上。
