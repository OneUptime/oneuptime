# 變數與篩選器

變數能將單一儀表板變成一個範本。在儀表板中加入一個 `service` 變數，相同的圖表便會針對 `checkout`、`payments` 或 `search` 重新繪製——檢視者只要從頂部的下拉選單挑選即可，您不必再建立三個幾乎相同的儀表板。

## 變數類型

在 **Dashboard → Settings → Variables** 下新增變數。每個變數都有一個名稱（在小工具中以 `{{name}}` 的形式使用）、一個選填的標籤，以及一個類型。

### Custom List

一個靜態下拉選單。選項由您自行輸入。

適用情境：選項數量少且固定時。例如值為 `prod, staging, dev` 的 `environment`，或值為 `us-east-1, eu-west-1, ap-south-1` 的 `region`。

### Query

選項來自針對您資料執行的查詢。

適用情境：選項會隨時間變動，而您希望下拉選單能即時更新。例如「過去 24 小時內出現過的每個客戶 ID」。查詢會針對您專案的資料執行，結果即成為下拉選單的內容。

### Text Input

一個自由文字欄位。檢視者輸入的任何內容都會被使用。

適用情境：您希望儀表板像搜尋工具一樣運作。可依 IP 位址、請求 ID 或任何其他自由格式的值進行篩選。

### Telemetry Attribute

選項為某個遙測屬性在儀表板時間範圍內的不重複值。

設定 **attribute key**（例如 `service.name`、`host.name`、`k8s.cluster.name`）。下拉選單會填入您的日誌、指標與追蹤中出現過的每個不重複值。

適用情境：選項與您隨遙測一同傳送的標籤相符時。這是最常見的類型，因為它會自動更新——當您部署一個標記為 `service.name = inventory` 的新服務時，該名稱便會出現在下拉選單中，您無須編輯儀表板。

## 多選

每個變數都可允許多重選取。開啟後，檢視者可挑選一個或多個值；儀表板會篩選出其中任一個值。

使用多選的時機：當您想在不離開儀表板的情況下，比較「checkout 與 payments 兩者一起」時。當所選值之間的運算不合理時（例如對平均值再取平均），請避免使用。

## 預設值

每個變數都可以有一個預設值。在檢視者變更之前，儀表板會以預設值繪製。對於公開儀表板，預設值就是訪客最先看到的內容。

## 如何在小工具中使用變數

凡是小工具接受篩選器之處——指標的 `WHERE`、清單的篩選器、日誌串流的屬性比對——您都可以使用 `{{variable_name}}`。

舉例來說，一個依服務篩選的圖表：

```
service.name = '{{service}}'
```

當下拉選單設為 `checkout` 時，圖表會篩選至 checkout 服務。當檢視者切換到 `payments` 時，圖表會針對 payments 重新繪製。

對於 **Telemetry Attribute** 變數，OneUptime 知道該變數對應到哪個屬性，並會將篩選器套用到每個使用相同屬性的小工具——您不必逐一手動編輯每個小工具。

## 時間範圍

儀表板標頭具有一個全域時間範圍。每個指標小工具都會針對這個視窗進行查詢。選項：

- **Presets**——過去一小時、24 小時、7 天、30 天、90 天（視您的資料保留期而定）。
- **Custom**——挑選一個開始與結束時間。

時間範圍是儀表板 URL 的一部分——分享 URL 即分享該視窗。在事件期間很有用：將時間範圍固定為「今天 10:00–10:30 UTC」，再把連結貼到事件頻道中。

## 重新整理間隔

在時間範圍旁，挑選小工具重新查詢的頻率：

- **Off**——小工具僅在頁面載入時查詢一次。
- **5s / 10s / 30s / 1m / 5m / 15m**——自動重新整理。

自動重新整理適合用於牆面顯示螢幕或即時事件檢視。當您正在調查時請將其關閉，這樣在您查看時畫面才會保持靜止。

## 整合運用

一個以服務為範本的儀表板通常包含：

1. 一個類型為 **Telemetry Attribute**、對應 `service.name` 的 `service` 變數。預設值：您最常觀察的服務。關閉多選（讓圖表一次只顯示一個）。
2. 一個類型為 **Custom List** 的 `environment` 變數。預設值：`prod`。
3. 一個類型為 **Telemetry Attribute**、對應 `k8s.cluster.name` 的 `cluster` 變數。開啟多選（讓您可以跨叢集比較）。
4. 在篩選器中參照這些變數的小工具。

結果就是：一個儀表板、涵蓋每個服務、頂部三個下拉選單。
